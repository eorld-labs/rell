# 研发记录索引

本目录用于保存世界模型项目的连续研发记录，目标是形成可复核、可追溯、可提交、可审查的工程实施证据链。

## 使用方式

1. 每天推进前，先明确一个最小研发目标。
2. 使用 `每日研发记录模板.md` 创建当日记录。
3. 记录输入材料、研发过程、产出文件、验证方式和验证结果。
4. 将当日工作与对应技术层、专利链特征、代码或文档路径绑定。
5. 使用中文 Git 提交摘要，并在提交正文写明技术对应和证据链。

## 每日最小闭环

```text
小目标 -> 实施改动 -> 验证结果 -> 研发记录 -> 中文提交 -> 远程推送
```

该闭环用于证明项目处于持续实施、验证和迭代状态，而不是仅进行概念整理或事后补写。
