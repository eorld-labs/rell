# 提交规范与专利链映射

## 一、目的

本项目的 Git 提交不仅用于代码版本管理，也用于形成“实施过程证据链”。因此，每次提交都应当明确对应到项目技术体系、专利链特征、代码模块、输出证据和验证方式。

## 二、提交摘要规则

每次提交摘要必须使用中文。

推荐格式：

```text
动词 + 技术对象 + 证据/能力
```

示例：

```text
初始化世界模型运行底座与技术证据链
补充计划预验真接口与审计输出
实现来源链校验与代理权预算裁决
增加候选态执行与经验回写证据
```

避免使用：

```text
init
update
fix
misc
WIP
```

## 三、提交正文规则

每次重要提交建议包含以下字段：

```text
技术对应：
- 对应技术层：
- 对应专利/技术链特征：
- 对应文档或代码模块：

证据链：
- 输入证据：
- 输出证据：
- 验证方式：
```

## 五、证据链映射

每次提交应尽量回答：

1. 本次提交对应哪个技术链特征。
2. 该技术特征在代码中由哪个模块实现。
3. 输入样例在哪里。
4. 输出证据在哪里。
5. 如何复现验证。

示例：

```text
技术对应：
- 对应技术层：运行时执行治理层
- 对应专利/技术链特征：计划预适配、来源链、代理权预算
- 对应文档或代码模块：demo_runtime/run_demo.py

证据链：
- 输入证据：demo_runtime/data/plan.json
- 输出证据：demo_runtime/output/precheck.json、decision_tokens.json
- 验证方式：python .\demo_runtime\run_demo.py
```

## 六、本地提交模板

仓库内提供 `.gitmessage` 作为提交模板。

本地可执行：

```powershell
git config commit.template .gitmessage
```

之后执行 `git commit` 时会自动带出模板。
