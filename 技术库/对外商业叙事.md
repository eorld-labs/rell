# 对外商业叙事

## 一、对外一句话

我们为 Agent、机器人和物理 AI 提供空间经验世界模型与运行时执行治理底座，使自动化执行体在真实环境中能够理解空间、复用经验、执行任务，并在来源、授权、预算、计划边界和物理后果约束下安全可控运行。

## 二、客户痛点

自动化执行体正在从“回答问题”走向“执行动作”。一旦它们能够调用工具、修改数据、控制设备、部署代码、执行支付或影响物理空间，客户就会遇到一组共同痛点：

1. 动作来源不清楚，难以判断是用户授权、模型推理、外部输入污染还是子任务扩散导致。
2. 权限是静态的，难以表达长期任务中的剩余代理权、预算消耗和撤销传播。
3. 多步计划执行到中后段才发现越界，前序步骤已经产生后果。
4. Agent 或机器人缺少长期经验，重复犯错，换空间后重新适配成本高。
5. 高风险工具调用和物理控制动作缺少运行时因果验真。
6. 对老人、儿童、患者、被监护人等主体缺少跨系统保护策略。
7. 出问题后缺少可追溯的来源链、裁决链和审计摘要。

## 三、我们的解决方案

我们提供三类基础能力：

### 3.1 空间经验世界模型

将空间语义、对象关系、任务历史、动作后果、人类偏好和补救经验组织为可检索、可调用、可回写的经验底座。

客户获得：

1. 更低的场景适配成本。
2. 可复用的任务经验和失败经验。
3. 面向机器人和 Agent 的环境理解接口。
4. 动作后果预测和偏好适配能力。

### 3.2 运行时执行治理

在动作进入真实系统或物理执行端前，对计划边界、来源链、代理权预算、人类真实意图和物理控制因果一致性进行验证。

客户获得：

1. 高风险动作准入控制。
2. 多步计划预验真。
3. 代理权预算和任务链降权。
4. 物理控制动作安全接管。
5. 可审计的裁决链。

### 3.3 SDK/API 与治理网关

通过 SDK、API、网关或私有化部署方式接入客户现有系统。

客户无需重构现有 Agent、机器人或工具平台，即可在执行链路前后增加空间经验和治理能力。

## 六、是否太底层

SDK、API 和专利许可确实是底层抓手，但底层不等于商业价值弱。

底层能力有三种风险：

1. 客户不理解。
2. 采购预算不明确。
3. 落地周期长。

但底层能力也有三种优势：

1. 一旦接入，替换成本高。
2. 可横向迁移到多个行业。
3. 能形成标准、接口和专利壁垒。

因此，对外不能只说“我们卖底层 SDK”。应当把底层能力包装成客户可采购的结果：

1. 降低机器人场景部署成本。
2. 降低 Agent 高风险动作事故率。
3. 给企业自动化提供审计和合规证据。
4. 给物理 AI 提供执行前准入和安全接管。
5. 给平台提供可嵌入的治理网关。

换句话说：

> 内核是 SDK/API/IP，客户买的是安全执行、快速部署、审计合规和经验复用。

## 七、推荐早期商业路径

### 阶段一：技术验证型合作

找 1 至 2 个合作方，接入其 Agent 或自动化链路，提供计划预验真、来源链和代理权预算。

目标不是大收入，而是证明客户愿意把执行链路接入我们。

### 阶段二：SDK/API 产品化

将计划预验真、来源链、代理权预算、审计摘要做成清晰 API。

形成可演示、可试用、可部署的产品包。

### 阶段三：行业样板

选择一个行业形成样板：

1. 企业 Agent 治理。
2. 智能家居/IoT 控制治理。
3. 养老或家庭机器人执行治理。
4. 云运维 Agent 安全治理。

### 阶段四：专利许可谈判

当行业开始意识到 Agent 和物理 AI 执行治理的重要性后，专利矩阵可以成为授权谈判筹码。

专利许可不一定最早变现，但可能成为中后期最高杠杆的商业资产。

## 八、对外表达禁区

避免说：

1. 我们做通用机器人大脑。
2. 我们做最大世界模型。
3. 我们替代 VLA。
4. 我们做 AI 伦理治理。
5. 我们只是做数据集。

建议说：

1. 我们为执行型 AI 提供运行时治理基础设施。
2. 我们让 Agent 和机器人在真实环境中可控执行。
3. 我们提供空间经验、计划预验真、来源链、代理权预算和审计闭环。
4. 我们不是替代模型，而是让模型输出进入真实世界前可验证、可裁决、可回写。
