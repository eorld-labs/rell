# P013 意图理解与任务规划 —— 立项书

---

## 一、哲学源头：人怎么听懂别人要他做什么

"帮我把客厅整理一下。"

一个8岁孩子不需要拆解教程。他基于过去三年在类似客厅里收到类似指令的经验——"整理"通常包括哪些动作、哪些东西属于"该收的"、哪些东西看起来像杂物其实是妈妈正在读的书不能动——直接形成了一套动作序列，然后去做。

这不是翻译，是**意图理解+基于经验的规划**。

P013要做的，就是让硅基生命学会同一件事：接收到人类的意图——无论用什么形式表达——基于当前的空间情境、已有的物理经验、和过去类似意图的执行结果，把它形成为一个可执行的任务计划。

## 二、技术骨架：P011的同构框架，但起点不同，终点不同

| | P011（物理经验） | P013（意图理解与任务规划） |
|------|------|------|
| 输入 | 当前空间情境+候选动作 | 人类意图信息（语言/手势/环境触发/日程推断） |
| 记录结构 | {情境，动作，物理后果} | {意图，任务计划，执行结果，人类反馈} |
| 类比检索什么 | 相似情境→动作后果预测 | 相似意图→任务计划模板 |
| 输出 | 下一步最优动作 | 可执行的多步子任务计划 |
| 反馈维 | 物理后果（成功/失败/偏移） | 任务层面的完成度+人类满意度 |

**同一个骨骼：{输入，计划/动作，结果，反馈}→预测→选择→执行→存储。P011管"物理世界怎么操作"，P013管"人类意图怎么变成操作计划"。**

## 三、在矩阵中的位置

```
P010（空间认知）→ "在哪"
```

## 五、独权核心骨架（升维版）

> S1. 获取与待执行任务相关联的人类意图信息；
>
> S2. 基于所述人类意图信息及目标空间的情境信息，将所述人类意图形成为包含多个可执行步骤的任务计划；
>
> S3. 执行所述任务计划；
>
> S4. 将本次{所述人类意图信息，所述任务计划，所述任务计划的执行结果，人类反馈}作为任务经验记录存储。

**独权1不出现"经验模型""拆解""检索"。** 保护的是"意图→计划→执行→存储"这个闭环。20年后不管用什么技术形成计划——只要做了这个闭环，就在射程内。

---

## 六、从权锁死当前最优——对抗"LLM+规划器"质疑

**从权A——类比复用历史计划，不依赖LLM推理**：

> 根据权利要求1所述的方法，其特征在于，步骤S2包括：基于所述任务经验记录中与当前人类意图相似的历史意图及对应的历史任务计划进行类比复用，以形成所述任务计划；所述形成不以语言模型推理或规则引擎分解为必要步骤。

**从权B——计划模板的结构**：

> 所述任务经验记录中的每条记录包括历史意图描述、历史任务计划、历史执行结果和历史人类反馈；所述类比复用基于当前人类意图信息与历史意图描述之间的相似度。

**从权C——失败不重来**：

> 当所述任务计划中的某一可执行步骤执行失败时，基于当前情境确定该步骤的替代动作，仅替换该失败步骤，而不重新形成全部任务计划。

## 七、与"LLM+规划器"的区别

| | LLM+规划器 | P013 |
|------|------|------|
| 拆解谁来做 | LLM推理——链式思维生成步骤 | 从权A：类比复用历史任务计划 |
| 计划从哪来 | 参数空间生成 | 真实交互验证过的历史计划 |
| 物理落地 | 拆对了但不知道茶几坐标 | 和情境信息关联——"这个用户家客厅" |
| 错了怎么办 | 换prompt | 从权C：只换失败那一步 |

## 与 P014 的边界补充

P013 的失败步骤替换发生在任务计划层，目标是基于当前情境和历史任务经验替换某个可执行步骤或其参数，且不重新形成全部任务计划。该步骤替换不主张在物理动作正在执行的连续链路中进行不停机补救切换。

执行动作中途检测到实际状态与预期状态偏离，并从历史补救经验中确定候选补救动作、在不中断当前执行链路的情况下切换，属于 P014。P013 可以调用 P014 的恢复结果作为后续任务经验记录，但不取得 P014 的执行中恢复主链。
