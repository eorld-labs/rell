### 1.1 D1（CN121998092A）对比分析

**概述**：意图解析→关键词匹配计划缓存→SLM渲染模板/LLM生成新计划→执行→异步收集→模板构造。

**与P013核心差异**：

| | D1 | P013 |
|------|-----|------|
| 意图解析 | SLM提取核心任务意图关键词 | 人类意图信息（不绑实现） |
| 计划形成 | 缓存命中→SLM渲染模板；缓存未命中→LLM推理生成 | 形成为任务计划（不绑实现）；从权2：类比复用历史计划 |
| 检索方式 | **关键词精确匹配** | 相似度检索 |
| 反馈维度 | LLM反思自判有效/无效——**无人类反馈** | {意图，计划，结果，**人类反馈**} |
| 技术路线 | 从名称到内容全部锚定LLM+SLM | 从权2明确排除LLM推理 |

**结论**：D1在结构上和P013最近，但走LLM路线。从权2（不以LLM推理为必要步骤）是防火墙。D1无法破坏独权1的新颖性（缺少人类反馈维），也不构成创造性威胁（技术路线根本不同）。

### 1.2 D2（CN121946550A）对比分析

**概述**：语义特征提取→从历史任务库检索经验片段→语言模型生成层级任务计划→技能编排执行→实时闭环修正。

**与P013核心差异**：

| | D2 | P013 |
|------|-----|------|
| 历史经验 | 预设历史任务库中的历史任务经验片段 | 结构化{意图，计划，结果，人类反馈}任务经验记录 |
| 计划生成 | **通过语言模型生成** | 形成为任务计划（不绑实现） |
| 反馈 | 实时传感数据闭环修正 | 人类反馈（沉默/纠正/质疑/中断） |
| 存储结构 | 历史任务经验片段（未定义结构化形态） | 三元组任务经验记录 |

**结论**：D2有历史经验检索+计划执行+修正闭环，和P013结构最近。但从权2和从权4联手排除——D2用语言模型生成计划，且无结构化人类反馈维。

## 四、背景技术段建议写法

```
现有技术中，大模型智能体或具身智能系统可通过语言模型或视觉-语言-动作模型，
基于用户输入的自然语言指令进行意图解析和任务分解，形成执行计划或动作序列。
该类方案通常将任务规划完全委托于语言模型或神经网络的前向推理，
缺少基于历史人类交互形成的信息进行结构化积累和基于相似历史意图的类比复用机制。
```
