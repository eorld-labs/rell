# P013 任务语义翻译工程证据说明

## 一、中文摘要

本工程证据包用于固定 P013“任务语义翻译”的最小可运行闭环。证据链通过独立校验脚本和结构化 JSON 输出，逐段证明：系统能够将自然语言请求统一收口为语义请求；将人类意图翻译为目标因果事实和可执行过程链；在用户显式给出空间路线时保留路线约束并完成因果校验；在对话教学和边教边动过程中形成新的任务经验记录；并通过统一入口、概念解析和提示词契约约束语言模型只参与语义理解而不直接下发执行。

该证据包的重点不是展示界面，而是将 P013 的核心技术点收束成可重复运行、可落盘、可复核、可直接对应权利要求步骤的工程输出。

## 二、验证命令

单独生成 P013 工程证据：

```powershell
python demo_runtime\rell_sample\validate_p013_task_semantics.py
```

全量样品检查并同步刷新证据：

```powershell
python demo_runtime\rell_sample\run_all_checks.py
```

## 四、证据文件索引

### 1. evidence_index.json

用途：机器可读的证据索引。  
对应内容：列出每个证据文件对应的技术特征、步骤、关键字段和代码来源。

### 2. 01_semantic_request_entry.json

用途：证明统一语义入口。  
对应步骤：S1 获取人类意图信息以及目标空间情境信息。  
关键字段：

- `task_execution.semantic_request.request_type`
- `task_execution.semantic_request.intent_frame.goal_fact`
- `state_query.semantic_request.request_type`
- `teaching.semantic_request.teaching_plan`

### 3. 02_goal_fact_and_plan.json

用途：证明目标事实翻译和过程链形成。  
对应步骤：S2 形成包含多个可执行步骤的任务计划。  
关键字段：

- `short_intent.goal_fact`
- `short_intent.candidate_process_chain`
- `long_run.intent_translation.causal_plan.process_chain`

### 4. 03_explicit_route_preservation.json

用途：证明显式路线保留与空间语义约束进入计划形成。  
对应步骤：S2 形成任务计划。  
关键字段：

- `routed_run.intent_translation.candidate_process_chain`
- `routed_run.intent_translation.intent_frame.spatial_constraints`
- `routed_run.intent_translation.intent_frame.concept_matches`

### 5. 04_teaching_feedback_and_experience.json

用途：证明教学反馈闭环和任务经验形成。  
对应步骤：S3 执行任务计划；S4 记录人类反馈并存储任务经验。  
关键字段：

- `dialogue_teaching.experience.causal_signature`
- `reused_run.intent_translation.causal_plan.reasoning`
- `blocked_stepwise.step_feedback`
- `finished_stepwise.experience_result.experience.process_chain`

### 6. 05_unified_entry_and_llm_boundary.json

用途：证明统一入口、概念解析和 LLM 边界。  
对应步骤：S2 形成任务计划。  
关键字段：

- `agent_preview.semantic_request.request_type`
- `agent_preview.route_result.intent_translation.goal_fact`
- `concept_resolution.resolved_concepts`
- `llm_prompt_contract.handoff_contract.direct_execution_allowed`

### 7. 06_task_experience_library.json

用途：证明任务经验库留存了结构化任务经验。  
对应步骤：S4 存储任务经验记录。  
关键字段：

- `experience_library.experiences`
- `latest_dialogue_experience.context.human_intent_ref`
- `latest_stepwise_experience.context.human_intent_ref`
- `concept_candidate_library.concept_candidates`

## 五、典型审查质疑对应

### 1. 被质疑只是自然语言理解或抽象规划

可引用证据：

- `02_goal_fact_and_plan.json`
- `04_teaching_feedback_and_experience.json`

答复重点：P013 不是停留在文本分类或抽象意图识别，而是将意图映射为目标事实、过程链、执行反馈和任务经验记录的闭环结构。

### 2. 被质疑依赖大模型直接生成计划

可引用证据：

- `01_semantic_request_entry.json`
- `05_unified_entry_and_llm_boundary.json`

答复重点：统一语义入口、概念解析和提示词契约共同限制模型只输出候选语义，不允许绕过验证层和编排层直接执行。

### 3. 被质疑缺乏历史任务经验复用

可引用证据：

- `04_teaching_feedback_and_experience.json`
- `06_task_experience_library.json`

答复重点：对话教学和边教边动都会形成结构化任务经验，并在后续任务中被因果求解层或经验库推理直接复用。
