# P010 最后一轮全文近似判断

## 一、结论先行

最后一轮全文没有出现单篇完全覆盖 P010 三柱链路的文献，不构成放弃理由。

但本轮确认了两个需要重点防御的生成侧/组织侧近似：

1. **CN113052152B**：基于视觉的室内语义地图构建，包含度量图、区域划分层、房间拓扑图、对象层、局部/全局场景图、对象类别/属性/位置关系。  
   这是本轮最接近“空间语义信息组织”的文献。

2. **CN118298231A**：基于语义关系图谱的室内多场景语义地图，包含空间知识图谱、先验关系图谱、三元组、场景连通关系、地标位置关系。  
   这是本轮最接近“空间关系 + 先验关系图谱”的文献。

两者共同说明：普通语义地图领域已经能够做到“室内区域划分、房间拓扑、对象关系、知识图谱、先验关系”。因此 P010 后续不能只靠“空间语义/空间关系”这几个词站住，必须把区别压在：

**空间语义信息被组织为空间语义先验，并由端侧空间使用主体加载和转换为主体侧空间认知模型。**

## 二、逐篇判断

### 1. WO2025136997A1 / US20250209724A1 / US12620164B2：Using Neural Radiance Fields for Label Efficient Image Processing

公开内容：
- 使用 NeRF / implicit field / DINOv2 / temporal masked autoencoder 等模型，从一个或多个图像生成 semantic map。
- 输出可包括 semantic BEV map、semantic segmentation map、3D scene understanding 任务输出。
- 应用场景包括自动驾驶、XR、机器人等。
- 技术重点是用少量标注或无监督预训练降低 BEV 语义映射的标注成本。

攻击点：
- 可攻击 P010 的“外部视觉输入生成空间语义信息”。
- 其“one or more images -> semantic map / BEV semantic map”表达很宽，且明确提到 XR 和 robotics。

防御结论：
- 该同族文献的发明点是模型训练和 BEV/semantic view mapping，不是空间语义先验的数据接口。
- 输出是 semantic BEV map、semantic segmentation map 或 3D scene understanding 任务结果，不是包括语义区域集合、空间关系集合和端侧转换信息的空间语义先验。
- 没有端侧空间使用主体加载该先验，也没有转换为主体侧空间认知模型。
- 分类：**橙偏红，生成侧强近似，但不破坏全链路。**

### 2. EP4115318A1：System and Method for Predicting a Map from an Image

公开内容：
- 从单目图像预测 occupancy grid map。
- 该 map 可以是 semantic map / semantic Bayesian occupancy grid map。
- 每个栅格可同时编码多个对象类别，并可编码几何信息。
- 使用 feature pyramid、dense transformer、topdown network，将图像平面特征映射到 BEV 空间。
- 主要服务自动驾驶/机器人平台的路径规划、碰撞避免、导航等。

攻击点：
- 可攻击“从图像直接预测语义地图/BEV 空间表示”。
- 比一般视觉 SLAM 文献更接近“外部图像 -> 空间语义表示”，因为其不依赖深度输入，强调单目图像直接生成 BEV 语义占据栅格。

防御结论：
- 其输出是 BEV semantic occupancy grid map，本质是导航/规划/避障用的占据栅格地图。
- 没有将区域功能属性和空间关系组织为端侧可加载的空间语义先验。
- 没有“端侧转换信息”，也没有主体侧空间认知模型。
- 应作为 CN118730072A 的英文同类增强文献处理。
- 分类：**橙偏红，生成侧/BEV 语义地图强近似。**

### 3. CN113052152B：一种基于视觉的室内语义地图构建方法、装置及设备

公开内容：
- 通过相机获取室内环境图像数据流，包含色彩图、深度图、相机位姿信息、相机参数信息、无人平台运动轨迹和姿态数据。
- 通过视觉 SLAM 构建室内度量图。
- 将室内环境划分为独立房间，得到区域划分层。
- 对不同区域识别场景类别标签，并结合区域划分层构建房间拓扑图。
- 由单张图像得到局部场景图，由同一区域多张局部场景图得到区域全局场景图。
- 将对象、对象类别、对象属性、对象间位置关系标注到度量图上，得到对象层。
- 将度量图、对象层、区域划分层和房间拓扑图逐层级联，完成室内语义地图构建。

攻击点：
- 这是本轮最接近 P010 “空间语义信息”部分的文献。
- 它已经公开区域划分、场景类别、房间拓扑、对象属性、对象间位置关系、局部/全局场景图、层级语义地图。
- 请求人可能主张：P010 的“语义区域集合、空间关系集合”已由 CN113052152B 的区域划分层、房间拓扑图、对象层和场景图公开。

防御结论：
- CN113052152B 是无人平台/室内机器人在线构建语义地图，依赖视觉 SLAM、深度图、位姿、运动轨迹和姿态数据。
- 其最终成果是多层级联的室内语义地图，用于无人平台环境理解、检索、匹配、推理、路径规划。
- 它没有把该语义地图进一步组织为“空间语义先验”这一可被端侧空间使用主体加载的数据结构。
- 它没有公开“端侧转换信息”，也没有公开端侧主体将该先验转换为主体侧空间认知模型。
- 它解决的是语义地图环境表达效果不足，不是端侧空间使用主体进入、呈现或使用目标空间前的冷启动认知问题。
- 分类：**红档生成/组织侧近似，但不覆盖使用侧内化。**

### 4. CN118298231A：一种基于语义关系图谱的室内多场景语义地图构建方法

公开内容：
- 基于激光雷达构建 2D 栅格地图。
- 使用双目相机获取图像数据，结合贝叶斯滤波和先验信息进行场景分类，获得场景语义标签。
- 检测和定位地标物体，获得地标语义标签。
- 基于 2D 栅格地图，将场景语义标签和地标语义标签构建为语义地图。
- 基于场景语义标签和地标语义标签，按三元组构建空间知识图谱。
- 利用目标检测数据集和常识信息构建先验关系图谱。
- 将空间知识图谱和先验关系图谱融合，形成基于语义关系图谱的多场景语义地图。

攻击点：
- 它已经公开“空间拓扑图 + 三元组 + 空间关系知识图谱 + 先验关系图谱 + 语义地图”。
- 容易被用于攻击 P010 中“空间语义先验”“语义区域之间的空间关系”“三元结构”等表达。

防御结论：
- 其先验关系图谱来自目标检测数据集、常识信息和物体共现统计，不是基于目标空间外部视觉输入生成的端侧可加载空间语义先验。
- 其技术目的为机器人目标搜索，使用地标和目标物体关系推断目标位置概率。
- 其输出仍是机器人语义图谱地图，不是用于端侧空间使用主体加载和内化的数据接口。
- 没有公开主体侧空间认知模型；图谱用于搜索任务，不是端侧主体内部的空间区域表、空间行动图、任务可达图等。
- 分类：**橙偏红，组织侧/先验关系图谱强近似。**

### 5. CN118999525A：一种视觉语义地图生成方法、装置、电子设备及存储介质

公开内容：
- 对二维图像进行语义分割，得到预设类型对象的掩膜图和像素边缘点。
- 选取像素边缘点的西格玛点，并转换到三维坐标系。
- 计算像素边缘点在三维坐标系中的均值坐标和协方差。
- 将建图区域划分为单位网格，根据建图点的坐标和协方差生成视觉语义地图。
- 后续可用视觉语义地图与待定位图像边缘点配准，确定位姿。

攻击点：
- 可攻击“二维图像语义分割 -> 三维/世界坐标 -> 视觉语义地图”。

防御结论：
- 该文献的核心是通过西格玛点和协方差传播提高视觉语义地图精度，用于视觉定位。
- 语义信息主要是预设类型对象的边缘点/地图点，不涉及区域功能属性、区域间空间关系、空间语义先验或端侧内化。
- 分类：**橙/黄，生成侧背景文献。**

- CN118298 的“先验”是常识/数据集共现先验，用于目标搜索，不是目标空间外部视觉输入所生成的空间语义先验。

### 组合二：EP4115318A1 / Qualcomm 同族 + CN113052152B

### 组合三：CN118298231A + CN121804500A

- P010 的技术问题不是“如何更快找到目标”，而是“如何让端侧空间使用主体在进入、呈现或使用目标空间前快速形成可调用的空间认知模型”。

## 四、对权利要求的影响

当前不建议因为最后一轮全文直接放弃三柱 10 条结构，也不建议立刻把独权改窄。

但 CN113052152B 和 CN118298231A 说明：如果后续审查员把生成侧独权7抓得很紧，最有效的备选收窄不是加入机器人、SLAM、BEV、NeRF 等技术细节，而是把下面这一组上提：

> 所述空间语义先验包括语义区域集合、空间关系集合以及端侧转换信息。

理由：
- “语义区域集合 + 空间关系集合”能承接 CN113/CN118298 中已有的区域、拓扑、关系图谱；
- “端侧转换信息”是这批文献普遍缺失的接口特征；
- 这样不会把母案锁死到机器人或具体建图算法上。

## 五、说明书必须补强的区别表述

建议在说明书无效防御或背景区别段中加入：

> 现有室内语义地图可以包括度量图、区域划分层、对象层、房间拓扑图、场景图或语义关系图谱，用于机器人导航、目标搜索、路径规划、语义检索或环境推理。本发明不以构建供机器人在线使用的语义地图为终点，而是将由目标空间外部视觉输入得到的空间语义信息进一步组织为空间语义先验，使其成为可由端侧空间使用主体加载并转换的数据结构。

> 现有语义关系图谱中的先验关系通常来源于通用数据集、物体共现统计或常识信息，用于推断目标物体可能位置。本发明中的空间语义先验来源于目标空间的外部视觉输入，反映目标空间自身的区域功能属性和空间关系，并包含端侧转换信息，以支持端侧主体形成主体侧空间认知模型。

> 端侧加载空间语义先验不同于读取、查询或渲染语义地图。端侧内化是指端侧空间使用主体将空间语义先验转换为其内部可调用的空间区域表、空间行动图、语义拓扑图、任务可达图、风险区域表、可挂载表面表、交互区域表或局部环境摘要。

## 六、最终检索收束判断

- CN113052152B：室内视觉语义地图层级结构；
- CN118298231A：空间知识图谱 + 先验关系图谱；

2. 在说明书中厚写“空间语义先验”和“主体侧空间认知模型”的区别；
3. 预留审查阶段备选收窄方案。
