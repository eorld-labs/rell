# P010 窄口径补刀检索结论

## 一、结论先行

本轮窄口径检索没有发现单篇直接覆盖 P010 三柱链路的文献。

尤其是以下组合式检索结果较弱：

- "generate semantic map" AND "load semantic map" AND (robot OR agent OR device)：无有效结果。
- "external visual input" AND "semantic map" AND (robot OR agent OR device)：无有效结果。
- "scene prior" AND "cognitive map" AND ("embodied AI" OR robot)：无有效结果。
- "spatial semantic prior" / "semantic spatial prior" / "spatial semantics prior"：仅命中视觉深度估计中的空间语义先验特征增强，不是 P010 的空间语义先验接口。

因此，当前没有因为本轮检索而放弃 P010 的理由。真正风险仍然是分段近似组合：

1. 视觉/BEV/单目图像生成语义地图；
2. 机器人加载、查询或使用语义地图进行导航；
3. 数字孪生、空间拓扑、认知数字孪生中的空间语义表达。

P010 的核心防线仍是：

外部视觉输入数据 -> 目标空间空间语义信息 -> 可加载的空间语义先验 -> 端侧转换为主体侧空间认知模型。

## 二、需优先拉全文的文献

### A-档：建议优先拉全文

1. WO2025136997A1 / US20250209724A1 / US12620164B2  
   Using neural radiance fields for label efficient image processing  
   命中点：apparatus to generate a semantic map from one or more images。  
   判断：对“从一个或多个图像生成语义地图”的生成侧有直接近似，应确认其 semantic map 的结构、用途和是否有端侧加载。

2. EP4115318A1  
   System and method for predicting a map from an image  
   命中点：从图像特征预测地图。  
   判断：可能接近视觉输入到地图/语义地图生成，需确认是否仅自动驾驶/地图预测，还是有空间语义区域和关系输出。

3. CN118730072A  
   语义地图生成方法、装置、设备及计算机可读存储介质  
   命中点：机器人相机图像 + 相机参数/第一位姿 + BEV 语义分割 + 局部/全局语义地图。  
   判断：已分析，橙偏红；对生成侧较强，但未公开空间语义先验和端侧内化。

4. CN113052152B  
   一种基于视觉的室内语义地图构建方法、装置及设备  
   命中点：室内环境相机图像流，包含彩色图、深度图、相机位姿信息、相机参数信息。  
   判断：与“图像 + 位姿/参数 -> 室内语义地图”接近，应拉全文确认是否已有区域功能属性或空间关系图。

5. CN118298231A  
   基于语义关系图的室内多场景语义地图构建方法  
   命中点：空间拓扑图、空间关系知识图谱、语义关系图。  
   判断：对 P010 的“语义区域之间的空间关系”和“语义拓扑图”从权有近似，应全文确认其关系图是否仅用于建图，还是存在可加载先验。

## 三、各检索组判断

### 1. BEV / bird's eye view / semantic map 组

该组命中大量自动驾驶、机器人和 BEV 感知文献。风险集中在“视觉输入生成语义地图”这一段。CN118730072A 不是孤例，后续还需要把 Qualcomm 的 WO2025136997A1/US20250209724A1、EP4115318A1、CN113052152B 等拉全文。

防御边界：

- BEV 语义分割结果不是空间语义先验；
- 语义地图不是空间语义先验；
- 从图像生成地图不是本案终点；
- 本案强调将空间语义信息组织为端侧可加载、可转换的数据结构。

### 2. preload / preloaded / map loading 组

该组没有命中“预加载语义地图并内化为主体认知模型”的强文献。命中结果多为：

- 自动驾驶或泊车中的预加载地图；
- 子地图加载到内存；
- BIM 模型流式预加载；
- AR/XR 房间空间映射；
- 语义地图分片或缓存。

防御边界：

- 加载地图不等于端侧内化；
- 预加载地图不等于空间语义先验；
- 子地图缓存、BIM 流式加载或停车场预加载地图，技术目的仍是定位、渲染或导航，不是让端侧空间使用主体形成主体侧空间认知模型。

### 3. prior / scene prior / cognitive map / internal representation 组

该组结果整体偏弱，没有直接命中“spatial semantic prior + cognitive map + embodied agent”的强组合。较近的是 CN115311538A 的 scene prior 目标搜索，以及 KR 文献中的 cognitive digital twin / spatial semantic information。

防御边界：

- scene prior 多为目标搜索或导航中的任务先验，不是目标空间外部视觉输入生成的空间语义先验；
- cognitive digital twin 多为数字孪生监控/事件推理，不是端侧主体把先验转换为内部可调用空间认知模型；
- internal representation 检索结果多跑偏到广告、知识库、模型压缩等领域。

## 四、对权利要求和说明书的影响

当前三柱 10 条结构可以继续保留，不建议因为本轮检索改成窄案。

但有两点需要在说明书中继续压实：

1. 明确“空间语义先验”不是语义地图、BEV 分割图、三维场景图、HD map 或局部/全局导航地图，而是包括语义区域集合、空间关系集合以及端侧转换信息的可加载数据结构。

2. 明确“端侧内化”不是加载、查询、缓存或渲染地图，而是端侧空间使用主体将空间语义先验转换为内部可调用的主体侧空间认知模型，例如空间区域表、空间行动图、语义拓扑图、任务可达图、风险区域表、可挂载表面表、交互区域表或局部环境摘要。

## 五、当前建议

1. 不继续大规模泛检。
2. 拉取 A-档 5 篇全文，确认是否有更接近 CN118730072A 的生成侧文献。
3. 如果 A-档全文中没有出现“空间语义先验 + 端侧转换信息 + 主体侧空间认知模型”，则可以基本收束检索。
4. 权利要求暂不收窄；但保留一个审查阶段备选方案：必要时将“语义区域集合、空间关系集合以及端侧转换信息”上提到生成侧独权。
