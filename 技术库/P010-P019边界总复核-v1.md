# P010-P019 技术边界总复核 v1

## 一、复核范围与结论

本复核以 P010-P019 当前技术库中的正式权利要求、立项书和已确认分案边界为依据，检查各案是否在输入、核心处理、输出、运行时位置和验证责任上发生不可接受的重叠。

总体结论：

1. P010-P019 可以组成连续技术链，没有发现两案独立权利要求完全相同的致命冲突；
2. 多个方案共享“经验、概念、事实、运行时快照、迁移”等基础对象，但共享数据结构不等于共享发明步骤；
3. 当前需要收紧的是功能边界和撰写表述，尤其是 P014 的执行主体转换、P017/P018 的运行时快照以及 P019/P012 的概念生命周期；
4. P020 已被定位为“原始教学输入到首次可信经验”，不应再把 P012 的多经验概念归纳或 P017 的跨空间跨本体迁移写回其独权主链。

## 二、总流通链

```text
P010：从外部视觉输入生成空间语义先验并在端侧形成空间认知模型
  -> P011：基于真实交互经验类比预测动作后果并反馈经验记录
  -> P012：从多条经验记录归纳概念标签和典型经验模式
  -> P013：基于人类意图和任务情境类比形成任务计划
  -> P014：执行中检测偏离并检索/切换补救动作
  -> P015：在人类约束可行的候选动作中学习和调用个体偏好
  -> P016：将物理过程模板实例化、执行阶段跃迁并双通道验真
  -> P017：将既有不变量经验适配到新空间、新本体和当前任务
  -> P018：在任务执行期间对状态查询、控制、教学、中断和切换进行仲裁
  -> P019：在端侧形成、落地、复用和回退概念候选
  -> P020：从多源教学材料编译候选经验并通过自主复做首次晋级
```

P020 与 P010-P019 的关系不是简单的后置模块：它是教学输入入口，形成的可信经验再进入 P012/P017/P019 等后续层；P016 和 P018 负责 P020 的验真与仲裁边界。

## 四、四个重点压力点

### 4.1 P010 与 P017/P018

P010 权利要求 7 使用了“控制导航、交互触发和任务执行行为”的表述，容易被误读为执行控制案。

建议解释为：P010 保护空间语义先验和主体侧空间认知模型“可被运行时调用”的能力；具体任务计划、跨本体适配、运行时仲裁和物理执行分别由 P013、P017、P018、P016 承担。

### 4.2 P013 与 P014

P013 权利要求 3 是任务计划中的某一步执行失败后，检索替代步骤或参数，并且不重新形成全部任务计划；P014 独权 1 是动作正在执行过程中检测偏离，并在不中断执行链路的情况下切换补救动作。

二者可按时间和对象区分：

- P013：步骤级计划替换，通常在一步失败或结果已知后发生；
- P014：动作级中途偏离恢复，强调当前动作执行链连续性和不停机切换。

若 P013 说明书把“执行中即时切换”写得过宽，会压缩 P014；若 P014 从权 5 把一般主体能力转换写得过宽，会压缩 P017。

### 4.3 P016 与 P017/P018

三案都使用因果事实和运行时快照，但功能不同：

- P016：过程模板如何以连续状态变量推进阶段，并用独立通道确认目标事实；
- P017：已有不变量经验如何在当前空间、本体和快照中绑定并判断能否迁移；
- P018：交互输入改变任务控制前如何读取快照并决定继续、暂停、切换或澄清。

运行时快照可以是共享基础设施对象，但权利要求的触发条件和处理结果不同，不应仅因字段相同就合并方案。

### 4.4 P012 与 P019/P020

- P012 的概念来自多条经验记录之间的共同情境、动作和后果模式，并用于跨域复用；
- P019 的概念是端侧高频交互概念候选，必须绑定当前任务快照并回到仲裁层；
- P020 的概念是与当前教学证据绑定的契约候选，用于首次经验编译和复做，不主张多经验公共归纳。

“概念”一词相同不构成冲突，关键是形成依据、生命周期和执行权限不同。

## 五、必须保留的接口边界

### 5.1 P010 -> P013/P017/P018

P010 输出空间区域、对象关系、通行/阻挡/绕行/交互可达等语义数据；P013 使用其形成任务计划，P017 使用其进行迁移绑定，P018 使用其生成运行时快照和仲裁依据。

### 5.2 P011/P020 -> P012/P019

P011 和 P020 可以产出交互经验或教学候选；P012 在多条可信经验上归纳公共概念，P019 在端侧对概念候选进行形成、落地、复用和回退。

### 5.3 P012/P017/P019 -> P016

这些方案只输出概念、经验、绑定候选或过程候选；P016 才将物理过程实例化并提交验真事实。

### 5.4 所有输入 -> P018

自然语言、教学输入、偏好、恢复候选、概念候选和迁移候选都不得直接改变执行控制，必须先进入 P018 状态优先仲裁。

## 六、需要收紧的三处表述

1. **P014 权利要求 5**：建议把“动作参数、执行方式或适用条件转换”限定为补救动作在能力变化下的准入调整，不要写成一般跨本体经验迁移。
2. **P013 说明书**：明确步骤替换发生在任务计划层，不能覆盖 P014 的执行中偏离和不停机切换。
3. **P010 权利要求 7**：在说明书中明确“调用空间模型”不等于取得执行权，具体执行仍由任务计划、P017 适配、P018 仲裁和 P016 验真完成。

## 七、复核结论

当前 P010-P019 的分工可以成立：

```text
空间从哪里来       P010
经验后果怎么预测   P011
经验如何抽象成概念 P012
任务如何形成计划   P013
执行中如何补救     P014
人类偏好如何约束   P015
物理过程如何验真   P016
既有经验如何迁移   P017
运行时输入如何仲裁 P018
端侧概念如何内化   P019
教学如何形成首份经验 P020
```

真正需要持续防守的不是“所有方案都不能共享字段”，而是：任何一个分案都不能越过自己的输入起点、核心处理和输出责任去主张另一案的完整主链。
