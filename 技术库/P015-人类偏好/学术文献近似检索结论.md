# P015 学术文献近似检索结论

## 一、检索口径

本轮检索围绕当前收束后的P015权利要求进行，重点不再检索泛泛的“human preference”或“RLHF”，而是拆成以下同义骨架：

1. state-action preference / action-preference function / evaluative feedback
2. human feedback after action / post-action feedback / preference drift
3. feasible actions / constraints / soft planning constraints / safe and preferable robot interaction
4. personalized agent memory / per-user memory / tool-calling constraints / preference-aware infeasibility
5. activity selection / social robot / explicit and implicit feedback

判断标准为：是否公开“动作执行后获取人类反馈→形成情境-动作粒度偏好记录→独立存储→后续动作确定时先经约束可行性限定、再按偏好确定动作”的完整组合。

## 二、高价值近似文献

### 1. Learning Personalized Agents from Human Feedback（PAHF）

链接：https://arxiv.org/html/2602.16173v1

判断：高价值近似，数字Agent侧最接近。

该文提出个性化Agent通过显式的per-user memory从pre-action和post-action feedback中持续学习。其post-action feedback用于纠正错误、适应偏好漂移，并更新显式记忆。

接近点：

- 明确有post-action feedback。
- 明确有per-user memory。
- 明确处理preference drift。
- 明确用于后续Agent行为调整。

区别点：

- 其记忆更接近用户偏好画像或偏好profile，不明确形成{当前情境，动作，人类偏好指示}的结构化偏好记录。
- 未强调偏好记录独立于奖励模型、策略模型和约束记录。
- 未公开“约束可行性判断先限定候选动作，偏好只在满足约束条件的候选动作中确定动作”。

结论：该文对P015数字Agent方向有明显解释压力。P015说明书应当强调偏好不是泛用户画像，而是情境-动作粒度的独立偏好记录；并强调约束在先、偏好在约束可行范围内生效。

### 2. Learning Human Preferences Over Robot Behavior as Soft Planning Constraints

链接：https://arxiv.org/abs/2403.19795

判断：高价值近似，物理机器人规划侧接近。

该文将人类偏好建模为soft planning constraints，区分required behavior和desired behavior，避免把任务成功要求和用户偏好混成一个标量奖励函数。

接近点：

- 直接区分必要约束和偏好性行为。
- 与P015“约束在先、偏好在约束范围内生效”的思想方向接近。
- 应用于机器人行为规划。

区别点：

- 其偏好以soft planning constraints进入规划框架，不是执行动作后形成情境-动作粒度偏好记录。
- 偏好主要通过查询选择潜在机器人行为学习，不是基于真实动作执行后反馈持续积累。
- 未公开多主人隔离、沉默反馈、时间衰减、人-主体关系连续性。

结论：该文可能成为审查员或第三方论证“约束与偏好分层”已知的材料。P015应把创造性落在“动作后反馈生成独立偏好记录+后续调用”上，而不是单独落在“偏好不同于硬约束”上。

### 3. Text2Interaction: Establishing Safe and Preferable Human-Robot Interaction

链接：https://arxiv.org/abs/2408.06105

判断：高价值近似，安全与偏好联合规划侧接近。

该文使用LLM生成任务计划、motion preferences和安全控制器参数，以同时满足任务完成和用户满意度，避免把任务成功和用户偏好简单加权为奖励。

接近点：

- 同时考虑安全/任务完成与人类偏好。
- 偏好可转化为运动层或控制层条件。
- 面向机器人真实执行。

区别点：

- 偏好主要来自自然语言输入并转成规划/控制代码，偏零样本即时生成。
- 未形成动作执行后的情境-动作偏好记录。
- 未公开偏好记录的长期独立存储、多人隔离、时间衰减和主体连续性。

结论：可作为P015背景技术中的“现有方案将偏好即时转化为规划或控制条件”的代表。P015区别在长期偏好记录层。

### 4. AWARE-US: Preference-Aware Infeasibility Resolution in Tool-Calling Agents

链接：https://arxiv.org/abs/2601.02643

判断：高价值近似，数字工具调用Agent侧接近。

该文处理工具调用Agent中的不可行查询：当约束导致无结果时，Agent根据用户偏好推断哪些约束最不重要并进行放松。

接近点：

- 明确是数字Agent和工具调用场景。
- 明确将infeasibility与preference-aware处理结合。
- 体现“约束不可行时结合偏好处理”的技术方向。

区别点：

- 其核心是查询修复和约束放松，不是动作执行后形成偏好记录。
- 偏好来自对话中的约束重要性推断，不是情境-动作粒度的后验偏好记录。
- 未公开“先对候选动作做可行性限定，再在满足约束条件的动作中按偏好确定动作”的通用执行闭环。

结论：该文说明P015必须保留数字世界覆盖。说明书中应当将“约束可行性”明确扩展为权限、API条件、业务流程、数据完整性、合规策略等数字约束。

### 5. Personalizing Activity Selection in Assistive Social Robots from Explicit and Implicit User Feedback

链接：https://link.springer.com/article/10.1007/s12369-024-01124-2

判断：中高价值近似，后执行反馈和个体偏好侧接近。

该文在社交机器人活动选择中，使用活动执行后的显性评分和隐性反馈更新用户偏好，使机器人更频繁选择用户偏好的活动。

接近点：

- 明确“活动执行后”获取显性/隐性反馈。
- 反馈用于更新用户偏好。
- 后续活动选择受偏好影响。

区别点：

- 偏好落点为活动Q值或活动偏好，不是通用{情境，动作，人类偏好指示}记录。
- 未强调约束可行性在先。
- 未覆盖物理动作、数字动作和混合执行主体的通用动作确定。

结论：该文对“动作后反馈更新偏好”单点有压力，但不足以覆盖P015的约束先行与独立偏好记录架构。

### 6. Personalized Modeling for Human-Robot Collaborative Manipulation

链接：https://www2.eecs.berkeley.edu/Pubs/TechRpts/2019/EECS-2019-153.pdf

判断：中高价值近似，可作为“可行动作中的偏好”背景文献。

该博士论文讨论协作机器人理解人的动作可行性和人体工学偏好，并提出预测人对不同可行动作偏好的代价函数，且可在线学习个体偏好。

接近点：

- 明确提到feasible actions中的偏好。
- 涉及个体化在线学习。
- 面向人机协作。

区别点：

- 偏好以人体工学代价函数形式存在，主要用于协作动作规划。
- 未公开动作执行后形成独立偏好记录。
- 未公开偏好记录与约束记录、物理经验记录、模型参数的独立存储边界。

结论：该文会削弱“可行动作中按偏好选择”本身的新颖性，P015不能只靠这一点；必须强调记录结构和动作后反馈闭环。

### 7. In-situ Value-aligned Human-Robot Interactions with Physical Constraints

链接：https://arxiv.org/html/2508.07606v2

判断：中高价值近似，物理约束+人类偏好融合侧接近。

该文将物理反馈、物理约束和人类偏好纳入LLM规划循环，使生成计划符合物理约束和人类偏好。

接近点：

- 有物理约束和人类偏好共同作用。
- 有动作执行后的物理反馈进入后续规划。
- 面向具身机器人。

区别点：

- 反馈主要进入LLM规划器和文本奖励形式。
- 未形成情境-动作-偏好指示的独立偏好记录。
- 未强调约束可行性限定与偏好记录调用的结构性分层。

结论：可作为“现有方案将物理约束与人类偏好共同输入规划器”的背景文献。

## 三、通用背景文献

### 1. A Survey of Reinforcement Learning from Human Feedback

链接：https://arxiv.org/html/2312.14925v1

该综述说明RLHF/PbRL的基本路线是从人类定性反馈推断目标或奖励，包括行为之间或给定状态下动作之间的偏好。该文属于基础背景，不直接公开P015组合。

### 2. Reinforcement Learning With Human Advice: A Survey

链接：https://www.frontiersin.org/journals/robotics-and-ai/articles/10.3389/frobt.2021.584075/full

该综述提到evaluative feedback可作为动作价值或action-preference function，并可把建议表达为动作偏好约束。该文会构成“动作偏好反馈”公知背景，但不覆盖P015的独立偏好记录和约束可行范围内调用。

## 四、综合判断

学术侧不存在“完全空白”。相反，以下局部要素均已有较多研究：

1. 人类反馈用于奖励模型或策略模型训练。
2. 给定状态或轨迹的人类偏好学习。
3. 偏好作为soft constraints或motion preferences进入机器人规划。
4. 后执行反馈用于更新用户偏好或个性化记忆。
5. 数字工具调用Agent中基于偏好处理不可行约束。
6. 可行动作中根据人体工学或用户偏好进行选择。

但当前未发现完整公开P015收束版组合的文献：

执行动作后获取人类反馈；基于反馈形成当前情境下该动作的人类偏好指示；将{当前情境，动作，人类偏好指示}作为独立偏好记录存储；后续动作确定时先基于约束可行性判断对候选动作进行可行性限定，再基于匹配的人类偏好指示在满足约束可行性条件的候选动作中确定待执行动作。

## 五、对权利要求和说明书的建议

1. 独权当前“约束可行性判断对候选动作进行可行性限定，偏好指示在满足约束条件的候选动作中确定动作”的表述应保留。

2. 说明书应明确“约束可行性”包括物理约束和数字约束。物理约束包括碰撞、力矩、路径、空间安全、对象状态；数字约束包括权限、API前置条件、业务流程、合规规则、数据完整性、不可逆操作确认。

3. 说明书应明确偏好记录不是用户画像、偏好profile、奖励模型参数、策略模型参数、软约束代码或提示词，而是可独立检索和更新的情境-动作粒度记录。

4. 对PAHF类文献，应重点区分“per-user memory/profile”与“{情境，动作，人类偏好指示}偏好记录”的结构差异。

5. 对Text2Interaction和soft planning constraints类文献，应重点区分“偏好即时进入规划/控制条件”与“执行后反馈形成可复用偏好记录”的时序差异。

6. 对AWARE-US类文献，应重点区分“不可行查询的约束放松”与“后续动作确定中的约束可行性限定+偏好记录调用”。

7. 对社交机器人activity selection类文献，应重点区分“活动Q值更新”与“跨物理/数字执行主体的情境-动作偏好记录”。
