# P018 英文专利库补强检索结果-v1

检索时间：2026-07-11

检索口径：专利库英文补强导出，按 P018 主链逐项筛选

## 一、结论

本轮英文专利库补强检索显示：单件文献尚未完整覆盖 P018 的整条主链。

P018 的核心组合仍然是：

`交互输入 -> 读取任务期运行时世界状态快照 -> 输入分流 -> 任务控制仲裁 -> 基于当前事实续推 -> 执行控制衔接 -> 快照释放`

英文专利文献中可以分别找到以下单点背景：

1. 任务中断与恢复。
2. 当前状态 / context snapshot / state snapshot。
3. 任务切换与 task progress。
4. 语音或自然语言输入控制机器人。
5. 工作流跳步、resume、checkpoint。
6. 持物安全、人工确认、异常接管。

但没有看到单件专利把上述要素同时组织成：

1. 任务执行期间先读同一任务期快照。
2. 以该快照同时做输入分流、任务控制仲裁、当前事实裁剪和快照释放。
3. 状态查询不改变执行控制链路。
4. 基于“当前事实是否成立”而不是“节点是否 done”裁剪步骤。

## 四、对 P018 的专利风险判断

### 1. 新颖性风险

中低。

原因是：

1. 没有单件文献看到完整主链。
2. 没有单件文献同时公开输入分流、任务控制仲裁、当前事实裁剪和快照释放。

### 2. 创造性风险

中。

原因是：

1. 审查员很容易把以下材料拼起来：
   - 状态快照 / context snapshot
   - task interruption / resume
   - current world state / current environment state
   - workflow checkpoint / step skipping
   - voice command / query-command split
2. 但要拼成 P018，仍然缺关键桥：
   - 状态查询不触发执行控制链路改变。
   - 任务控制必须先过同一任务期快照仲裁。
   - 裁剪依据是当前事实，不是节点 done。
   - 快照是任务期临时体，任务结束要释放。

### 3. 最危险的组合方式

最像审查员拼接的组合是：

1. `CN121374575A` 的 context snapshot + interrupt instruction。
2. `CN121050946A` 的 state checkpoint + jump from current step。
3. `US11768504B2` 的 interrupt service request + task preemption。
4. `US11348269B1` 的 resume uncovered areas。
5. `CN121672073B` 的 system state snapshot。
6. `CN121365683A` 的 instruction/state-data interface。

即便这样拼，仍然缺少 P018 的核心统一入口和事实裁剪逻辑。

## 五、建议保留的答辩切割点

1. 状态查询输入只返回快照信息，不改变执行控制链路。
2. 任务控制输入必须先经过任务期运行时世界状态快照仲裁。
3. 后续待执行步骤集合基于当前事实成立与否裁剪，不是按 workflow done 跳过。
4. 快照只在任务期存在，任务完成或取消后释放，防止状态污染。
5. 持物或资源占用场景下允许人工确认、释放或暂停切换。

## 六、结论

本轮英文专利库补强检索的作用不是推翻 P018，而是证明：

1. 单点背景很多。
2. 主链组合仍然稀缺。
3. 只要把“状态优先仲裁闭环”守住，P018 的母案支撑仍然成立。
