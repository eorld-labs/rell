# P018 英文近似检索加强计划

检索对象：一种面向自动化执行体的状态优先任务交互仲裁方法及系统。

检索日期：2026-07-11。

本计划用于英文专利、论文、学术库和技术文献的近似检索。中文检索由另一条线进行，本文件只覆盖英文检索方向。

## 一、核心技术方案拆解

P018 的主链不是普通的自然语言任务理解，也不是普通任务规划，而是：

1. 自动化执行体处于任务执行期间；
2. 新交互输入到来；
3. 在改变执行控制行为之前，先读取当前任务期运行时世界状态快照；
4. 区分状态查询输入和任务控制输入；
5. 对任务控制输入进行运行时仲裁，输出继续、暂停、暂停并切换、合并、终止或人工确认；
6. 允许继续或切换时，基于当前事实裁剪已满足、已完成或目标事实已成立的步骤；
7. 从当前世界状态续推执行，而不是从任务初始状态重跑；
8. 任务结束、取消或终止后释放任务期快照。

检索时不要只搜自创表达 `state-first arbitration` 或 `runtime world state snapshot`，应拆成行业常见词：

- robot task interruption / task resumption
- task switching / task arbitration
- current state / world state / task state
- workflow state / checkpoint / resume
- human-robot instruction correction
- dialog state / task monitor
- autonomous agent workflow / human-in-the-loop / user input node

### 0.1 检索替换规则

当某一组检索结果过少时，优先按以下方向替换：

1. `arbitration` 替换为 `coordination`、`mediation`、`management`、`control`、`interruption handling`。
2. `snapshot` 替换为 `runtime state`、`task state`、`context`、`working memory`、`belief state`、`blackboard`。
3. `task control input` 替换为 `user command`、`human instruction`、`intervention`、`correction`、`new instruction`。
4. `trim completed steps` 替换为 `skip completed steps`、`prune actions`、`resume from current state`、`plan repair`。
5. `automated executor` 替换为 `robotic system`、`autonomous agent`、`AI agent`、`software agent`、`embodied agent`。

当某一组检索结果过多时，优先加入以下锚点：

1. 机器人方向加入：`robot`、`robotic system`、`service robot`、`manipulation`。
2. 数字执行体方向加入：`AI agent`、`workflow`、`checkpoint`、`human-in-the-loop`。
3. 仲裁方向加入：`interruption`、`task switching`、`resume`、`user instruction`。
4. 状态方向加入：`current state`、`runtime state`、`world state`、`task progress`。

## 四、初步命中与风险判断

### 1. US12240120B2 - Method of operating one or more robots

来源：Google Patents。

链接：https://patents.google.com/patent/US12240120

要点：机器人执行第一任务时，收到停止该任务并执行附加任务的指令；附加任务完成后，机器人回到第一任务的位置并继续第一任务。

风险等级：中高。

相近点：

- 涉及机器人任务中断；
- 涉及附加任务；
- 涉及回到原任务继续执行。

区别点：

- 当前摘要更偏“停止第一任务、执行附加任务、回到原任务位置继续”；
- 尚未看到“新输入必须先读取任务期运行时世界状态快照”；
- 尚未看到把输入区分为状态查询输入/任务控制输入；
- 尚未看到基于当前事实裁剪已满足步骤；
- 尚未看到状态查询直接从快照返回。

对 P018 的影响：需要在说明书中强调 P018 不是单纯任务打断后回到原位置，而是所有交互输入进入执行控制前都以当前任务期世界状态快照为第一判断依据。

### 2. Interruptible Task Execution with Resumption in Golog

来源：ECAI 2016 / BibBase。

链接：https://bibbase.org/network/publication/gierse-niemueller-claen-lakemeyer-interruptibletaskexecutionwithresumptioningolog-2016

要点：提出 IntRGolog，扩展 Golog，使机器人能够中断任务、恢复任务，并通过新构造确定和满足任务需求。

风险等级：中高。

相近点：

- 机器人任务中断；
- 恢复任务；
- 恢复时可能需要中间步骤。

区别点：

- 侧重逻辑程序语言 Golog 的可中断执行结构；
- 不以人机交互输入的运行时仲裁为主链；
- 未见“状态查询输入”和“任务控制输入”分流；
- 未见“任务期运行时世界状态快照”的生命周期隔离和释放；
- 未见“基于当前事实删除已满足步骤”的专门交互仲裁机制。

对 P018 的影响：权利要求 1 中“状态查询输入/任务控制输入分流”和“基于快照裁剪后续待执行步骤集合”要保留，不建议再放宽。

### 3. Robot Task Interruption by Learning to Switch Among Multiple Models

来源：IJCAI 2018。

链接：https://www.ijcai.org/proceedings/2018/686

要点：移动机器人基于观察在多个任务模型之间切换，使机器人能响应环境中的人和对象。

风险等级：中。

相近点：

- 任务中断；
- 任务切换；
- 基于观察进行切换。

区别点：

- 核心是基于多个任务模型学习任务选择策略；
- 不是用户交互输入触发的状态优先仲裁；
- 不覆盖状态查询和任务控制输入分流；
- 不强调当前快照作为改变执行控制行为前的第一门槛。

对 P018 的影响：说明书中应区分“基于效用/策略的任务模型切换”和“交互输入进入执行系统前的状态优先仲裁”。

### 4. US20180178379A1 - Task planning device

来源：Google Patents。

链接：https://patents.google.com/patent/US20180178379A1

要点：任务规划设备包括模拟器、系统状态获取单元和任务计划生成单元，基于获取的状态和用户指令动态生成机器人系统任务计划。

风险等级：中。

相近点：

- 获取系统状态；
- 基于状态和用户任务指令生成任务计划；
- 机器人系统/虚拟空间。

区别点：

- 主体是任务计划生成；
- 没有任务执行期间的新交互输入仲裁；
- 没有状态查询/任务控制输入分流；
- 没有暂停、切换、合并、终止、人工确认的仲裁结果集合；
- 没有基于当前事实裁剪后续步骤集合。

对 P018 的影响：背景技术可把这类方案归为“基于状态的动态任务规划”，并强调 P018 解决的是任务执行期间交互输入进入控制链路前的仲裁。

### 5. US11086938B2 - Interpreting human-robot instructions

来源：Google Patents。

链接：https://patents.google.com/patent/US11086938B2/en

风险等级：中。

相近点：

- 自然语言机器人指令；
- 层级规划；
- 状态和动作抽象。

区别点：

- 核心是语言命令解释和奖励函数映射；
- 不主张任务执行期间的新输入仲裁；
- 不主张状态查询直接读运行时快照；
- 不主张基于当前事实裁剪已完成步骤。

对 P018 的影响：说明书中应坚持“语义理解只是入口，执行控制必须回到状态优先仲裁”，不要把主案写成自然语言控制机器人。

### 6. US20190339693A1 - Autonomous robot with on demand teleoperation

来源：Google Patents。

链接：https://patents.google.com/patent/US20190339693A1/en

要点：机器人自主执行任务；当确定没有可用策略执行下一任务时，进入人工介入/远程操作模式。

风险等级：中。

相近点：

- 自主机器人；
- 无策略时请求人工介入；
- 远程操作和恢复自主控制。

区别点：

- 核心是自主模式与人工介入模式切换；
- 不涉及所有交互输入统一进入状态优先仲裁；
- 不涉及状态查询输入；
- 不涉及基于当前事实裁剪续推。

对 P018 的影响：权利要求中“请求人工确认”只是仲裁结果之一，不能让主案被理解为单纯人工接管/远程操作。

### 7. US20160042735A1 - Dialog Flow Management In Hierarchical Task Dialogs

来源：Google Patents。

链接：https://patents.google.com/patent/US20160042735A1/en

要点：层级任务对话管理中，task monitor 可查询任务状态，例如 created、active、suspended、finished，并返回任务执行状态。

风险等级：中。

相近点：

- 对话系统；
- 任务监控；
- 当前任务状态；
- 活动/挂起/完成等状态。

区别点：

- 偏对话任务状态管理；
- 不聚焦机器人或自动化执行体的真实执行控制；
- 不主张状态快照作为改变执行控制行为前的第一判断依据；
- 不主张基于当前事实裁剪任务步骤。

对 P018 的影响：状态查询从权要保留“直接读取任务期运行时世界状态快照中的对应事实或字段”，以区别普通对话状态监控。

## 五、初步结论

目前英文初检没有看到一个单件同时完整覆盖 P018 主链的公开：

1. 任务执行期间接收新交互输入；
2. 改变执行控制行为前必须读取当前任务期运行时世界状态快照；
3. 区分状态查询输入和任务控制输入；
4. 对任务控制输入输出继续、暂停、切换、合并、终止或人工确认的仲裁结果；
5. 允许继续或切换时，基于当前事实删除已满足/已完成/目标事实已成立的步骤；
6. 以当前世界状态续推，不从初始状态重跑；
7. 任务完成、取消或终止后释放任务期快照。

但以下方向有组合攻击风险：

- `US12240120B2`：机器人任务中断、附加任务、回到原任务继续；
- `Interruptible Task Execution with Resumption in Golog`：可中断任务执行与恢复；
- `US20180178379A1`：基于系统状态和用户指令动态生成任务计划；
- `US20160042735A1`：对话任务状态监控；
- `Interruptable Autonomy / CoBot`：执行期间状态查询、取消运行任务、新任务预约；
- `Tell me when and why to do it`：自然语言运行时更新规划器模型；
- 自然语言控制机器人闭环任务规划文献：执行过程中的意外事件处理和重规划；
- mixed-initiative / shared autonomy 文献：执行期间人类干预、主动权切换和状态感知；
- `TCA / TDL`：机器人任务控制架构、执行监控、暂停/终止/重试/异常处理；
- `RAPs`：反应式动作包、当前世界模型、自适应执行；
- `PRS / UM-PRS`：当前 beliefs / intentions 驱动的过程推理和实时控制；
- `3T / ATLANTIS`：规划层、序列层、技能层分层执行架构；
- `ROGUE`：面向 asynchronous user requests 的规划与执行交织；
- `CASPER / continual planning`：执行中持续规划、当前状态更新和计划修补；
- `adjustable autonomy / traded control / sliding autonomy`：任务执行期间人机控制权交接和自主等级调节；
- `Pike / human-robot team executives`：人机团队计划识别与执行并行；
- 传统 `workflow management / dynamic workflow change`：流程实例状态、暂停/恢复、人工任务、动态流程变更；
- `CLARAty / PLEXIL / NASA plan execution executives`：当前机器人/环境状态驱动的规划、执行、监控；
- `Chekhov / temporal plan execution monitoring`：时间计划执行监控、在线计划修改和鲁棒执行；
- `Soar / BDI / cognitive architecture / blackboard`：工作记忆、信念、黑板状态和动作选择；
- `event calculus / situation calculus / fluent calculus`：动作、前提、效果、事实变化和世界状态形式化；
- `CN122201291A`：对话线程与任务线程融合、状态感知和可中断交互；
- `CN116787419A`：行为树、节点状态返回、blackboard 状态通信；

因此，P018 权利要求不建议再压缩到“任务中断恢复”或“基于状态重新规划”。最稳区别口径应是：

- `ROGUE / TCA / CLARAty` 组合用于攻击“执行中请求、当前状态、执行监控和重规划”；
- `CoBot / hierarchical task dialogs / adjustable autonomy` 组合用于攻击“状态查询、取消任务、切换任务、人工介入”；
- `workflow / RPA long-running workflows` 组合用于攻击“暂停、恢复、人工任务、流程状态释放”；
- `BDI / event calculus / RAPs` 组合用于攻击“事实状态、动作效果、当前信念驱动后续动作”。

因此，P018 的申请文本应避免把上述任一单点作为核心创造性，而应把核心创造性固定为“面向自动化执行体的新交互输入状态优先任务交互仲裁闭环”。该闭环同时要求：控制前读取任务期世界状态快照、输入分流、任务控制仲裁、当前事实裁剪续推和任务期快照释放。
