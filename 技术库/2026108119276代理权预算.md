# 说明书初稿

## 发明名称

一种面向自动化执行主体的代理权预算治理方法和系统

## 技术领域

本发明涉及人工智能安全、自动化执行主体治理、访问控制、工具调用安全、任务链安全、工作流安全、具身智能安全、车载代理安全、物联网代理安全和自动化系统运行时控制技术领域，尤其涉及一种在自动化执行主体执行工具调用、接口调用、任务派生、跨执行主体传递、数据动作、资源动作、经济动作、权限动作或物理动作之前、过程中或之后，基于代理权预算对象或代理权预算约束确定预算适配结果，并据此生成预算裁决对象或预算裁决结果，以约束当前动作或后续动作范围的方法和系统。

## 背景技术

随着人工智能代理、RPA、企业工作流、多 Agent 协作系统、云运维代理、代码代理、财务代理、设备代理、具身执行端代理、车载代理和物联网代理的发展，自动化执行主体逐渐从执行单次固定命令，发展为能够基于任务目标、模型推理、规则编排、上下文状态或外部事件持续规划、调用工具、拆分任务、派生子任务、持有工具令牌、异步执行、跨端执行，并对数字资源、经济资源、组织流程或物理世界产生影响。

现有权限控制方案，例如 RBAC、ABAC、OAuth、JWT、能力票据、访问令牌、云 IAM 策略、零信任访问评估、工作流审批和工具调用风控，主要用于判断某一主体在某一时刻是否具有访问权限、令牌是否有效、动作是否命中风险规则或流程是否通过审批。此类方案通常能够解决“是否有权访问”或“当前动作是否可执行”的问题，但难以表达自动化执行主体在长任务、多工具调用、多 Agent 协作或异步执行过程中的剩余可执行动作量、持续执行时间、跨主体传播范围、后续工具调用范围以及降权或撤销触发条件。

现有配额、限流、成本预算或资源预算方案，通常用于限制接口调用频率、算力消耗、云资源用量、费用支出或数据流量。此类方案能够对资源用量进行计数或告警，但其控制目标主要是资源或成本约束，并未将动作语义、授权目标、任务链状态、后果边界、执行主体自主等级和跨执行主体传递关系统一映射为自动化执行主体的代理权预算。现有最小权限、令牌 scope、能力票据衰减或动态策略刷新方案，能够缩小访问范围或更新令牌有效性，但通常不针对每一动作事件形成预算消耗、预算适配结果、预算裁决对象以及当前或后续动作边界控制的连续闭环。

例如，在企业财务场景中，财务 Agent 获得处理报销的授权后，可能持续读取票据、核对收款方、发起支付、调用审批接口和生成报表。即使某一笔支付动作符合授权，其仍然可能消耗金额预算、收款方预算、数据外发预算或审批链预算，并影响后续动作范围。现有支付限额或审批规则通常不能完整表达该 Agent 在整个任务链中的剩余代理权。

又例如，在云运维场景中，云运维 Agent 被授权在维护窗口内重启指定服务，但其可能进一步发起跨项目访问、权限扩大、数据库删除、实例扩缩容或配置变更。现有 IAM 策略可判断某接口是否可调用，变更单可判断某流程是否被审批，但难以根据动作事件持续扣减高风险 API 预算、冻结后续任务链、撤销共享工具令牌或将子任务降权。

再例如，在多 Agent 协作场景中，父 Agent 可将任务拆分给多个子 Agent 或外部代理。若父任务授权被完整传递至子任务，则容易造成授权扩大、工具令牌误用或任务链污染扩散。若用户或企业审批人撤销授权，已经派生的异步任务、缓存任务、兄弟任务、子 Agent 会话或共享工具令牌可能未同步失效。

此外，在具身执行端、车载代理、物联网代理、工业控制代理、家庭自动化或城市基础设施自动化场景中，自动化执行主体的动作可能进一步影响物理设备、空间范围、速度、力矩、人员距离、设备能力或后果边界。现有物理安全控制或设备参数限制通常作用于底层设备控制层，难以表达代理主体自身的可执行范围、自主等级、跨主体传递和撤销传播。

因此，有必要提供一种技术方案，将自动化执行主体的授权从一次性、静态化的权限表示，转换为机器可解析、可计算、可消耗、可冻结、可撤销、可传播收缩和可审计的代理权预算。该方案在动作事件发生时确定预算适配结果，生成预算裁决对象或预算裁决结果，并据此约束当前动作或后续动作范围，从而降低自动化执行主体越权执行、任务链污染扩散、授权撤销不同步、工具令牌持续滥用和高风险动作失控的系统安全风险。

## 发明内容

### 发明目的

本发明的目的在于提供一种面向自动化执行主体的代理权预算治理方法和系统，使自动化执行主体的授权由静态权限或单次审批结果，转化为可计算、可消耗、可收缩、可撤销、可传播和可审计的代理权预算。系统在自动化执行主体发起动作事件时，根据代理权预算对象或代理权预算约束确定预算适配结果，基于预算适配结果生成预算裁决对象或预算裁决结果，并根据预算裁决对象或预算裁决结果约束当前动作或后续动作范围。

### 技术方案

为实现上述目的，本发明提供一种面向自动化执行主体的代理权预算治理方法，包括：

S1，基于授权锚点、任务目标、工具能力、策略配置或预授权信息中的至少一种，确定用于限定自动化执行主体可执行范围的代理权预算对象或代理权预算约束。

其中，代理权预算对象可以为显式结构化对象，代理权预算约束可以为状态表、策略对象、能力票据、令牌约束、权限边界、预算字段集合、配置项、规则集合或内部状态中的至少一种。代理权预算对象或代理权预算约束用于限定自动化执行主体可执行动作范围、工具能力、资源访问范围、时间窗口、动作次数、金额、数据外发量、自主执行等级、任务链深度、跨执行主体传递范围或动作后果边界。

S2，响应于所述自动化执行主体发起高风险动作、工具调用、任务派生或跨执行主体传递中的至少一种动作事件，根据所述动作事件的动作语义、资源范围、风险等级、后果边界、上下文状态、历史消耗记录以及所述代理权预算对象或代理权预算约束中的至少一种，确定预算消耗、预算余额、预算状态或预算适配结果中的至少一种。

其中，预算适配结果是动作事件与代理权预算对象或代理权预算约束之间进行运行时适配判断后得到的中间技术结果。预算适配结果可表现为预算消耗、预算余额、预算状态、预算越界标签、预算充足性结论、预算风险等级、状态转移结果、原因码、内部策略命中结果、能力票据更新结果或接口响应结果。

S3，基于所述预算消耗、预算余额、预算状态或预算适配结果中的至少一种，生成预算裁决对象或预算裁决结果。

其中，预算裁决对象可以为结构化对象，预算裁决结果可以为内部策略结果、接口响应码、令牌更新指令、权限更新指令、状态机转移、能力票据更新、任务状态更新或动作控制指令。

S4，根据所述预算裁决对象或预算裁决结果，约束所述自动化执行主体的当前动作或后续动作范围。

所述约束可包括放行、限权放行、参数裁剪、预算扣减、预算冻结、要求重新验真、撤销工具令牌、降低自主执行等级、冻结任务链、隔离子执行主体、阻断动作、回滚任务、审计增强、限制工具调用能力、限制接口调用次数、限制参数范围、缩短令牌有效期、降低可访问资源范围、触发人工复核中的至少一种。

在一种实施方式中，S2 与 S3 可由同一运行时模块完成；在另一种实施方式中，S2 可由预算适配模块完成，S3 可由预算裁决模块完成。无论采用单模块实现或多模块实现，均不影响本发明通过预算适配结果和预算裁决结果约束当前或后续动作范围的技术效果。

### 有益效果

与现有技术相比，本发明至少具有以下有益效果：

1. 将静态权限转换为可计算、可消耗、可撤销的代理权预算，使自动化执行主体的授权边界能够在运行时被动态管理。

2. 支持 AI Agent、RPA、工作流、多 Agent、云运维代理、代码代理、财务代理、具身执行端代理、车载代理和物联网代理等长任务场景中的持续授权治理。

3. 即使某一动作符合授权，也可根据动作事件消耗预算，并基于剩余预算或预算适配结果调整后续动作范围。

4. 通过预算适配结果和预算裁决对象，将动作语义、资源范围、风险等级、后果边界、历史消耗和任务链状态转换为可执行的工具令牌收缩、接口限制、参数裁剪、任务链冻结或自主等级降低。

5. 通过子预算继承，避免父任务授权在子任务、异步任务、外部代理或工作流节点中被扩大。

6. 通过授权撤销传播，使用户撤销、审批失效、输入污染、预算耗尽、工具异常或后果越界能够传播至相关子任务、兄弟任务、缓存任务、异步任务、共享工具令牌或跨执行主体会话。

7. 通过任务链连锁降权，抑制提示注入、输入污染、任务漂移和异常工具调用在多节点任务链中的扩散。

8. 通过预算轨迹摘要，记录预算生成、分配、消耗、继承、冻结、撤销、传播、恢复和裁决过程，支持安全审计、合规复核、事故复盘和争议追溯。

9. 可将配额、令牌、策略、状态机、能力票据、任务图和调用链中的至少一种实现形式统一映射为代理权预算约束、预算适配结果或预算裁决结果，从而适配 SDK、工具网关、API 网关、MCP 工具层、企业 Agent 平台、边缘节点和具身执行端代理节点等不同部署形态。

10. 可同时适配数字执行端的 Agent 数字执行质量预算和具身执行端的物理行为预算，为后续自动化系统的运行时安全控制提供统一基础。

预算轨迹摘要是指记录预算生成、分配、消耗、继承、冻结、撤销、传播、恢复和裁决过程的结构化摘要对象。预算轨迹摘要可包括预算对象摘要、动作事件摘要、消耗记录、余额、状态转移、裁决结果、子预算继承关系、撤销传播路径、工具令牌状态、原因码、时间戳、签名、哈希或审计索引。

#### 人类真实意图因果验真路径

人类真实意图因果验真路径是指基于自动化执行主体发起的高风险动作请求的动作语义、授权意图锚点、运行时人机状态和动作后果边界，计算该动作请求与人类真实意图之间的因果一致性，并据此生成裁决令牌或代理权裁决结果的并行验证机制。

该路径可与本发明的代理权预算治理路径共享动作声明对象、动作语义、作用对象、执行上下文或动作后果中的至少一种输入。人类真实意图因果验真路径从意图验真维度判断动作请求是否符合授权意图，代理权预算治理路径从预算治理维度判断动作事件对代理权预算的消耗、适配和后续动作边界影响。该路径以动作语义、授权意图锚点、运行时人机状态和动作后果边界等技术表征为依据，输出因果一致性评分、断裂标签、原因码、裁决令牌或代理权裁决结果。

### 实施例一：通用代理权预算治理流程

如图1和图2所示，在本实施例中，代理权预算治理系统部署于自动化执行主体的运行时链路中。该系统可以部署于 SDK、工具网关、API 网关、MCP 工具层、插件运行时、企业 Agent 平台、工作流编排平台、系统服务、安全沙箱、边缘代理节点、具身执行端代理节点、车载代理节点或云端审计平台。

系统首先基于授权锚点、任务目标、工具能力、策略配置或预授权信息，确定代理权预算对象或代理权预算约束。以企业 Agent 为例，授权锚点可以包括用户授权、企业审批、任务配置、工具令牌、策略配置、变更单、工单、监管策略或预授权规则。代理权预算对象可包括执行主体标识、任务目标、可调用工具集合、资源范围、时间窗口、动作次数上限、数据外发上限、自主执行等级、任务链深度上限和撤销条件。

当自动化执行主体发起动作事件时，预算适配模块根据动作语义、目标资源、风险等级、后果边界、上下文状态、历史消耗记录和预算约束，确定预算适配结果。例如，当动作事件为读取低敏感等级数据且预算余额充足时，预算适配结果可以为允许并扣减数据访问预算；当动作事件为修改生产配置且未处于维护窗口时，预算适配结果可以为维护窗口越界和预算适配失败；当动作事件为派生子 Agent 时，预算适配结果可以为生成受限子预算。

预算裁决模块基于预算适配结果生成预算裁决对象或预算裁决结果。预算裁决结果可以为放行、限权放行、参数裁剪、预算扣减、预算冻结、要求重新验真、撤销工具令牌、降低自主执行等级、冻结任务链、隔离子执行主体、阻断动作或审计增强。

动作约束模块根据预算裁决对象或预算裁决结果，约束当前动作或后续动作范围。例如，系统可以缩短工具令牌有效期、移除高风险工具、限制接口调用次数、降低可访问资源范围、冻结同一任务链、要求人工复核或阻断高风险动作。

在本实施例中，即使动作事件被放行或限权放行，系统仍可根据该动作事件对应的预算消耗更新预算适配结果，并基于更新后的预算适配结果调整后续动作范围。例如，同一自动化执行主体连续发起工具调用后，即使每次调用均未触发阻断，其剩余工具调用预算、金额预算或自主执行等级也可持续下降，进而影响后续工具调用的放行范围或参数边界。

在一种实现方式中，预算适配可通过规则、状态机或策略表实现。在另一种实现方式中，预算适配可通过固定扣减、比例扣减、分级扣减、风险加权、查表或实时计算实现。在又一种实现方式中，代理权预算可嵌入工具令牌、能力票据、会话令牌、API key 附加声明、MCP 工具调用上下文、插件运行时上下文或企业策略对象中。

如图5所示，代理权预算对象或代理权预算约束可具有有效、部分消耗、限权、待重新验真、冻结、撤销、过期、隔离或恢复等状态。预算状态可随动作事件、预算适配结果、授权撤销事件、重新验真结果或人工复核结果发生转移。所述状态机仅为一种示例，系统也可通过策略表、令牌字段、数据库状态或内部状态变量表示预算状态。

### 实施例二：同源输入、双路径治理

如图3所示，在本实施例中，同一动作声明对象或动作事件可同时触发人类真实意图因果验真路径和代理权预算治理路径。

动作数字研判模块首先形成动作声明对象。动作声明对象可包括动作事件标识、动作语义、作用对象、目标资源、执行上下文、授权锚点、风险等级和动作后果边界。该动作声明对象一方面进入人类真实意图因果验真路径，用于判断动作请求是否符合人类授权意图；另一方面进入代理权预算治理路径，用于判断动作事件消耗多少代理权预算、是否超出预算约束以及是否影响后续动作范围。

以财务 Agent 发起转账动作为例，动作声明对象包括金额、收款方、用途、任务来源、审批链、执行时间和预期后果。人类真实意图因果验真路径可判断该转账是否符合授权意图、审批链和运行时人机状态。代理权预算治理路径可判断该转账消耗多少金额预算、是否超过单笔或总额预算、是否为新收款方、是否降低后续支付能力以及是否触发重新验真。

在一种实施方式中，人类真实意图因果验真路径与代理权预算治理路径可并行执行或串行执行。在并行执行方式中，两者通过图3所示的共享动作声明对象读取动作语义、作用对象和执行上下文，以避免重复解析；其各自输出的裁决结果可按照图3所示的组合控制路径合并为最终执行控制结果。

两条路径可分别输出因果验真裁决结果和预算裁决对象或预算裁决结果。系统可将两个裁决合并为最终执行控制结果。例如，若因果验真通过但预算余额不足，则系统可限额放行或要求人工复核；若预算充足但因果验真失败，则系统可阻断动作并冻结任务链；若两者均通过，则系统可放行动作并扣减预算。

本实施例中，代理权预算治理路径可独立运行，也可与人类真实意图因果验真路径协同运行。动作事件符合授权意图时仍可消耗预算；动作事件出现预算适配失败、输入污染、授权撤销或后果越界时，还可触发冻结、降权、令牌撤销或重新验真。

### 实施例三：企业财务 Agent 预算治理

在本实施例中，自动化执行主体为企业财务 Agent。用户或企业审批系统授权财务 Agent 在预设时间窗口内处理报销任务。系统基于审批链、报销任务目标、支付工具能力和企业财务策略生成代理权预算对象。

该预算对象可包括：总金额预算、单笔金额预算、收款方白名单、支付工具集合、可读取票据范围、可访问财务系统范围、审批链要求、执行时间窗口、数据外发上限、重新验真条件和撤销条件。

当财务 Agent 发起票据读取、发票核验、报销分类、收款方核对、付款请求或报表生成等动作事件时，预算适配模块根据动作语义、金额、收款方、审批状态、数据敏感等级、历史消耗记录和剩余预算确定预算适配结果。

若付款金额在单笔预算和总金额预算内，收款方属于白名单，审批链未失效，则系统生成放行并扣减金额预算的预算裁决结果。若收款方为新账户、金额超出单笔预算、任务来源存在输入污染或审批链撤销，则系统生成预算冻结、要求人工复核或撤销支付工具令牌的预算裁决结果。

系统还可生成预算轨迹摘要，记录预算对象摘要、每次支付动作、金额消耗、收款方状态、审批链状态、裁决原因码、工具令牌状态和时间戳，用于企业合规复核和争议追溯。

### 实施例四：云运维 Agent 预算治理

在本实施例中，自动化执行主体为云运维 Agent。企业授权云运维 Agent 在指定维护窗口内对指定项目执行服务重启、实例健康检查和配置回滚。系统生成代理权预算对象，限定项目范围、云资源范围、API 类型、维护窗口、操作次数、生产环境权限、高风险 API 调用预算和回滚要求。

当云运维 Agent 发起重启服务、修改配置、扩缩容、删除资源、修改权限、读取日志或调用云 API 等动作事件时，预算适配模块根据动作语义、目标资源、维护窗口、资源敏感等级、高风险 API 类型、历史操作次数和回滚要求确定预算适配结果。

若动作事件为在维护窗口内重启授权服务，则系统可放行并扣减操作次数预算。若动作事件为删除生产数据库、跨项目访问、扩大实例权限、关闭审计日志或在非维护窗口修改生产配置，则系统可输出预算越界标签，并生成撤销高风险 API 工具令牌、降为只读、冻结任务链、要求变更单或人工复核的预算裁决结果。

在一种实施方式中，系统可在动作执行前通过沙箱、静态分析、变更预演、数字孪生或规则推演预测动作后果，并根据预测后果确定预算消耗或裁决结果。例如，若变更预演显示服务不可恢复或回滚脚本缺失，则系统可提高预算消耗等级或直接冻结任务链。

### 实施例五：代码代理质量与部署预算

在本实施例中，自动化执行主体为代码代理。企业授权代码代理修改某一仓库中的指定模块。系统生成代理权预算对象，限定仓库范围、目录范围、文件类型、修改行数、测试通过率、静态扫描结果、部署权限、生产配置修改预算、凭证访问限制和人工审查要求。

当代码代理发起读取代码、修改文件、生成补丁、运行测试、提交分支、创建合并请求、修改 CI/CD 配置或触发部署等动作事件时，预算适配模块根据动作语义、文件路径、变更规模、测试结果、静态扫描结果、依赖风险、是否触碰凭证或安全策略，确定预算适配结果。

若代码修改位于授权目录、测试通过且未触碰敏感配置，则系统可允许生成提交或合并请求，并扣减代码变更预算。若代码代理试图修改生产凭证、安全策略、部署脚本或跨仓库文件，系统可降低自主执行等级、冻结部署令牌、要求人工审查或阻断动作。

本实施例中，测试通过率、静态扫描结果、代码质量、结果可回滚性和人工审查要求可以作为 Agent 数字执行质量预算的预算维度。数字执行质量预算可与代理权预算对象、预算适配结果和后续动作边界控制联动。

### 实施例六：多 Agent 子预算继承

如图6所示，在本实施例中，父 Agent 接收用户或企业系统授权后，将任务拆分给多个子 Agent、工作流节点、外部代理或异步任务。系统基于图6所示的子预算继承模块，由父预算对象生成一个或多个子预算对象或子预算约束。

父预算对象可包括任务目标、总金额预算、可调用工具集合、数据外发上限、任务链深度上限和撤销条件。子预算对象可包括父预算标识、子预算标识、子执行主体标识、继承范围、衰减规则、隔离范围、共享工具令牌规则、撤销传播关系和子预算状态。

子预算对象的授权范围、工具集合、资源集合、时间窗口、金额上限、数据外发上限、自主执行等级、任务链深度和跨执行主体传递范围，不超过父预算剩余范围。例如，父 Agent 获得 10000 元采购预算，其派生的供应商比价子 Agent 只获得查询供应商和生成报价摘要的预算，不能获得付款工具预算；付款子 Agent 只能在审批链完成后获得有限金额预算。

若某一子 Agent 发生输入污染、提示注入、异常工具调用或预算越界，系统可根据预算继承关系冻结该子预算，降低兄弟任务权限，撤销共享工具令牌，或要求父任务重新验真。

### 实施例七：授权撤销传播

如图6所示，在本实施例中，授权主体、企业审批人、策略系统或安全模块触发授权撤销、预算冻结或预算降权事件。系统基于任务图、调用链、令牌关系、预算继承关系或跨执行主体传递关系，通过图6所示的撤销传播模块和传播路径，将撤销或降权事件传播至相关对象。

传播对象可包括当前会话、子 Agent、兄弟任务、异步任务、缓存任务、定时任务、共享工具令牌、外部代理、工作流节点、具身执行端代理或车载代理。传播方式可包括预算状态同步、令牌撤销、策略刷新、任务链冻结、预算对象失效、接口调用阻断、事件总线推送、异步消息、状态订阅、轮询校验、策略版本号更新或令牌失效列表；其中，共享工具令牌、受限子预算对象和预算轨迹摘要可分别对应图6中的共享工具令牌节点、子执行主体节点和预算轨迹摘要节点。

例如，企业审批人撤销财务 Agent 的付款授权后，系统生成预算撤销事件，并传播至已派生的付款子任务、缓存任务和共享支付工具令牌。相关子任务在下一次动作事件发生前或发生时校验预算状态，若发现预算已撤销，则拒绝支付动作并生成审计摘要。

在另一示例中，云运维 Agent 的父任务被安全系统判定为输入污染，系统可将降权事件传播至相关工作流节点和高风险 API 工具令牌，使后续动作转入只读模式或人工复核模式。

### 实施例八：任务链连锁降权

在本实施例中，系统在任务链中检测到输入污染、提示注入、任务目标漂移、异常工具调用、子执行主体异常、动作后果越界或预算适配失败后，基于任务图、调用链、共享工具令牌关系或预算继承关系，对相关节点执行连锁降权。

连锁降权可包括当前节点隔离、子节点冻结、兄弟节点降权、共享工具令牌撤销、父任务重新验真、任务链审计增强、外部代理隔离或预算对象状态更新。

例如，多 Agent 任务链中的一个子 Agent 被外部输入诱导调用数据外发工具，系统可将该输入源标记为不可信，冻结该子 Agent 的数据外发预算，撤销共享外发工具令牌，并降低同一任务链下兄弟 Agent 的自主执行等级。若父任务仍需继续执行，则系统可要求重新验真或人工复核。

本实施例中，系统可根据预算继承关系、风险等级和后果边界，选择性冻结当前节点、相关子节点、共享工具或高风险后续动作。

### 实施例九：预算轨迹摘要

在本实施例中，系统生成预算轨迹摘要。预算轨迹摘要用于记录代理权预算治理过程中的关键状态和裁决信息。

预算轨迹摘要可包括：预算对象摘要、预算约束摘要、动作事件摘要、预算消耗记录、预算余额、预算状态转移记录、预算适配结果、预算裁决结果、子预算继承关系、授权撤销传播路径、工具令牌状态、原因码、时间戳、签名字段、哈希字段或审计索引。

在一种实现方式中，系统为每次动作事件生成动作事件摘要和预算适配摘要，并将其与预算对象摘要、裁决结果和时间戳关联。若发生撤销传播，系统记录撤销事件的触发主体、传播路径、受影响预算对象、受影响工具令牌和最终状态。若发生子预算继承，系统记录父预算标识、子预算标识、继承范围和衰减规则。

预算轨迹摘要围绕预算生成、分配、消耗、继承、冻结、撤销、传播、恢复和裁决形成结构化审计对象，可记录动作事件与预算状态转移、预算裁决结果和撤销传播路径之间的对应关系，并用于企业合规复核、争议追溯、安全审计、事故复盘和跨平台协同。

### 实施例十：Agent 数字执行质量预算

如图7所示，在本实施例中，代理权预算用于约束数字执行端的结果质量和后续动作能力。适用对象包括代码 Agent、内容 Agent、数据 Agent、财务 Agent、云运维 Agent、企业流程 Agent 和其他数字动作执行主体。

数字执行质量预算可包括事实性预算、来源可信度预算、测试通过率预算、数据敏感等级预算、代码质量预算、配置风险预算、外发范围预算、结果可回滚性预算和人工审查预算。

例如，内容 Agent 被授权生成面向客户的公开材料。系统可将来源可信度、事实性校验、敏感信息外发、人工审核状态和发布渠道作为预算维度。若内容 Agent 使用不可信来源、包含敏感信息或未通过事实性校验，系统可降低发布权限、要求人工审查或禁止自动发布。

又例如，数据 Agent 被授权生成分析报告。系统可将数据访问范围、数据脱敏要求、外发范围、模型调用次数和报告可信度作为预算维度。当数据 Agent 试图外发未脱敏数据或访问超范围数据集时，系统可冻结数据外发预算并生成审计摘要。

### 实施例十一：具身智能物理行为预算

具身行为预算可包括空间范围、时间窗口、动作次数、速度、力矩、接近人员距离、自主等级、任务阶段、主权控制状态和动作后果边界。

例如，家庭机器人被授权在客厅区域执行清洁任务。预算对象限定空间范围、时间窗口、可操作设备、接近人员距离和自主等级。当机器人代理试图进入儿童房、触碰未授权设备或在夜间执行高噪声动作时，系统可输出预算越界标签，并限制动作范围或要求重新授权。

又例如，车载代理被授权执行低风险远程控制任务。预算对象限定远程动作范围、车辆状态、时间窗口、主权控制状态和后果边界。当车载代理试图在车辆行驶中执行不适配动作，或跨越授权时间窗口执行远程控制时，系统可收缩代理权边界、阻断动作请求或要求重新验真。

## 系统实施例

如图1所示，本发明还提供一种面向自动化执行主体的代理权预算治理系统。该系统包括预算确定模块、预算适配模块、预算裁决模块和动作约束模块。

预算确定模块用于基于授权锚点、任务目标、工具能力、策略配置或预授权信息中的至少一种，确定用于限定自动化执行主体可执行范围的代理权预算对象或代理权预算约束。

预算适配模块用于响应于自动化执行主体发起高风险动作、工具调用、任务派生或跨执行主体传递中的至少一种动作事件，根据动作事件的动作语义、资源范围、风险等级、后果边界、上下文状态、历史消耗记录以及代理权预算对象或代理权预算约束中的至少一种，确定预算消耗、预算余额、预算状态或预算适配结果中的至少一种。

预算裁决模块用于基于预算消耗、预算余额、预算状态或预算适配结果中的至少一种，生成预算裁决对象或预算裁决结果。

动作约束模块用于根据预算裁决对象或预算裁决结果，约束自动化执行主体的当前动作或后续动作范围。

在一种实施方式中，系统还包括子预算继承模块、撤销传播模块、任务链降权模块、工具令牌收缩模块、预算轨迹摘要模块、后果预测模块、沙箱执行模块、恢复预算模块或审计接口模块中的至少一种。

子预算继承模块用于在自动化执行主体进行任务拆分、派生子执行主体、调用外部代理、委托工作流节点、生成异步任务或共享工具令牌时，基于父任务对应的代理权预算对象或代理权预算约束生成子预算对象或子预算约束。

撤销传播模块用于响应授权撤销、预授权锚点失效、预算耗尽、预算过期、任务目标漂移、输入污染、提示注入、工具调用异常、动作后果越界、子执行主体异常或上下文突变中的至少一种异常事件，生成预算撤销事件或预算降权事件，并基于任务图、调用链、令牌关系、预算继承关系或跨执行主体传递关系进行传播。

任务链降权模块用于在任务链中任一节点发生异常时，对当前节点、子节点、兄弟节点、父任务、共享工具令牌或外部代理执行降权、冻结、隔离、令牌撤销或重新验真。

预算轨迹摘要模块用于生成预算轨迹摘要，以记录预算对象、动作事件、预算消耗、预算状态转移、裁决结果、子预算继承关系、撤销传播路径、工具令牌状态和原因码。

上述模块可位于同一设备，也可分布于 SDK、工具网关、API 网关、云端平台、边缘节点、终端服务、企业 Agent 平台、工作流平台或具身执行端代理节点。各模块之间可通过预算对象、预算适配结果、预算裁决对象、令牌更新指令、策略更新指令或预算轨迹摘要进行数据交互。
