# P020 自然语言基准集 v1

## 目标

本基准用于测量人类语言经机器字典、RCIR、Grounding和交互合同进入具身规划的正确性。它不以回复文本相似度作为主要指标，也不以动作最终成功掩盖语义错误。

## 分层

### 语义层

检查以下结构化结果：

- 言语行为；
- 事件算子及顺序；
- EntityRef角色绑定；
- 时间范围；
- 空间关系；
- 人类报告事件；
- `runtime_fact_committed=false`证据边界。

### 交互层

检查以下控制结果：

- 唯一角色能够进入规划；
- 多候选角色必须产生澄清合同；
- 未知表达只能形成候选过程模板；
- 缺少物理或因果条件时不得启动动作；
- 显式当前约束优先于类别歧义和历史候选。

当前覆盖：

10. 未知概念安全边界。

## 指标

- `semantic_accuracy`：语义层通过数 / 语义层总数；
- `interaction_contract_accuracy`：交互层通过数 / 交互层总数；
- `overall_pass_rate`：全部通过数 / 全部案例数；
- `unsafe_false_positive_count`：不应执行却进入运动的案例数；
- `physical_fact_boundary_violation_count`：语言或候选感知直接提交物理事实的案例数。

## 扩展合同

扩展到100至300条时必须遵守：

- 新变体映射到既有概念、算子、角色和关系，不新增逐句运行时分支；
- 同一意图的变说法应共享结构化期望；
- 预期结果依据RCIR合同确定，不能依据当前程序输出倒推；
- 失败必须归因到词典覆盖、作用域、指代、Grounding、询问、规划或P016验真之一；
- 修复后保留失败案例，形成永久回归；
- 每次报告记录基准版本、代码提交和分类指标。
