6. API 服务：`demo_runtime/api_server.py`
7. 研发记录制度：`研发记录/研发记录制度.md`
8. 当日研发记录：`研发记录/2026-06-29.md`

## 四、后续使用方式

后续每次提交至少补一行或补一列相关证据，形成：

> 母案特征 -> 代码实现 -> 输出证据 -> 验证命令 -> 研发记录 -> Git 提交

这样才能保证在专利审查、非正常复核或业务争议中，能够快速给出可查、可验、可追溯的对应链。
