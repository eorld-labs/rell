# 教学舱：基于概念契约与因果算子的具身智能学习架构

## 一、文档定位

教学舱是 RELL 体系中面向人类示范、真实观测和经验编译的学习入口。它不是新的执行层，也不替代 P016、P017、P018、P019、P003 或本体画像。

教学舱负责把不同来源的观测材料，逐步编译为：

1. 事件候选；
2. 语义角色候选；
3. 概念契约候选；
4. 因果前提、结果和事实失效关系；
5. 可由具体本体重新落地的经验不变量。

教学舱的输出始终是候选或待验证经验。只有重新绑定当前空间、通过 P018 仲裁、经 P016 双通道验真并完成必要的人类验收后，经验才可以晋升为端侧可信经验。

### 1.1 架构必须承载的三种演进终局

教学舱不是只为当前三维样品设计的交互功能。其数据结构、证据边界和晋级路径必须同时承载三种最终运行形态。

#### 终局一：真实环境中的师徒式教学

机器人在家庭、工厂、商业空间或公共环境中工作。终端用户像教人一样，通过语言、示范、纠正、打断、边界指示和结果确认教会机器人。

该形态要求：

- 教学与当前世界状态实时绑定；
- 机器人能够解释不会的原因并请求补教；
- 教学过程不能绕过本体边界、安全保护和执行验真；
- 示范结束后必须由机器人自主重新绑定并复做；
- 经验在本地形成，并可在权限允许时继续晋级为可迁移经验。

#### 终局二：GPU 驱动的离线规模化学习

系统可以像当前大模型训练一样，批量读取真实视频、语音、文本、真机日志、公开数据集和高物理仿真数据，并在 GPU 或其他并行计算资源上完成对象候选、事件切分、语言对齐、因果结构发现和契约编译。

GPU 在这里是观测解析和候选编译的加速资源，不是新的事实来源，也不改变知识边界。离线批处理输出的仍然是：

- 候选概念；
- 候选语义角色；
- 候选因果契约；
- 候选过程模板；
- 候选不可行条件；
- 对应的来源链、不确定性和适用范围。

无论离线计算规模多大，这些结果都不能直接成为某台机器人当前世界中的已成立事实。进入端侧后仍需当前空间重新绑定、本体适配、P018 仲裁和 P016 验真。

#### 终局三：普通人成为分布式机器人训练员

普通人可以使用手机、领夹相机、头戴设备、控制器或机器人第一视角，通过视频流、语音和动作完成教学，不要求理解机器人编程、关节控制或概念契约格式。

系统负责把人的自然示范编译为结构化候选；训练员负责表达目标、演示过程、指出错误和确认结果。该形态除教学能力外，还要求：

- 训练员身份、授权范围和来源链；
- 视频、语音、场景和对象的隐私及数据权利；
- 示范质量、适用范围和历史可信度；
- 恶意、错误或相互冲突教学的隔离；
- 候选经验的沙箱复做、端侧验真和分级晋升；
- 训练成果、贡献记录和后续版本之间的可追溯关系。

普通训练员可以生成候选经验，但不能直接获得修改公共概念、覆盖机器人当前事实或绕过安全治理的权力。

### 1.2 三种终局共享同一学习内核

三种形态的区别只应存在于输入方式、计算位置、实时性和治理强度，不应形成三套互不兼容的知识体系。

```text
现场用户实时教学 -----\
真实视频/语料/GPU批处理 ----> 统一观测证据包
大众训练员视频流与动作 ---/          |
                                      v
                           事件与语言候选
                                      |
                                      v
                           概念契约与因果算子
                                      |
                                      v
                         经验不变量与适用边界
                                      |
                                      v
                    当前空间重新绑定 + 本体适配
                                      |
                                      v
                         P018 仲裁 + P016 验真
                                      |
                                      v
                            分级可信经验资产
```

共同内核至少包括：

1. 统一观测证据包；
2. 统一的事件、角色和概念候选结构；
3. 统一的 `requires / produces / destroys / verification` 因果契约；
4. 统一的经验不变量、不可行条件和适用范围；
5. 统一的来源链、可信等级和晋级状态机；
6. 统一的当前世界重新绑定、本体适配、仲裁和验真边界。

这组共同内核是教学舱能否从单机样品演变为规模化学习基础设施的决定性条件。

## 二、核心原则

### 2.1 教学输入不是视频本身

视频、语音、遥操作记录和仿真日志都只是观测来源。教学舱真正接收的是统一的观测证据包，而不是某一种固定文件格式。

每个观测证据包至少应带有：

- `source_type`：观测来源类型；
- `source_identity`：设备、会话或数据集来源；
- `timestamp`：时间戳和时间顺序；
- `frame_of_reference`：空间参考系及其可信度；
- `body_reference`：示范者或执行体本体参考系；
- `observations`：图像、深度、位姿、语音、动作或状态候选；
- `world_revision`：对应的世界状态版本；
- `provenance`：来源、处理过程和修改记录；
- `uncertainty`：感知、对齐和空间落地的不确定性；
- `evidence_level`：该输入允许承担的证据等级。
- `pedagogical_signals`：示范、纠正、边界指示、负例和中断等教学信令。

缺少这些字段时，输入仍可以用于生成候选，但不得直接生成已成立的事实或可信经验。

### 2.2 示范轨迹不是经验本体

系统不应把某一次示范中的绝对坐标、关节角、固定时长、按键序列或单一本体轨迹作为经验本体。

教学舱应从示范中提取：

- 语义角色及其绑定条件；
- `requires / produces / destroys / verification` 因果契约；
- 拓扑关系不变量；
- 试探方向和物理约束不变量；
- 事实终止条件不变量；
- 本体适配所需的能力和边界；
- 失败、恢复和请求教学的入口。

### 2.3 失败信息只沉淀为有适用范围的不可行条件

失败示范或失败执行的完整传感器轨迹、全量日志和单次本体动作不得作为可迁移经验长期保存。教学舱应从失败材料中编译结构化的 `infeasibility_summary`，交由 P019 和 P017 判断是否进入概念适用边界或经验适用约束。

不可行条件至少说明：

- 哪一条前提事实未满足；
- 哪一个验真条件未通过；
- 哪一个物理约束、安全边界或治理条件被违反；
- 哪一个语义角色绑定不满足当前场景；
- 该判断适用于哪个对象类型、空间关系、本体画像和世界版本；
- 证据来源、置信度和复现次数。

```json
{
  "infeasibility_summary": {
    "concept_id": "concept_fill_container",
    "failed_requirement": "container_placeable_below_source",
    "negative_constraint": "container_height > available_nozzle_clearance",
    "scope": {
      "source_type": "fixed_water_dispenser",
      "executor_profile": "current_executor_profile",
      "world_revision": "world_rev_42"
    },
    "evidence": {
      "verification_failed": "container_aligned_below_nozzle",
      "confidence": 0.94,
      "support_count": 1
    },
    "disposition": "candidate_constraint_pending_revalidation"
  }
}
```

一次失败只能形成有作用域的候选约束，不得直接改写全局概念。经过重复验证、跨实例复现或人工确认后，才可以扩大其适用范围。这样既避免保存失败轨迹，也避免系统重复尝试已经有充分证据证明不可行的路径。

### 2.4 证据等级不能被跨越

不同数据源可以共同参与概念提取，但不能因为多个低等级信号相互重复，就自动越过物理验真边界。

统一边界如下：

```text
观测证据
  -> 候选事件
  -> 概念/角色候选
  -> 因果契约候选
  -> 当前空间和本体重新绑定
  -> P018 仲裁
  -> P016 执行与物理验真
  -> 可信经验
```

视频可以证明“看起来发生了什么”，不能单独证明“机器人已经完成了什么物理事实”。

## 三、统一教学输入模型

### 3.1 ObservationSourceAdapter

每类输入源通过观测源适配器进入教学舱。适配器只负责格式归一化和证据标注，不负责直接生成执行命令。

```json
{
  "source_type": "first_person_rgbd_audio",
  "source_identity": "head_camera_session_001",
  "time_window": {
    "start": 1710000000.120,
    "end": 1710000018.640,
    "clock_quality": "synchronized"
  },
  "frame_of_reference": {
    "type": "body_frame",
    "quality": "tracked"
  },
  "body_reference": {
    "body_type": "human",
    "body_profile_available": true
  },
  "pedagogical_signals": {
    "signal_types": ["demonstration"],
    "target_experience_ref": null,
    "interruption_occurred": false,
    "clarification_occurred": false,
    "outcome": "completed_successfully"
  },
  "observations": [
    {
      "kind": "object_candidate",
      "entity_type": "fillable_container",
      "confidence": 0.91,
      "spatial_binding": "candidate_only"
    },
    {
      "kind": "language_event",
      "text": "现在拿起杯子",
      "time_range": [1710000002.4, 1710000003.6],
      "confidence": 0.96
    }
  ],
  "evidence_level": "candidate_generation_only"
}
```

教学信令采用 `signal_types` 而不是一组可能互相矛盾的互斥布尔值，因为一次教学片段可能同时是“纠正”和“边界指示”。建议值包括：

- `demonstration`：首次示范；
- `correction`：纠正已有候选经验；
- `boundary_indication`：指出物理、安全或治理边界；
- `negative_example`：明确说明当前做法不可行；
- `confirmation`：确认某个目标事实或教学意图。

`outcome` 与教学意图区分保存，可取 `completed_successfully`、`failed`、`interrupted`、`unknown`。纠正不会自动创建新概念，负例也不会直接进入正向经验；两者首先被编译为带作用域的候选修改或不可行条件。

### 3.2 事件候选与事实的区别

适配器可以输出：

- 手接近对象；
- 物体候选被抓取；
- 执行体位移；
- 物体相对支撑面发生变化；
- 语言中出现动作或状态描述；
- 传感器检测到接触、受力或液体流动。

这些都首先是候选事件。只有在当前世界中重新绑定，并由执行过程和验真通道确认后，才可以写入运行时事实。

## 四、可作为教学输入的数据源

### 4.1 第一视角真人教学

包括当前已经跑通的第一视角教学舱、真人接管机器人视角和带语音的实时教学。

优点：

- 具有明确的主体前方、左右和身体参考系；
- 语言与动作可以实时对齐；
- 可以在遇到缺口时立即澄清；
- 可以直接进入 P016 验真和经验晋升流程。

限制：

- 示范者和目标机器人本体可能不同；
- 人类动作中隐含的力量、接触和身体约束未必可见；
- 需要把人类视角转换为概念角色，而不是复制人的轨迹。

当前阶段应将其作为最高优先级的结构化教学入口。

### 4.2 领夹相机或可穿戴相机

领夹相机、头戴相机、胸前相机等都可以作为第一视角观测源。它们不要求相机本身理解语义，只需要提供稳定的时间序列和尽可能可靠的空间线索。

建议能力分层：

- 仅 RGB：用于语言对齐、对象候选和事件候选；
- RGB-D：增加深度、距离和支撑关系候选；
- RGB-D + IMU：增加身体朝向、移动方向和视角变化；
- RGB-D + 语音时间同步：支持较稳定的示范事件切分。

领夹视频可以进入概念编译，但在缺少深度、接触、力或结果验真的情况下，只能形成候选经验，不能凭视频本身晋升为可信经验。

### 4.3 真机自身传感器和执行日志

真实机器人可以提供：

- 相机、深度、触觉、力矩和接触传感器；
- 末端位姿和执行体状态；
- 控制器反馈的动作阶段；
- 碰撞、限位、保护和停止事件；
- 目标事实的传感器证据。

这类数据对 P016 物理验真和 P008/P6 本体边界最有价值，但仍不能把真机的原始关节轨迹直接存为可迁移经验。真机日志应被编译为本体相关的约束和验真摘要。

### 4.4 仿真环境数据

仿真数据可以用于：

- 大量生成事件候选；
- 验证概念契约的前提和结果；
- 压力测试障碍、遮挡、打断和恢复；
- 测试不同本体画像下的经验适配；
- 验证世界版本、路径撤销和事实生命周期。

仿真成功不能直接证明真实世界成功。仿真数据的来源、物理模型和传感器模型必须进入证据 provenance，并在迁移到真机时重新进行物理验真。

### 4.5 第三方真实视频或公开数据集

真实视频可以作为候选知识来源，但不能默认等价于机器人自己的教学。

它适合提供：

- 动作和事件的视觉先验；
- 对象功能角色的候选；
- 语言与动作的弱对齐；
- 常见任务的阶段分解候选；
- 失败模式和环境变化样本。

它通常缺少：

- 机器人本体参考系；
- 可执行的空间坐标；
- 真实接触和受力信息；
- 执行结果的双通道验真；
- 示范者的完整意图和隐含前提。

因此第三方视频只能进入“候选概念、候选过程模板或云端补给”，不能直接写入端侧可信经验，更不能直接获得执行权。

### 4.6 语言、文本和操作记录

自然语言、工单、操作规程、字幕和人工标注可以提供语言触发、目标事实和步骤顺序候选，但不能单独证明物理因果。

这类输入适合补充：

- 语言适配器；
- 概念别名；
- 语义角色提示；
- 任务目标和偏好约束；
- 澄清问题模板。

它们必须与观测或执行验真结合后，才能参与经验晋升。

## 五、输入源的统一可信等级

建议把输入源分为五个证据等级：

| 等级 | 来源 | 可以做什么 | 不能做什么 |
|---|---|---|---|
| L0 | 文本、字幕、语言描述 | 触发概念和提出目标 | 证明物理事实 |
| L1 | 单目或第三方视频 | 生成对象、事件和步骤候选 | 直接绑定当前对象并执行 |
| L2 | 第一视角 RGB-D/IMU 视频 | 生成空间关系和身体参考系候选 | 替代执行验真 |
| L3 | 真机多传感器日志或仿真物理日志 | 支持接触、位姿和结果候选 | 绕过当前 P018 仲裁 |
| L4 | 当前本体执行 + P016 双通道验真 + 必要人工验收 | 晋升为可信经验 | 跨环境永久复用而不重新绑定 |

等级不是数据质量的绝对排名，而是当前输入可以承担的证明责任。L1 视频可能比低质量 L3 日志更清晰，但仍不能替代真实执行事实。

### 5.1 多源冲突的证据仲裁

当多个输入源对同一个候选事实给出不同结论时，仲裁不能只比较一个置信度数字。系统应同时比较：

1. 是否属于当前世界版本和当前任务期；
2. 是否直接观测目标事实，还是由语言或模型推断；
3. 证据等级及其允许承担的证明责任；
4. 观测时间、新鲜度和是否已被状态变化撤销；
5. 来源独立性、传感器健康状态和适用范围；
6. 同等级、同新鲜度证据的置信度。

在上述条件相同且证据均有效时，证明责任优先级为：

```text
L4 > L3 > L2 > L1 > L0
```

但该顺序不是“历史高等级证据永久覆盖当前低等级观察”。低等级证据不能直接把高等级已验真事实改写为相反事实，却可以表明原事实可能已经过期、对象已经移动或世界版本已经变化。此时必须把旧事实标记为 `stale` 或 `conflicted`，触发当前本体重新观察、澄清或重新验真。

高置信度 L1 不得直接覆盖仍然有效的 L3/L4 事实；同样，旧世界版本的 L4 也不得覆盖当前任务期的新观察。最高可用等级无法产生当前候选时，系统可以使用低等级证据生成候选和澄清问题，但不得自动把它晋升为运行时事实。

## 六、教学舱的标准处理流程

```text
观测源适配器
  -> 时间和参考系归一化
  -> 候选对象、事件和语言对齐
  -> 概念包与语义角色匹配
  -> requires / produces / destroys 实例化
  -> 当前空间、本体和状态重新绑定
  -> P018 判断继续、澄清、暂停或切换
  -> P016 执行和双通道验真
  -> 失败解释、请求补教或恢复
  -> 经验不变量编译
  -> 自主复做和人工验收
  -> 本地可信经验持久化
```

输入源只改变前端证据的获取方式，不改变后端的概念、状态、经验和执行边界。

## 七、与现有方案的关系

- P003：统一不同观测源的空间参考、对象和关系表达；
- P008/P6：提供执行体可进入的空间边界、本体能力和动作限制；
- P016：执行过程、事实跃迁和双通道验真；
- P017：跨空间、跨本体迁移所需的经验不变量；
- P018：运行时状态优先的继续、暂停、切换、澄清和回退；
- P019：端侧概念候选、证据包、角色绑定、概念内化和云脑缺口回退。

教学舱不改变这些层的职责，只把更多类型的观测材料接入同一条受约束的流通逻辑。

### 7.1 教学舱不拥有持久认知更新权

教学舱可以在一次教学会话内保存临时帧、时间对齐、纠正上下文和编译中间状态，但不拥有跨会话的持久认知记忆，也不直接修改公共概念。

失败信息流向如下：

```text
失败观测或负例
  -> 教学舱编译 infeasibility_summary
  -> P019 判断概念适用范围、角色绑定或候选优先级是否需要调整
  -> P017 判断经验适用条件、失败恢复入口或迁移边界是否需要更新
  -> P018 在当前任务期使用有效约束完成仲裁
```

一次失败首先附着在具体经验和具体作用域上。只有达到重复证据、跨实例复现、人工确认或既定晋级规则后，才能反向影响更公共的概念契约。教学舱负责“从观测到结构化候选”的编译，P019/P017 负责“候选如何形成持久知识”的治理。

## 八、工程推进顺序

### 阶段一：结构化第一视角教学

继续使用当前教学舱，完成高频概念、过程契约、P016 验真、经验持久化和冷启动重绑定。

### 阶段二：领夹相机半自动编译

加入视频和语音导入，自动生成事件候选和语言时间对齐；关键语义角色、终止事实和风险边界由人确认。

### 阶段三：真机观测适配

接入本体传感器、控制器反馈和 P6/P008 本体画像，验证同一概念契约在不同真机上的重新落地。

### 阶段四：仿真与真实数据混合验证

使用仿真扩大障碍、遮挡和打断测试，用真实执行验真校正仿真假设，区分仿真候选经验和真实可信经验。

### 阶段五：第三方视频和数据集补给

把第三方数据用于概念先验、候选过程和失败模式补给；所有进入端侧的内容仍需经过当前空间绑定、P018 仲裁和 P016 验真。

### 8.1 跨本体可迁移晋级前的适配预测试

候选经验在被标记为“可跨本体迁移”或进入公共经验池之前，应选择一个与示范本体不同的目标本体，使用其本体画像进行不执行动作的契约预测试：

- `requires` 涉及的可达、抓取、承载、移动和操作能力是否具备；
- `verification` 依赖的视觉、深度、触觉、力矩、流量或其他传感器是否具备，或者是否存在等价验真通道；
- `destroys` 指向的事实类型是否存在于目标本体接入的事实系统；
- 安全、空间准入和治理约束是否存在已知冲突；
- 经验中的过程不变量能否由目标 Adapter 映射为合法的过程候选。

预测试只证明“当前已知能力与经验契约之间没有已知矛盾”，不证明真实执行一定成功，也不生成任何已成立的世界事实。

本测试不是 `trusted_local_experience` 的硬门槛。同一本体完成自主复做、P016 验真和人工验收后，仍可成为本地可信经验；只有当它继续声明跨本体可迁移、进入共享经验池或向公共概念晋级时，才必须通过目标本体预测试。未通过时标记缺失的本体适配信息，进入补教、Adapter 补全或澄清流程。

## 九、验收标准

教学舱不能只以“视频识别准确率”验收，应至少分别统计：

1. 对象候选是否正确；
2. 语义角色是否正确；
3. 目标事实和缺失前提是否正确；
4. 当前空间和本体是否可以重新绑定；
5. P018 是否正确处理澄清、打断和障碍；
6. P016 是否确认了真实结果；
7. 经验是否只保存了可迁移不变量；
8. 在新对象、新空间和新本体上的首次执行是否成功；
9. 无法执行时是否能解释原因并进入补教或恢复流程。
10. 纠正、负例和边界指示是否被编译为有作用域的约束，而未污染正向经验；
11. 多源证据冲突时是否正确处理世界版本、新鲜度和证明责任；
12. 声明跨本体可迁移的经验是否通过了本体适配预测试。

## 十、结论

后续真实视频、领夹相机、真机日志、仿真数据和第三方数据都可以成为教学输入源，但它们不是同一种“教会”。

它们可以共同进入教学舱的候选生成和概念编译流程，最终是否成为可执行经验，必须由当前空间、本体、运行时状态和物理验真共同决定。

教学舱真正实现的是：

```text
多源观测输入
  -> 统一证据表达
  -> 概念契约与因果结构
  -> 当前世界重新绑定
  -> 本体可执行性判断
  -> 物理事实验真
  -> 可迁移经验沉淀
```

这使系统可以逐步从“真人在机器人视角下教学”，扩展到“领夹视频辅助教学”，再扩展到“真机和真实世界数据共同补给”，而不牺牲端侧概念、状态优先和事实验真的边界。

最终，教学舱应同时成为现场机器人与用户之间的师徒接口、离线规模化因果结构编译入口，以及普通人参与机器人训练的统一通道。三者共享同一份概念契约、经验不变量和事实验真规则，从而使学习规模可以扩大，而机器人的事实边界和执行安全不会随规模扩张而失控。
