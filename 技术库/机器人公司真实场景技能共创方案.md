# 机器人公司真实场景技能共创方案

## 一、方案定位

本方案面向机器人公司、具身智能团队、机器人系统集成商和真实场景客户，提供一种轻量、低资本、可持续积累的市场打开方式。

本项目不购买机器人本体、不制造机器人本体、不训练通用机器人基座模型，而是作为机器人公司的真实场景训练与落地伙伴，帮助其将机器人从实验室、展会和演示视频带入真实任务场景，并把每一次真实执行沉淀为可复用技能、经验库和治理证据。

一句话定位：

> 让机器人公司提供机器人，我们提供真实场景训练方法、经验闭环、技能沉淀和运行治理，双方共同把机器人落地。

## 二、机器人公司的现实痛点

当前机器人公司普遍面临以下问题：

1. 机器人本体能力不断增强，但真实场景任务不够清晰。
2. 展示能力强，持续落地能力弱。
3. 能完成单次演示，但难以沉淀为可复用技能。
4. 场景数据采集成本高，且缺少结构化经验回写机制。
5. 真机训练成本高，失败风险高，客户现场容错空间有限。
6. 缺少动作前准入、执行中接管、执行后审计的治理能力。
7. 很多行业客户不是缺机器人，而是缺“可交付、可维护、可复用”的机器人任务能力。

因此，机器人公司真正需要的不只是更多硬件，而是：

> 真实场景、真实任务、真实反馈、可复用技能和可控执行。

## 三、合作价值

### 3.1 对机器人公司的价值

1. 获得真实场景任务样例。
2. 获得可复用技能包。
3. 获得真实执行数据和经验记录。
4. 获得失败补救与偏好反馈记录。
5. 获得可用于客户交付的审计报告。
6. 降低独自摸索场景落地的成本。
7. 增强产品展示、售前、交付和行业方案能力。

### 3.2 对本项目的价值

1. 不承担机器人本体成本。
2. 不承担硬件维护成本。
3. 获得真实场景经验数据。
4. 沉淀行业技能包和经验库。
5. 验证 P010-P015 经验世界模型闭环。
6. 验证运行时治理六项技术。
7. 形成专利实施证据、案例证据和商业合作证据。

### 3.3 对场景客户的价值

1. 获得更贴近真实业务的机器人能力。
2. 降低一次性采购后无法落地的风险。
3. 明确机器人能做什么、不能做什么、如何审计。
4. 让机器人任务从演示走向持续运营。

## 六、六个经验闭环如何参与

### 6.1 空间语义自动生成

把真实场景中的对象、区域、路径、约束和任务相关关系转化为空间语义对象。

### 6.2 物理经验内化

把机器人在真实环境中的每次动作，记录为 `{情境, 动作, 后果}`。

### 6.3 概念归纳跨域迁移

从多个相似任务中抽象出可跨场景复用的概念、步骤和技能模式。

### 6.4 任务语义翻译

把人类自然任务目标转化为机器人可执行、可验证、可审计的任务步骤。

### 6.5 执行恢复

把失败、偏离、卡住、异常和人工接管沉淀为补救经验。

### 6.6 人类偏好

把用户的接受、拒绝、纠正、习惯和禁忌沉淀为偏好层，让技能更贴近真实使用场景。

## 七、运行时治理如何参与

机器人进入真实场景后，动作不应直接从模型输出进入执行端，而应经过运行时治理。

治理链路包括：

1. 计划预适配：判断任务步骤是否越界。
2. 来源链：判断动作来源、任务来源和输入来源是否可信。
3. 代理权预算：判断当前机器人是否还有执行额度。
4. 代理权边界治理：判断高风险动作是否仍在授权边界内。
5. 物理执行链路验真：判断控制指令、设备状态、环境状态和物理后果是否一致。
6. 保护策略声明：面对老人、儿童、病患等受保护主体时调整动作边界。

这使机器人真实训练不是盲目试错，而是：

> 有边界、有记录、有补救、有审计的真实世界训练。

## 八、首批适合共创的场景

优先选择低成本、低风险、可快速验证、能形成技能包的场景。

### 8.1 家庭/办公整理类

任务示例：

1. 识别桌面物品。
2. 将低风险物品归位。
3. 对高敏物品生成候选动作。
4. 根据人类偏好调整整理方式。

适合验证：

- 空间对象语义
- 候选态
- 偏好记录
- 经验回写

### 8.2 园区/场馆巡检类

任务示例：

1. 巡检固定路线。
2. 识别异常对象。
3. 生成告警候选。
4. 记录处置结果。

适合验证：

- 任务语义翻译
- 来源链
- 审计摘要
- 补救经验

### 8.3 仓储辅助类

任务示例：

1. 识别待处理物品。
2. 判断可搬运与不可搬运对象。
3. 对异常对象阻断处理。
4. 形成搬运/分拣技能包。

适合验证：

- 代理权预算
- 物理执行链路验真
- 技能包复用
- 概念迁移

### 8.4 养老陪护辅助类

任务示例：

1. 日常提醒。
2. 异常状态上报。
3. 陪伴任务边界控制。
4. 保护策略下的动作限制。

适合验证：

- 人类偏好
- 保护策略声明
- 审计记录
- 受保护主体治理

## 九、商业模式

### 9.1 项目制共创

按场景收取共创服务费，交付技能包、经验记录和审计报告。

### 9.2 技能包授权

对沉淀出的可复用技能包进行授权，按机器人数量、场景数量或使用周期收费。

### 9.3 经验库授权

对行业经验库进行授权，机器人公司或场景客户可调用经验模板、补救模板和偏好模板。

### 9.4 平台分成

后续建立开放平台后，第三方贡献技能包，本项目提供验证、分发和治理，收取交易抽佣。

### 9.5 私有化部署

为机器人公司或行业客户部署私有经验库、治理引擎和审计系统。

### 9.6 专利许可

对涉及经验闭环、技能复用、运行时治理和空间对象治理的商业使用提供专利许可。

## 十、合作流程

### 阶段一：场景筛选

1. 确定真实场景。
2. 确定机器人类型。
3. 确定任务目标。
4. 确定安全边界。

### 阶段二：任务拆解

1. 定义空间对象。
2. 定义任务步骤。
3. 定义可执行动作。
4. 定义阻断、候选和人工确认条件。

### 阶段三：真实训练

1. 机器人执行任务。
2. 记录情境、动作和后果。
3. 收集失败、补救和偏好反馈。
4. 输出治理和审计记录。

### 阶段四：技能封装

1. 抽象可复用步骤。
2. 形成技能包。
3. 绑定适用场景和限制条件。
4. 生成验证报告。

### 阶段五：复用和授权

1. 在同类场景复测。
2. 调整参数和偏好。
3. 授权机器人公司或场景客户使用。
4. 持续回写新经验。

## 十二、对外沟通话术

可以对机器人公司这样表达：

> 你们提供机器人和基础能力，我们帮助你们把机器人放进真实场景，拆任务、跑闭环、沉淀技能、生成审计，并把一次落地变成可复用的行业技能资产。

可以对场景客户这样表达：

> 我们不是只让机器人来演示一次，而是把任务、经验、失败补救、人类偏好和安全边界一起沉淀，让机器人能力能够持续复用和可控运营。

## 十三、结论

真实场景技能共创是本项目在缺少资本和本体能力时打开市场的有效路径。

它把本项目的弱项避开：

1. 不买机器人。
2. 不造机器人。
3. 不训练通用大模型。
4. 不承担完整硬件交付。

同时把本项目的强项放大：

1. 经验闭环。
2. 技能包沉淀。
3. 运行时治理。
4. 审计证据。
5. 专利组合。
6. 开放平台潜力。

这条路线的核心是：

> 帮机器人公司解决场景和落地问题，同时把每一次落地变成我们自己的经验库、技能包和治理资产。
