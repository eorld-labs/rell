# 分案种子：教学示范因果归因与关键前提识别

## 一、状态

- 当前性质：P020 说明书扩展种子，不编号、不进入 P020 独立权利要求必要主链；
- 是否立项：待针对性检索后决定；
- 主要风险：多示范共同特征统计与 P012 重叠，反事实因果归因方向现有技术密集。

## 二、技术问题

一次教学示范中会同时出现多项事实和动作，其中部分事实是目标结果成立所必需的因果前提，部分事实可由等价条件替代，另一些只是教师习惯、身体结构或当前场景产生的偶然伴随状态。

普通 `requires/produces/destroys/verification` 契约能够表达已知前提和结果，但不必然说明：

1. 哪些前提是不可缺失的因果瓶颈；
2. 哪些前提可以由等价角色或状态替代；
3. 哪些状态只与成功同时出现，但删除后不影响目标结果；
4. 上述判断来自教师解释、对照示范、真实干预还是模型推断。

本种子拟解决：如何从教学示范及其对照证据中形成带证据来源和适用范围的因果归因结构，使经验在复用时优先保持真正必要的因果前提，而不把偶然伴随状态固化为经验不变量。

## 三、不能采用的简化推论

以下推论不成立：

```text
某事实在一次成功示范中出现
  != 该事实是成功的必要原因
```

以下推论也不充分：

```text
某事实在多次成功示范中高频出现
  != 删除该事实必然导致失败
```

多次共现可能来自教师习惯、同一本体结构、场景固定布局或未观察到的共同原因。因果归因至少需要额外的对照、干预、反事实预测、教师显式解释或既有因果契约证据。

## 四、建议的归因类别

对于每个候选前提事实，生成以下状态之一：

- `causally_necessary`：在当前作用域内，移除该前提且其他关键条件保持时，目标结果不成立；
- `functionally_substitutable`：该前提可以由承担等价语义角色或物理功能的另一条件替代；
- `supporting_but_not_necessary`：提高成功率或稳定性，但不是目标事实成立的必要条件；
- `incidental_cooccurrence`：与成功同时出现，但没有足够证据表明影响结果；
- `unresolved`：当前证据不足，不能完成归因。

## 五、归因证据来源

归因至少基于以下一种或多种证据：

1. **教师显式对照说明**：例如“左右手都可以，关键是杯口必须位于出水口下方”；
2. **配对正负示范**：两次示范仅改变一个候选前提，结果分别成功和失败；
3. **受控物理干预**：在安全边界内改变候选前提，并由 P016 或等同执行闭环验真结果；
4. **仿真反事实预测**：在已标记模型有效范围内移除或替换候选前提，预测目标事实是否仍成立；
5. **跨示范不变性与替代性证据**：多条示范中目标结果稳定，而某些角色或动作发生替换；
6. **既有概念/因果契约**：已有验真契约为候选归因提供先验，但不能代替当前作用域复核。

证据强度应区分真实受控干预、仿真反事实、教师说明和统计共现。低等级证据只能形成归因候选，不得直接修改公共概念或经验。

## 七、候选技术主链

```text
获取同一教学目标的成功示范和至少一种归因证据
  -> 提取成功过程中同时成立的候选前提事实
  -> 对候选前提执行教师对照、配对示范、受控干预或反事实预测
  -> 判断前提的必要性、可替代性、支持性或偶然伴随性
  -> 生成带作用域、证据等级和置信度的因果归因结构
  -> 从经验不变量中排除偶然伴随状态
  -> 在新场景中优先验证必要前提，并允许等价前提替换
  -> 执行后根据物理验真结果更新归因状态
```

## 八、与现有方案的边界

### 8.1 与 P020

P020 负责从原始教学输入形成候选经验并完成首次可信晋级。P020 可以记录教师对关键原因的显式说明，或输出归因候选，但不要求每次教学都完成反事实因果识别。

本种子位于 P020 的因果契约实例化之后、经验不变量最终提炼之前，用于进一步区分必要、可替代和偶然事实。

### 8.2 与 P012

P012 已覆盖从多个交互经验记录中提取情境、动作和后果共同特征，统计后果频率或分布稳定性，并形成概念标签及典型经验模式。

因此，本种子不能仅以“多次成功示范中出现频率高”为保护中心。区别点必须落在：

- 对候选前提进行显式干预或反事实对照；
- 输出必要性、可替代性或偶然伴随性的归因结构；
- 将归因结构用于排除伪不变量或选择等价前提。

### 8.3 与 P016

P016 负责受控物理干预后的执行和目标事实验真。本种子可以使用 P016 结果作为归因证据，但不重复保护连续物理过程和双通道验真本身。

### 8.4 与 P017

P017 负责已有经验在新空间、新本体中的适配。归因结构可以帮助 P017 区分必须保持的因果关键事实与可替代条件，但本种子不保护迁移绑定算法。

### 8.5 与 P019

P019 负责端侧概念适用范围和候选晋级。归因结果进入公共概念前仍需 P019/P012 的多证据治理，不能由一次反事实预测直接改写端侧概念。

## 九、保护价值与风险

### 9.1 价值

- 防止把教师的手势、左右手习惯或场景偶然状态写成经验必要条件；
- 提升跨对象和跨本体复用时的约束精度；
- 让经验说明“为什么有效”，而不只说明“做了什么”；
- 为主动试验、对照教学和因果可解释性提供结构化接口。

### 9.2 风险

- 反事实因果推理和因果发现已有大量算法及专利；
- 多示范频率统计与 P012 边界接近；
- 若独权只写“对前提排序”，容易被认为是抽象规则或常规权重计算；
- 若要求真实物理干预，可能增加实施成本和安全风险；
- 单次示范无对照时，因果必要性通常无法被可靠证明。

## 十、建议

1. 不把本环节加入 P020 独立权利要求必要主链；
2. 在 P020 说明书中保留教师显式因果说明、对照示范和归因候选接口；
3. 作为不编号分案种子开展“机器人示教 + 前提干预 + 必要/可替代/偶然事实分类”的针对性检索；
4. 检索重点避开泛化的因果发现，聚焦教学示范、经验不变量净化和角色替代；
5. 在工程上先实现 `unresolved` 和 `teacher_asserted` 两种低风险状态，后续再加入受控反事实验证。
