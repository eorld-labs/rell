# P020 与 P012、P017 边界复核 v1

## 一、复核结论

P012、P017 和 P020 均出现“概念”“经验”和“迁移”用语，但三案应分别保护不同的技术阶段：

```text
原始教学材料
  -> P020：教学编译与经验候选晋级
  -> 已验证交互经验记录集合
  -> P012：跨记录概念归纳与概念驱动复用
  -> 已形成的可迁移经验记录
  -> P017：在新空间、新本体中的绑定、适配、准入和执行
```

P020 的独立权利要求不得以跨空间跨本体迁移执行为保护中心，也不得以多条经验归纳公共概念为保护中心。

## 二、六维边界表

| 维度 | P020 具身智能教学方法 | P012 概念归纳跨域迁移 | P017 真实经验迁移 |
|---|---|---|---|
| 起始输入 | 视频、语音、动作、遥操作、真机/仿真日志等原始教学材料 | 多条已经形成的 `{情境, 动作, 后果}` 交互经验记录 | 一条或多条已经形成且包含不变量契约的待迁移经验记录 |
| 必要数量 | 一次教学会话或一个教学片段即可形成候选 | 必须基于多个交互经验记录提取跨记录共同特征 | 已有经验即可进入迁移，不要求重新从原始教学材料编译 |
| 核心处理 | 教学信令分类、事件-语言对齐、角色槽位、因果契约、正向不变量和失败边界编译 | 从多记录的情境、动作和后果对应关系归纳概念标签及典型经验模式 | 用当前空间语义、本体画像和运行时快照绑定不变量并判断可执行性 |
| 主要输出 | 候选经验、不可行条件摘要、教学证据和晋级状态 | 概念标签、典型情境/动作/后果模式及概念驱动的可迁移经验 | 当前环境绑定候选、执行可行性结果和可交付执行闭环的过程实例 |
| 验证责任 | 自主复做、物理事实验真和必要人类验收决定是否成为可信经验 | 后续多条交互结果更新概念置信度、拆分、合并或待验证 | 在目标环境中完成适配准入，并由执行闭环回写事实和结果 |
| 明确不保护 | 多经验归纳公共概念；既有经验的一般跨空间跨本体迁移算法 | 单次原始视频/语音如何编译教学经验；教学权和自主复做晋级 | 原始示范的时间对齐和教学信令；从教学材料首次形成经验 |

## 三、P020 与 P012 的边界

### 3.1 P012 的必要主链

P012 独立权利要求以以下链路为中心：

```text
多个交互经验记录
  -> 提取情境、动作、后果的跨记录共同特征
  -> 形成概念标签和典型经验模式
  -> 当前情境匹配概念标签
  -> 确定跨空间、跨主体或跨任务的可迁移经验
```

P012 权利要求 6 进一步覆盖：来源主体与当前主体能力不同时，基于概念标签和当前主体能力约束，把来源主体的典型动作参数转换为当前主体可执行动作参数。

### 3.2 P020 必须避开的表述

P020 独权不应写：

- 从多条教学经验中提取共同特征并形成概念标签；
- 基于概念标签确定跨域可迁移经验；
- 基于概念标签把来源本体动作参数转换成目标本体动作参数；
- 根据后续多个经验记录拆分、合并或更新公共概念。

### 3.3 P020 可以保留的概念处理

P020 可以在一次教学会话中：

- 使用既有概念契约解释候选事件；
- 把事件和语言落入既有语义角色槽位；
- 在既有概念无法覆盖时生成 `concept_contract_candidate`；
- 把候选交给 P019 或 P012 决定是否形成端侧概念或跨记录公共概念。

这里的输出是“与当前教学证据绑定的概念契约候选”，不是 P012 所述“基于多个经验记录归纳形成的概念标签及典型经验模式”。

## 四、P020 与 P017 的边界

### 4.1 P017 的必要主链

P017 独立权利要求以以下链路为中心：

```text
获取已经形成的待迁移经验记录及经验不变量契约
  -> 获取当前空间语义、本体能力画像和任务意图
  -> 生成任务期运行时世界状态快照
  -> 将不变量契约绑定到当前空间、本体和快照
  -> 生成绑定候选和执行可行性结果
  -> 执行闭环执行并更新产出/销毁事实
```

P017 权利要求 2-5 已经限定不变量类型、排除绝对坐标和单一本体轨迹，并完成空间映射、本体能力匹配和验真通道匹配。

### 4.2 P020 必须避开的表述

P020 独权不应把以下内容设为必要主链：

- 对已经形成的经验记录执行一般性的跨空间跨本体适配；
- 基于当前空间语义、本体画像和运行时快照生成迁移绑定候选；
- 将经验不变量映射为目标本体可执行动作；
- 以目标空间或目标本体中的执行可行性判断作为本案唯一输出。

### 4.3 P020 可以保留的复做验证

P020 为了证明“教学已经形成经验”可以要求：

1. 结束教师控制权；
2. 丢弃示范对象实例绑定、按键序列和具体轨迹；
3. 在教学验证上下文中重新观察并绑定当前目标；
4. 由机器人自主生成复做过程；
5. 完成目标物理事实验真；
6. 根据验真和必要人类验收决定经验晋级。

该复做是教学经验形成的验证步骤，不主张在任意新空间、新本体中的一般迁移算法。经验晋级后若需要进入不同空间或不同本体，交由 P017 完成。

## 五、P020 独权建议主链

```text
S1 获取原始教学输入并形成带来源、时间、参考系和证据等级的观测证据包；
S2 识别教学信令，并生成与语言时间窗口对齐的候选事件；
S3 将候选事件映射至既有概念契约的语义角色槽位，或形成与当前教学证据绑定的概念契约候选；
S4 实例化 requires/produces/destroys/verification 因果契约；
S5 将成功过程编译为经验不变量，将失败过程编译为带作用域的不可行条件摘要；
S6 撤销教师控制并清除示范实例绑定和具体轨迹；
S7 在教学验证上下文中重新观察目标并由执行体自主复做；
S8 根据物理事实验真和必要人类验收更新候选经验的晋级状态。
```

独权不要求跨本体预测试。跨本体预测试、跨空间重绑定和共享经验池准入仅作为与 P017 组合实施的从属方案或说明书接口。

## 六、三案可组合但不可混同

### 6.1 P020 -> P012

多个经 P020 验真晋级的交互经验可以作为 P012 的输入，由 P012 归纳公共概念和典型经验模式。

### 6.2 P020 -> P017

P020 形成的可信经验包含不变量契约时，可以作为 P017 的待迁移经验，由 P017 在新空间、新本体中适配执行。

### 6.3 P012 -> P020

P012 已形成的概念和典型经验模式可以作为 P020 教学编译时的概念先验，但 P020 不重新主张该概念的跨记录归纳过程。

### 6.4 P012 -> P017

P012 的概念标签可帮助识别新环境中的同类对象或模式；P017 仍负责把具体经验不变量绑定到当前空间和本体。

## 七、撰写红线

1. P020 的发明名称和独权前序应突出“教学输入编译与经验晋级”，不突出“跨域迁移”。
2. P020 的“概念契约候选”必须与当前教学证据绑定，不表述为多经验归纳得到的公共概念。
3. P020 的“重新绑定”限于清除示范实例和轨迹后的自主复做验证；一般跨环境绑定属于 P017。
4. P020 的本体适配预测试降为组合实施接口，不作为独权必要特征。
5. P020 的技术效果优先写数据效率、因果可解释性、失败边界留存和可信晋级，不以跨空间跨本体迁移成功率作为第一技术效果。
