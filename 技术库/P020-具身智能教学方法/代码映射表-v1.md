# P020 具身智能教学方法代码映射表 v1

## 一、映射原则

本表只把当前代码中能够被验证的行为标记为已实现。尚未接入真实视频、音频时间码、GPU 批处理或多源适配器的部分，保留为部分实现或待实现，不以架构文档中的预留结构代替工程证据。

状态含义：

- **已实现**：当前样品有直接代码路径和验证断言；
- **部分实现**：有可运行的最小形态，但输入源、时间精度或适用范围受限；
- **待实现**：只有文档设计或接口预留，尚无可验证运行路径。

## 三、代码与专利边界

### 3.1 当前已经形成的 P020 证据链

```text
start_embodied_teaching
  -> build_live_first_person_observation_packet
  -> build_task_perception_result / ground_task_observations
  -> begin_teaching_control
  -> finish_embodied_teaching
  -> compile_demonstration_experience
  -> begin_learned_replay
  -> step_motion_command + P016 verification
  -> evaluate_learned_replay
  -> persist_trusted_experience
```

### 3.2 P012/P017 不应在本映射中冒充已实现

- 多条经验归纳公共概念、典型情境/动作/后果模式，属于 P012，当前不作为 P020 的已实现特征；
- 已形成经验在新空间、新本体中的一般迁移适配、能力匹配和目标本体执行，属于 P017，当前不作为 P020 的独权完成条件；
- 当前 P020 的 F9 仅指清除示范实例后的教学验证复做，不等同于 P017 的跨环境迁移。

## 四、下一阶段工程补齐顺序

1. 将 F1 从单一第一视角/模拟 RGB-D 扩展为真实领夹视频、真机传感器和离线数据的 `ObservationSourceAdapter` 集合；
2. 将 F3 从 session-window 对齐升级为视频帧时间码、音频时间戳和动作事件区间对齐；
3. 为 F2 增加纠正、边界指示和负例的真实交互入口；
4. 将 F7 的不可行条件从单次候选摘要扩展为带复现次数和证据版本的约束聚合；
5. 与 P017 联调跨本体适配预测试，但不把该功能回写为 P020 独权必要特征；
6. 与 P012 联调多条可信经验形成公共概念的输入接口，但不在 P020 内重复实现概念归纳。

## 五、映射结论

当前代码已经支撑 P020 的核心主链：

```text
教学接入
  -> 候选概念/角色
  -> 因果契约
  -> 不变量与失败边界
  -> 清除示范绑定
  -> 自主复做
  -> 物理验真
  -> 本地可信经验晋级
```

最主要的工程缺口不是经验晋级，而是 F1/F3 的多源输入与精确时间对齐。这两个缺口应作为后续教学舱工程扩展，不应被当前第一视角样品的运行结果掩盖。

## 六、扩展治理归属

| 扩展问题 | P020 当前责任 | 主要下游责任 |
|---|---|---|
| 新教学与已有经验冲突 | 输出来源、差异、教学信令和候选状态 | P019/P012 负责概念或经验分支、合并、待验证；P018 负责运行时冲突暂停 |
| 经验过期、降级和撤销 | 输出最后验证和失败摘要，不直接删除 | P017/P018 根据环境、能力和世界版本失效；治理层处理撤销 |
| 教师信誉和多教师质量 | 记录来源授权、证据等级、负例和验真结果 | 来源链/治理层进行信誉、权重、贡献和恶意输入管理 |
| 教学安全 | 教学权门控、沙箱复做、P016 验真、候选不直接执行 | P018 和治理层负责持续运行时保护与权限裁决 |
