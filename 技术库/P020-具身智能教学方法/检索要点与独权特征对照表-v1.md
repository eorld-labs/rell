# P020 检索要点与独权特征对照表 v1

## 一、检索目标

检索不是确认是否存在“机器人示教”“视频学习”或“跨本体迁移”，而是寻找是否已有单一对比文件或可被合理组合的对比文件，公开了以下完整技术链：

```text
多源教学输入归一化
  + 教学信令分类
  + 事件与语言时间对齐
  + 语义角色和因果契约实例化
  + 正向不变量与失败边界分别编译
  + 当前环境重新绑定
  + 目标本体适配预测试
  + 运行时仲裁
  + 物理验真后分级晋级
```

## 六、新颖性判断规则

1. 单一对比文件必须直接、无歧义地公开独立权利要求全部必要特征，才可能破坏新颖性；
2. 摘要中的“因果”“泛化”“跨机器人”不能替代对具体处理步骤的公开；
3. 仅公开轨迹抽象但未公开失败边界、当前重绑定和物理验真晋级，不等同于 P020 完整链；
4. 仅公开技能下载后的兼容性检查，但未公开从多源教学输入到概念契约的编译，不等同于 P020；
5. 仅公开语言和视频对齐，但输出为动作策略或分类标签，不等同于概念契约和经验不变量。

## 七、创造性判断规则

若多个文件分别公开 F1-F5、F6-F9 和 F10-F12，需要继续判断：

1. 是否存在把这些模块按本方案顺序组合的技术启示；
2. 现有方案是否反而倾向于直接训练策略或复制轨迹；
3. 将失败轨迹改编译为带作用域因果边界是否产生避免重复失败和防止过度泛化的技术效果；
4. 把目标本体预测试、运行时仲裁和物理验真放在经验晋级之前，是否解决候选污染当前事实的问题；
5. 三种教学形态共享统一契约和晋级状态机，是否降低了异构数据接入与跨本体复用成本。

## 九、当前待验证假设

1. 单一现有方案同时覆盖“教学信令、因果契约、不变量编译、失败边界、本体预测试和验真晋级”的概率较低；
2. 风险最大的组合来自“从视频语言生成任务图”与“跨本体示教迁移”的拼接；
3. P020 的创造性强度主要取决于 F6-F12 是否被现有技术给出明确组合启示；
4. 检索完成前，不把“单次示范跨对象、跨空间、跨本体成功”作为无条件技术效果，而表述为满足契约和验真条件后的可迁移能力。
