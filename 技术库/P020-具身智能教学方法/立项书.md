# P020 具身智能教学方法立项书

## 一、拟定名称

一种基于概念契约与因果算子的具身智能教学方法、系统、设备及存储介质。

## 二、立项结论

建议立项。

本方案不是把人类示范轨迹直接用于行为克隆，也不是仅对教学视频进行动作识别，而是把来自现场交互、第一视角视频、语音、真机传感器、仿真日志、外部真实视频或文本语料的教学输入，统一编译为可审计的候选事件、语义角色、概念契约、因果契约、不可行条件和经验不变量，并在目标执行体上完成当前空间重新绑定、本体适配预测试、运行时仲裁和物理事实验真后，分级晋升为可信经验。

该技术主链可以同时支撑：

1. 用户在真实环境中像教人一样教机器人；
2. 通过 GPU 批量处理真实视频、语料、真机或高物理仿真数据；
3. 普通人通过视频流、语音和动作成为机器人训练员。

三种运行形态共享同一教学输入模型、概念契约、经验不变量和事实验真边界，具备独立技术方案的完整性。

## 三、拟解决的技术问题

现有具身智能教学常将人类示范表示为视频帧、动作序列、控制指令、关节轨迹或策略训练样本，导致：

1. 示范中具体对象、绝对坐标和单一本体动作与经验结构混合，换对象、换空间或换本体后难以复用；
2. 语言描述、视觉事件与物理结果缺少统一的因果对应关系；
3. 正例、纠正、边界指示和负例未被区分，失败信息要么被丢弃，要么以原始轨迹形式污染经验库；
4. 离线视频或仿真结果可能被误当作当前机器人世界中的已成立事实；
5. 教学结果缺少从候选经验到可信经验的本体适配、运行时仲裁和物理验真链；
6. 现场教学、GPU 离线学习和大众训练员平台容易形成互不兼容的三套数据与学习体系。

P020 拟解决：如何把异构教学输入统一编译为可跨对象、跨空间、跨本体复用且可被物理验真的结构化经验候选，并在不允许教学输入直接取得执行权的条件下，使候选经验在目标执行体上安全晋升和持续修正。

## 四、核心技术主链

```text
异构教学输入
  -> 生成带来源、时间、参考系和证据等级的观测证据包
  -> 解析对象、动作、状态、语言和空间关系候选
  -> 读取教学信令并区分示范、纠正、边界指示和负例
  -> 对齐视觉事件序列与语言时间窗口
  -> 匹配概念核并落入语义角色槽位
  -> 实例化 requires / produces / destroys / verification
  -> 将失败编译为带作用域的不可行条件摘要
  -> 将成功过程编译为拓扑、物理约束和事实终止条件不变量
  -> 在目标执行体和当前世界中重新绑定
  -> 进行本体能力与验真通道预测试
  -> 进入状态优先的运行时仲裁
  -> 执行并完成数字通道与物理通道验真
  -> 自主复做与必要的人类验收
  -> 晋升为本地可信经验或可迁移经验候选
```

## 五、建议独立权利要求的必要技术特征

独立权利要求不应只写“接收示范并生成机器人技能”，至少应包含：

1. 获取至少一种教学输入，并生成观测证据包，所述观测证据包包含来源标识、时间信息、参考系信息和证据等级；
2. 从观测证据包生成候选事件，并将候选事件与语言教学信号在时间窗口内对齐；
3. 基于教学信令区分首次示范、纠正、边界指示或负例；
4. 将候选事件映射至概念契约的语义角色，并实例化包含前提事实、产生事实、失效事实和验真条件的因果契约；
5. 将经验证的示范过程编译为不含绝对坐标、单一本体关节轨迹和固定执行时长的经验不变量；
6. 将失败结果编译为带对象、空间、本体或世界版本作用域的不可行条件摘要，而不把失败轨迹作为可迁移经验保存；
7. 在目标执行体的当前世界状态中重新绑定语义角色，并依据目标执行体画像检查能力和验真通道；
8. 将编译结果作为候选重新进入运行时仲裁，不允许教学输入直接取得执行权；
9. 在目标执行体执行后，依据物理事实验真结果和必要的人类验收决定经验晋级状态。

其中第 4、5、6、7、8、9 项共同构成区别于普通示教、行为克隆、视频理解和技能分发的保护中心。

## 六、主要技术效果

1. 将示范中的具体轨迹与可迁移因果结构分离；
2. 使同一教学结果可以在新对象、新空间和不同本体上重新落地；
3. 使正例、纠正、边界指示和负例进入不同编译路径；
4. 保留失败的因果边界而不长期保存失败轨迹；
5. 防止视频、语言或仿真候选直接污染当前世界事实；
6. 使现场教学、GPU 批处理和大众训练共享统一经验格式；
7. 使每条经验具备来源、适用范围、验真依据和晋级过程；
8. 使机器人在不会时能够解释缺口、请求补教、自主复做并形成经验。

## 七、与既有方案的边界

- P016：保护物理过程模板、阶段跃迁、连续状态变量和双通道因果验真；P020 使用其验真结果完成教学经验晋级，但不重复保护底层执行闭环。
- P017：保护经验不变量的跨空间、跨本体迁移；P020 保护如何从多源教学输入中编译出该类不变量并完成教学晋级。
- P018：保护任务期运行时世界状态优先的交互仲裁与切换；P020 的候选教学结果必须回到该仲裁链，但不重复保护仲裁状态机本身。
- P019：保护端侧概念形成、角色落地、事实对齐、经验检索和失败回退；P020 保护教学输入如何形成概念契约和经验候选，P019 决定其如何进入端侧持久认知。
- P003/P008/P6：分别提供统一空间表达、本体空间准入和策略预适配边界，作为 P020 当前空间重新绑定和本体预测试的支撑。

## 八、三种实施形态

### 8.1 现场师徒式教学

用户在机器人当前环境中通过第一视角控制、语言、纠正和结果确认完成教学。机器人保持安全与本体边界，教学后自主复做。

### 8.2 GPU 离线规模化编译

GPU 对视频、语料、仿真和真机日志批量进行候选事件提取、语言对齐和契约编译。输出只具有候选资格，进入端侧后必须重新绑定和验真。

### 8.3 分布式大众教学

普通训练员通过手机、领夹相机、控制器或机器人第一视角进行教学。系统记录来源、授权、质量和适用范围，并通过沙箱复做、目标本体适配和分级验真控制晋升。

## 九、当前工程支撑

当前样品已具备：

1. 第一视角真人教学控制；
2. 教学期间持续生效的本体、碰撞和保护边界；
3. 示范动作的事实验真；
4. 示范轨迹向经验不变量的编译；
5. 教学信令结构化；
6. 失败动作向带作用域不可行条件摘要的编译；
7. 候选经验自主复做和人工验收；
8. 本地可信经验持久化；
9. 冷启动后基于当前观察重新绑定，且不复用示范轨迹；
10. 世界版本或策略变化时撤销教学权和复做作业。

## 十、当前检索重点

优先检索以下近似方案：

1. 从多模态人类示范中生成机器人技能或任务图；
2. 从视频和语言中抽取前提、效果或因果图；
3. 跨本体机器人示教和技能迁移；
4. 机器人技能包、技能胶囊和经验共享；
5. 失败示范、负例和安全边界学习；
6. 示范结果在目标机器人上的仿真预测试或运行时验证；
7. 私有经验、云端经验和分布式机器人训练员。

检索判断必须以权利要求特征组合为准，不能因为摘要中出现“视频教学”“因果学习”或“跨机器人迁移”就认定覆盖本方案。
