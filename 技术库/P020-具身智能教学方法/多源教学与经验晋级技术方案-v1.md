# P020 多源教学与经验晋级技术方案 v1

## 一、方案目标

建立一种与教学输入来源无关的具身智能学习架构，使现场用户、离线数据处理系统和普通训练员产生的教学材料，可以进入同一条“观测证据、概念契约候选、因果算子、经验不变量、教学验证复做、运行时仲裁、物理验真、经验晋级”链路。

方案不承诺一次示范无条件获得普适能力。一次示范首先形成候选经验；只有其因果结构、适用范围和验真条件在目标环境中成立时，才取得相应等级的复用资格。

## 四、教学信令

教学信令至少包括：

- `demonstration`：首次示范；
- `correction`：针对已有经验进行纠正；
- `boundary_indication`：指出安全、物理或治理边界；
- `negative_example`：明确指出当前做法不可行；
- `confirmation`：确认目标事实或教学意图。

教学意图和结果状态分开保存。一段教学可以同时属于纠正和边界指示，但只能有一个当前结果状态，例如成功、失败、中断或未知。

不同信令进入不同编译路径：

| 教学信令 | 编译结果 | 禁止行为 |
|---|---|---|
| 首次示范 | 新候选经验 | 直接晋升公共概念 |
| 纠正 | 对既有候选的差异项 | 未验证即覆盖原经验 |
| 边界指示 | 候选物理/安全约束 | 绕过治理层生效 |
| 负例 | 带作用域不可行条件 | 保存为正向过程模板 |
| 确认 | 人类验收证据 | 替代物理验真 |

## 五、概念契约编译

### 5.1 事件候选

信号层可以输出手接近、对象位移、接触、抓取、释放、液体流动、执行体导航和状态变化等候选事件。事件候选必须保留时间范围、来源、置信度和参考系。

### 5.2 语言对齐

自然语言只触发概念解析，不作为概念真实来源。系统把语言片段与同一时间窗口内的事件候选对齐，形成动作、对象、来源、目标、材料、工具和空间区域等语义角色候选。

### 5.3 角色落地

角色分别维护 `mention_status` 和 `grounding_status`。语言明确提及不等于当前世界中已经唯一绑定。多候选、遮挡、观察过期或参考系不明时，继续观察或请求澄清。

### 5.4 因果契约

```json
{
  "operator": "move_object",
  "semantic_roles": {
    "object": {"required_type": "graspable_object"},
    "source_location": {"required_type": "support_surface", "optional": true},
    "target_location": {"required_type": "spatial_target", "optional": true}
  },
  "effect_contract": {
    "requires": ["object_spatially_grounded", "object_within_reach", "gripper_available"],
    "produces": ["object_in_gripper"],
    "destroys": ["gripper_empty", "object_on_support"],
    "verification": ["gripper_closed_around_object", "object_follows_end_effector"]
  }
}
```

`destroys` 表示旧事实不再成立，不表示销毁物理对象。概念编译只形成状态投影，事实提交必须等待执行验真。

## 六、经验不变量编译

### 6.1 正向经验

成功示范不保存完整轨迹，而提取：

1. 拓扑关系不变量；
2. 试探方向和物理约束不变量；
3. 事实终止条件不变量；
4. 类型化对象、本体和空间角色槽位；
5. 失败恢复和补教入口。

禁止进入可迁移经验的字段至少包括：

- 绝对世界坐标；
- 单一本体关节角；
- 固定执行时长；
- 教师按键序列；
- 单次示范轨迹；
- 教学会话、复做作业或临时对象实例标识。

### 6.2 失败边界

失败不保存为负向轨迹，而被编译为：

```json
{
  "infeasibility_summary": {
    "concept_id": "concept_fill_container",
    "failed_requirement": "container_placeable_below_source",
    "failed_verification": "container_aligned_below_nozzle",
    "negative_constraint": "container_height > available_nozzle_clearance",
    "scope": {
      "object_type": "fillable_container",
      "source_type": "fixed_water_dispenser",
      "executor_profile": "current_executor_profile",
      "world_revision": "world_rev_42"
    },
    "evidence": {
      "confidence": 0.94,
      "support_count": 1
    },
    "disposition": "candidate_constraint_pending_revalidation"
  }
}
```

一次失败只形成有作用域的候选约束。扩大到公共概念边界之前，需要重复验证、跨实例复现、人工确认或满足预设晋级规则。

### 6.3 因果归因扩展接口

`requires` 表示当前契约声明的前提，但单次成功示范中的事实共现不能自动证明该事实具有因果必要性。P020 可以记录教师显式说明或对照示范产生的归因候选，将前提暂时标记为必要、可替代、支持性、偶然伴随或未决；反事实干预、多示范归因和公共归因结构形成不作为 P020 独权必要步骤。

详细技术构思见 `分案种子-教学示范因果归因与关键前提识别.md`。其中多记录共同特征统计必须与 P012 切边，跨环境使用归因结构必须与 P017 切边。

## 七、多源证据仲裁

证据等级可表示为：

```text
L4 当前本体执行 + 物理验真 + 必要人工验收
L3 真机多传感器或有来源标记的物理仿真日志
L2 第一视角 RGB-D/IMU/同步语音
L1 单目或第三方视频
L0 文本和语言描述
```

等级表示证据可以承担的证明责任，不是永久覆盖顺序。仲裁还必须检查：

1. 当前任务期和世界版本；
2. 证据新鲜度；
3. 直接观察还是推断；
4. 来源独立性和传感器健康；
5. 适用范围；
6. 同等级有效证据的置信度。

低等级证据不能直接覆盖仍有效的高等级事实，但可以使旧事实进入 `stale` 或 `conflicted`，触发重观察、澄清或重新验真。旧世界版本的高等级证据不得覆盖当前世界的新观察。

## 八、与 P017 组合的目标本体适配预测试

本节是 P020 可信经验形成后的下游组合接口，不属于 P020 独立权利要求的必要主链。P017 以已经形成且包含不变量契约的经验为输入，负责其在新空间、新本体中的绑定、能力匹配、准入和执行。

在经验声明可跨本体迁移或进入共享经验池前，使用目标本体画像进行不执行动作的预测试：

1. `requires` 涉及的可达、抓取、承载、移动和操作能力是否具备；
2. `verification` 所需传感器或等价验真通道是否具备；
3. `destroys` 指向的事实类型是否被目标事实系统支持；
4. 空间准入、安全和治理约束是否存在已知冲突；
5. 过程不变量是否能由目标 Adapter 映射为合法过程候选。

预测试只排除已知矛盾，不产生世界事实，也不保证执行成功。同一本体已经完成复做和验收时，可以先形成本地可信经验；跨本体或公共晋级才强制执行本测试。

## 九、经验晋级状态机

```text
raw_observation
  -> structured_teaching_candidate
  -> candidate_experience
  -> teaching_control_revoked
  -> demonstration_binding_discarded
  -> teaching_context_reobserved
  -> autonomous_replay_verified
  -> trusted_local_experience
```

`trusted_local_experience` 后续进入 `portable_experience_candidate`、跨空间或跨本体执行时，转交 P017 的迁移适配链；多个可信交互经验进一步归纳公共概念时，转交 P012。

任一阶段发生世界版本变化、目标绑定失效、策略变化、能力缺口或验真失败时，旧候选资格被撤销或降级，系统从最后一个已验真的事实重新编译或执行。

## 十、三种运行形态

### 10.1 实时端侧教学

要求低延迟、持续状态绑定、即时澄清和教学权可撤销。适合家庭、工厂和商业现场。

### 10.2 GPU 离线编译

允许大规模并行生成候选事件、候选契约和候选不变量。离线输出不直接取得任何具体机器人的执行权。

### 10.3 分布式训练员

增加训练员身份、数据授权、贡献来源、质量等级、恶意输入隔离、沙箱复做和收益归属等治理信息。训练员拥有候选生成权，不拥有公共概念改写权或机器人安全绕过权。

## 十一、经验私有与共享扩展接口

P020 预留经验流通元数据，但不把经验授权商业模式作为本案保护中心：

- `owner_domain`；
- `visibility`；
- `cloud_export_allowed`；
- `portable_core`；
- `private_overlay_ref`；
- `allowed_executor_profiles`；
- `license_scope`；
- `revocation_version`；
- `derivation_chain`。

该方向可与后续“经验主权与受控流通”保护方案衔接。P020 主要保护经验如何被教出、编译和晋级。

经验版本冲突、经验过期/撤销、跨教师质量聚合和收益归属属于 P019/P012/P017/P018 及来源链治理层的后续职责。P020 只输出来源、教学信令、验证摘要、边界提示和候选状态，并在首次复做和物理验真前持续阻止候选直接取得执行权。

## 十二、第一阶段工程范围

1. 保持现有第一视角教学闭环；
2. 完成教学信令结构化；
3. 完成失败边界摘要；
4. 完成可信经验持久化和冷启动重绑定；
5. 增加领夹视频/语音观测包导入接口；
6. 增加候选事件时间轴和人工确认界面；
7. 增加与 P017 本体适配预测试的接口测试，但不并入 P020 独权主链；
8. 再接入 GPU 批处理和第三方数据适配器。

## 十三、验收维度

- 对象和事件候选准确性；
- 语言时间对齐准确性；
- 语义角色和因果契约正确性；
- 失败边界是否保留作用域；
- 经验是否排除不可迁移字段；
- 清除示范实例绑定后，教学验证复做能否重新观察当前目标；
- 与 P017 组合时，不同本体的已知矛盾能否提前发现；
- 打断、障碍和世界版本变化是否撤销旧计划；
- 物理事实是否经过独立通道验真；
- 新对象、新空间和新本体上的首次执行成功率；
- 无法执行时是否能够解释、澄清和请求补教。

## 十四、分案种子：由教学证据形成对象或动作概念候选

本节作为 P020 的后续分案支撑披露，不并入 P020 主案“原始教学输入到首次可信经验”的必要主链。其拟解决的问题是：如何把一次或少量教学中的多模态证据，编译为可由端侧概念层接收的结构化概念候选，而不是仅形成某次任务的过程经验。

### 14.1 分案种子主链

1. 从视频、语音、动作、触觉、执行日志、仿真数据或外部模型输出中提取对象候选、交互事件、语言提及、结果观察和证据来源；
2. 对语言提及和现实落地分别维护状态，使“教师说到某对象”不等于“当前物理世界已唯一绑定该对象”；
3. 将教学中的对象实例、对象角色、前提事实、结果事实和验真结果编译为逐条声明，而不是给整个概念设置一个不可分解的总置信度；
4. 区分由教学支持的感知锚点、结构或拓扑约束、可变特征、功能候选、情境角色和待验证边界；
5. 从成功与失败教学证据中识别本质约束候选与偶然伴随特征，并保留相应证据来源、适用范围和验证要求；
6. 当现有概念与当前候选具有结构、功能或拓扑相似性时，生成继承项、差异项和待验证项，不把基底概念未经当前证据支持的功能或物理阈值直接继承为事实；
7. 根据未落地角色、证据冲突或声明缺口，生成重新观察、澄清、补充示范或受治理约束的低风险交互验真请求；
8. 将形成的对象或动作概念保持为候选，经人工审核、当前本体能力检查和物理验真后，才允许进入端侧概念库的候选版本或晋级流程。

### 14.2 可输出的结构

教学编译器可以输出以下至少一种内容，而不要求所有实施方式采用固定字段名：

- 感知召回锚点及其证据来源；
- 结构、拓扑和可变特征声明；
- 对象、来源、目标、工具、支撑、容器、材料或障碍等语义角色；
- 语言提及状态、现实落地状态、候选绑定及澄清策略；
- `requires / produces / destroys / verification` 形式或等同形式的事实契约；
- 功能候选与当前本体、任务情境、治理边界之间的适用条件；
- 每条声明的证据状态、适用范围、验证要求和来源引用；
- 类比基底、继承项、差异项、纠正记录和候选版本来源。

### 14.3 外部模型和合成数据边界

外部视觉模型、语言模型、生成图像、真实视频、公共数据集和仿真数据均可作为教学候选生产来源。它们可以用于生成感知锚点、反例、差异假设或因果草案，但不得因来源于模型或数据集就自动形成当前世界事实、已验证功能、量化物理阈值或直接执行权。

合成图像适合扩大外观、姿态、背景和光照变化，用于发现可变特征和构造反例；真实相机观测用于当前对象身份和空间落地；承重、盛液、可抓取、安全施力等功能事实仍需由真实交互、人工验收或等同物理验真支持。

### 14.4 与 P019、P012 和 P020 主案的边界

- P020 主案：从教学输入形成候选经验，并经独立复做和物理验真完成首次可信晋级；
- 本分案种子：从教学证据形成结构化对象或动作概念候选；
- P019：端侧接收概念候选后，完成当前实例落地、事实缺口对齐、本地复用、降级和回退；
- P012：基于多条可信经验或跨主体证据进行公共概念的归纳、合并、拆分和跨域复用。

本分案种子不保护概念库的一般生命周期治理，也不把单次教学直接等同于公共概念。其保护中心是“多模态教学证据经过逐条证据编译和受控验证形成概念候选”的过程。
