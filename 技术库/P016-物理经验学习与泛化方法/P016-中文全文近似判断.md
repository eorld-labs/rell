# P016 中文全文近似判断

> 阶段：中文近似全文复核  
> 输入文件：`中文近似全文.txt`  
> 处理日期：2026-07-06  
> 目的：判断已拉取全文是否破坏 P016 的新颖性/创造性，并为权利要求收口提供依据。

## 一、总判断

全文复核后，当前没有发现单篇文献同时覆盖 P016 的四个核心支柱：

1. P-A：物理动作被定义为包含阶段序列、连续物理状态变量和阈值跃迁条件的结构化过程。
2. P-B：机器人执行体系解耦为必然层、因果层和偶然层。
3. P-E：过程模板通过参数槽和偶然绑定生成不同环境/对象下的物理动作实例。
4. P-H：因果事实通过至少两个独立观测通道分别判断，并在冲突时通过仲裁确定最终成立状态。

但单点近似非常密集。尤其是：

1. `CN116133800A` / `CN115666871A` 已公开“技能模板 + 子任务 + 状态机 + 转换条件”，会直接压缩 P-A 的泛化空间。
2. `CN121893266A` / `CN122231846A` 已公开“参数化技能模板/参数化轨迹模板”，会压缩 P-E 的泛化空间。
3. `CN121960707A` / `CN118709769A` / `WO2025260330A1` 已公开“机器人技能知识表征/层次化知识表示”，会压缩 P-B 的泛化空间。
4. `CN121584889A` / `CN121989250A` / `CN103068535A` / `CN122223975A` 已公开“多源验证、执行验证、失败恢复或事件真实性验证”，会压缩 P-H 的泛化空间。

因此，P016 不应再主张泛泛的“机器人技能模板、状态机、任务分解、参数化模板、多传感器验证”。本案要站稳，必须把保护重心锁定为：

1. 以物理动作执行过程为对象，而不是泛化任务或技能。
2. 阶段关联连续可观测物理量。
3. 跃迁条件基于观测值与阈值比较，并可带持续时长/连续确认窗口。
4. 模板复用的是阶段结构和跃迁逻辑结构，不是轨迹点、动作原语或语义任务。
5. 参数槽由偶然层绑定，至少包括传感器映射、阈值数值、对象引用和偏好阈值。
6. 验证对象是“因果事实成立状态”，不是普通多传感器融合或视觉二值验证。

## 二、红灯全文判断

### 1. CN116133800A：机器人演示学习的技能模板

风险等级：红灯  
命中支柱：P-A、P-E  
近似程度：最高

全文命中点：

1. 权利要求 1 明确公开：接收机器人任务的技能模板；技能模板定义多个子任务的状态机；定义子任务之间的转换条件；执行时机器人通过状态机转换完成任务。
2. 权利要求 2 进一步公开：本地演示数据可以对应多个转换条件。
3. 权利要求 4-6 公开：技能模板可以是硬件不可知的；技能模板为每个子任务指定需要的输入传感器流。
4. 说明书还强调：技能模板可用于不同机器人模型，子任务可引用其他技能模板。

对 P016 的压力：

1. 如果 P016 权利要求 1 只写“阶段序列 + 跃迁/转换条件”，会被该文献强烈近似。
2. 如果 P016 权利要求 8 只写“技能模板 + 参数化复用”，也会被该文献强烈近似。

关键区别：

1. 该文献的“转换条件”是技能模板中子任务之间的切换条件，服务于演示学习和控制策略改进。
2. 未见其将每个阶段绑定到“物理可观测且连续变化的物理量”。
3. 未见其要求跃迁条件由连续状态变量观测值与阈值比较确定。
4. 未见“必然层-因果层-偶然层”三层架构。
5. 未见偶然层提供传感器映射、阈值绑定、对象引用绑定。
6. 未见对“因果事实成立状态”的双通道独立判断和仲裁。

撰写建议：

1. 权利要求 1 必须保留“连续状态变量 + 物理可观测 + 阈值比较 + 阶段跃迁”。
2. 避免把本案称为普通“技能模板状态机”，应统一使用“物理动作的结构化过程定义”。
3. 权利要求 8 的模板应限定为“过程模板”，并写明模板包含阈值标识、对象引用标识、传感器映射等参数槽。

### 4. CN121960707A：基于格兰杰因果检验的机器人技能知识表征

风险等级：红灯  
命中支柱：P-B，边缘触及 P-A  
近似程度：中高

3. 通过格兰杰因果检验确认变量之间的因果关系，构建因果关系网络，形成因果规则器。
4. 说明书强调将连续视频流转化为结构化时序数据，并进行因果验证。

对 P016 的压力：

1. 已公开“阶段变量 + 动作变量 + 因果网络 + 技能知识表征”。
2. 会压缩 P016 中“因果层/因果规则/阶段动作关系”的宽泛写法。

## 五、对权利要求 v2 的具体调整建议

### 5.1 权利要求 1

当前方向是对的，建议继续坚持以下限定：

1. “物理动作的结构化过程定义”优于“技能模板”。
2. “阶段关联连续状态变量”必须保留。
3. “连续状态变量为物理可观测且连续变化的物理量”必须保留。
4. “跃迁条件基于观测值与预设阈值比较”必须保留。
5. “确认窗口”和“失败检测/恢复过程”适合作为从属权利要求保留。

可进一步补强：

1. 增加“当前执行阶段”的实时观测来源可为执行器内置传感器、外部感知传感器或二者之一。
2. 增加“阈值由当前执行环境下的偶然绑定数据提供”的从属限定，以把权利要求 1 和权利要求 8 串起来。

### 5.2 权利要求 8

建议把权利要求 8 作为 P016 的第二独立支点，继续强化三层结构：

1. 必然层：只存物理过程模板的不变阶段结构、连续状态变量类型、跃迁条件逻辑结构。
2. 因果层：存每个物理动作的因果前提事实、因果产出事实、因果销毁事实。
3. 偶然层：存传感器映射、阈值数值、对象引用、偏好附着。

可进一步补强：

1. 明确偶然层绑定值不改变必然层的阶段序列和跃迁条件逻辑结构。
2. 明确同一过程模板在不同对象/环境下生成不同实例，但共享过程结构。

### 5.3 权利要求 5-7

P-H 需要避开多源验证现有技术，建议坚持：

1. 验证对象是“因果事实”，不是普通状态、事件、传感器数据或执行结果。
2. 两个观测通道各自独立生成成立判断。
3. 冲突时通过仲裁策略确定最终成立状态。
4. 验证结果用于阶段推进、中断恢复或过程链继续。

可进一步补强：

1. 把仲裁策略写成可配置：通道优先级、置信度、时间窗口、重采样、人工确认中的至少一种。
2. 把“恢复入口阶段匹配”与连续状态变量状态特征绑定，形成和 P-A 的闭环。

### 8.6 学术检索后的总收口意见

英文学术线进一步支持“母案合墙”的必要性：

1. P-A 单独做，会被 hybrid automata、FSM、BT、phase-conditioned imitation learning 压缩。
2. P-B 单独做，会被 action schema、TAMP、preconditions/effects、skill effect models 压缩。
3. P-E 单独做，会被 parameterized skills、DMP、skill libraries、generalized action plans 压缩。
4. P-H 单独做，会被 multimodal execution monitoring、grasp outcome prediction、failure detection/recovery 压缩。

因此 P016 的母案独权不应追求单堵墙概念，而应追求如下组合：

1. 获取物理动作的结构化过程定义。
2. 过程定义包含阶段序列、连续状态变量类型和阈值跃迁逻辑。
3. 必然层保存上述过程模板的不变结构。
4. 因果层保存动作的前提事实、产出事实和销毁事实。
5. 偶然层保存传感器映射、阈值、对象引用、偏好阈值和执行器资源映射。
6. 执行中由连续物理观测触发阶段跃迁。
7. 关键因果事实由至少两个独立观测通道分别判断。
8. 冲突时由仲裁策略给出最终事实状态。
9. 事实状态用于阶段推进、恢复入口选择和过程链继续。

### 9.2 US20210362331A1：Skill templates for robotic demonstration learning

风险等级：红灯  
命中支柱：P-A、P-E  
近似关系：`CN116133800A` 的英文/海外同族强近似。

全文命中点：

1. 技能模板定义多个子任务的状态机。
2. 子任务之间具有 transition conditions。
3. 技能模板指示哪些子任务需要 local demonstration learning。
4. 每个演示子任务的机器学习模型使用 input sensor streams 生成机器人命令。
5. 说明书明确 skill template 可 hardware agnostic。
6. 示例包括 connector insertion、grasp、force-guided task 等真实接触操作。

对 P016 的压力：

1. “技能模板 + 状态机 + 子任务 + 转换条件”已被明确公开。
2. “传感器流 + 本地适配 + 跨机器人硬件不可知”也已公开。
3. 因此 P016 不能主张泛泛的技能模板、状态机、转换条件或硬件无关技能适配。

区别点：

1. 该文献核心是演示学习和控制策略 refinement。
2. 未见必然/因果/偶然三层架构。
3. 未见过程模板的阈值标识、对象引用、传感器映射由偶然层绑定。
4. 未见因果事实双通道独立验真和仲裁。

v3 收口建议：

1. 避免使用“skill template”作为独权中心词。
2. 使用“物理动作的结构化过程定义/过程模板”。
3. 把阶段跃迁限定为连续物理量观测值与阈值比较。
4. 把阈值来源与偶然绑定挂钩。

### 9.3 WO2016034269A1：Electronic minimanipulation libraries

风险等级：红灯祖先  
命中支柱：P-B、P-E，背景触及经验库  
近似关系：比普通技能库更靠近“经验库/过程链/前提后置条件”。

全文命中点：

1. 公开 electronic minimanipulation libraries，每个 minimanipulation 是 sensing-and-action sequence。
2. 每个 minimanipulation 编码 preconditions，以产生 desired functional results / postconditions。
3. 每个 minimanipulation 引用一组 variables，这些变量可预设或通过 sensing operations 获得。
4. 执行 action sequence 后改变变量值，以表示 postconditions。
5. minimanipulations 可由人类导师重复观察获得，并确定变量可接受值范围。
6. minimanipulations 可组合成严格顺序、并行或偏序的多阶段 end-to-end tasks。
7. 组合由 robotic planning 执行，并考虑 component minimanipulations 的 preconditions/postconditions。
8. case-based reasoning 可从人类、其他机器人或同一机器人过去经验中获取 reusable robotic plans，包括成功经验和失败经验。
9. 权利要求 1 公开 robotic control platform，包括 sensors、actuators、electronic library database、minimanipulations、robotic planning module、robotic interpreter module。

对 P016 的压力：

1. “可复用微操作库 + sensing/action sequence + preconditions/postconditions + variables + planning + past experience”已经非常接近经验库底层思想。
2. 如果 P016 的因果层只写“动作前提/产出事实”，会被该文献和 TAMP 学术线共同压缩。
3. 如果 P016 的过程模板只写“可复用操作序列”，也会被该文献压缩。

区别点：

1. 该文献的核心是 minimanipulation library，主要是预定义/学习的 sensing-and-action sequences。
2. P016 的核心应是物理过程模板的不变阶段结构、连续变量类型和跃迁条件逻辑结构。
3. P016 的三层不是普通 library/preconditions/postconditions，而是按执行稳定性来源拆分：必然层、因果层、偶然层。
4. P016 的偶然层不仅提供变量值，还提供传感器映射、阈值数值、对象引用、偏好附着、执行器资源映射。
5. P016 的因果事实必须通过独立观测通道验真和仲裁，而不是仅由 postcondition 变量更新表示。

v3 收口建议：

1. 因果层不要只写 precondition/postcondition，应写“因果前提事实、因果产出事实、因果销毁事实”并与验真绑定。
2. 过程模板不要写成 action sequence，应写“阶段序列 + 连续状态变量类型 + 跃迁条件逻辑结构”。
3. 经验库不要主张普通“过去成功/失败经验复用”，要主张“偶然绑定和验真结果更新过程实例参数”。

### 9.4 CN113156926B：机器人的有限状态机建立方法

风险等级：红灯  
命中支柱：P-A  
近似关系：形式上非常接近“连续变量 + 阈值 + 阶段切换”。

全文命中点：

1. 应用于机器人的连续跳跃模型。
2. 机器人状态包括加速相、腾空相和减速相。
3. 每种状态划分为规划进入阶段、实际进入阶段、状态持续阶段、状态结束与跳转阶段。
4. 检测不同阶段之间的状态切换，得到状态检测结果。
5. 根据状态检测结果更新机器人状态信息，并控制机器人在各阶段切换。
6. 使用质心竖直方向速度、速度方向、速度变化方向与预设速度阈值判断阶段。
7. 使用机器人足底与地面的竖直方向作用力、作用力变化方向与预设作用力阈值判断阶段。
8. 通过压力传感器或关节末端转矩传感器计算竖直方向作用力。

对 P016 的压力：

1. 该文献已经明确公开“物理机器人过程中的阶段划分 + 连续物理变量 + 阈值判断 + 阶段切换”。
2. P016 权利要求 1 如果只保留“阶段序列 + 连续状态变量 + 阈值跃迁”，会面临该文献的强近似压力。

区别点：

1. 该文献限定在连续跳跃模型和加速/腾空/减速相。
2. 它是机器人运动控制状态机建立方法，不是经验复用方法。
3. 未见过程模板、偶然绑定、因果层和因果事实验真。
4. 未见同一过程模板跨对象/跨环境实例化。
5. 未见因果事实用于过程链继续和恢复入口匹配。

v3 收口建议：

1. 权利要求 1 最好和权利要求 8 的偶然绑定形成交叉限定。
2. 仅以 P-A 单独独权时，应加入“结构化过程定义中的阈值标识由偶然绑定数据提供”。
3. 说明书中应明确 P016 不是特定运动相控制，而是跨物理动作的经验复用过程结构。

### 9.5 CN121696965A：多模态感知与阶段条件的机器人模仿学习

风险等级：橙红  
命中支柱：P-A、P-H 边缘  
近似关系：贴近阶段检测、多模态感知、恢复入口。

全文命中点：

1. 采集视觉、分布式触觉、末端六维力、本体感知传感器数据。
2. 定义任务阶段并生成阶段标签。
3. 结合多模态数据进行阶段边界检测，并将阶段标签编码为阶段嵌入向量。
4. 构建多模态融合 Transformer，输出预测动作序列。
5. 阶段边界检测包括阶段转换检测函数。
6. 基于触觉 L1 范数与进入接触阈值/释放阈值进行接触状态检测。
7. 基于力觉数据进行力矩突变检测。
8. 实时推理包括阶段检测、多模态数据预处理、模型推理、时序集成和安全检查。
9. 实施例中提及恢复模式：任务失败后从特定阶段重新开始，无需从头执行。

对 P016 的压力：

1. “多模态数据 + 阶段边界检测 + 阈值 + 阶段嵌入 + 实时阶段检测”已被公开。
2. “失败后从特定阶段恢复”也有公开痕迹。
3. P016 不能把“阶段检测/恢复入口”写得太宽。

区别点：

1. 该文献核心是模仿学习模型训练和 Transformer 多模态动作预测。
2. 阶段标签用于模型条件输入，不是必然层过程模板。
3. 阈值主要用于接触/运动检测，不是由偶然层绑定到过程模板参数槽。
4. 未见因果事实独立通道判断、冲突仲裁和过程链继续。

v3 收口建议：

1. P016 的恢复入口应写成“通过连续状态变量与各阶段预设状态特征匹配，并结合因果事实验真结果确定”，避免泛泛“从特定阶段恢复”。
2. P016 的多模态感知不应主张模型融合，而应主张独立通道对因果事实成立状态分别判断。

### 9.6 CN118322196A：参数化动作基元的机器人自适应装配

风险等级：橙灯  
命中支柱：P-E  
近似关系：参数化动作原语/强化学习方向近邻。

全文命中点：

1. 手动指定机器人动作原语库，并通过强化学习学习参数以参数化。
2. 参数化动作基元结合人类执行任务的数据、机器学习处理和模型训练获取人类技能。
3. 将技能赋予机器人，实现任务执行拟人化。
4. 学习过程分为基元划分、模型学习和自主执行。
5. 基本动作包括抬起、推动、抓取等人工定义行为。
6. 基元参数是策略网络的学习输出。
7. 说明书提到参数化基元可跨不同机器人、任务和环境通用。

对 P016 的压力：

1. 参数化动作原语和跨任务复用已有公开。
2. P016 不能把“参数化动作/技能复用”作为 P-E 的宽泛主张。

区别点：

1. 该文献核心是 RAPS/强化学习动作原语参数学习。
2. P016 不是学习动作原语参数，而是用偶然绑定填充过程模板参数槽。
3. P016 复用的是过程阶段结构和跃迁逻辑，不是原语动作或轨迹。

v3 收口建议：

1. 权利要求中避免“动作原语库”的语言。
2. 使用“过程模板的参数槽”，并限定参数槽类型为传感器映射、阈值标识、对象引用、偏好阈值、执行器资源映射。

### 9.7 对 P016 v3 的最终影响

这 5 个全文会导致 v3 做三处收紧：

1. P-A 不能只靠“阶段 + 连续变量 + 阈值跃迁”。
   - 原因：`CN113156926B` 已经公开该形式。
   - 处理：把阈值来源、过程模板、偶然绑定、恢复入口和事实验真串起来。

2. P-B/P-E 不能只靠“经验库 + 前提/后置条件 + 可复用操作单元”。
   - 原因：`WO2016034269A1` 已经公开 minimanipulation library、preconditions/postconditions、variables、planning、past experience。
   - 处理：强调三层不是普通库结构，而是按执行稳定性拆分；偶然绑定不是普通变量赋值，而是当前环境下的传感器/阈值/对象/偏好/执行器映射。

3. P-H 不能只靠“多模态阶段检测/失败后恢复”。
   - 原因：`CN121696965A` 已经公开多模态阶段边界检测和从特定阶段恢复。
   - 处理：强调“同一因果事实由独立观测通道分别判断，冲突仲裁，结果用于阶段推进/恢复入口/过程链继续”。

因此，v3 的主独权建议采用组合式写法：

1. 获取物理动作的结构化过程定义，过程定义包含阶段序列、连续状态变量类型、跃迁条件逻辑结构和参数槽。
2. 通过偶然绑定数据填充参数槽，至少包括传感器映射、阈值数值、对象引用和执行器资源映射。
3. 执行过程中实时获取当前阶段连续状态变量的观测值，并基于绑定后的阈值触发阶段跃迁。
4. 每个物理动作声明因果前提事实、因果产出事实和因果销毁事实。
5. 对关键因果事实通过至少两个独立观测通道生成成立判断，冲突时仲裁。
6. 因果事实最终成立状态用于阶段推进、恢复入口确定或过程链继续。
