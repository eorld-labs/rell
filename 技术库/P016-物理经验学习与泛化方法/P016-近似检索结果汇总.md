## 一、本案技术骨架

P016 当前保护重点不是泛泛的“机器人学习”，而是以下四个支柱：

1. P-A：物理动作被定义为结构化过程，过程包含阶段序列、连续状态变量和跃迁/转换条件。
2. P-B：机器人知识组织被解耦为必然层、因果层和偶然层，编排基于因果依赖而非单纯状态匹配。
3. P-E：过程模板通过参数槽和偶然绑定生成不同环境/对象下的过程实例。
4. P-H：因果事实通过两个独立观测通道进行成立状态判断，并在冲突时通过仲裁策略确定最终结果。

后续全文判断应重点问：

1. 是否出现“技能模板/状态机/子任务/转换条件”的组合。
2. 是否出现“阶段 + 连续观测变量 + 条件触发阶段切换”的组合。
3. 是否出现“模板 + 参数绑定/偶然绑定 + 跨对象/跨环境复用”的组合。
4. 是否出现“因果事实/因果依赖 + 过程链/编排”的组合。
5. 是否出现“两个独立通道分别判断同一事实 + 一致性/仲裁”的组合。

## 五、初步结论

1. P016 的最大红灯不是“倒水场景”，而是 `CN116133800A` 和 `CN115666871A` 这类“机器人演示学习的技能模板”。它们已公开“技能模板 + 子任务 + 状态机 + 转换条件”的组合。

3. `CN118709769A/WO2025260330A1` 是 P-B 方向的重要近邻，因为覆盖机器人技能的层次化知识表示。
4. `CN121584889A`、`CN121989250A`、`CN103068535A` 是 P-H 方向的重要近邻，尤其涉及多源验证、视觉/力传感、执行失败和恢复。

## 六、对权利要求的即时调整建议

### 6.1 权利要求 1

为避开“技能模板定义状态机和转换条件”的现有近邻，建议进一步强调：

1. 阶段关联的是“物理可观测且连续变化的物理量”。
2. 跃迁条件基于连续状态变量的观测值与阈值比较。
3. 阈值可以来自偶然绑定，而不是写死在技能模板内。
4. 失败事件与恢复过程应作为从属或备选实施例保留。

### 6.2 权利要求 2

为避开普通“技能层次化知识表示”，建议强调：

1. 必然层存储物理动作的不变阶段结构。
2. 因果层存储动作执行前提、执行产出和销毁事实。
3. 偶然层存储传感器映射、阈值绑定、对象绑定和偏好绑定。
4. 编排引擎基于因果事实产出/消费关系搜索过程链。

### 6.3 权利要求 3

为避开普通“技能模板/参数化轨迹模板”，建议强调：

1. 模板复用的不是轨迹点，而是阶段结构和跃迁条件逻辑结构。
2. 参数槽至少包括阈值标识、对象引用、传感器引用和力阈值。
3. 通过偶然绑定生成过程实例。
4. 同一模板的不同实例共享阶段结构，仅绑定值不同。

### 6.4 权利要求 4

为避开普通多传感器融合/交叉验证，建议强调：

1. 验证对象是“因果事实”的成立状态。
2. 两个通道各自独立生成成立判断。
3. 判断不一致时进入预设仲裁策略。
4. 验证结果用于过程阶段推进、中断恢复或过程链继续执行。
