## 一、对四个问题的结论

### 1. 当前权利要求是否站在未来二十年产业发展站位

当前 `权利要求书-v2.md` 已经比普通机器人技能专利更靠近长期底层，因为它不把保护点押在某一个模型、某一种 VLA、某一种强化学习或某一种本体上，而是押在真实世界操作能力的稳定结构上：

1. 物理动作被拆成阶段。
2. 阶段由连续物理变量观测驱动。
3. 阶段跃迁由阈值/确认窗口触发。
4. 技能可通过必然层、因果层、偶然层进行解耦。
5. 不同环境、不同对象、不同机器人通过偶然绑定实例化。
6. 关键因果事实由多通道验真和仲裁保证。

这个站位可以抵抗“后面出来新模型就过时”的风险。未来模型可能从 VLA 变成世界模型、端到端策略、扩散策略、具身基础模型、神经符号模型或其他范式，但真实落地仍然绕不开：

1. 真实物理过程如何被组织。
2. 当前动作执行到了哪个阶段。
3. 什么观测事实证明阶段完成。
4. 什么条件触发进入下一阶段。
5. 同一经验如何迁移到不同物体和环境。
6. 执行结果如何被可信验证。

需要补强的是：当前权利要求 v2 仍然偏“单机器人单动作执行”和“过程模板组织”。如果要站到二十年，需要在说明书中增加三个未来兼容层：

1. 模型无关层：本方法可接入 VLA、世界模型、强化学习策略、传统控制器或规则引擎，模型只负责生成动作候选或参数候选，不能替代过程结构。
2. 本体无关层：同一过程模板可通过偶然绑定映射到不同机器人本体、传感器和执行器。
3. 经验资产层：阶段结构、阈值、失败条件、恢复入口和因果事实验真记录可沉淀为可复用经验资产。

### 2. 四堵墙作为母案合并，未来再分案是否可行

可行，而且更符合当前 P016 的实际新颖性环境。

原因是单堵墙附近现有技术很密：

1. P-A 附近已有技能模板、状态机、转换条件、奖励状态机、阶段条件模仿学习。
2. P-B 附近已有层次化知识表示、因果技能表征、任务/技能/动作知识库。
3. P-E 附近已有参数化技能模板、参数化动作原语、参数化轨迹模板。
4. P-H 附近已有多源验证、视觉验证、传感器一致性验证、失败恢复。

如果把四堵墙拆开分别申请，单案可能都要面对强近似文献；但四堵墙合并后，母案的新颖性/创造性不是来自单个概念，而来自以下组合：

1. 物理过程阶段结构提供必然层基础。
2. 因果层把动作前提、产出、销毁事实串成过程链。
3. 偶然层把传感器、阈值、对象、偏好映射到当前环境。
4. 双通道验真保证因果事实可被真实世界确认。

因此，母案建议保留一个“连续系统”的总权利要求组，未来再通过分案拆出：

1. 分案 A：过程混合自动机与阶段跃迁。
2. 分案 B：必然/因果/偶然三层经验组织。
3. 分案 C：过程模板与偶然绑定实例化。
4. 分案 D：因果事实双通道验真与恢复入口匹配。

注意：母案中四堵墙要形成单一发明构思，不能像四个并列小发明堆在一起。建议统一母题为：

> 面向真实世界物理动作的经验复用与可信执行方法。

### 4. 是否需要海外学术论文和 WIPO 自主检索

需要。中文专利检索只能证明国内近似环境，不能证明 P016 的全球技术边界。尤其 P016 涉及以下跨国通用技术：

1. Hybrid automata / timed automata。
2. Behavior trees。
3. Task and motion planning。
4. STRIPS/PDDL action schema。
5. Options / skill policies。
6. Dynamic movement primitives。
7. Affordance-based manipulation。
8. Skill libraries and skill effect models。

这些技术不会都以中文专利形式出现。必须用英文术语做变体检索。

## 二、海外检索中需要关注的技术族

### 1. Skill template / robotic demonstration learning

代表文献：

1. `US20210362331A1`：Skill templates for robotic demonstration learning。
2. 中文同族/近似：`CN116133800A`、`CN115666871A`。

风险：

1. 已覆盖技能模板、子任务、状态机、转换条件、传感器流、硬件不可知。
2. 对 P016 的“过程模板/阶段跃迁”构成强近似压力。

区别方向：

1. P016 不是演示学习模板。
2. P016 的阶段由连续物理变量观测驱动。
3. P016 的模板参数槽由偶然层绑定。
4. P016 有因果事实验真。

### 2. Manipulation library / minimanipulation / reusable primitive

代表文献：

1. `WO2016034269A1`：Robotic manipulation methods and systems。

风险：

1. 这类文献可能公开机器人操作库、可复用小操作、电子化 manipulation library。
2. 对“技能库/经验库/动作原语复用”构成祖先型压力。

区别方向：

1. P016 不保护普通操作库。
2. P016 保护物理过程模板的阶段结构、阈值跃迁和偶然绑定。

### 3. Behavior trees / task planning / action schemas

代表方向：

1. Behavior trees for robot control。
2. PDDL/STRIPS action schema。
3. Task and motion planning with preconditions and effects。

风险：

1. 前提/效果、动作编排、失败恢复在学术界非常成熟。
2. 如果 P016 的因果层只写“前提和结果”，会被认为是动作 schema 的常规设计。

区别方向：

1. P016 的因果层必须和物理过程阶段、偶然绑定、因果事实验真结合。
2. 单独“前提/效果”不作为主要创造性来源。

### 4. Hybrid automata / timed automata

代表方向：

1. 混合自动机描述离散状态与连续变量。
2. timed automata / hybrid automata verification。

风险：

1. “阶段 + 连续变量 + guard 条件”的数学形式已有较长历史。
2. P016 若只写过程混合自动机，容易被混合自动机现有技术吸收。

区别方向：

1. P016 不主张抽象混合自动机本身。
2. P016 主张在机器人真实物理动作经验复用中，把连续变量跃迁、三层经验组织、偶然绑定和事实验真合在一起。

### 5. Dynamic movement primitives / parameterized action primitives

代表方向：

1. DMP 技能迁移。
2. parameterized action primitives。
3. trajectory template mapping。

风险：

1. 对 P-E 的“参数化复用”有持续压力。
2. 轨迹模板、动作原语参数化是成熟方向。

区别方向：

1. P016 模板复用的不是轨迹，而是过程结构和跃迁逻辑。
2. 偶然绑定不是轨迹缩放/旋转，而是传感器映射、阈值、对象引用、偏好阈值等运行绑定。

## 四、对 P016 母案写法的建议

### 4.1 母案主线

母案不应写成“四个模块并列”。建议写成：

> 一种面向真实世界物理动作的经验复用与可信执行方法，通过过程模板表达物理动作的不变结构，通过因果事实表达动作间可组合关系，通过偶然绑定适配不同对象、环境和机器人本体，通过独立观测通道验真保证执行结果可信。

### 4.2 独立权利要求建议

建议保留两个主独立权利要求：

1. 方法一：物理动作执行方法，覆盖阶段、连续变量、阈值跃迁、失败恢复、因果事实验真。
2. 方法二：机器人经验组织与复用方法，覆盖必然层、因果层、偶然层、过程模板、偶然绑定、过程链搜索。

后续分案可拆：

1. 过程混合自动机分案。
2. 三层经验组织分案。
3. 偶然绑定实例化分案。
4. 因果事实验真分案。

### 4.3 说明书必须补的长期兼容实施方式

1. 模型接入：VLA、世界模型、强化学习策略、扩散策略、传统控制器均可作为动作候选生成器。
2. 本体映射：不同机器人本体通过偶然层提供传感器映射和执行器资源映射。
3. 经验沉淀：每次执行可沉淀阈值、持续窗口、失败条件、恢复入口、通道置信度。
4. 多行业迁移：制造、家政、仓储、医疗辅助、巡检、服务机器人均可用同一过程框架表达。
