# P016 无效宣告攻防分析

> 对应版本：`权利要求书-v9-定稿.md`、`P016-中文全文近似判断.md`、`P016-竞对规避攻防分析.md`  
> 目的：从授权后无效宣告视角预判对手攻击路径，并提前给出说明书布防和答辩口径。  
> 总体判断：P016 授权后的主要无效风险不在单篇新颖性，而在创造性组合攻击。应通过说明书强调“最终成立状态回流”这一闭环效果，使各技术特征形成相互配合关系，而非简单拼接。

## 一、无效攻击总览

对手可能使用的无效理由主要包括：

1. 新颖性：主张单篇现有技术已经公开全部技术特征。
2. 创造性：以多个现有技术组合，主张本案只是状态机、参数化模板、前提/效果、双通道验证的常规拼接。
3. 不清楚：攻击“过程模板”“偶然绑定”“目标因果事实”“最终成立状态”等术语边界不明。
4. 不支持：攻击权利要求概括过宽，说明书只给了少量场景。
5. 必要技术特征缺失：攻击权利要求未限定具体模型、具体传感器、具体阈值计算方式，无法解决技术问题。
6. 单一性或发明点分散：攻击本案把多个独立方案合在一起。
7. 修改超范围：后续审查或无效答辩中若引入说明书没有明确记载的限定，可能被攻击超范围。

其中最危险的是第 2 项：创造性组合攻击。

## 二、新颖性攻击与防守

### 2.1 攻击路线：用状态机/阶段阈值文献攻击 P-A

可能对比文件：

1. `CN113156926B`：连续跳跃模型、阶段划分、速度/作用力阈值、阶段切换。
2. hybrid automata / finite state machine 相关论文。
3. `CN121696965A`：多模态阶段边界检测、触觉阈值、力觉突变和恢复模式。

对手攻击：

1. 本案“阶段序列 + 连续状态变量 + 阈值跃迁”已被公开。
2. 本案不过是把已有阶段切换用于一般物理动作。

防守要点：

1. V9 主独权不是单独保护阶段阈值切换。
2. 本案的阶段跃迁必须与过程模板、偶然绑定、目标因果事实最终成立状态回流共同工作。
3. `CN113156926B` 是特定连续跳跃控制状态机，不公开“参数槽不包含具体阈值数值/对象/传感器标识”，也不公开“偶然绑定数据填充参数槽生成当前环境实例”，更不公开“从过程模板关联因果事实声明中确定目标因果事实并将最终成立状态用于阶段推进/恢复/后续动作触发”。

说明书布防：

1. 强调阶段跃迁只是闭环中间环节。
2. 明确本案不是特定运动相控制，而是经验复用过程结构。
3. 描述同一过程模板在不同环境下通过偶然绑定生成不同实例。

### 2.2 攻击路线：用技能模板/演示学习文献攻击过程模板

可能对比文件：

1. `US20210362331A1` / `CN116133800A`：skill template、state machine、subtasks、transition conditions、sensor streams。
2. `CN115666871A`：分布式机器人演示学习，技能模板、子任务、传感器输出。

对手攻击：

1. 技能模板、状态机和转换条件已公开。
2. 技能模板可跨机器人硬件复用，也包含传感器流。

防守要点：

1. 本案过程模板不是演示学习模板；其参数槽不包含具体阈值数值、具体对象标识和具体传感器标识。
2. 本案偶然绑定数据至少包括传感器映射、阈值绑定和对象绑定，用于生成当前执行环境下物理动作实例。
3. 对比文件的 local demonstration learning 和 control policy refinement 与本案“过程模板 + 偶然绑定 + 因果事实最终成立状态回流”不同。

说明书布防：

1. 明确本案可以接入演示学习模型，但发明点不在演示学习或控制策略训练。
2. 说明过程模板保留不变结构，偶然绑定承担当前环境适配。

### 2.3 攻击路线：用 minimanipulation library 攻击经验库/因果事实

可能对比文件：

1. `WO2016034269A1`：electronic minimanipulation libraries、preconditions/postconditions、variables、robotic planning、past experience reusable plans。
2. TAMP / action schema / skill effect models 学术文献。

对手攻击：

1. 可复用微操作库、前提/后置条件和变量已公开。
2. 本案只是把 preconditions/postconditions 改名为因果事实。

防守要点：

1. 本案不单独保护前提/效果，而是从过程模板关联的因果事实声明中确定目标因果事实。
2. 本案通过至少两个独立观测通道确定目标因果事实最终成立状态。
3. 最终成立状态回流到阶段推进、恢复执行或后续物理动作触发。
4. `WO2016034269A1` 的 variables/postconditions 更多是动作库和规划表达，不公开本案这种“过程模板实例执行中事实状态回流控制”的闭环。

说明书布防：

1. 写明目标因果事实是执行闭环控制对象，不只是规划标签。
2. 写明最终成立状态回流控制流程，避免被理解为普通后置条件。

### 2.4 攻击路线：用多源验证/执行监控文献攻击验真

可能对比文件：

1. `CN121584889A`：多源数据交叉验证、一致性判定、恢复策略。
2. `CN103068535A`：抓取中力反馈失败判断、第二传感器验证成功夹持。
3. `CN121989250A`：规划-执行-验证闭环，多模态大模型视觉验证。
4. execution monitoring / grasp outcome prediction 学术文献。

对手攻击：

1. 多传感器验证、视觉/力觉验证、执行成功检测已公开。
2. 本案的至少两个独立观测通道没有新颖性。

防守要点：

1. 本案不保护普通多源验证。
2. 本案的验证对象是“从过程模板关联的因果事实声明中确定的目标因果事实”。
3. 验证结果不是仅报警或输出成功/失败，而是用于阶段推进、恢复执行或后续物理动作触发。
4. 说明书中不要把“跨层具身验真”作为本案发明点，以免被自有公开或现有验真技术攻击。

说明书布防：

1. 强调目标因果事实来源。
2. 强调最终成立状态回流。
3. 将验真模块写成可调用既有能力或接口，而不是本案独立创新核心。

## 三、创造性组合攻击与防守

### 3.1 最强组合攻击 A

组合方式：

1. `CN116133800A/US20210362331A1` 提供技能模板、状态机、转换条件、传感器流。
2. `WO2016034269A1` 提供 preconditions/postconditions、变量、规划和经验库。
3. `CN121584889A` 或 `CN103068535A` 提供多源验证和恢复。

对手论点：

本领域技术人员面对机器人执行不可靠问题，会自然地将技能模板、前提/效果规划和多传感器验证组合，得到本案。

防守口径：

该组合缺少本案的核心耦合关系：

1. 对比文件 1 的技能模板用于演示学习，不公开参数槽不包含具体对象/阈值/传感器标识，并由偶然绑定生成当前环境实例。
2. 对比文件 2 的前提/后置条件用于 minimanipulation planning，不公开从过程模板关联的因果事实声明中确定目标因果事实，并将其最终成立状态回流到阶段推进/恢复/后续触发。
3. 对比文件 3 的多源验证用于设备或动作结果验证，不公开其验证对象由过程模板关联因果事实声明确定。

技术效果差异：

本案不是简单提高验证准确率，而是实现：

1. 过程结构跨环境复用。
2. 当前环境下实例化。
3. 执行中阶段推进。
4. 事实状态回流。
5. 恢复或后续动作触发。

这些效果依赖各特征之间的数据流闭环，不能由简单并列组合得到。

### 3.2 最强组合攻击 B

组合方式：

1. `CN113156926B` 提供连续状态变量、阈值和阶段切换。
2. `CN121893266A` 或 `CN122231846A` 提供参数化技能模板/轨迹模板。
3. `CN121696965A` 提供多模态阶段检测和恢复模式。

对手论点：

已有阶段阈值切换、参数化模板、多模态恢复，本案只是扩大应用场景。

防守口径：

1. `CN113156926B` 是特定连续跳跃模型，不涉及模板参数槽和偶然绑定。
2. 参数化技能/轨迹模板关注动作参数或轨迹映射，不关注过程模板中跃迁条件逻辑结构与当前环境绑定。
3. `CN121696965A` 是模仿学习模型训练，阶段标签用于模型条件输入，不是目标因果事实最终成立状态回流控制。
4. 本案的区别不在单个阶段或单个模板，而在过程模板、偶然绑定、阶段跃迁和事实状态回流之间的闭环。

### 3.3 最强组合攻击 C

组合方式：

1. Hybrid automata 提供连续变量和离散跃迁。
2. TAMP/action schema 提供 precondition/effect。
3. Behavior tree/execution monitoring 提供执行监控和恢复。

对手论点：

本案是学术成熟技术在机器人执行系统中的常规整合。

防守口径：

1. 学术框架分别提供形式化建模、任务规划、执行监控，但未给出本案中特定的数据结构耦合：
   - 过程模板具有参数槽；
   - 偶然绑定填充传感器映射、阈值绑定、对象绑定；
   - 阶段跃迁使用绑定后的传感器和阈值；
   - 目标因果事实由过程模板关联声明确定；
   - 最终成立状态用于阶段推进、恢复或后续动作触发。
2. 本案解决的是跨环境物理经验复用和可信回流，不是单纯的控制建模或任务规划。

## 四、不清楚攻击与防守

### 4.1 可能攻击：“过程模板”不清楚

防守：

说明书应定义过程模板为用于描述物理动作不变过程结构的数据结构，至少包括阶段序列、连续状态变量类型、跃迁条件逻辑结构和参数槽。

补强：

说明过程模板可实现为状态图、行为树、任务图、技能图、工作流、DSL、数据库记录等。

### 4.2 可能攻击：“偶然绑定数据”不清楚

防守：

说明书应定义偶然绑定数据为在当前执行环境下为过程模板中的参数槽提供具体绑定值的数据，至少包括传感器映射、阈值绑定和对象绑定。

补强：

列举运行期、部署期、场景加载期、任务编排期等绑定时机。

### 4.3 可能攻击：“目标因果事实”不清楚

防守：

说明书应定义目标因果事实为从过程模板关联的因果事实声明中确定的、在物理动作实例执行过程中需要确认其成立状态的事实。

补强：

说明其可以表示为谓词、键值状态、布尔标志、事件记录、任务状态、动作成功标志或世界状态图谱节点。

### 4.4 可能攻击：“最终成立状态”不清楚

防守：

说明书应定义最终成立状态为基于至少两个独立观测通道生成的成立判断确定的目标因果事实状态。

补强：

明确最终成立状态可包括成立、不成立、未知、待重新观测、冲突需仲裁等状态。

## 五、不支持攻击与防守

### 5.1 可能攻击：权利要求覆盖所有物理动作，但说明书只有倒水

防守：

说明书必须至少写抓取、倒水、插入/装配三个实施例，并列出更多动作类型。

补强：

每个实施例都要体现：

1. 过程模板。
2. 偶然绑定。
3. 连续状态变量。
4. 阶段跃迁。
5. 目标因果事实。
6. 最终成立状态回流。

### 5.2 可能攻击：偶然绑定过宽

防守：

说明书要列出具体数据结构和绑定示例：

1. 阈值标识到具体阈值。
2. 对象引用到识别对象。
3. 传感器引用到当前传感器。
4. 可选执行器资源、坐标系、时间窗口、偏好阈值。

### 5.3 可能攻击：独立观测通道过宽

防守：

说明书要列举通道类型，并说明独立的判断依据：

1. 执行器内置读数。
2. 力觉/触觉。
3. 编码器/电气状态。
4. 外部视觉。
5. 深度/环境传感。
6. 多模态感知模型。

同时说明独立性可以来自传感来源、模型分支、判断逻辑或时间窗口。

## 六、必要技术特征攻击与防守

### 6.1 可能攻击：没有限定具体阈值如何计算，不能解决问题

防守：

本案解决的问题不是具体阈值学习，而是过程模板与当前环境绑定。具体阈值可由人工设置、历史经验、用户偏好、模型估计、环境识别或标定过程提供。

说明书补强：

给出多种阈值来源，并说明不同来源均可作为偶然绑定数据。

### 6.2 可能攻击：没有限定具体机器人控制器，不能执行物理动作

防守：

本案位于过程执行与经验复用层，具体控制器可以是现有控制器、策略模型、VLA、世界模型、行为树或传统控制器。本案的创新不在底层控制器。

说明书补强：

写明动作候选或控制命令可由多种控制器生成，但受过程模板、偶然绑定和最终成立状态约束。

### 6.3 可能攻击：没有限定具体验真算法

防守：

本案不以具体验真算法为创新点；本案要求的是从过程模板关联的因果事实声明中确定目标因果事实，并使最终成立状态回流控制流程。具体验真算法可为现有模块或外部接口。

说明书补强：

写明事实验真模块可以是内部模块、外部服务或已配置验真能力。

## 七、单一性攻击与防守

可能攻击：

过程模板、偶然绑定、阶段跃迁、因果事实验真属于多个发明构思。

防守：

V9 已经不是多独权并列，而是一个最小闭环：

1. 过程模板提供可复用结构。
2. 偶然绑定将结构实例化到当前环境。
3. 阶段跃迁执行实例。
4. 目标因果事实验真确认执行结果。
5. 最终成立状态回流控制下一步。

各步骤之间存在数据依赖和控制依赖，属于一个总的发明构思。

说明书补强：

在发明内容中写明：

> 上述各步骤形成闭环数据流，任一环节缺失均会导致物理动作经验难以跨环境复用、执行阶段难以可靠推进、或执行结果难以回流到后续控制。

## 八、修改超范围风险

后续审查或无效答辩中可能需要加入限定。为避免超范围，说明书必须提前记载：

1. 过程模板的多种实现形式。
2. 偶然绑定的多种绑定时机和数据形式。
3. 阈值的多种类型。
4. 连续状态变量的多种来源。
5. 目标因果事实的多种表示形式。
6. 独立观测通道的多种独立性来源。
7. 最终成立状态的多种回流用途。
8. 事实验真模块可为外部接口或既有能力。
9. 后续物理动作触发可由过程链、VLA、世界模型、行为树或规划器实现。

## 九、建议写入说明书的答辩型段落

### 9.1 闭环不是简单拼接

本发明中，过程模板、偶然绑定数据、阶段跃迁和目标因果事实的最终成立状态并非彼此独立的功能模块。偶然绑定数据用于将过程模板中的参数槽具体化为当前执行环境下的传感器、阈值和对象；绑定后的传感器和阈值用于触发阶段跃迁；阶段执行过程中需要确认的目标因果事实由过程模板关联的因果事实声明确定；目标因果事实的最终成立状态又用于阶段推进、恢复执行或后续物理动作触发。因此，各技术特征之间存在明确的数据依赖和控制依赖。

### 9.2 与普通状态机不同

普通状态机通常将状态和转换条件固定在特定任务或特定控制过程内。本发明的过程模板不包含具体阈值数值、具体对象标识和具体传感器标识，而是通过偶然绑定数据在当前执行环境下生成物理动作实例，从而实现同一过程结构在不同对象、传感器和阈值条件下复用。

### 9.3 与普通前提/效果规划不同

普通前提/效果规划主要用于任务选择或动作编排。本发明中，目标因果事实不仅用于描述动作之间的依赖关系，还在物理动作实例执行过程中被至少两个独立观测通道判断，并将最终成立状态回流到阶段推进、恢复执行或后续物理动作触发。

### 9.4 与普通多源验证不同

普通多源验证通常验证传感数据一致性、设备状态或动作成功概率。本发明的验证对象是从过程模板关联的因果事实声明中确定的目标因果事实，验证结果作为最终成立状态参与物理动作实例的后续控制。

## 十、无效风险等级

| 风险项 | 等级 | 说明 |
|---|---|---|
| 单篇新颖性 | 中低 | 尚未发现单篇覆盖完整 V9 闭环。 |
| 创造性组合 | 中高 | 最可能被状态机/模板/前提效果/多源验证组合攻击。 |
| 不清楚 | 中 | 术语较新，需说明书强定义。 |
| 不支持 | 中 | 需多个实施例支撑物理动作广度。 |
| 必要技术特征缺失 | 中低 | V9 已保留必要闭环，但说明书需说明算法/控制器可替换。 |
| 单一性 | 低至中 | V9 是单一闭环，比 V4 多独权风险低。 |
