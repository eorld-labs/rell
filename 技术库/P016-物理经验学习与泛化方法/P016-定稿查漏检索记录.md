# P016 定稿查漏检索记录

> 日期：2026-07-06  
> 对象：`权利要求书-v9-定稿.md`、`P016-说明书-v2-正式提交稿.md`  
> 目的：在正式提交前围绕 V9 主闭环和说明书新增种子进行查漏检索，判断是否存在单篇覆盖完整闭环或强组合攻击需要进一步修改权利要求/说明书。

## 一、检索结论

本轮查漏未发现单篇文献同时覆盖 V9 的完整闭环：

1. 过程模板定义物理动作不变过程结构。
2. 过程模板包括阶段序列、连续状态变量类型、跃迁条件逻辑结构和参数槽。
3. 参数槽不包含具体阈值、具体对象和具体传感器。
4. 当前执行环境下的偶然绑定数据至少提供传感器映射、阈值绑定和对象绑定。
5. 执行中基于绑定传感器获取连续状态变量观测值，并基于绑定阈值触发阶段跃迁。
6. 从过程模板关联的因果事实声明中确定目标因果事实。
7. 至少两个独立观测通道分别生成目标因果事实成立判断并确定最终成立状态。
8. 最终成立状态回流至阶段推进、恢复执行或后续物理动作触发。

但本轮确认：单点近似和组合攻击仍然较强，尤其是技能模板/状态机、minimanipulation library、连续变量阈值状态机、执行监控、前提/效果学习、确认窗口和技能参数绑定方向。当前 V9 的“最小必要闭环”不建议再放宽。

## 三、红灯/橙灯近似文献复核

### 3.1 US20210362331A1 / CN116133800A：skill template + state machine + transition conditions

来源：Google Patents  
链接：<https://patents.google.com/patent/US20210362331A1/en>

命中点：

1. 公开 skill template，用于 robotic demonstration learning。
2. skill template 定义 state machine、subtasks 和 transition conditions。
3. 示例中 connector insertion skill template 包括视觉流、力传感流、接触力转换条件、失败后回退。
4. skill template 可硬件无关，并可层级组合。

关键摘录位置：

1. skill template 可用于适配机器人和环境，并指定视觉/力感知流：Google Patents lines 231-243。
2. skill template 定义 state machine 和 transition conditions：lines 430-432、607-608、950-958。
3. transition condition 可要求物理接触力，失败时回到前一 movement subtask：lines 445-453、833-836。

风险等级：红灯。

对 P016 的影响：

该文献压缩“技能模板 + 状态机 + 转换条件 + 传感器流 + 失败回退”的宽泛空间。P016 必须坚持：

1. 过程模板不是普通 skill template/state machine。
2. 过程模板包含连续状态变量类型、跃迁条件逻辑结构和参数槽。
3. 参数槽由偶然绑定填充，至少包括传感器映射、阈值绑定和对象绑定。
4. 验证对象是目标因果事实，且最终成立状态回流控制。

结论：不破坏 V9 完整闭环，但会强烈攻击宽泛过程模板/状态机写法。当前说明书已补“区别于普通状态图或行为树”的段落，处理正确。

### 3.2 WO2016034269A1：electronic minimanipulation libraries

来源：Google Patents  
链接：<https://patents.google.com/patent/WO2016034269A1/en>

命中点：

2. 公开 preconditions/postconditions 指向物理世界属性。
3. 公开 case-based robotic learning、参数调整、过去成功计划复用。
4. 公开 minimanipulations 可顺序、并行或偏序组合。
5. 公开 sensory feedback 和 performance monitoring。

关键摘录位置：

1. robotic planning 组合 minimanipulations 时考虑 preconditions/postconditions：lines 233、4169。
2. preconditions/postconditions 指向物理世界属性，软件算法直接影响机器人和物理世界：lines 1008-1009。
3. case-based reasoning 包含动作序列和参数，参数包括距离、力、方向、位置等物理量：lines 812-817、3018-3031。
4. minimanipulations 可顺序、并行或偏序执行：lines 1014、3189。
5. 任务数据包含时序变量、性能度量，执行中有 sensory feedback/performance monitoring：lines 3846-3850。

风险等级：红灯祖先。

对 P016 的影响：

该文献对“经验库、前提/后置条件、变量、过程链、并行/偏序、历史经验复用”形成很强压力。P016 的区别必须落在：

1. 过程模板定义连续物理观测驱动的阶段跃迁结构。
2. 偶然绑定不是普通变量值调整，而是传感器映射、阈值绑定和对象绑定。
3. 目标因果事实需要两个独立观测通道分别判断并确定最终成立状态。
4. 最终成立状态用于阶段推进、结构化恢复或后续动作触发。

结论：不覆盖 V9，但会攻击过程链编排和经验库分案种子。说明书新增“过程链编排引擎、资源冲突、回溯恢复”是必要补强。

### 3.5 CLASP 2026：skill schemas + object parameter binding + skill composition

来源：arXiv  
链接：<https://arxiv.org/html/2606.08169v1>

命中点：

1. VLM 生成 skill schemas，描述 skill parameters、preconditions 和 semantic properties。
2. 执行阶段通过 detected objects 绑定 task parameters。
3. 使用 task-parameterized KMPs 以对象坐标系泛化技能。
4. 支持 skill composition，并用 compatibility constraint 判断组合是否有效。
5. 当技能缺失时触发 active skill acquisition。

关键摘录位置：

1. 摘要公开 schema、parameters、preconditions、parameter bindings 和 skill composition：lines 55-58。
2. 学习阶段生成 JSON schema，执行阶段绑定 detected objects 到 task parameters：lines 66-68、125-127。
3. 执行流程中 skill matching、composition feasibility 和 active acquisition：lines 105-116。
4. skill composition 通过 compatibility constraint 判断兼容性：lines 128-138。

风险等级：橙红，尤其影响 P-E 和过程链/技能组合种子。

对 P016 的影响：

CLASP 很新，且与“技能 schema、对象参数绑定、技能组合”相近。P016 的区别在：

1. CLASP 重点是 task-parameterized movement primitives 和 VLM 选择/组合技能。
2. P016 重点是过程模板中的连续状态变量类型、跃迁条件逻辑结构、参数槽，以及偶然绑定中的传感器映射、阈值绑定和对象绑定。
3. CLASP 未公开目标因果事实由双独立观测通道分别判断并仲裁，也未将最终成立状态回流到阶段推进/恢复入口/后续触发。

结论：不破坏 V9，但建议说明书保留“底层控制器/VLA/世界模型只是动作候选生成器，本案不限定底层控制器”的表述；当前正式稿已保留。

ASPIRE 与 P016 的市场叙事接近，均指向机器人经验沉淀和真实世界能力复用。但 ASPIRE 的重点是 code-as-policy 控制程序生成、失败修复和技能库持续扩展，不是“过程模板参数槽不含具体值 + 当前执行环境偶然绑定传感器/阈值/对象 + 连续物理观测驱动阶段跃迁 + 目标因果事实双通道成立判断 + 最终成立状态回流”的权利要求闭环。

## 五、是否发现单篇覆盖 V9 完整闭环

未发现。

最强单篇近似为 `US20210362331A1` 和 `WO2016034269A1`：

1. `US20210362331A1` 覆盖 skill template、state machine、transition conditions、sensor streams、failure return，但缺少参数槽不含具体值、偶然绑定三类映射、因果事实双通道判断和最终成立状态回流。
2. `WO2016034269A1` 覆盖 minimanipulation library、preconditions/postconditions、variables、case-based experience、sequential/parallel/partial-order composition，但缺少当前执行环境下的传感器/阈值/对象偶然绑定、连续物理观测驱动阶段跃迁、目标因果事实双通道成立判断和最终成立状态回流。
3. `CN113156926B` 覆盖连续变量阈值阶段切换，但缺少经验复用、偶然绑定和因果事实验真回流。
4. `ConditionNET` 覆盖 preconditions/effects execution monitoring 和 recovery，但缺少过程模板偶然绑定、双独立通道仲裁和阶段跃迁闭环。
5. `CLASP` 覆盖 skill schema、object parameter binding、skill composition，但缺少阈值跃迁、传感器映射、目标因果事实双通道判断和最终成立状态回流。
6. `ASPIRE` 覆盖闭环执行引擎、多模态轨迹、失败诊断修复和可复用技能库，但缺少 P016 所要求的过程模板参数槽、偶然绑定三类映射、绑定阈值驱动阶段跃迁以及目标因果事实最终成立状态回流。

## 六、创造性组合风险

最强组合攻击可能为：

1. `US20210362331A1` 提供 skill template、state machine、transition conditions、sensor streams、failure return。
2. `WO2016034269A1` 提供 preconditions/postconditions、variables、case-based experience、sequential/parallel/partial-order composition。
3. `ConditionNET` 或 `CN121584889A` 类文献提供 execution monitoring、多源验证和 recovery。
4. `CN113156926B` 或接触检测论文提供连续变量阈值阶段切换和确认窗口。
5. `ASPIRE` 或同类技能库系统提供闭环执行轨迹、失败修复和可复用技能沉淀。

防守口径：

上述组合仍需事后拼接多个方向，且缺少本案的特定数据依赖和控制依赖：

1. 过程模板中的参数槽不包含具体阈值/对象/传感器。
2. 偶然绑定至少同时提供传感器映射、阈值绑定和对象绑定。
3. 绑定后的传感器和阈值用于连续物理观测驱动阶段跃迁。
4. 目标因果事实来自过程模板关联声明。
5. 至少两个独立观测通道分别生成该目标因果事实的成立判断。
6. 最终成立状态直接回流阶段推进、恢复执行或后续动作触发。

因此，当前 V9 的创造性核心应继续表述为“真实物理动作经验复用与可信执行闭环”，而不是任何单点模块。

1. 普通状态图/行为树与过程模板的区别。

5. 结构化恢复物理动作。

结合本轮查漏检索，建议：

1. 暂不修改 V9 独权。
2. 暂不再向独权加入过程链、确认窗口或结构化恢复，以免把主权利要求变长并引入新近似。
3. 说明书当前支持已经足够作为分案种子。
4. 提交前可由代理人按格式拆分为：说明书、权利要求书、摘要、附图说明。

### 8.3 中文最终查漏结论

本轮中文最终查漏未发现新的单篇破坏性文献。中文侧最强风险仍是已纳入分析的 `CN116133800A/CN115666871A` 技能模板同族方向；新增命中文献主要分布在上层任务规划、状态机控制、行为树装配、多机器人调度、工业控制闭环和数字孪生校验方向。

对 V9 的影响判断：

1. 不建议修改 V9 独权。
2. 不建议把独权扩展为普通技能库、普通任务规划、普通状态机或普通执行监控。
3. 继续坚持“过程模板参数槽不含具体值 + 当前环境偶然绑定三类映射 + 绑定传感器/阈值驱动阶段跃迁 + 过程模板关联目标因果事实 + 双独立观测通道判断 + 最终成立状态回流控制”的完整闭环。
4. 当前正式说明书中新增的数字孪生/仿真/软件智能体段落应保持“轻挂”定位，仅作为物理动作实例的预演、验证、记录或回放，不上升为主发明点。
