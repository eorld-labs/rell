# 面向真实世界自动化执行体的空间经验世界模型与运行时治理底座

## 一、项目总定位

本项目面向自动化执行体从“生成内容”走向“执行动作”的下一阶段需求，建设一套可被 Agent、机器人、车载代理、物联网代理、企业自动化系统和未来物理 AI 调用的空间经验世界模型与运行时治理底座。

本项目不以制造机器人本体为第一目标，也不以发布通用大模型或端到端机器人大脑为第一目标。项目的核心目标是：为能够影响真实数字系统、真实空间、真实设备和真实人的自动化执行体，提供空间理解、经验积累、动作后果预测、计划边界预验真、代理权治理、来源链验证、物理控制验真和审计闭环能力。

可以概括为：

> 让自动化执行体在真实世界中能够理解空间、积累经验、执行任务、修正行为，并在全过程中保持可控、可审计、可治理。

## 二、核心判断

当前业界正在快速推进 VLA、机器人基础模型、世界模型、仿真数据和 Physical AI 平台。上述路线主要回答“模型如何理解、生成或预测世界”以及“机器人如何根据视觉和语言生成动作”的问题。

本项目选择不同但互补的站位：

> 不与大厂正面竞争最大模型，而是建设世界模型进入真实执行场景时必须依赖的空间经验层和执行治理层。

大模型、VLA 或机器人策略模型可以生成候选计划、候选动作或动作参数；本项目负责回答这些动作进入真实系统之前和之后的一组关键问题：

1. 当前空间、对象、区域和权限边界是什么。
2. 过去相似情境下执行过什么动作，产生过什么后果。
3. 当前计划在第几步可能越过预算、时间、资源、数据或物理安全边界。
4. 当前动作的授权来源、任务派生来源、输入影响来源和裁决来源是否完整可信。
5. 当前执行体的代理权预算是否足够，是否应当降权、冻结或重新验真。
6. 当前高风险动作是否仍符合人类真实意图。
7. 当前物理控制指令是否与责任主体、设备状态、环境状态和物理后果边界因果一致。
8. 动作执行后是否实际达到预期后果或预期安全状态。
9. 新后果、新反馈、新补救是否应当回写为可复用经验。

因此，本项目的核心不是“替代 VLA”，而是成为 VLA、Agent 和机器人控制栈进入真实世界时的运行底座。

## 三、统一母问题

现有技术成果可统一到一个母问题下：

> 自动化执行体进入真实世界后，如何在空间中理解、行动、学习、协同，并保持安全、可控和可审计。

该母问题包括两个并行目标：

1. 能力提升：执行体要能够利用空间语义、历史经验、人类反馈和真实后果逐步提升任务执行能力。
2. 运行治理：执行体在调用工具、控制设备、进入物理空间或影响人类时，必须受到来源、授权、预算、计划边界、物理后果和保护策略约束。

项目的价值恰恰来自这两个目标的统一：不是只让自动化执行体变强，也不是只限制它，而是让它在真实世界中可持续、负责任地变强。

## 四、三层技术体系

### 4.1 空间治理与对象秩序层

对应既有技术：P005、P006、P007。

该层来源于记忆空间向多主体复杂空间演化时的治理需求。其核心任务是将三维空间从“可展示场景”升级为“可运行、可授权、可审计的空间秩序环境”。

主要能力包括：

1. 空间对象实体化与统一治理。
2. 空间对象、区域、关系和制度规则的挂载。
3. 对象状态、区域规则、权限边界和治理因子的表达。
4. 运行主体的主动性等级控制。
5. 候选态、正式态、确认流程和审计链。
6. 多主体、多对象、多区域下的冲突识别和治理裁决。

该层在早期不需要完整实现复杂多机器人治理系统，但必须从底座第一天开始埋入字段、对象结构和接口钩子。否则后期接入机器人、Agent 或物理 AI 时，会缺少动作能否发生、由谁确认、落入何种状态、如何审计的基础。

### 4.2 空间经验世界模型层

对应既有技术：P010 至 P015。

该层来源于记忆空间向具身智能和世界模型方向的自然延伸。记忆空间沉淀的空间语义数据，不应只服务于展示或回忆，而应成为执行主体进入空间、理解空间、使用空间和积累经验的先验。

主要能力包括：

1. 基于外部视觉输入生成空间语义先验，并在端侧内化为主体侧空间认知模型。
2. 基于真实交互积累 `{情境, 动作, 后果}` 经验记录。
3. 基于经验记录进行候选动作后果预测和经验自进化。
4. 从多条交互经验中归纳概念标签和典型经验模式，实现跨空间、跨主体迁移。
5. 基于人类意图、情境、历史任务计划、执行结果和人类反馈类比形成当前任务计划。
6. 在执行中发生偏离时，基于 `{偏离情境, 补救动作, 补救结果}` 进行不中断补救切换。
7. 基于 `{情境, 动作, 人类偏好指示}` 建立独立偏好层，并与约束可行性判断协同调用。

该层是本项目区别于普通空间数据库、普通机器人控制系统和普通 Agent 平台的能力核心。它使执行主体不是一次性执行命令，而是在真实环境中持续积累、迁移、修正和适应。

### 4.3 运行时执行治理层

对应新增治理技术：代理权边界治理、物理执行链路验真、代理权预算、来源链、计划预适配、保护策略声明。

该层解决 Agent、机器人、车载代理、IoT 代理或企业自动化系统在真实环境中安全可控运行的问题。

主要能力包括：

1. 人类真实意图验真：判断高风险动作是否仍落入人类真实、当前、知情、可感知且可负责的授权边界。
2. 物理控制因果验真：判断物理控制指令是否与控制责任主体状态、设备状态、环境状态和物理后果边界因果一致。
3. 代理权预算治理：将静态授权转化为可计算、可消耗、可撤销、可传播收缩的代理权预算。
4. 动作来源链完整性验证：将动作授权来源、任务派生来源、输入影响来源、工具调用来源和裁决来源转化为执行准入条件。
5. 执行计划边界预验真：在多步计划执行前判断第几步会越过预算、时间、资源、数据、物理安全或合规边界，并触发重新规划。
6. 保护策略声明：面向未成年人、老人、被监护人、病患、异常状态用户等受保护主体，跨内容、广告、消费、AI 回答、工具调用、设备控制和具身动作调整行为边界。

该层应当被定位为“运行时执行治理基础设施”，而不是泛泛的伦理治理或合规宣言。其输出应当是机器可解析、可执行的裁决令牌、预算对象、来源链对象、计划预验真结果、控制裁决对象、保护策略声明对象和审计摘要。

## 五、连续执行链路

本项目可形成如下统一执行链路：

1. 获取空间语义、对象关系、区域规则和治理字段。
2. 接收人类意图、任务目标、外部事件或自动化触发条件。
3. 形成候选任务计划或候选动作。
4. 对多步计划进行步骤级边界预验真。
5. 对动作来源链进行完整性和可信性验证。
6. 对代理权预算进行适配、扣减、降权或冻结判断。
7. 对高风险动作进行人类真实意图验真。
8. 对物理控制动作进行因果一致性和安全接管判断。
9. 根据治理结果生成裁决令牌或控制裁决对象。
10. 执行动作、调用工具、控制设备或影响空间对象。
11. 采集实际后果、执行反馈、人类反馈和补救结果。
12. 将新记录回写为空间经验、任务经验、偏好记录、补救经验和审计摘要。

该链路体现了本项目的根本差异：动作不是直接从模型输出到执行端，而是经过空间经验和运行时治理底座，形成“计划前可预判、动作前可准入、执行中可接管、执行后可回写”的闭环。

## 六、产品化方向

本项目的早期产品不宜命名为“机器人大脑”或“通用世界模型”。更适合的商业表达包括：

1. 空间经验世界模型 SDK。
2. 自动化执行治理网关。
3. 机器人长期经验与环境适配中间件。
4. Agent 动作来源链与代理权预算服务。
5. 多步计划预验真服务。
6. 物理控制因果验真模块。
7. 空间对象治理与审计运行时。
8. 面向物理 AI 的执行治理底座。

对外可概括为：

> 为 Agent、机器人和物理 AI 提供可调用的空间经验、动作准入、计划预验真、代理权治理和审计闭环能力。

## 七、市场价值

未来越来越多系统会从“建议型 AI”走向“执行型 AI”。凡是自动化执行体能够调用工具、修改资源、外发数据、发起支付、部署代码、控制设备、进入空间或影响人类，都需要类似本项目的底座能力。

潜在客户包括：

1. 机器人公司：需要空间经验、任务经验、失败补救、人类偏好和环境适配能力。
2. Agent 平台公司：需要工具调用治理、来源链、代理权预算、多步计划预验真和审计能力。
3. 企业自动化客户：需要财务 Agent、云运维 Agent、代码 Agent、数据 Agent 的动作准入和任务链治理。
4. 智能家居和 IoT 公司：需要设备控制授权、空间对象权限、物理控制验真和受保护主体策略。
5. 车载代理和低空设备公司：需要高风险物理控制指令因果验真和安全接管闭环。
6. 医疗、养老、教育、家庭空间、园区、工厂、仓储等行业客户：需要执行体在复杂空间中安全服务人的能力。

商业模式可包括：

1. SDK/API 授权。
2. 私有化部署。
3. 按执行体、场地、任务量或 API 调用订阅。
4. 行业解决方案交付。
5. 经验库和治理规则库授权。
6. 专利/IP 授权。
7. 与机器人公司、Agent 平台或物理 AI 公司联合产品分成。

## 八、社会价值

本项目的社会价值不是抽象宣言，而是直接来自自动化执行体进入真实世界后的风险降低：

1. 降低 Agent 越权调用工具、外发数据、修改生产系统或错误支付的风险。
2. 降低机器人、车辆、IoT 设备执行危险物理动作的风险。
3. 保护老人、儿童、患者、被监护人和异常状态用户。
4. 让高风险自动化动作具备来源可追溯、边界可验证、后果可审计的技术痕迹。
5. 让人类在自动化执行系统中保持最终主权控制边界。
6. 让 AI 和机器人不是只追求能力提升，而是在可控、可审计和可恢复的框架下进入真实社会。

可对外表达为：

> 本项目不是阻止 AI 执行，而是让 AI 在真实世界中负责任地执行。

## 九、落地节奏

### 阶段一：总纲与对象体系

目标：统一母项目口径，完成核心对象定义和接口草案。

产出包括：

1. 项目总纲。
2. 总架构图。
3. 统一术语表。
4. 核心对象模型：空间对象、经验记录、任务记录、偏好记录、补救记录、来源链对象、代理权预算对象、计划预验真对象、裁决令牌、审计摘要。
5. API 草案和数据样例。

### 阶段二：最小工程底座

目标：不做大模型，先做可运行的数据结构、接口和记录闭环。

最小链路包括：

1. 空间语义输入。
2. 情境生成。
3. 候选动作或候选计划输入。
4. 计划预验真。
5. 来源链校验。
6. 代理权预算裁决。
7. 动作执行记录。
8. 后果与反馈回写。
9. 审计摘要生成。

### 阶段三：真实数字环境 Demo

目标：证明项目不是理论系统，而是已经能在真实数字环境中形成执行闭环。

可选场景：

1. 文件系统或内容空间管理。
2. 3D 空间对象编辑。
3. 智能家居 API 控制。
4. 云资源或企业系统模拟运维。
5. Agent 工具调用治理。

重点不是任务多复杂，而是展示：

1. 计划执行前可预验真。
2. 动作来源可追溯。
3. 代理权预算可扣减和冻结。
4. 动作后果可回写为经验。
5. 审计摘要可复核。

### 阶段四：轻量物理执行体接入

目标：用低成本硬件验证物理 AI 方向，而不是一开始承担机器人本体研发。

可选接入：

1. 智能灯、门锁、传感器、插座等 IoT 设备。
2. 小车或低成本机械臂。
3. 简单空间传感器。
4. 仿真环境加少量真实设备。

验证重点：

1. 物理动作前的因果验真。
2. 安全接管后的实际状态反馈。
3. 执行偏离后的补救记录。
4. 人类偏好与保护策略对动作边界的影响。

### 阶段五：对外合作与行业试点

目标：形成可被机器人、Agent 和物理 AI 公司调用的基础设施能力。

试点方向：

1. 与机器人公司合作，提供空间经验和执行治理接口。
2. 与 Agent 平台合作，提供来源链、预算和计划预验真服务。
3. 与智能家居或 IoT 公司合作，提供设备控制验真和保护策略声明。
4. 与养老、教育、园区、工厂或仓储场景合作，建设行业示范。

## 十、研发原则

1. 不先追求大模型规模，先追求真实执行闭环。
2. 不先制造机器人本体，先做可被机器人和 Agent 调用的底座。
3. 不把治理做成口号，治理对象必须机器可解析、可裁决、可审计。
4. 不把经验库做成普通日志，经验必须能参与后果预测、计划形成、补救和偏好调用。
5. 不把空间语义只用于展示，空间语义必须进入任务、动作、约束和治理链路。
6. 不孤立讲专利点，要把专利点放回连续系统。
7. 先用真实数字环境跑通，再接轻量物理执行体，再进入更完整的物理 AI 场景。

本项目可以将现有技术成果统一为三个技术域。

### 11.1 空间治理与对象秩序

包含：

1. P005 空间对象制度流转方法。
2. P006 空间对象治理。
3. P007 空间制度运行。

作用：为真实空间中的对象、区域、规则、主体、权限、确认、候选态和审计提供秩序层。

### 11.2 空间经验世界模型

包含：

1. P010 空间语义自动生成。
2. P011 物理经验内化机制。
3. P012 概念归纳跨域迁移。
4. P013 任务语义翻译。
5. P014 执行恢复。
6. P015 人类偏好。

作用：为执行主体提供空间先验、真实交互经验、后果预测、概念迁移、任务类比、执行补救和人类偏好适配能力。

### 11.3 运行时执行治理

包含：

1. 代理权边界治理。
2. 物理执行链路验真。
3. 代理权预算。
4. 来源链。
5. 计划预适配。
6. 保护策略声明。

作用：为自动化执行体在真实数字环境和真实物理环境中的高风险动作提供来源、预算、计划、人类意图、物理因果和保护策略治理。

## 十二、项目一句话

本项目建设面向真实世界自动化执行体的空间经验世界模型与运行时治理底座，使 Agent、机器人和物理 AI 能够在真实空间中理解对象、积累经验、执行任务、适应人类，并在动作来源、代理权预算、计划边界、物理后果和保护策略约束下安全可控地运行。
