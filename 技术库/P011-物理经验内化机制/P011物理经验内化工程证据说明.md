# P011 物理经验内化工程证据说明

## 一、中文摘要
本工程证据包用于固定 P011“物理经验内化机制”的最小可运行闭环。证据链通过独立校验脚本和结构化 JSON 输出，逐段证明：系统能够将真实教学形成的经验沉淀为`{情境, 动作, 后果}`三元结构；经验记录中不仅保留过程链，还保留因果签名和经验不变量契约；边教边动过程中，系统会依据当前任务期状态快照主动反馈缺失前提，而不是猜测执行；当经验形成后，后续任务能够优先从经验库中复用该经验，并将经验中的后果约束转化为当前任务的预期后果与过程链来源；同时，经验库明确剥离绝对坐标、机器人专用关节角和固定执行时长等不可迁移细节，只保留可跨空间复用的经验约束。

本证据包的重点不是展示页面效果，而是把 P011 的核心工程事实收束成可重复运行、可落盘、可复核、可直接对应权利要求步骤的输出。

## 四、证据文件索引
### 1. evidence_index.json

用途：机器可读的证据索引。  
对应内容：列明每个证据文件对应的技术特征、权利要求步骤、关键字段和代码来源。

### 2. 01_experience_triplet_structure.json

用途：证明经验记录采用情境、动作、后果三元结构。  
对应步骤：获取基于真实交互积累的经验模型记录。  
关键字段：
- `dialogue_experience.context`
- `dialogue_experience.action`
- `dialogue_experience.outcome`
- `dialogue_experience.causal_signature`
- `dialogue_experience.invariant_contract`

### 3. 02_dialogue_teaching_creation.json

用途：证明对话教学可直接形成结构化经验记录并入库。  
对应步骤：将本次情境、目标动作和实际后果作为新记录反馈入经验模型。  
关键字段：
- `dialogue_teaching.decision`
- `dialogue_teaching.experience.context.human_intent_ref`
- `dialogue_teaching.experience.action.parameters.source`
- `dialogue_teaching.experience.outcome.outcome_type`

### 4. 03_stepwise_teaching_feedback.json

用途：证明边教边动时，系统会基于当前任务期状态反馈缺失前提，并在成功后固化经验。  
对应步骤：执行目标动作并将真实交互结果反馈入经验模型。  
关键字段：
- `blocked_step.step_feedback`
- `executed_steps.step_feedback`
- `finished_session.experience_result.experience.process_chain`
- `finished_session.release_result.release_status`

### 5. 04_experience_reuse_reasoning.json

用途：证明后续任务能够优先复用经验库中的历史经验。  
对应步骤：基于经验模型对候选动作在当前情境下的预期后果进行预测。  
关键字段：
- `reused_run.intent_translation.causal_plan.reasoning`
- `reused_run.intent_translation.candidate_process_chain`
- `reused_run.audit_summary.outcome`

### 6. 05_non_transferable_field_stripping.json

用途：证明经验库明确区分可迁移经验约束与不可迁移执行细节。  
对应步骤：经验模型中的记录区别可迁移经验约束与不可迁移执行细节。  
关键字段：
- `dialogue_experience.invariant_contract.storage_policy`
- `dialogue_experience.invariant_contract.forbidden_storage`
- `dialogue_experience.invariant_contract.invariant_dimensions`

### 7. 06_experience_library_snapshot.json

用途：证明经验库中持续保留多条可复用经验记录，并伴随概念候选等后续内化线索。  
对应步骤：更新后的经验模型可供后续任务检索使用。  
关键字段：
- `experience_library.experiences`
- `concept_candidate_library.concept_candidates`
- `latest_dialogue_experience.experience_id`
- `latest_stepwise_experience.experience_id`

## 五、典型审查质疑对应
### 1. 被质疑只是在保存教学文本，而不是形成可复用经验
可引用证据：
- `01_experience_triplet_structure.json`
- `02_dialogue_teaching_creation.json`

答复重点：P011 不是保存原始话术，而是把教学结果沉淀为包含情境、动作、后果、因果签名和不变量契约的结构化经验记录。

### 2. 被质疑经验复用其实还是规则求解
可引用证据：
- `04_experience_reuse_reasoning.json`

答复重点：后续任务的因果求解链中明确出现 `source = experience_library`，表明系统优先复用了历史经验记录，而不是只依赖固定过程模板。

### 3. 被质疑边教边动并没有真实反馈闭环
可引用证据：
- `03_stepwise_teaching_feedback.json`

答复重点：系统在缺失前提时返回 `needs_more_teaching` 和 `missing_before_step`，在目标事实成立后才固化经验并释放任务期快照，形成了完整反馈闭环。

### 4. 被质疑“不保存绝对坐标等字段”只是负面限定
可引用证据：
- `05_non_transferable_field_stripping.json`

答复重点：经验记录采用“只保留可迁移经验约束”的积极存储策略，明确把绝对坐标、关节角和固定执行时长排除为复用必要内容，以支持跨空间复用。
