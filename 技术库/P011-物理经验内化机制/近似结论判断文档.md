## 二、D1（CN118836863A）对比分析

### 2.1 概述

{场景抽象信息, 可行路径} → 存储为历史经验 → 新场景通过相似度指标(SIM)检索最相似历史场景 → 获取对应的历史经验 → 启发当前场景路径规划。

### 2.2 权利要求对比

| 对比维度 | D1 | P011 |
|----------|-----|------|
| 经验结构 | **两元**：{场景，路径}——无后果描述 | **三元**：{情境，动作，后果} |
| 储存 | 云端数据库 | 经验模型（数据库或等同结构） |
| 检索 | 相似度指标SIM（障碍物向量欧氏距离） | 类比推断 |
| 失败处理 | 相似度不够则重新规划并存储为新经验 | 从权4：只修正与本次情境相关的记录 |
| 跨主体共享 | 多台机器人的经验存储在云端 | 从权6：通过通信网络共享 |
| 真实vs仿真 | 来自实际采摘作业 | 从权7：明确区分真实交互 |

### 2.3 与P011各项权利要求的对比

| P011权利要求 | D1是否公开 | 判断 |
|-------------|:--:|------|
| 独权1（经验情境→类比预测→确定动作→执行反馈） | ❌ | D1的经验结构缺少**后果维**，其"历史经验"仅包含{场景，路径}，不包含"动作执行后的后果"。P011的三元组以后果预测为核心，D1复用的是路径本身而非后果预测 |
| 从权2（类比推断不基于第一性原理规则计算） | ✅ | D1使用SIM相似度指标进行场景匹配，属于基于相似度的类比方式。但D1的应用场景是路径规划而非动作后果预测 |
| 从权3（跨场景迁移） | ✅ | D1的云端经验库支持不同果园场景的经验复用 |

### 2.4 结论

**不构成对独权1的新颖性破坏。** D1的经验结构缺少后果维度——P011的核心是"用经验预测动作后果"，D1是"用经验复用路径本身"。

**从权4（只修正一条）是对D1最强的不侵权区分点。** D1的数据库维护策略是整体置换，不存在记录粒度的修正。

## 三、D2（CN114161419A）对比分析 ⚠️ 最威胁

### 3.1 概述

建立情景记忆模型M（四元组`<ot, at, rt, ot+1>`）→ 余弦距离检索匹配事件 → 情景记忆引导深度强化学习动作选择 → 更新情景记忆。

### 3.2 权利要求对比

| 对比维度 | D2 | P011 |
|----------|-----|------|
| 经验结构 | **四元组**：{场景编码, 动作, 奖赏值rt, 下一场景} | **三元组**：{情境，动作，后果} |
| 检索方式 | 任务编码器编码→余弦距离 | 类比推断 |
| 决策方式 | 策略网络生成aπ + 记忆提取am → 混合决策 | 纯基于经验预测选动作 |
| 更新方式 | 四元组添加到记忆M | 只修正与本次情境相关的记录（从权4） |
| 后果形态 | **二值奖赏rt∈{0,1}** | 结构化后果描述 |
| 经验来源 | 机器人真实交互 | 真实交互 |

### 3.3 与P011各项权利要求的对比

| P011权利要求 | D2是否公开 | 判断与防御说明 |
|-------------|:--:|------|
| 独权1（经验情境→类比预测→确定动作→执行反馈） | ⚠️ | D2的四元组`{ot, at, rt, ot+1}`在结构上最接近P011的三元组。但P011的"后果"与D2的"奖赏值rt"存在范围差异：rt∈{0,1}为二值布尔，而说明书应将P011的"后果"定义为结构化后果描述（如"杯子掉落""抽屉卡住""路径偏移3cm"） |
| 从权2（类比推断不基于第一性原理规则计算） | ⚠️ | D2使用余弦距离进行相似度检索——这属于类比检索。但D2的最终动作选择融合了RL策略网络的输出aπ，存在神经网络计算的混合。P011是**纯类比**，不混合模型推理 |
| 从权3（跨场景迁移） | ✅ | D2的记忆模型M存储不同场景的事件 |
| 从权4（自进化——仅修正一条） | ❌ | **关键差异。** D2的更新是将四元组"添加到情景记忆模型M中"，每次添加M即改变。P011从权4要求"仅修正与本次情境相关的历史记录或其权重，而不修改无关记录"——D2没有这个粒度控制 |
| 从权5（影子执行） | ❌ | 无 |
| 从权6（经验共享） | ❌ | 无跨主体共享 |
| 从权7（真实交互≠仿真） | ⚠️ | D2使用的是机器人真实交互数据，与其RL环境融合。需确认D2说明书是否明确区分"来自真实物理交互" |

### 3.4 说明书补丁建议 ⚠️

在说明书中加入以下区分措辞：

> 现有情景记忆引导的强化学习方法中，经验通常以四元组`{状态, 动作, 奖赏值, 下一状态}`的形式存储于记忆模型中，其后果表征为二值的奖赏值（成功/失败）。该类方法的最终动作选择融合了强化学习策略网络的前向推理，经验模型服务于神经网络训练，且经验更新通常以整体添加方式改变记忆模型。
>
> 本发明中的经验模型以`{情境描述, 动作, 后果}`三元组为基本结构，其中后果为结构化的后果描述而非二值奖赏值；本发明通过类比检索而非神经网络前向计算进行后果预测；且经验更新采用记录粒度的局部修正而非整体模型更新。

### 3.5 结论

**新颖性不受破坏。创造性存在中等风险（主要针对独权1和从权2），通过从权4和说明书的后果定义补丁可有效防御。** D2与P011的核心差异在于：(1) 后果的粒度（结构化后果 vs 二值奖赏）；(2) 决策机制（纯类比 vs RL混合）；(3) 更新粒度（记录级 vs 模型级）。

## 四、D3（CN118003303A）对比分析

### 4.1 概述

### 4.2 与P011的核心区别

| 对比维度 | D3 | P011 |
|----------|-----|------|
| 预测方式 | LSTM推理 + DQN决策（神经网络计算） | 类比检索（非神经网络） |

### 6.3 数字执行体方向的共同区别

数字执行体方向的近似文献主要集中于三类：工具调用路径复用、GUI操作轨迹记忆、智能体任务/安全案例库。该类方案通常记录“任务-路径-工具”“子任务节点-操作序列”“历史轨迹-修复策略”等信息，用于复用成功路径、生成操作序列或进行安全校验。

其共同区别在于：未以{情境描述，动作，后果}作为基础经验单元，未基于相似情境记录对候选动作的预期后果进行类比预测，也未以实际后果回写来形成动作后果经验模型的自进化闭环。

即便CN121935243A公开“条件、操作、效果”的知识三元组，该三元组用于领域知识/工艺规程的记忆管理，并经强化学习奖励信号更新决策策略；不同于P011中由真实交互积累的经验三元组直接参与候选动作后果预测。

### 6.4 补充结论

**数字执行体中文补检未发现破坏P011新颖性的文件。** 创造性风险主要来自CN121351971A与CN121279405A的组合式质疑，但二者均缺少P011的“动作后果预测”核心链路。

建议在说明书中补充如下区分措辞：

> 现有数字执行体或LLM智能体记忆方案中，历史经验通常表现为工具调用路径、任务轨迹、GUI操作序列、成功规划路径、修复策略或上下文记忆，用于路径复用、工具选择、任务规划或安全校验。该类方案关注“如何复用已成功的执行路径”或“如何管理任务记忆”，并不以动作执行后的可观测后果为基础经验单元。
>
> 本发明中的经验模型以{情境描述，动作，后果}记录为基本结构，其中后果为动作作用于真实物理环境或真实数字环境后产生的可观测状态变化、执行结果或约束影响；本发明基于相似情境记录对候选动作的预期后果进行类比预测，并在真实执行后将实际后果反馈入经验模型，从而形成面向动作后果预测的经验自进化闭环。

### 7.3 最强英文近似：Microsoft action store

US20260111363A1 / WO2026089797A1 是本轮英文补检中最需要注意的同族文件。该方案公开了 hierarchical multi-layer action store，动作可以由物理机器人或数字智能体执行，执行信号中可以包含动作执行结果，并可据此更新动作映射、动作层级、动作版本或性能指标。

但该方案与 P011 的核心区别仍然清楚：

| 对比维度 | Microsoft action store | P011 |
|----------|------|------|
| 基础记录单元 | action、sub-action、action mapping、memory layer、performance metric | {情境描述，动作，后果}经验记录 |
| 检索目的 | 为任务找到可执行动作或动作版本 | 为当前情境找到相似历史经验并预测候选动作后果 |
| 反馈用途 | 更新动作库、动作层级、动作映射、动作版本或性能指标 | 将真实执行后的实际后果反馈进经验模型 |
| 决策机制 | action store + GMLM/RAG/embedding/trajectory optimization | 基于相似情境记录的类比后果预测 |
| “后果”地位 | 作为动作库管理信号或 performance data | 作为经验三元组的核心字段和预测对象 |

因此，Microsoft 文献可能用于质疑“数字执行体 + 动作执行反馈 + 动作库更新”的宽泛组合，但不足以直接覆盖 P011 的“动作后果经验模型”和“类比预测候选动作后果”。

### 7.4 英文数字执行体方向的共同区别

英文数字执行体近似文献主要分为五类：action store/skill registry、embodied-agent memory、interaction history graph、GUI/workflow automation、case-based/transactional reasoning。该类方案通常将历史信息用于动作库管理、行为仿真、路径探索、工作流复用、会话意图判断或案件评分。

其共同区别在于：未将{情境描述，动作，后果}作为统一的基础经验单元；未以相似情境记录为依据，对候选动作的预期后果进行类比预测；也未在真实物理环境或真实数字环境执行动作后，将实际后果作为动作后果经验反馈写入经验模型。换言之，现有英文方案多数回答“做哪个动作/调用哪个工具/复用哪条路径/评分哪个案例”，而 P011 回答的是“在这个情境下，做某个动作会产生什么后果，并据此选择动作”。

### 7.5 补充结论

**数字执行体英文补检未发现破坏 P011 新颖性的文件。** 创造性风险主要来自 E1（Microsoft action store）与 E2/E3/E7 的组合式质疑：即“动作库/经验记忆/历史交互图/结果反馈”是否可组合到 P011。防御重点应放在“后果作为经验字段和预测对象”以及“基于相似情境的类比预测”这两个硬差异上。

建议在说明书中补充如下英文近似防御措辞：

> 现有英文数字执行体方案中，action store、skill registry、interaction history graph、experience memory store、workflow automation、case-based reasoning engine 等结构通常用于存储可执行动作、操作路径、交互轨迹、多模态经历、会话历史或领域案例，并根据成功/失败反馈、性能指标或评分结果调整动作库、工作流、记忆图、神经网络策略或案例评分模型。
>
> 本发明中的经验模型以{情境描述，动作，后果}为基础经验记录，其中“后果”不是动作库的性能指标、案例评分或工作流完成状态，而是动作作用于真实物理环境或真实数字环境后产生的可观测状态变化、执行结果或约束影响。本发明基于相似情境下的历史经验记录，对候选动作的预期后果进行类比预测，并根据真实执行后的实际后果对经验模型进行反馈更新，从而形成面向动作后果预测的经验自进化机制。
