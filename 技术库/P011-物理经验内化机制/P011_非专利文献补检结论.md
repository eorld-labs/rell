# P011 非专利文献补检结论

---

## 一、补检目的

此前主要基于专利库和自拟检索词进行近似排查。本轮从 P011 方案本身出发，脱离专利检索词，补查论文、开源项目和学术语境，重点寻找是否存在不同术语表达下的近似技术。

本轮发现：学术语境中最接近 P011 的不是 “case-based reasoning”，而是以下术语族：

1. affordance learning / robotic affordances。
2. object-action-effect relation。
3. contextual affordance / action-effect prediction。
4. Object-Action Complexes, OAC。
5. deep episodic memory。
6. learned action models / preconditions and effects。
7. LLM agent reflection / skill library。

---

## 二、总体判断

非专利文献中确实存在与 P011 词义和机制高度相关的研究传统，尤其是机器人 affordance learning 中的 “object-action-effect” 三元关系，以及 contextual affordance 中的 “state-object-action-effect” 表达。

但本轮未发现完全覆盖 P011 母案完整链路的非专利文献。主要区别仍在：

1. P011 的经验记录以 `{情境描述，动作，后果}` 为基础，并面向候选动作后果预测。
2. P011 强调经验来自真实物理环境或真实数字环境中的真实交互。
3. P011 包含执行后实际后果反馈入经验模型的自进化闭环。
4. P011 覆盖数字执行体和真实数字环境，而 affordance/OAC 文献基本集中于机器人/物理对象。
5. 很多学术方案采用神经网络、仿真、抽象规划模型或技能库，不是记录级经验模型。

---

## 三、最接近的学术语境：Affordance / Object-Action-Effect

### 3.1 近似点

机器人 affordance 学派常将 affordance 表示为 object-action-effect 或类似关系，用于表达“某对象在某动作下会产生什么效果”。这与 P011 的 `{情境描述，动作，后果}` 在结构上接近。

代表性线索：

- IJCAI 2021 综述《Building Affordance Relations for Robotic Agents - A Review》提到机器人可在任务开始前知道 push → roll 关系，并以一定概率将其用于未知环境泛化。
- IROS 2015《Contextual Affordances for Action-Effect Prediction in a Robotic-Cleaning Task》明确使用 `<state, object, action, effect>`，并提出 effect = f(state, object, action)。
- 多篇 affordance/OAC 文献将机器人与对象、动作、效果之间的关系用于规划、控制或预测。

### 3.2 风险等级

**中高。**

这是目前非专利文献里对 P011 最需要认真防御的方向。它与 D2、Microsoft action store 不同，是真正从“动作会产生什么效果”这个语义逼近 P011。

### 3.3 区别点

| 对比维度 | Affordance / OAE 文献 | P011 |
|----------|------|------|
| 基础表达 | object-action-effect，或 state-object-action-effect | `{情境描述，动作，后果}` |
| 重点对象 | 物理机器人与物体/场景可供性 | 物理执行体 + 数字执行体 |
| 经验来源 | 常含仿真、自探索、训练数据、演示或模型学习 | 明确真实物理环境或真实数字环境中的真实交互 |
| 使用方式 | affordance 分类、效果预测、规划特征、模型训练 | 相似经验检索/类比预测候选动作后果 |
| 更新方式 | 多为模型训练、分类器/神经网络更新或规划模型更新 | 实际后果反馈入经验模型，可记录级局部修正 |
| 数字空间 | 通常不覆盖 GUI/API/真实数字环境 | 明确覆盖真实数字环境 |

### 3.4 说明书防御建议

正式说明书必须新增一段与 affordance/OAC 的区别：

> 现有机器人可供性学习通常将可供性表示为对象、动作和效果之间的关系，用于判断某对象是否支持某类操作，或通过模型预测动作效果。该类方法通常面向机器人与物理对象的交互，且其关系多作为分类器、神经网络、规划模型或可供性模型的训练对象。本发明中的经验模型以真实交互形成的 `{情境描述，动作，后果}` 经验记录为可检索单元，不限于物理对象可供性，还包括真实数字环境中的页面状态、接口状态、数据状态与数字动作后果；本发明在候选动作选择前基于相似历史经验记录对预期后果进行类比预测，并在真实执行后将实际后果反馈入经验模型。

## 四、Deep Episodic Memory 方向

### 4.1 代表文献

Rothfuss 等，2018，IEEE RA-L：

《Deep Episodic Memory: Encoding, Recalling, and Predicting Episodic Experiences for Robot Action Execution》

### 4.2 近似点

该文献公开：

1. 用深度神经网络表示 robot experiences in an episodic-like memory。
2. 编码观察到的动作到潜在向量空间。
3. 检索最相似的既往 episodes。
4. 重构原始 episode。
5. 预测未来视频帧。

词面上非常接近 “episodic memory + recalling + predicting + robot action execution”。

### 4.3 风险等级

**中。**

标题和摘要非常近，但技术本体不同。

### 4.4 区别点

| 对比维度 | Deep Episodic Memory | P011 |
|----------|------|------|
| 经验形式 | 视频 episode 的潜在向量表示 | `{情境描述，动作，后果}` 经验记录 |
| 预测对象 | future frames / action experience reconstruction | 候选动作的预期后果 |
| 决策链路 | action matching/retrieval，偏动作识别/记忆 | 后果预测→动作确定→真实反馈 |
| 更新方式 | 深度网络训练/表示学习 | 实际后果反馈经验模型 |
| 数字环境 | 不涉及真实数字环境 | 明确覆盖 GUI/API/数字执行体 |

### 4.5 防御建议

说明书中可增加：

> 现有深度情景记忆方法可以将视频、动作片段或传感序列编码为潜在向量，并据此检索相似片段或预测未来帧。本发明并非以视频帧重构或潜在表示学习为核心，而是以动作执行前情境、动作和动作后果之间的可检索对应关系为核心，用于候选动作后果预测和真实后果反馈更新。

## 五、Object-Action Complexes / Learned Action Models

### 5.1 近似点

OAC（Object-Action Complexes）和 learned action models 文献强调：

1. 对象与动作不可分离。
2. 动作具有 effects。
3. 可学习 action preconditions and effects。
4. 可用于规划与机器人控制。

该方向与 P011 在“动作效果可被学习并用于规划”这一点上接近。

### 5.2 风险等级

**中。**

OAC 是概念上较近的学术传统，尤其容易被无效请求人用于普通技术常识或组合动机。

### 5.3 区别点

1. OAC 更多是感知-动作抽象、规划接口或符号/亚符号桥接。
2. learned action models 多学习 preconditions/effects 并生成 PDDL 或规划模型。
3. P011 不以建立通用规划域模型为必要，而是基于真实交互经验记录预测候选动作后果。
4. P011 覆盖真实数字环境与业务数字动作。

### 5.4 防御建议

> 现有动作模型学习或对象-动作复合体通常致力于学习动作前提、动作效果或规划域模型，以供符号规划器使用。本发明不要求生成显式规划域模型，也不要求抽取通用动作规则；本发明在当前情境下基于历史经验记录中的动作-后果对应关系预测候选动作后果，并通过实际后果反馈修正经验模型。

---

## 六、Predictive State Representations / World Models

### 6.1 近似点

PSR、world model、model-based RL、learned transition model 方向都关注：

1. 从 action-observation 序列学习环境动态。
2. 预测未来状态。
3. 在 learned model 中规划动作。

### 6.2 风险等级

**中低。**

这些方向宽泛且历史很长，但更偏动态系统建模/策略学习，而非 P011 的可检索经验三元组。

### 6.3 区别点

1. PSR/world model 是生成式/状态转移模型。
2. P011 是记录级经验类比后果预测。
3. P011 不要求学习完整环境模型。
4. P011 的真实数字环境与业务动作后果覆盖更具体。

### 6.4 防御建议

说明书继续强调：

> 本发明不要求建立完整世界模型、状态转移模型或 POMDP/PSR 模型；本发明可以在缺少完整环境模型的情况下，基于相似历史经验记录预测候选动作后果。

## 七、LLM Agent 记忆与反思方向

### 7.1 代表文献/项目

1. Reflexion: Language Agents with Verbal Reinforcement Learning。
2. Voyager: An Open-Ended Embodied Agent with Large Language Models。
3. Generative Agents。
4. 各类 agent memory、reflection、skill library、episodic memory buffer。

### 7.2 近似点

LLM agent 方向常见机制：

1. 执行任务。
2. 接收环境反馈或错误。
3. 生成反思文本。
4. 存入 episodic memory buffer 或 skill library。
5. 后续任务检索记忆或技能。

### 7.3 风险等级

**中。**

这一方向对“数字执行体”很重要，但多数方案不是动作后果经验三元组。

### 7.4 区别点

| 对比维度 | LLM Agent memory/reflection | P011 |
|----------|------|------|
| 记忆内容 | 反思文本、错误总结、任务轨迹、技能代码 | `{情境描述，动作，后果}` 或等同对应关系 |
| 使用目的 | 改进提示词、复用技能、提升任务成功 | 预测候选动作预期后果 |
| 后果地位 | 反馈信号或反思输入 | 经验记录核心字段和预测对象 |
| 更新方式 | 追加反思、修复代码、技能库增长 | 实际后果反馈经验模型，可局部修正 |

### 7.5 防御建议

> 现有 LLM 智能体反思或技能库方法通常将执行失败、环境反馈或任务轨迹总结为自然语言反思或可复用技能，用于后续提示或技能调用。本发明不是一般反思记忆或技能库，而是将动作执行前情境、动作以及动作后果建立可检索对应关系，用于预测候选动作的预期后果。

---

## 八、RPA / Workflow Mining / Business Process Learning

### 8.1 近似点

学术和产业中有大量工作研究：

1. 从用户操作日志学习工作流。
2. 从事件日志发现流程。
3. 推荐下一步操作。
4. 自动化重复业务流程。

### 8.2 风险等级

**中低。**

它们对数字空间业务场景有相关性，但通常偏流程挖掘/下一步推荐，而非后果预测。

### 8.3 区别点

1. workflow mining 关注流程结构、节点、频率、路径。
2. P011 关注候选动作在当前情境下的预期后果。
3. workflow 推荐不必预测“点击/提交/API 调用会产生什么结果”。
4. P011 可覆盖业务流程，但核心不是业务流程本身。

### 8.4 防御建议

维持业务逻辑剥离章节中的边界：单纯流程发现/日志分析不是核心；若系统基于历史动作后果预测候选动作影响并据此选择动作，则进入 P011。

## 十、对 P011 说明书的新增建议

此前说明书提纲已经覆盖 RL、路径复用、action store、CBR、数字执行体等区别。基于本轮非专利补检，建议新增一个专门区别段：

### 与机器人可供性学习和动作效果预测的区别

建议正式写入：

> 现有机器人可供性学习、对象-动作-效果关系、上下文可供性或对象-动作复合体等方法，通常用于表示物理对象在特定动作下可能产生的效果，或用于训练分类器、神经网络、规划模型或动作模型。本发明与其不同之处在于：本发明以真实交互形成的 `{情境描述，动作，后果}` 经验记录或其可关联等同结构作为可检索经验单元；所述情境描述不限于物理对象属性，也包括真实数字环境中的页面状态、接口状态、数据状态、权限状态和业务状态；所述动作不限于机器人动作，也包括 GUI 操作、API 调用、工具调用、文件操作和业务级数字动作；所述后果不限于物理对象状态变化，也包括页面变化、接口返回、错误信息、数据库变化、流程状态变化等数字后果。本发明在动作确定前基于相似历史经验记录预测候选动作的预期后果，并在真实执行后将实际后果反馈入经验模型。

## 十三、继续补检后的新增判断：Web Agent 经验记忆方向

### 13.1 新增高相关线索

继续从“数字执行体在真实数字环境中通过经验预测动作后果”这一语义出发补检后，发现 2025 年前后 LLM Web Agent 方向出现更贴近数字空间的表达，代表线索为：

《WebATLAS: An LLM Agent with Experience-Driven Memory and Action Simulation》。

其公开语义包括：

1. LLM web agent 面向新的 web 环境执行长程导航和任务完成。
2. 使用 experience-driven memory。
3. 结合 look-ahead action simulation。
4. 通过记住过去失败并预测未来动作后果来适应未见网站。

这一路线比一般 Reflexion、Voyager、skill library 更接近 P011 的数字执行体部分，因为它已经进入“真实网页环境、过往经验、未来动作后果预测”这一组合。

### 13.2 风险等级

**中高，但主要攻击数字执行体实施例，不足以整体击穿母案。**

理由是：其非常适合被无效请求人拿来主张“数字智能体也会保存经验并预测点击/操作后果”。但从目前摘要和公开信息看，它仍更像 web agent 任务完成框架，经验记忆和动作模拟服务于网页导航；未见其覆盖 P011 的物理执行体与数字执行体统一机制，也未见其将 `{情境描述，动作，后果}` 作为跨物理/数字空间统一的可检索经验单元并形成真实执行后的经验模型反馈更新闭环。

### 13.3 与 P011 的区别点

| 对比维度 | Web Agent 经验记忆/动作模拟 | P011 |
|----------|------|------|
| 应用对象 | 主要是 Web/浏览器任务智能体 | 物理执行体 + 数字执行体 |
| 环境范围 | 网站、网页导航、WebArena 类任务 | 真实物理环境或真实数字环境，包括 GUI、API、工具、文件、业务系统、虚拟世界实例 |
| 经验形态 | 多为网页交互经验、失败记忆、自然语言或任务轨迹摘要 | `{情境描述，动作，后果}` 或可关联等同结构 |
| 预测方式 | look-ahead action simulation / 未来动作后果模拟 | 基于相似历史经验记录对候选动作后果进行类比预测，可采用检索、统计、模型或组合方式 |
| 反馈闭环 | 可能记住失败、改进后续任务 | 实际后果反馈入经验模型，影响后续相似情境的后果预测和动作确定 |
| 母案范围 | 数字网页智能体子领域 | 跨物理空间和数字空间的通用经验内化机制 |

### 13.4 对说明书的处理建议

说明书中应单独增加“与 Web Agent 经验记忆/动作模拟的区别”，不要只并入 LLM 反思记忆。建议写入：

> 现有 Web 智能体或 LLM 智能体可以在网页任务中记录历史失败、任务轨迹或页面交互经验，并在后续任务中进行动作模拟或计划生成。该类方法通常面向网页导航、网页任务完成或提示词/计划改进。本发明并不限定于网页任务，也不是单纯记忆失败经验或模拟未来操作序列；本发明以真实交互形成的情境-动作-后果对应关系作为经验模型基础，在动作确定前基于相似历史经验预测候选动作的预期后果，并在真实物理环境或真实数字环境执行后将实际后果反馈入经验模型。即使候选动作由 LLM、Web Agent 或工具调用框架生成，只要其动作选择、排序、过滤或参数调整基于上述预期后果预测完成，仍属于本发明的实施方式。

| D-W2 | WebNavigator: Global Web Navigation via Interaction Graph Retrieval | 中 | 建议摘要+必要时全文 | 重点是 Interaction Graph、retrieval、pathfinding、global navigation，更像网站拓扑/路径记忆，不是后果经验三元组，但会攻击“数字空间检索记忆”。 |

| D-W4 | A History-Aware Visually Grounded Critic for Computer Use Agents / HiViG | 中 | 建议摘要+关键段 | 有 pre-execution action evaluation、real GUI trajectories、history-aware critic，接近“执行前评估动作”，但核心是视觉接地批评器/拦截错误，不是相似经验后果预测。 |
| D-W5 | Securing Computer-Use Agents: A Unified Architecture-Lifecycle Framework | 中低 | 不必全文，作为背景 | 是综述/框架型可靠性文献，覆盖 CUA 的浏览器、桌面、移动应用、文件系统、终端和工具后端；可作为数字执行体背景，但不是直接近似方案。 |
| D-W6 | Dyna-Think: Synergizing Reasoning, Acting, and World Model Simulation in AI Agents | 中 | 建议摘要+关键段 | 涉及 world model simulation 和 action outcome/process simulation，容易作为组合动机材料，但偏世界模型/推理动作协同。 |
| D-W7 | Grounded Test-Time Adaptation for LLM Agents | 中 | 建议摘要+关键段 | 若其中有非参数经验检索或 action simulation，可作为数字侧辅助近似；目前看不是最强。 |
| D-W8 | GUI Agents survey / Computer-use agents survey | 中低 | 保留引用 | 适合作为“领域已有大量 agent memory/GUI agent”的背景，不作为核心对比文献。 |

结论：数字侧必须重点拉 WebATLAS 全文，其他 Web Agent / CUA 文献按“摘要 + 关键段”处理即可。除非全文中出现明确的 `{context/state, action, outcome}` 可检索经验单元和真实执行反馈更新，否则不需要动权利要求。

物理侧继续以 affordance/OAE/action-effect prediction 为主风险。其中《Contextual Affordances for Action-Effect Prediction in a Robotic-Cleaning Task》《Building Affordance Relations for Robotic Agents - A Review》《Cross-Embodied Affordance Transfer through Learning Affordance Equivalences》《AutoGPT+P: Affordance-based Task Planning with Large Language Models》等可作为物理侧背景/近似族群。其共同缺口仍是：主要面向物理对象可供性、任务规划或机器人操作，不覆盖真实数字环境中的 GUI/API/工具/业务后果，也未见跨物理与数字执行体的统一经验内化闭环。

### 13.6 WebATLAS 全文初读结论

全文链接：https://arxiv.org/html/2510.22732v2

公开时间：arXiv v1 提交于 2025-10-26，v2 修订于 2025-12-19。若 P011 申请日晚于该日期，应按现有技术风险处理。

#### 13.6.1 与 P011 高度接近的公开内容

WebATLAS 摘要和方法部分明确公开以下内容：

1. 使用 experience-driven memory 与 look-ahead action simulation，使 LLM web agent 通过记住过去失败并预测未来动作后果来适应未见网站。
2. 从 interaction experience 学习轻量内部环境模型，并在真实执行前进行 hypothetical action rollouts。
3. 构建 persistent cognitive map，存储 interaction outcomes as experience-based memory。
4. Actor 提出候选动作，Critic 对每个候选动作进行 outcome-aware simulation，并选择最安全且推进目标的动作。
5. 多层记忆包括 working memory、cognitive map 和 semantic memory。
6. cognitive map 记录状态转移、action-to-outcome deltas、新可用动作和环境约束。
7. 在执行过程中，当观察与预期不一致或遇到新动态时，可更新 cognitive map、episodic memory 或 semantic memory。

这些内容已经覆盖了数字执行体侧的若干关键语义：经验记忆、候选动作、动作后果预测/模拟、真实执行前选择、记忆更新。因此 WebATLAS 是目前发现的数字侧最强非专利近似文献。

#### 13.6.2 仍可区分的核心边界

| 对比维度 | WebATLAS | P011 |
|----------|------|------|
| 技术场景 | LLM web agent，WebArena-Lite，网页导航和任务完成 | 真实物理环境或真实数字环境，覆盖物理执行体、GUI、API、工具调用、文件操作、业务系统和虚拟世界实例 |
| 经验结构 | cognitive map，以状态转移图、action-to-outcome deltas、自然语言摘要和 semantic memory 为主 | `{情境描述，动作，后果}` 或可关联等同结构作为统一可检索经验单元 |
| 预测方式 | 在 cognitive space 中做 look-ahead action simulation / tree search | 基于相似历史经验记录对候选动作后果进行类比预测，可用检索、统计、模型或组合方式 |
| 后果地位 | interaction outcome / state transition / observation delta，用于网页导航计划 | 后果是经验记录核心字段和预测对象，覆盖物理状态变化与数字后果 |
| 更新闭环 | cognitive map、episodic memory、semantic memory 可在线更新 | 实际后果反馈入经验模型，并影响后续相似情境下的后果预测和动作确定 |
| 母案覆盖 | 数字网页智能体子领域 | 跨物理执行体和数字执行体的统一经验内化机制 |

#### 13.6.3 风险判断

**风险等级：数字侧中高，整体母案中等。**

WebATLAS 对 P011 的数字执行体实施例构成实质近似，尤其可以被用于攻击以下特征：

1. 数字执行体在网页/数字环境中形成经验记忆。
2. 记录动作执行后的结果或状态变化。
3. 基于经验记忆预测候选动作的后果。
4. 在真实执行前选择较优或较安全动作。
5. 执行后更新记忆。

但其不足以直接完整击穿 P011 母案，原因是：

1. WebATLAS 明显绑定 Web agent / web navigation / WebArena-Lite，不是跨物理和数字空间的通用机制。
2. 其核心记忆形态是 cognitive map 和 web 环境状态转移，不是以 `{情境描述，动作，后果}` 为统一基础的母案经验模型。
3. 其动作主要是 click/type/navigation 等网页动作，未覆盖物理执行主体、机器人真实物理交互、API/工具/文件/业务动作等更广数字后果。
4. 其贡献重点在 inference-time actor-critic web agent、LAS、网页任务计划效率，不是提出“物理经验/数字经验内化”为统一技术机制。

#### 13.6.4 对 P011 的处理建议

1. **不建议因 WebATLAS 直接修改独权。** 独权应继续守“真实物理环境或真实数字环境 + 真实交互经验 + 候选动作后果预测 + 反馈更新”的完整闭环。
2. **说明书必须写入 WebATLAS 式方案的区别。** 不能只写“区别于 LLM 反思”。
3. **数字实施例不要只写网页点击。** 必须覆盖 GUI 表单、API 调用、工具调用、文件操作、业务系统状态变化、虚拟世界角色行为。
4. **经验模型定义要高于 cognitive map。** 说明经验模型可用图、表、向量库、日志索引或自然语言摘要实现，但本质是可检索的情境-动作-后果对应关系。
5. **后果字段要写宽。** 包括页面变化、接口返回、错误信息、权限拒绝、数据库变化、流程状态变化、资源变化、物理状态变化等。
6. **反馈更新要写成真实执行后的实际后果更新。** 不是单纯 planner digest、失败记忆或网页 cognitive map 增量。

#### 13.6.5 建议写入说明书的针对性段落

> 在一些 Web 智能体方案中，系统可以通过探索网页环境形成认知地图，记录网页状态转移、点击或输入动作引起的页面变化，并在执行前对候选网页动作进行前瞻模拟，以提高网页任务完成率。该类方案通常服务于网页导航或网页任务完成，其经验主要表现为网页状态转移图、网页交互摘要或网页环境约束。本发明与其不同之处在于：本发明不限定于网页导航任务，也不限定于浏览器页面状态转移；本发明以真实交互形成的情境描述、动作和后果之间的对应关系作为经验模型基础，所述情境、动作和后果既可以来自真实数字环境，也可以来自真实物理环境；所述后果不仅包括页面跳转或页面元素变化，还包括接口返回、工具调用结果、文件变化、数据库状态、业务流程状态、权限反馈、物理对象状态变化等。本发明在动作确定前基于相似历史经验记录预测候选动作的预期后果，并在真实执行后基于实际后果更新经验模型。

## 十四、给 P011 的最终策略建议

### 14.1 不建议因非专利文献直接收窄权利要求

现有权利要求的主线仍应保持母案宽度。非专利文献确实提示了两个强近似方向：
