一、哲学源头：人不靠计算物理规则活着

一个8岁孩子走进陌生房间，走向沙发，拿起杯子，返回。

他不靠——他没有——力学建模、传感器精度、深度相机点云。他靠的是过去3年摔过的跤、掉过的杯子、碰过的桌角压缩成的一层"影子执行"能力：在行动之前，在脑子里先跑一遍动作的后果。试错没有成本，只费一点葡萄糖。

这就是智人从茹毛饮血到成为智人的那一跳：把试错从物理空间搬到神经空间。

P011要做的事，就是让硅基生命学会同一件事。

三、核心方法论：试错内化循环

不是"装一个物理引擎"。是让机器走完这个循环：

物理尝试（实际操作或低精度模拟）
  → 观察后果 → 记录 {情境, 动作, 后果} 三元组
       ↓
  三元组库持续积累
       ↓
  新情境 → 不计算物理 → 检索最接近的历史三元组
       ↓
  基于历史预测后果 → 影子执行 → 选最优动作 → 真执行
       ↓
  真执行的结果反馈回三元组库 → 持续修正


机器不需要"物理规则本能"——它需要"物理经验的内化"。规则是公式，必须穷举所有变量才能算对。经验是三元组，只需要"上次和这次差不多"。



四、与纯传感器路线的本质区别

| | 传统机器人路线 | 试错内化路线 |
|------|-------------|------------|
| 怎么知道杯子只能向上拿 | 力学计算 + 深度相机精确建模 | 过去100次抓取杯子，只有垂直向上的成功了 |
| 进新空间怎么适应 | 重新扫描建模，重新计算 | P010告诉它空间结构 → 检索"上次遇到类似布局怎么走的" |
| 精度要求 | 极高——建模误差导致动作失败 | 极低——只要"差不多像上次成功那次"就行 |
| 核心能力 | 计算 | **记忆 + 类比** |

这不是放弃精度——是承认生物不靠精度活着。

八、专利保护方向建议

待保护核心：一种基于空间语义先验和经验三元组的物理操作后果预测与动作选择方法。

关键保护点：

1. 用一个预置或积累的物理经验模型（不依赖当前空间数据训练），基于P010提供的当前空间的空间语义先验，检索最接近的历史{情境-动作-后果}三元组
2. 基于检索到的历史后果，对当前情境下候选动作的物理后果进行非计算式的类比预测
3. 执行后，将本次{情境-动作-实际后果}作为新三元组反馈入模型——一次失败修正一个库，而非修改一个方程
4. 经验模型跨空间可迁移——在空间A学到的"抽屉向外拉"，适用于空间B/C/D...

与P010的关系：P011作为独立新案，在说明书实施例中引用P010生成的空间语义先验作为情境输入——不主张P010的生成方法，主张的是"拿到空间语义先验之后，怎么用经验来预测动作后果"。
