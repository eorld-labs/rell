说明书

发明名称：一种基于交互经验类比检索的后果预测与经验自进化方法、系统及存储介质
技术领域

本发明涉及智能体控制、机器人控制、数字执行体控制、经验学习、动作后果预测及交互反馈技术领域，尤其涉及一种基于交互经验类比检索的后果预测与经验自进化方法、系统及存储介质。


背景技术

随着机器人、软件智能体、工具调用智能体和虚拟角色等执行主体的发展，执行主体需要在真实物理环境或真实数字环境中根据当前情境选择动作并完成任务。现有技术中，执行主体可依赖预设规则、动力学模型、几何模型、仿真引擎、策略网络、工作流模板、动作库或工具调用序列确定待执行动作。

对于物理执行主体而言，现有方案可对物体形状、材质、摩擦、接触、运动约束或动力学参数进行建模，并基于模型计算动作后果。该类方案对环境建模精度和传感器精度依赖较高，在环境变化、模型不完备或对象属性难以准确获取时，存在预测准确性、泛化能力或部署成本方面的限制。

对于数字执行主体而言，现有方案可记录页面轨迹、工具调用序列、工作流路径、错误反馈、任务摘要或交互日志，并在后续任务中复用。该类方案主要关注路径复用、步骤推荐、工具选择、任务总结或提示词改进，未必将动作执行后的可观测后果作为可检索经验字段，也未必在动作确定前针对候选动作的预期后果进行预测。

现有强化学习、情景记忆或经验回放方案可将状态、动作、奖励、下一状态等数据作为训练样本，通过策略网络、价值网络或其他模型进行动作选择。该类方案中的后果多表现为奖励值、成功失败标签或下一状态，主要用于模型训练或策略优化。

现有端到端视觉-语言-动作（VLA）模型可通过大量数据训练，使执行主体根据视觉输入和语言指令直接输出动作。该类方案的知识通常存在于神经网络权重中，训练成本较高，且具体交互经验难以作为独立经验记录被检索、并入或通过通信网络共享。

现有案例推理方法可将问题和解决方案作为主要案例结构，通过检索相似历史案例并复用或适配其中的解决方案来求解当前问题。该类方法可包括检索历史案例、复用或适配案例、测试或修正方案以及保留新案例等过程。

现有 k-NN、向量数据库、相似度检索或检索增强生成方法，可根据查询样本与历史样本之间的特征距离、向量距离或语义相似度，返回相似样本、分类标签、回归值、文本片段或参考资料。

现有机器人可供性学习、对象-动作-效果关系、上下文可供性、对象-动作复合体或动作模型学习方案，可表示物理对象、动作和效果之间的关系，并用于机器人操作、规划或动作效果预测。该类方案可面向物理机器人与物理对象之间的可供性关系，或者用于训练分类器、神经网络、动作模型或规划域模型。

现有 Web 智能体或 LLM 智能体方案可在网页任务中记录页面交互轨迹、失败经验、网页状态转移或认知地图，并在执行前对候选网页动作进行模拟或评估。该类方案可用于网页导航或网页任务完成，其经验可表现为网页状态转移图、网页交互摘要、认知地图或页面环境约束。

为此，有必要提供一种以真实交互形成的情境描述、动作和后果之间的对应关系为基础，面向候选动作后果预测，并在真实执行后基于实际后果持续更新的经验机制。


发明内容

一、要解决的技术问题

本发明至少用于解决以下技术问题之一：

1. 在不依赖完整物理规则、完整环境模型或高保真仿真引擎的情况下，预测候选动作在当前情境下可能产生的后果。
2. 将真实交互产生的后果沉淀为可复用、可迁移、可局部修正的经验模型。
3. 使不同物理空间或数字环境中的经验能够在相似情境中复用，并在真实执行后继续反馈更新。
4. 使物理执行主体和数字执行体能够共用动作后果经验机制。

二、技术方案

为解决上述技术问题，本发明提供一种基于交互经验类比检索的后果预测与经验自进化方法，包括：

获取目标空间的经验情境描述信息；

基于经验模型，对候选动作在所述经验情境描述信息所表征的当前情境下的预期后果进行预测；其中，所述经验模型为基于真实交互积累的多个情境描述、动作和后果记录的集合；

至少基于所述预期后果，从所述候选动作中确定目标动作；

执行所述目标动作，并将本次经验情境描述信息、目标动作和实际后果作为新记录反馈入所述经验模型。

可选地，所述预测包括：基于所述经验情境描述信息，在所述经验模型中检索与当前情境相似的历史记录；基于检索到的历史记录中的动作与后果之间的对应关系，对所述候选动作的预期后果进行类比推断。

可选地，所述经验模型中的记录来自多个不同空间的真实交互，所述经验模型被配置为使在任一空间中积累的情境描述、动作和后果记录，适用于其他空间中具有相似情境描述的候选动作的后果预测。

可选地，当所述实际后果与所述预期后果不一致时，所述经验模型仅修正与本次情境相关的历史记录或其权重，而不修改所述经验模型中与本次情境无关的其他记录。

可选地，在确定目标动作之前，基于所述预期后果对候选动作进行影子执行，以在真实执行前验证所述候选动作的预期后果链是否满足预设安全约束或任务约束；若不满足所述约束，则从候选动作中剔除该候选动作或调整其执行参数。

可选地，在反馈更新后，将更新后的经验模型或其增量部分通过通信网络共享至至少一个其他执行主体。

可选地，所述真实交互为在真实物理环境或真实数字环境中执行并产生可观测后果的动作。所述真实数字环境包括实际运行的软件系统、网页、应用、操作系统、API、数据库、工具链、文件系统、业务系统或虚拟世界运行实例。

本发明还提供一种经验模型的积累方法，包括：

获取多个空间中执行主体执行真实交互所产生的经验情境描述信息、执行的动作和实际后果记录；

将所述记录按照经验情境描述信息的相似度组织为经验模型；

输出所述经验模型，所述经验模型被配置为被执行主体在预测候选动作的后果时检索使用。

本发明还提供一种基于经验模型的动作后果预测方法，包括：

获取经验模型，所述经验模型包含基于真实交互积累的多个情境描述、动作和后果记录；

获取目标空间的经验情境描述信息以及待评估的候选动作；

基于所述经验情境描述信息，在所述经验模型中检索情境相似度较高的历史记录；

基于检索到的历史记录中的动作与后果之间的对应关系，对所述候选动作的预期后果进行类比推断。

本发明还提供一种基于交互经验类比检索的后果预测与经验自进化系统，包括：

情境获取模块，用于获取目标空间的经验情境描述信息；

经验预测模块，用于基于经验模型，对候选动作在当前情境下的预期后果进行预测；其中，所述经验模型为基于真实交互积累的多个情境描述、动作和后果记录的集合；

动作确定模块，用于基于所述预期后果确定目标动作；

执行反馈模块，用于执行所述目标动作，并将本次经验情境描述信息、目标动作和实际后果作为新记录反馈入所述经验模型。

本发明还提供一种计算机可读存储介质，其上存储有计算机程序或指令，所述计算机程序或指令被处理器执行时，实现上述任一方法。

三、有益效果

与现有技术相比，本发明至少具有以下有益效果：

1. 可以降低对精确物理建模、完整因果规则和高保真仿真的依赖。
2. 可以使执行主体基于真实交互经验预测候选动作后果。
3. 可以使经验跨空间、跨主体、跨物理场景和数字场景复用。
4. 可以通过实际后果反馈实现经验模型自进化。
5. 可以通过局部修正降低错误更新对无关经验的影响。
6. 可以通过影子执行降低真实执行中的失败风险和安全风险。
7. 可以使物理执行主体和数字执行体共用动作后果经验机制。

具体实施方式

下面结合附图和实施例对本发明进行进一步说明。应当理解，所描述的实施例仅用于解释本发明，而不用于限定本发明。除非另有明确限定，本发明中的技术特征可以相互组合。

一、术语和适用对象

本发明中的执行主体10可以包括物理执行主体11和数字执行主体12。所述物理执行主体11可以包括机器人、机械臂、移动机器人、无人机、自动驾驶设备、工业控制设备或其他能够在真实物理环境21中执行动作的设备。所述数字执行主体12可以包括软件智能体、GUI 智能体、工具调用智能体、数字员工、虚拟角色、游戏智能体、API 调用智能体或其他能够在真实数字环境22中执行动作的执行主体。

在一些实施例中，物理执行主体11可以包括由软件智能体、控制程序或数字执行体控制的机器人、机械臂、无人机或其他物理设备。此时，数字执行体可以用于生成、选择或下发控制指令，物理执行主体11通过物理机构在真实物理环境21中产生动作后果。数字执行主体12也可以直接在真实数字环境22中执行 GUI 操作、API 调用、工具调用、文件操作或业务流程动作，并在真实数字环境22中产生可观测数字后果。

本发明将物理执行主体11和数字执行主体12统一称为执行主体10。物理执行主体11和数字执行主体12均满足以下共同技术条件：动作执行前存在可描述情境，执行主体10能够执行动作，动作后产生可观测后果，并且情境、动作和后果之间的对应关系能够被记录、检索并用于后续候选动作后果预测和经验模型反馈更新。

本发明中的目标空间20可以是真实物理环境21，也可以是真实数字环境22。真实物理环境21包括真实设备、真实物体或真实空间构成的运行环境。真实数字环境22包括实际运行的软件系统、网页、应用、操作系统、API、数据库、工具链、文件系统、业务系统或虚拟世界运行实例。

本发明中的真实交互，是指执行主体在真实物理环境或真实数字环境中执行动作并产生可观测后果的交互。真实交互区别于仅根据预设规则、游戏引擎规则、物理动力学规则或仿真参数计算后果的仿真动作。

二、经验模型

所述经验模型50为基于真实交互积累的多个经验记录51的集合。每条经验记录51至少包括情境描述字段511、动作字段512和后果字段513。

在一些实施例中，经验模型50可以作为执行主体10的经验型世界模型或具身智能世界模型的组成部分，用于表征目标空间20中情境、动作和后果之间的可检索对应关系，并用于对候选动作60或动作链执行后的后续情境、状态变化或风险结果进行预测。

所述情境描述用于描述动作发生前的环境、对象、状态、任务约束或上下文。在真实物理环境中，情境描述可以包括空间语义信息、物体类别、物体相对位置、可操作部件、接触关系、遮挡关系、任务目标、安全边界、执行主体位姿、末端执行器状态、载荷、速度或接触状态。在真实数字环境中，情境描述可以包括 GUI 页面状态、控件位置、控件语义、焦点状态、输入字段内容、DOM 信息、辅助功能树信息、API 状态、工具可用性、权限状态、任务上下文、数据对象状态、业务流程状态、虚拟世界对象状态或角色状态。

在一些实施例中，经验情境描述信息中的空间语义部分可以由基于跨平台空间交互协议编码的空间语义先验提供。所述跨平台空间交互协议可以使不同品牌、不同架构或不同运行环境的执行主体10采用统一的数据格式描述和交换空间语义信息，使经验记录51中的情境描述字段511具有跨主体可解析性和跨平台可迁移性。

所述动作可以包括物理动作、数字动作或复合动作。物理动作可以包括抓取、推动、拉动、旋转、移动、避障、点击实体按钮、开关设备、改变姿态或调节力控参数。数字动作可以包括点击、输入、拖拽、选择、调用工具、调用 API、修改文件、提交表单、执行脚本、发送消息、更新数据库记录或触发业务流程。复合动作可以包括由多个子动作构成的动作序列。

所述后果为动作执行后在真实物理环境或真实数字环境中产生的可观测影响。物理后果可以包括目标物体位置变化、姿态变化、打开、关闭、倾倒、滑落、破损、卡滞、碰撞、力反馈异常、路径偏移、到达目标或未到达目标。数字后果可以包括页面元素变化、页面跳转、字段值变化、接口返回、错误信息、权限拒绝、数据库变化、文件变化、工具执行结果、业务流程状态变化、虚拟世界对象状态变化、资源消耗或安全告警。

经验记录51可以采用表结构、图结构、向量索引、日志索引、事件链、自然语言记录、结构化字段、半结构化字段或其组合实现。经验记录51可以包括权重字段514、可信度字段515、来源字段516和时间字段517。经验记录51不要求情境描述、动作和后果物理存储于同一数据对象中，只要能够建立可检索的对应关系即可。

在不同实现中，情境描述也可以被称为状态、上下文、观察、任务上下文、环境描述、页面状态、业务状态或场景描述；动作也可以被称为操作、指令、工具调用、API 调用、控制量、执行步骤或子任务；后果也可以被称为结果、反馈、状态变化、返回值、执行影响、状态差异、异常、效果或 outcome。上述名称变化均可对应于情境、动作和后果之间的对应关系。

所述后果不等同于单一奖励值、动作评分、工作流完成标签或动作库性能指标。所述后果是用于预测和比较候选动作影响的经验字段。

三、方法实施例

如图1所示，在一个实施例中，基于交互经验类比检索的后果预测与经验自进化方法包括以下步骤。

步骤 S101，获取目标空间20的经验情境描述信息。

所述经验情境描述信息用于描述执行动作前的当前情境。对于物理执行主体11，经验情境描述信息可以来自视觉感知、空间语义识别、力觉感知、定位信息或任务信息。对于数字执行主体12，经验情境描述信息可以来自页面截图、DOM、辅助功能树、应用状态、API 元数据、日志、数据库状态、文件状态或业务上下文。

步骤 S102，获取候选动作60。

候选动作60可以来自执行主体10的动作库、任务规划模块、人工输入、示教数据、工具注册表、API 列表、历史经验中出现过的动作、LLM、VLM、VLA、RAG、规则引擎或规划器。候选动作60可以为单步动作，也可以为动作序列。

在一些实施例中，即使候选动作60由端到端模型、规划器或规则系统生成，系统仍可以基于经验模型50对候选动作60的预期后果70进行预测，并至少基于所述预期后果70确定目标动作61。

步骤 S103，基于经验模型50对候选动作60在当前情境下的预期后果70进行预测。

在一些实施例中，系统基于当前情境描述，在经验模型50中检索情境相似度最高或满足相似度阈值的历史经验记录51。相似度可以基于向量相似度、结构化字段匹配、对象关系图相似度、GUI 状态相似度、任务目标相似度、约束相似度或其组合计算。

检索到的历史经验记录51包括与候选动作60相同的动作，也可以包括与候选动作60相似的动作。系统基于历史记录中的动作与后果之间的对应关系，对候选动作60的预期后果70进行类比推断。在一些实施例中，系统可以根据当前情境与历史情境之间的差异，对历史后果进行映射、修正或置信度调整。

在一些实施例中，所述类比推断不依赖于基于第一性原理的因果规则计算，且不以对检索到的历史记录整体进行改编以生成当前情境下的目标动作61或动作方案为必要步骤。所述类比推断可以在保留历史记录中的动作与后果对应关系的基础上，对候选动作60的预期后果70进行推断。

步骤 S104，至少基于所述预期后果70，从候选动作60中确定目标动作61。

在一些实施例中，系统根据预期后果70是否满足任务目标、是否满足安全约束或权限约束、后果置信度、执行成本、时间成本、资源消耗、可逆性或失败代价，从候选动作60中选择目标动作61。动作确定可以包括选择一个目标动作61、对多个候选动作60排序、剔除风险动作、调整动作参数或输出风险提示。

步骤 S105，执行目标动作61，并采集实际后果71。

在真实物理环境21中，实际后果71可以通过传感器、摄像头、力传感器、位置传感器、状态检测器或任务完成检测器采集。在真实数字环境22中，实际后果71可以通过页面状态读取、DOM 变化检测、API 返回、日志解析、数据库状态读取、文件系统状态读取、工具返回值或业务流程状态读取采集。

步骤 S106，将本次经验情境描述信息、目标动作61和实际后果71作为新记录反馈入经验模型50。

在一些实施例中，如果实际后果71与预期后果70一致，系统可以新增经验记录51或提高相关记录的可信度。如果实际后果71与预期后果70不一致，系统可以新增经验记录51、修正相似历史记录的权重、降低错误预测来源记录的可信度，或者仅更新与本次情境相关的记录，而不修改经验模型50中与本次情境无关的其他记录。

实施例一：动作后果预测方法

如图3所示，在一些实施例中，系统基于当前情境检索多个相似历史记录，并对候选动作生成预期后果。

对于候选动作 A，系统可以在历史记录中检索与当前情境相似且动作为 A 或动作与 A 相似的记录，提取所述记录中的后果字段，并根据相似度、记录时间、来源主体可信度、后果稳定性或任务约束，对不同后果赋予不同权重。系统可以输出候选动作 A 的后果分布、最可能后果、风险后果、置信度或建议执行参数。

在一些实施例中，后果预测可以通过最近邻映射、加权投票、统计聚合、规则过滤、图路径推断、向量检索增强生成、语言模型推理或其组合实现。相似度公式可以根据情境描述类型、动作类型、后果类型或任务约束确定，并用于基于真实交互形成的情境、动作和后果之间的对应关系，对候选动作的预期后果进行预测并基于实际后果反馈更新。

实施例二：影子执行

在一些实施例中，在真实执行目标动作之前，系统可以基于预期后果进行影子执行。影子执行可以基于经验模型中的历史后果链推演动作可能造成的后续情境。

例如，若候选动作为点击提交，其预期后果为页面进入确认页，则系统可以继续基于确认页情境检索后续经验记录，判断是否会出现权限拒绝、不可逆提交或需要人工审批。如果影子执行的后果链触发安全约束、权限约束或任务失败条件，系统可以剔除该候选动作、调整动作参数或要求人工确认。

在一些实施例中，所述影子执行可以不生成完整仿真环境，而是基于经验模型50中的历史后果记录推演候选动作60或动作链的预期后果70。

影子执行可以使用真实交互经验中的后果链进行低成本推演，也可以与规则引擎、仿真引擎、LLM 或其他预测模块结合使用。

实施例三：经验模型积累方法

在一些实施例中，经验模型的积累方法包括：获取多个空间中执行主体执行真实交互所产生的经验情境描述信息、执行的动作和实际后果记录；将所述记录按照经验情境描述信息的相似度组织为经验模型；输出所述经验模型，所述经验模型被配置为被执行主体在预测候选动作的后果时检索使用。

所述多个空间可以包括多个真实物理空间、多个真实数字环境或二者的组合。不同执行主体可以将各自积累的经验记录上传至共享经验库，也可以仅上传经验模型的增量部分、摘要部分、高置信度部分或脱敏部分。

在一些实施例中，经验记录51或经验模型50的共享可以携带空间语义先验的协议版本、坐标系标识、语义区域标识、空间关系标识或端侧转换标识，使接收经验记录51的执行主体10能够解析经验情境描述信息中的空间语义部分，并将所述经验记录51用于自身目标空间20中的候选动作后果预测。

在一些实施例中，接收经验模型50或其增量部分的其他执行主体10，可以将接收到的经验模型50或其增量部分并入自身经验模型50，并基于并入后的经验模型50执行候选动作后果预测。

在一些实施例中，基于交互经验类比检索的后果预测与经验自进化系统包括情境获取模块30、候选动作获取模块31、经验模型存储模块32、相似检索模块33、后果预测模块34、动作确定模块35、影子执行模块36、执行控制模块37、后果采集模块38、经验反馈更新模块39和经验共享模块40。

经验模型存储模块32用于存储基于真实交互积累的多个情境描述、动作和后果记录。

相似检索模块33用于基于当前情境描述从经验模型50中检索相似历史记录。

后果预测模块34用于基于历史动作与后果之间的对应关系预测候选动作60的预期后果70。

执行控制模块37用于控制执行主体10执行目标动作61。

后果采集模块38用于采集目标动作61执行后的实际后果71。

经验反馈更新模块39用于将当前情境、目标动作61和实际后果71反馈入经验模型50。

经验共享模块40用于通过通信网络41向其他执行主体共享经验模型50或其增量部分。

上述模块可以部署于机器人本体、边缘计算设备、云端服务器、智能体平台、浏览器扩展、桌面自动化系统、移动终端、工业控制系统或多主体协作网络中。


应用实施例

应用实施例一：机械臂抽屉操作

机械臂面对一个抽屉，当前任务为打开抽屉。系统获取的情境描述包括目标对象为抽屉、抽屉具有可抓取把手、把手位于抽屉正面、抽屉可能具有滑轨结构。候选动作包括向外拉、向内推、向上抬和旋转把手。

经验模型中可以包括以下经验记录：抽屉正面、把手可抓取、滑轨结构的情境下，向外拉的后果为抽屉打开；向内推的后果为抽屉未打开并产生阻力；向上抬的后果为抽屉卡滞。

系统确定当前情境与历史抽屉情境相似，预测向外拉的预期后果为抽屉打开，预测向内推的预期后果为未打开并产生阻力，预测向上抬的预期后果为卡滞，并选择向外拉作为目标动作。

若实际执行后抽屉未打开且检测到锁止状态，系统将带锁抽屉、把手可抓取、向外拉、未打开且受锁止约束作为新经验记录写入经验模型。

应用实施例二：机械臂杯子抓取

机械臂面对位于桌面边缘的杯子，杯口向上。候选动作包括垂直上提、水平推和侧向夹取。

历史经验记录显示，在类似杯子靠近桌边的情境下，水平推可能导致杯子倾倒或掉落，侧向夹取在杯壁湿滑时可能导致滑落，垂直上提在夹持稳定时通常能够移取杯子。系统预测垂直上提的预期后果为杯子被成功拿起，预测水平推的预期后果为杯子位置偏移或掉落风险，预测侧向夹取的预期后果为滑落风险，并选择垂直上提作为目标动作。

应用实施例三：移动机器人避障

移动机器人在通道中检测到临时障碍物。情境描述包括通道宽度、障碍物位置、左右可通行空间、任务目标和安全约束。候选动作包括左绕、右绕、停止等待和直接通过。

经验模型中记录了类似通道下左绕成功通过、右侧窄缝导致碰撞风险、直接通过导致接触障碍物的历史经验。系统预测左绕的预期后果为通过障碍区域，右绕的预期后果为碰撞风险较高，直接通过的预期后果为接触障碍物，停止等待的预期后果为任务延迟，并基于任务目标和安全约束选择左绕。

执行后，系统将实际路径偏移、是否碰撞、是否到达目标、耗时等实际后果写回经验模型。

应用实施例四：GUI 智能体填写表单

GUI 智能体在真实运行的订单退货页面中执行任务。当前页面包含订单号输入框和提交按钮，当前订单号为空。候选动作包括输入订单号、点击提交、点击取消和切换页面。

经验模型中可以包括以下经验记录：订单退货表单且订单号为空时，点击提交的后果为页面提示订单号必填；订单退货表单且订单号为空时，输入有效订单号的后果为输入框状态变为已填写；订单退货表单且订单号已填写时，点击提交的后果为进入退货确认页面。

系统预测在订单号为空的当前情境下直接点击提交会触发错误提示，并先执行输入订单号。输入完成后，当前情境更新为订单号已填写，系统再预测点击提交会进入退货确认页面，进而执行点击提交。

若提交后页面返回权限错误，则系统将订单号已填写、点击提交、权限错误作为新经验记录写入经验模型。

应用实施例五：工具调用智能体生成配送标签

工具调用智能体的任务为生成配送标签。可用工具包括订单查询 API、地址校验 API 和物流标签 API。当前任务上下文中存在订单号，但缺少收件地址。

经验模型中记录，在缺少收件地址的情境下直接调用物流标签 API 会返回参数缺失错误；在具有订单号但缺少地址的情境下调用订单查询 API 可以返回地址信息；在获得地址信息后调用地址校验 API 可返回地址有效或地址无效；在地址有效后调用物流标签 API 可生成标签。

系统预测直接调用物流标签 API 的后果为参数缺失错误，预测调用订单查询 API 的后果为返回地址信息，并选择调用订单查询 API。若订单查询 API 返回地址字段为空，则系统将有订单号、缺少地址、调用订单查询 API、地址为空作为实际后果写入经验模型。

应用实施例六：业务系统数字执行体

数字执行体在客服工单系统中处理退款请求。情境描述包括工单类型、客户状态、订单状态、退款金额、当前权限、审批状态和历史沟通记录。候选动作包括查询订单、提交退款、升级人工、请求客户补充信息和等待客户确认。

历史经验显示，在订单状态为已发货且退款金额超过阈值的情境下，直接提交退款会返回审批拒绝；先升级人工审批可使流程进入人工审核状态；若客户地址信息缺失，请求客户补充信息会使工单进入等待客户状态。

系统基于上述历史动作与后果对应关系，预测候选动作的预期后果，并选择满足任务目标且不违反权限约束的目标动作。执行后，系统将业务流程状态变化、审批结果、权限反馈或客户响应状态作为实际后果写回经验模型。

应用实施例七：真实数字环境与仿真环境区分

真实数字环境包括实际网页应用、实际 SaaS 系统、实际 API 服务、实际数据库、实际操作系统、实际文件系统、实际工具链或实际运行中的虚拟世界实例。执行主体在真实数字环境中执行动作后，会使真实运行系统的状态产生可观测变化，例如页面跳转、接口返回、数据库记录变化、文件生成、业务流程状态改变或虚拟世界对象状态改变。

仿真环境可以包括离线测试沙盒、游戏引擎基于预设规则计算的模拟场景、数字孪生仿真或仅根据预设参数计算动作结果的环境。本发明所述真实交互区别于仅在仿真规则中计算结果的动作。

应用实施例八：经验共享

执行主体 A 在仓库 A 中积累纸箱湿滑、侧向夹取、纸箱滑落的经验记录，并将该经验记录或其增量部分共享至执行主体 B。执行主体 B 在仓库 B 中遇到相似纸箱情境时，检索到该经验记录，预测侧向夹取存在滑落风险，改用底部托举动作。

经验共享可以基于通信网络、云端经验库、边缘节点或主体间同步机制实现。共享时可以保留来源主体、时间戳、环境类型、可信度和适用条件，以便接收主体在融合经验时进行加权。

应用实施例九：局部修正

系统预测向外拉抽屉的预期后果为抽屉打开，但实际后果为未打开并检测到锁止。系统不修改所有抽屉相关记录，而是仅新增或修正与带锁抽屉、阻力异常或锁止状态相关的记录或权重。

通过局部修正，系统可以保留普通抽屉向外拉能够打开的经验，同时增加带锁抽屉直接向外拉无法打开的经验，从而避免一个异常后果影响无关经验。

应用实施例十：多主体经验融合

在一些实施例中，经验模型50可以接收来自多个执行主体10的经验记录51。不同执行主体10可以具有不同的硬件结构、传感器配置、控制精度、软件版本、权限范围或运行环境。系统在接收经验记录51时，可以读取来源字段516、时间字段517、环境类型、执行主体类型、情境描述字段511、动作字段512、后果字段513、权重字段514或可信度字段515。

当多个经验记录51对应相似情境和相似动作，且后果一致或近似时，系统可以对所述多个经验记录51进行合并、去重或权重叠加。当多个经验记录51对应相似情境和相似动作，但后果存在差异时，系统可以基于来源主体可信度、记录时间、执行次数、历史成功率、情境相似度、动作相似度、后果一致性或环境适配条件，对冲突经验记录进行仲裁，并生成融合后的经验记录、并列保留的条件化经验记录或待验证经验记录。融合后的经验记录可以用于后续候选动作60的预期后果70预测。

在一些实施例中，当某一执行主体10上传的经验记录51在后续验证中多次导致错误预测时，系统可以降低该执行主体10对应来源字段516的可信度、隔离该来源的部分经验记录51，或者仅在满足特定环境适配条件时使用该来源的经验记录51。

应用实施例十一：经验记录生命周期管理

在一些实施例中，经验记录51可以包括时效标识、版本标识、适用区域标识、适用对象标识、可信度衰减参数、验证状态标识或休眠标识。系统可以基于记录时间、执行次数、最近验证结果、环境变化信息、对象状态变化信息、接口版本变化信息或业务规则变化信息，对经验记录51进行生命周期管理。

当检测到目标空间20中的区域布局、对象位置、对象状态、可操作部件、数字页面结构、API 返回格式、权限规则或业务流程发生变化时，系统可以确定受影响的经验记录51，并对所述经验记录51进行降权、休眠、作废、标记为待验证或触发重新采集。所述处理可以基于适用区域标识、对象标识、接口标识、业务流程标识或空间语义标识进行局部执行。

在一些实施例中，系统可以根据经验记录51的验证结果动态调整权重字段514或可信度字段515。多次被实际后果71验证一致的经验记录51可以提高权重或延长有效期；多次与实际后果71不一致的经验记录51可以降低权重、缩短有效期、进入休眠状态或被替换为新的经验记录51。

应用实施例十二：跨执行主体形态的经验转换

在一些实施例中，来源执行主体和目标执行主体可以具有不同的执行形态。来源执行主体可以为机械臂、移动机器人、无人机、车辆、工具调用智能体、GUI 智能体或虚拟角色；目标执行主体可以为不同结构、不同自由度、不同软件接口或不同动作集合的执行主体10。

系统可以基于动作语义等价关系、执行主体能力约束、动作参数范围、可执行动作集合、后果维度可比性或任务目标相似性，将来源执行主体形成的经验记录51转换为目标执行主体可检索使用的经验记录51。所述转换可以包括将来源动作字段512映射为目标执行主体的可执行动作类型、将来源动作参数范围调整为目标执行主体的可执行参数范围、将来源后果字段513转换为目标执行主体可观测的后果维度，或者为转换后的经验记录51增加适用条件和可信度字段515。

在一些实施例中，机械臂执行主体形成的“抓取把手、向外拉、抽屉打开”的经验记录51，可以被转换为人形机器人执行主体可检索使用的“握持把手、向外拉、抽屉打开”的经验记录51；工具调用智能体形成的“缺少地址、调用标签接口、返回参数缺失”的经验记录51，可以被转换为另一工具调用环境中的同类接口调用经验记录51。

应用实施例十三：基于经验不确定性的主动探索

在一些实施例中，系统可以基于经验模型50中的记录数量、后果一致性、后果置信度、历史失败率、后果分歧程度、任务风险等级或相似记录覆盖范围，确定当前情境是否属于经验不足或不确定情境。

当当前情境对应的相似经验记录51数量低于阈值、多个相似经验记录51的后果差异超过阈值，或者候选动作60的预期后果70置信度低于阈值时，系统可以生成探索动作或验证动作。所述探索动作或验证动作可以包括低力度触碰、低风险试探、只读接口调用、预检查 API 调用、权限检查、局部移动、参数探测或请求人工确认。

系统执行探索动作或验证动作后，可以将探索前情境、探索动作和实际后果71作为新的经验记录51反馈入经验模型50。由此，经验模型50可以在经验不足的情境区域中增加经验记录51，并降低后续相似情境下候选动作后果预测的不确定性。

应用实施例十四：基于经验的动作链安全预判

在一些实施例中，影子执行模块36可以基于经验模型50对多个候选动作60构成的动作链进行安全预判。所述动作链可以包括多个物理动作、多个数字动作或物理动作与数字动作的组合。

系统可以针对动作链中的每一步动作，基于当前情境或上一步预期后果70形成的后续情境，在经验模型50中检索相似历史记录，并预测该步动作的预期后果70。系统可以检测预期后果链中是否出现碰撞、卡滞、物体不稳、超出力矩范围、不可逆状态变化、权限拒绝、数据覆盖、流程终止、安全告警或其他约束违反情形。

当预期后果链满足安全约束和任务约束时，系统可以允许执行目标动作61或动作链；当预期后果链不满足安全约束或任务约束时，系统可以剔除对应候选动作60、调整动作参数、缩短动作链、请求人工确认或生成替代动作链。

应用实施例十五：情境描述特征抽象与泛化

在一些实施例中，系统可以从经验记录51中提取用于相似检索的情境描述特征。所述情境描述特征可以包括对象类别、对象功能、对象状态、空间关系、可操作部件、任务约束、权限状态、接口状态、页面结构、业务状态或历史后果关联特征。

系统可以根据多个经验记录51中的后果一致性、特征出现频率、特征变化幅度、任务相关性或跨空间复用结果，区分跨场景稳定特征和场景特定特征。跨场景稳定特征可以被赋予较高检索权重，场景特定特征可以被赋予较低检索权重或作为二级过滤条件。

在一些实施例中，当两个经验记录51在尺寸、品牌、页面样式、接口名称或局部坐标上存在差异，但在对象功能、动作语义、空间关系或后果类型上相似时，系统可以基于抽象后的情境描述特征将二者确定为相似记录。所述抽象后的情境描述特征可以用于提高跨空间、跨主体或跨平台的经验检索覆盖率。


与现有技术的进一步区别

现有强化学习或情景记忆方案通常将经验作为训练样本，用于更新策略网络、价值网络或其他模型参数。其记录中的后果可能表现为奖励值、下一状态或成功失败标签。本发明中的经验模型以真实交互形成的情境描述、动作和后果之间的对应关系作为可检索经验基础，后果为可用于预测候选动作影响的经验字段。本发明可以基于相似经验记录对候选动作的预期后果进行类比预测，并在实际执行后按记录粒度反馈更新。

现有路径复用、工具调用路径复用或 GUI 工作流复用方案通常存储成功路径、动作序列、工具序列、工作流或操作轨迹，用于在相似任务中复用路径。本发明基于相似情境下历史动作与后果之间的对应关系，预测候选动作在当前情境下的预期后果。

现有 action store 或 skill registry 通常用于存储可执行动作、动作版本、动作层级、性能指标或访问策略，并根据执行反馈调整动作库。本发明中的后果作为经验记录的字段和预测对象，可区别于动作库性能指标或动作版本管理信号。

现有 case-based planning 或 decision pattern map 通常基于案例、状态空间模式或历史方案进行动作建议。本发明的基础单元为由真实交互形成的情境描述、动作和后果记录；本发明用于预测候选动作的预期后果，并根据预期后果选择、排序、剔除或调整动作。

现有案例推理方法通常以问题和解决方案为主要案例结构，通过检索相似历史案例并复用或适配其中的解决方案来求解当前问题。本发明中的经验记录以真实交互形成的情境描述、动作和后果之间的对应关系为基础；系统基于历史动作与后果之间的对应关系预测当前候选动作的预期后果，并至少基于该预期后果确定目标动作。本发明在真实执行后将实际后果反馈入经验模型，并可仅修正与本次情境相关的记录或权重，使经验模型在后续相似情境中改变后果预测和动作确定。

进一步地，案例推理中的检索目标通常为可复用或可适配的解决方案，输出为当前问题的方案；本发明中的检索目标为历史真实交互中的动作与后果对应关系，输出为候选动作的预期后果。所述预期后果可用于动作选择、排序、剔除、参数调整或影子执行，并在真实执行后基于实际后果对相关经验记录或权重进行局部更新，而不以整体复用或适配历史解决方案为必要步骤。

现有 k-NN、向量数据库或相似度检索方法通常根据查询样本与历史样本之间的距离或相似度，返回相似样本、分类标签、回归值、文本片段或参考内容。本发明中的相似检索用于后果类比推断；系统检索与当前情境相似的历史交互记录，并基于历史交互记录中的动作与后果之间的对应关系，预测当前候选动作的预期后果。所述预期后果可进一步用于目标动作确定、候选动作排序、候选动作剔除、动作参数调整、风险提示或影子执行。本发明可采用 k-NN、向量数据库或相似度检索作为底层检索工具。

现有机器人可供性学习、对象-动作-效果关系、上下文可供性或对象-动作复合体等方法，通常用于表示物理对象在特定动作下可能产生的效果，或用于训练分类器、神经网络、规划模型或动作模型。该类方法可面向机器人与物理对象之间的可供性关系。本发明以真实交互形成的情境、动作和后果对应关系作为可检索经验基础；情境不限于物理对象属性，还包括真实数字环境中的页面状态、接口状态、工具状态、数据状态、权限状态和业务状态；动作不限于机器人动作，还包括 GUI 操作、API 调用、工具调用、文件操作和业务级数字动作；后果不限于物理对象状态变化，还包括页面变化、接口返回、错误信息、数据库变化和流程状态变化。

现有 Web 智能体或 LLM 智能体可以在网页任务中记录历史失败、任务轨迹、网页状态转移或页面交互经验，并在后续任务中进行动作模拟、计划生成或提示词改进。该类方案可用于网页导航或网页任务完成，其经验可表现为网页状态转移图、网页交互摘要、认知地图或网页环境约束。本发明以真实交互形成的情境、动作和后果对应关系作为经验模型基础，在动作确定前基于相似历史经验预测候选动作的预期后果，并在真实物理环境或真实数字环境执行后将实际后果反馈入经验模型。
