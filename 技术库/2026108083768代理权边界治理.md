# 说明书初稿

## 名称

一种面向自动化执行主体的人类真实意图验真协议及代理权动态边界控制方法和系统

## 技术领域

本发明涉及人工智能安全、计算机访问控制、人机交互安全、自动化流程治理、工具调用治理、具身智能安全、物联网控制安全、车联网安全和隐私计算技术领域，尤其涉及一种在自动化执行主体发起工具调用、应用程序接口调用、外部系统操作、资金交易、数据外发、设备代理动作、具身执行端动作或车载代理动作时，对其动作请求与人类真实意图之间的一致性进行运行时因果验真，并据此输出裁决令牌、代理权裁决结果或审计摘要对象，以动态调整代理权边界、执行边界或主权控制边界的方法及系统。

## 背景技术

随着大语言模型、AI Agent、RPA、工作流引擎、自动化脚本、具身智能、车载代理、物联网代理和自动驾驶技术的发展，软件系统、具身执行端和联网设备代理已从单纯的信息生成或被动响应，逐渐演进为能够规划任务、调用工具和执行动作的自动化执行主体。AI Agent 为其中的典型实施形态之一，其可在用户授权后自动处理邮件、管理日程、调用企业系统、执行数据库查询、提交代码、发起支付、请求物联网设备代理执行动作，甚至与具身执行端或车载代理联动。RPA、企业工作流、云运维脚本、具身执行端代理、车载代理和设备控制代理亦存在相同的代理动作风险。

传统访问控制通常采用基于角色的访问控制、基于属性的访问控制、访问令牌、权限白名单、人工审批、工具调用日志或事后审计流程。该类方案可以判断某个主体是否具备某项权限，但难以判断某一次具体代理动作是否仍然符合人类用户原始、当前、知情、可感知且可负责的真实意图，尤其难以应对以下情况：

1. 自动化执行主体根据概率推理、规则编排或历史任务自主规划，可能在合法权限范围内做出实质违背用户意愿的动作。
2. 多 Agent 协作、RPA 编排、链式工具调用或工作流嵌套会导致用户意图在传递过程中被简化、扭曲、丢失或扩大。
3. 人在回路确认在部分场景中容易退化成形式化点击，不能证明用户理解动作后果，也不能证明动作执行链路与授权意图之间仍然保持一致。
4. 提示注入、间接提示注入、记忆投毒、规则注入、上游任务污染或外部依赖污染可能诱导自动化执行主体越权调用工具。
5. 人类授权主体与自动化执行主体、真实执行端往往不在同一设备、同一网络或同一物理环境中，传统同端确认难以覆盖云端 Agent、远程具身执行端、车载代理和多端审批场景。
6. 人类授权可能发生在动作执行之前较长时间，如自动续费、定时任务、审批后延迟执行或长期预授权，传统按动作发生时刻检测用户是否在场的方案难以区分用户因睡眠、离线或不可交互导致的物理静默与因远程控制、自动化脚本或非自主操作导致的物理静默。
7. 事后审计无法在动作执行前阻止不可逆后果，如资金转出、数据外发、生产系统配置修改、设备代理动作或车载代理动作。

现有技术中还出现了若干针对 Agent、自动化工具调用、交易授权、可信执行或通信鉴权的安全方案。该类方案可通过审计 Agent 的推理链、工具调用记录和用户任务目标，判断 Agent 行为与文本任务目标之间的语义对齐程度；也可通过轻量级风险模型或大语言模型对 Agent 工具调用进行分级检测，以识别越权调用或异常工具使用；还可通过意图凭证、交易凭证、支付凭证、消费意图令牌、可信执行环境、规则校验、数据字段授权、通信意图辅助信息或服务信息一致性校验，验证授权记录、交易参数、数据传输、通信服务或业务请求是否符合预设规则。

上述现有方案能够在各自场景中提供权限控制、语义对齐、凭证防篡改、交易参数匹配或规则一致性校验。然而，该类方案通常侧重数字语义、身份权限、交易参数或通信协议层面的判断，缺少将自动化执行主体的动作数字上下文与人类运行时人机状态、设备在场状态、物理或具身交互状态进行跨层因果关联的机制；对于预授权后延迟执行、用户处于静默或不可交互状态时的异步授权锚定，以及人类交互端与 Agent、具身执行端、车载代理或云端执行环境不在同一设备或同一物理环境中的多端协同场景，也缺少动作语义、授权锚点、执行端状态和动作后果之间的跨端因果一致性判断。此外，现有方案的输出多为单次告警、拦截、授权通过、交易处理或审计报告，难以对自动化执行主体的后续动作范围、自主执行等级、工具令牌或主权控制边界进行连续性的动态治理。

因此，需要一种在不对现有 Agent、RPA、工作流、设备代理或车载代理架构进行大幅重构的前提下，通过 SDK、工具网关、动作验真层、系统服务、边缘代理节点、具身执行端代理节点或车载代理节点，对自动化执行主体的高风险动作进行人类真实意图验真、动态限权、主权控制和审计留痕的技术方案。

## 发明内容

### 要解决的技术问题

本发明要解决的技术问题是：在自动化执行主体发起高风险动作请求时，通过人类真实意图验真协议确定该请求是否仍然落入人类授权意图、运行时人机状态、设备在场状态、物理或具身交互状态、业务或环境上下文以及可接受动作后果共同限定的代理权边界、执行边界或主权控制边界内，并以可计算的因果一致性结果表征该请求是否对应人类真实、当前、知情、可感知且可负责的意图；在出现权限漂移、目标错配、提示注入、规则污染、预授权锚点失效、跨端因果不一致或后果越界时，对自动化执行主体进行动态收缩、降级、冻结、撤销代理权或收紧主权控制边界。

### 技术方案

为解决上述问题，本发明提出一种面向自动化执行主体的人类真实意图验真协议及代理权动态边界控制方法。参见图2，该方法包括以下步骤。

1. 在自动化执行主体的高风险动作执行链路中的动作请求入口或执行前置控制点，接收、观察、挂起或拦截高风险动作请求，形成表征所述高风险动作请求的动作声明对象，并获取第一特征序列。所述第一特征序列用于表征高风险动作请求的动作语义、作用对象、预期后果和执行上下文，可包括目标工具、目标接口、动作类型、动作参数摘要、资源范围、调用链上下文、父执行主体或子执行主体标识、任务来源、外部输入可信度、动作风险等级、可回滚性和动作预计影响。

2. 响应于第一特征序列命中预设风险触发条件，生成意图验真请求对象，并获取与高风险动作请求相关联的第二特征序列。所述第二特征序列用于表征人类授权意图、运行时人机状态、设备在场状态、物理或具身交互状态、业务或环境上下文以及动作可执行后果中的至少一种，可包括授权意图锚点、意图脉冲响应、用户运行时交互状态、设备在场状态、触控状态、按键状态、姿态状态、环境感知状态、物理操作缺失状态、前台焦点、多端确认状态、跨端确认状态、授权链路完整性、预授权锚点有效性、业务上下文、环境上下文、沙箱预执行结果和动作后果边界。意图脉冲可以作为风险触发后的可选增强确认方式，不作为本发明判断人类真实意图或形成第二特征序列的必要限定。

3. 将第一特征序列与第二特征序列进行时间对齐、动作链路阶段对齐、上下文对齐和授权链路对齐，通过预设的因果验真机制计算高风险动作请求与人类真实意图之间的因果一致性。所述因果一致性可表示为评分、等级、布尔结论、断裂指数、原因码、解释标签或结构化证据摘要。

4. 基于因果一致性生成裁决令牌、代理权裁决结果或审计摘要对象中的至少一种。所述裁决令牌、代理权裁决结果或审计摘要对象用于动态调整自动化执行主体对本次高风险动作请求或后续动作请求的代理权边界、执行边界或主权控制边界，或者用于事前控制、事中复核或事后追溯。其可包括放行、限权、延迟、参数裁剪、重新验真、人工复核、沙箱预执行、会话隔离、任务链冻结、令牌撤销、动作阻断、回滚、主权开关收紧、跨执行主体连锁降权、审计增强或后续动作范围约束。

上述四步构成本发明的基本技术流程。动作声明对象、意图验真请求对象、具身响应对象、裁决令牌对象和审计摘要对象可以作为上述流程中的结构化对象或输出对象实现；审计留痕属于代理权治理输出的一种实现形态，可以与裁决令牌或代理权裁决结果组合输出，也可以作为审计摘要对象单独输出。

参见图1，本发明还提出一种面向自动化执行主体的人类真实意图验真协议及代理权动态边界控制系统。所述系统包括动作数字研判模块、意图与具身后验模块、因果验真模块以及裁决与审计模块。动作数字研判模块用于在动作请求入口或执行前置控制点形成动作声明对象并获取第一特征序列；意图与具身后验模块用于响应于第一特征序列命中预设风险触发条件，生成意图验真请求对象并获取第二特征序列；因果验真模块用于对第一特征序列和第二特征序列进行时间、动作链路阶段、上下文和授权链路对齐，并计算因果一致性；裁决与审计模块用于生成裁决令牌、代理权裁决结果或审计摘要对象中的至少一种，以动态调整代理权边界、执行边界或主权控制边界。

本发明还提供一种用于确定自动化执行主体的高风险动作是否源于人类真实意图的因果验真逻辑。对于自动化执行主体发起的、具有外部后果、资源状态改变、经济后果、数据外发或物理世界影响的高风险动作，可以通过将动作语义、授权锚点、运行时人机或具身状态和动作后果进行因果对齐，将人类真实、当前、知情、可感知且可负责的意图表征为可计算的因果一致性结果，以识别自动化越权、输入污染、代理链路断裂或后果越界等异常情形。

在具身智能、物联网或车载代理场景中，本发明关注的是自动化执行主体的动作请求、授权锚点、运行时人机或具身状态和动作后果之间的因果一致性，以及由该一致性驱动的代理权边界、执行边界或主权控制边界调整；执行端的底层闭环控制、执行机构安全控制或安全动作效果验证可由相应执行端系统实现，并非本发明的必要限定。

本发明中的人类真实意图不用于直接测量用户心理活动，也不以作出法律责任判断为必要条件。在技术上，其等价于自动化执行主体的高风险动作请求的动作语义、预设授权意图锚点和运行时人机或具身状态之间，在时间、动作链路阶段、上下文和授权链路四个维度上的匹配度量化指标。该指标可通过评分、等级、断裂指数、原因码、状态标签或结构化证据摘要表示，并用于驱动裁决令牌、代理权裁决结果或审计摘要对象的生成。

参见图7，在一种实施方式中，本发明支持同步具身验证路径：在高风险动作发生前、发生中或短时窗口内，采集人类授权主体的实时具身交互状态、设备在场状态、触控状态、按键状态、姿态状态、多端确认状态或物理操作缺失状态，并与动作语义、执行上下文和后果边界进行时间、阶段和授权链路对齐。

在另一种实施方式中，本发明支持异步授权锚定路径：当人类授权发生在动作执行之前，且动作执行时用户可能处于静默、离线、睡眠或不可交互状态时，系统并非仅依据当前缺少具身交互作出异常结论，而是验证该动作是否落入预授权锚点所限定的授权目标、授权范围、授权时段、授权风险等级、可撤销状态、后果边界和授权链路完整性之内。

在另一种实施方式中，本发明支持多端协同验证路径：当人类授权主体与自动化执行主体或真实执行端不在同一设备、同一网络或同一物理环境中时，在人类交互端采集授权意图、具身交互或确认状态，在执行端或执行环境采集动作声明、执行上下文、设备状态、环境状态和动作后果，并通过跨端因果一致性计算判断二者是否在时间、授权、状态和后果上相互匹配。

对于自动化执行主体发起的高风险动作请求，在验证其是否源于人类真实意图而非自动化越权、输入污染或代理链路断裂时，可在相应场景下采用以下验证条件之一或其组合：将动作语义与运行时人类具身或授权状态进行因果对齐；验证该动作在授权锚点、授权链路和后果边界的限定范围内；或者建立人类交互端与执行端之间的跨端因果一致性。缺少与具体场景相匹配的上述验证条件时，难以在逻辑上排除权限漂移、非自主操作、代理链路断裂和后果越界。本发明提供的同步具身验证路径、异步授权锚定路径和多端协同验证路径分别用于满足上述验证条件。仅进行数字侧规则匹配、身份认证、语义审计或事后日志记录的方式，由于未将动作语义、授权锚点、运行时人机或具身状态和动作后果纳入因果对齐闭环，难以完整判定高风险动作请求是否仍然源于人类真实意图。

### 有益效果

与现有技术相比，本发明至少具有以下效果。

1. 将自动化执行主体治理从静态权限分配升级为运行时人类真实意图验真，能够识别权限漂移、目标错配、授权链路断裂和后果越界。
2. 通过动作数字研判、意图与具身后验、跨层因果对齐和代理权治理输出形成闭环，避免仅凭单一权限、单次身份认证、形式化审批或事后日志判断高风险动作。
3. 支持同步具身验证、异步授权锚定和多端协同验证三类路径，能够覆盖手机确认、云端 Agent、长期预授权、远程具身执行端、车载代理和多端审批等场景。
4. 通过裁决令牌、代理权裁决结果或审计摘要对象的结构化输出，使 SDK、工具网关、API 网关、系统服务、边缘节点、设备代理节点、具身执行端代理或车载代理节点可执行放行、限权、冻结、撤销、阻断、回滚或审计增强。
5. 通过工具网关或 SDK 方式实施，不需要整体重构 Agent、RPA、工作流、具身执行端代理或车载代理平台，具有明确的工程实施路径；其中 SDK 是便于集成的实施路径之一，并不限定本发明的其它实施形态。
6. 支持对资金、隐私、云资源、数据库、代码仓库、物联网代理动作、具身执行端动作和车载代理动作等多类高风险动作进行统一治理。
7. 能够结合提示注入检测、沙箱预执行、回滚和结构化审计，形成事前预防、事中控制和事后复核的闭环。
8. 将动作语义、授权意图锚点、运行时人机或具身状态、动作链路阶段和动作后果边界纳入同一因果验真机制，能够识别动作参数合法、语义合理但人类授权状态、具身交互状态或运行时上下文发生断裂的异常场景，并输出可解释的因果一致性结论。
9. 支持预授权后的异步授权锚定路径和多端协同验证路径，并将输出类型从单次授权通过、单次交易处理或审计报告扩展为可用于动态调整后续代理权边界、执行边界和主权控制边界的裁决令牌、代理权裁决结果和审计摘要对象。
10. 将提示注入、规则污染、任务链污染和外部依赖不可信作为输入可信度因子纳入因果一致性计算，使得在自动化执行主体的上游输入或执行规则已被污染的情况下，即使单个动作参数处于合法范围内，也能够降低因果一致性并输出隔离、降权或阻断执行的代理权裁决结果。
11. 本发明的四步主干和三类路径不依赖于特定大语言模型类型、Agent 框架、传感器型号、软件平台、操作系统或硬件架构。在自动化执行主体发起高风险动作请求且需要验证其是否由人类真实意图驱动的场景中，可通过接入动作声明对象、意图验真请求对象、具身响应对象、裁决令牌对象和审计摘要对象中的至少部分对象，实现统一的因果验真和代理权边界控制，而不需要共享相同的模型实现、软件形态或硬件环境。

指基于因果一致性生成的、用于表达自动化执行主体本次或后续动作是否允许执行、限权执行、延迟执行、重新验真、人工复核、沙箱预执行、会话隔离、任务链冻结、令牌撤销、动作阻断、回滚或审计增强的裁决结果。代理权裁决结果可由裁决令牌承载，也可作为单独的结构化输出对象。

具身响应对象可包括用户触控轨迹、按键状态、姿态状态、设备在场状态、前台焦点、环境状态、物理操作缺失状态、意图脉冲响应、多设备联合确认、跨端确认状态或机器人、车辆、IoT 设备的物理控制反馈。

在一种机器可解析实现中，具身响应对象可包括响应标识 response_id、关联请求标识 request_id、采集设备标识 sensing_device_id、采集时间窗口 sensing_window、触控轨迹摘要 touch_trace_digest、按键状态 key_state、姿态状态 posture_state、设备在场状态 device_presence_state、前台焦点 foreground_focus_state、环境状态摘要 environment_state_digest、物理操作缺失状态 physical_silence_state、意图脉冲响应 pulse_response_digest、多端确认状态 multi_endpoint_confirmation_state、跨端确认状态 cross_endpoint_confirmation_state、可信度评分 confidence_score 和采集完整性标识 capture_integrity_flag 中的至少一种。

裁决令牌对象可包括令牌标识、动作标识、执行主体标识、授权范围、参数边界、有效期、允许/拒绝/限权状态、后续动作边界、后续动作约束、撤销标志、降权标志、原因码和审计摘要索引。审计摘要对象可包括动作声明摘要、第一特征序列摘要、第二特征序列摘要、因果一致性、因果断裂标签、裁决结果、动作执行结果、回滚结果、校验字段和敏感内容外流边界。

在一种机器可解析实现中，裁决令牌对象可包括如下字段中的至少一种：令牌标识 token_id、关联动作标识 action_id、执行主体标识 actor_id、裁决状态 decision_state、代理权边界 boundary_scope、参数边界 parameter_boundary、授权范围 authorization_scope、有效时间窗口 validity_window、后续动作约束 subsequent_action_constraints、自主执行等级 autonomy_level、撤销条件 revocation_condition、降权标志 downgrade_flag、撤销标志 revocation_flag、原因码 reason_code、因果一致性评分 causal_score、因果断裂标签 causal_break_tags、审计摘要索引 audit_summary_ref 和签名字段 signature。所述 boundary_scope 可用于限定可继续调用的工具、接口、设备控制能力或资源范围；所述 validity_window 可用于限定裁决令牌对本次或后续动作的生效时长；所述 revocation_condition 可用于在检测到用户状态变化、授权撤销、输入污染、跨端不一致或后果越界时撤销令牌或收缩代理权边界。

裁决令牌对象不限定为特定 JSON、JWT、证书、区块链凭证或网络报文格式，其可实现为数据结构、策略对象、状态标记、能力票据、调用凭证、应用程序接口响应码、权限更新指令、加密凭证或签名凭证，并可被 SDK、工具网关、应用程序接口网关、系统服务、边缘节点、设备控制器、机器人控制器或车载控制节点执行。

裁决令牌、代理权裁决结果和审计摘要对象可独立输出，也可组合输出。审计摘要对象不替代裁决令牌的治理功能，也不要求审计摘要必然伴随裁决令牌；只输出裁决令牌、只输出代理权裁决结果、只输出审计摘要对象，或同时输出其中任意两种或三种，均属于本发明代理权治理输出的实施形态。

通过上述对象，AI Agent、RPA、具身机器人、车载代理、自动驾驶控制主体或 IoT 控制节点均可按统一语义接入本发明，而不需要共享相同的软件框架或硬件形态。

### 实施例三：动作请求入口或执行前置控制点

参见图2和图4，在本实施例中，动作请求入口或执行前置控制点是指高风险动作对真实系统、外部对象、用户权益或物理环境产生实际影响之前，能够接收、观察、拦截、挂起、改写或裁决动作请求的逻辑位置。该逻辑位置不限定于某一种软件层级、通信协议或硬件节点，而是由高风险动作执行链路本身决定。

在软件自动化场景中，动作请求入口或执行前置控制点可以体现为工具调用层、应用程序接口网关、插件运行时、工作流执行器、函数调用代理、自动化动作节点、企业代理网关、系统服务、安全沙箱入口、消息队列消费者或数据库代理。作为示例，AI Agent 在调用外部工具前、RPA 在执行写入步骤前、工作流在触发审批或支付节点前，均可将动作请求提交至本发明的验真协议层。

在一种非侵入式或低侵入式实施方式中，系统可通过自动化执行主体外部的包装器、工具网关、API 网关、插件运行时、函数调用代理、消息队列代理、数据库代理、浏览器插件、终端系统服务或安全沙箱入口采集动作请求、执行上下文、调用链摘要、前台焦点、用户交互状态或结果回传，而不要求读取自动化执行主体的全部内部推理过程、模型参数或私有日志。作为示例，AI Agent 外部包装器可在函数调用前生成动作声明对象；API 网关可从请求体、响应码和调用链标识中生成第一特征序列；终端系统服务或浏览器插件可提供前台焦点、触控状态和设备在场状态摘要；数据库代理或消息队列代理可在写入、删除、外发或状态改变动作发生前触发验真流程。

在具身智能、物联网或车辆控制场景中，动作请求入口或执行前置控制点可以体现为设备控制器、边缘节点、机器人主权开关、车载控制节点、自动驾驶控制节点或其它能够在物理动作发生前进行裁决的控制接口。作为示例，机器人在执行开门、移动、抓取或释放动作前，车辆控制主体在执行远程接管、车辆解锁、靠边停车或变道动作前，均可先生成动作声明对象并进入人类真实意图验真流程。

在另一种非侵入式或低侵入式实施方式中，系统可通过边缘网关、设备控制接口监听、机器人控制器日志、车载网关、CAN 总线摘要、OBD 诊断接口、车载安全域、IoT 网关、摄像头、麦克风、雷达、惯性传感器或独立可信传感器模块采集执行端状态、环境状态、设备在场状态或物理控制反馈，而不要求替换原有机器人控制器、车载控制器、底层固件或工业控制协议。上述采集结果可被封装为第二特征序列、具身响应对象或审计摘要对象，并与动作声明对象进行因果对齐。

上述各类部署位置均为动作请求入口或执行前置控制点的具体实施形态。其共同技术本质在于：在高风险动作由自动化执行主体转换为真实执行效果之前，将动作语义、上下文、授权边界、用户状态和动作后果统一纳入因果验真和裁决控制闭环。

### 实施例四：基于工具调用网关的代理动作验真

参见图3和图4，在本实施例中，系统部署为工具调用网关。AI Agent、RPA 机器人、企业工作流或云运维代理在调用邮件系统、日程系统、支付系统、数据库、CRM、代码仓库或云资源之前，将高风险动作请求提交至工具调用网关，并由网关形成动作声明对象。

以财务 Agent 请求发起一笔转账为例，第一特征序列可包括目标工具、动作类型、金额区间、收款方标识、用途摘要、调用链上下文、任务来源、资源影响、可回滚性和风险等级。当高风险动作请求命中高风险条件时，系统查询用户授权意图锚点，并生成意图验真请求对象。授权意图锚点可包括用户此前授权的报销额度、授权对象、授权时间窗口、授权审批记录、授权撤销记录和后果边界。

在本实施例中，预设风险触发条件可由金额区间、收款方可信度、用途摘要、授权额度、执行时段、任务来源、外部输入可信度、可回滚性和后果边界共同确定。作为示例，同一财务 Agent 读取公开账务备注或查询低敏感报销状态时，可不触发人类真实意图验真；当其试图发起资金转账、修改收款账户、外发敏感财务数据、调用不可逆支付接口或在非典型时段执行高金额付款时，即可触发意图与具身后验。作为另一示例，当动作金额处于授权额度内但收款方、用途摘要、调用来源或执行时间与授权意图锚点显著不一致时，也可触发验真。

对于处于模糊边界的请求，系统可向最终授权主体发起意图脉冲请求。作为示例，可在用户手机上展示上下文绑定的确认摘要或随机微交互区域，要求用户在短时间内完成确认。系统检测响应时间、触控轨迹、设备在场状态和前台焦点。若响应时间低于预设合理反应时间窗口、轨迹呈现脚本特征或设备状态与确认事件不一致，则降低因果一致性。

基于上述因果一致性计算，系统可输出如下裁决：若授权意图锚点、用户状态和动作请求一致，则放行；若授权边界模糊但用户完成有效意图脉冲，则限权执行或人工复核；若存在明显权限漂移、意图锚点失效或输入污染，则冻结当前任务链并撤销支付工具令牌。

在本实施例的一种组合实施方式中，以 Agent 工具调用网关为实施载体，系统实现动作声明对象、意图验真请求对象、具身响应对象、裁决令牌对象和审计摘要对象的组合流程。动作数字研判模块将 Agent 的工具调用请求封装为动作声明对象，并提取第一特征序列；意图与具身后验模块生成意图验真请求对象，获取包括授权意图锚点、用户运行时交互状态、设备在场状态和业务上下文的第二特征序列，并以具身响应对象封装物理或具身交互状态的采集结果；因果验真模块计算所述第一特征序列与第二特征序列之间的因果一致性；裁决与审计模块基于因果一致性生成裁决令牌对象和审计摘要对象，对本次工具调用和后续工具调用进行代理权边界动态调整。

### 实施例五：基于 SDK 的轻量化集成

参见图4，在本实施例中，本发明以 SDK 形式嵌入 AI Agent 框架、RPA 平台、工作流平台或自动化脚本框架中。开发者在定义工具调用函数、自动化步骤或设备控制动作时，通过 SDK 声明工具风险等级、可回滚性、资源范围和默认授权策略。

当自动化执行主体调用工具时，SDK 自动生成动作声明对象，并调用本发明的真实意图验真接口。对于低风险动作，如读取公开日程，SDK 可直接放行；对于中风险动作，如发送外部邮件，SDK 可要求用户确认邮件摘要；对于高风险动作，如删除数据库记录、转账、外发敏感数据或控制物理设备，SDK 可先触发沙箱预执行、意图与具身后验或人工复核。

在一种实现中，SDK 或工具网关可维护风险分级策略表。所述风险分级策略表可将动作类型、资源敏感度、金额区间、目标对象可信度、执行时段、可回滚性、授权锚点完整性、上游输入可信度和历史因果一致性映射为不同验真强度。作为示例，读取公开信息、查询低敏感状态或执行可回滚低影响动作时，可仅生成审计摘要或直接放行；发送外部消息、更新中敏感字段或调用可回滚业务接口时，可触发轻量确认、异步授权锚定或后台沙箱预执行；转账、删除数据、修改权限、外发敏感内容、控制机器人或控制车辆时，可触发同步具身验证、人工复核、限权执行或阻断执行。

该实施例可作为 AI 应用、企业 Agent 平台、开源 Agent 框架、RPA 平台和企业工作流平台的安全插件实现。上述 SDK 形态仅为一种实施方式，本发明还可通过工具网关、动作验证层、系统服务或边缘控制节点等形态实施。

### 实施例六：提示注入与间接攻击隔离

参见图6，在本实施例中，售后退款 Agent 从客户上传文件中读取内容后，计划调用内部退款 API 处理一笔退款请求，该调用属于具有经济后果和账户状态改变效果的高风险动作请求。

动作数字研判模块在退款 API 调用前形成动作声明对象，并提取第一特征序列。所述第一特征序列可包括目标工具为内部退款 API、动作类型为退款执行、退款金额区间、收款账户或用户账户标识、调用来源为客户上传文件解析结果、任务来源、上游输入可信度、可回滚性和风险等级。

意图与具身后验模块响应于第一特征序列命中预设风险触发条件，生成意图验真请求对象，并获取第二特征序列。所述第二特征序列可包括客服人员或审批主体的退款审批权限构成的授权意图锚点、当前会话的退款处理上下文、客户上传文件作为外部输入来源的可信度、授权链路完整性以及必要时的审批确认状态或意图脉冲响应。

因果验真模块将第一特征序列与第二特征序列进行时间对齐、动作链路阶段对齐、上下文对齐和授权链路对齐，并分析客户上传文件中是否包含覆盖系统指令、诱导工具调用、要求忽略安全策略或诱导越权退款的隐藏指令。若检测到提示注入、间接提示注入、规则污染或外部依赖不可信，则将该输入源标记为不可信，降低退款动作请求与人类真实意图之间的因果一致性。

裁决与审计模块基于因果一致性生成裁决令牌、代理权裁决结果或审计摘要对象。所述裁决令牌或代理权裁决结果可包括隔离当前会话、降权为人工审核、冻结退款任务链、撤销退款工具令牌、阻断退款 API 调用或记录审计摘要。由此，本实施例在提示注入与间接攻击场景中仍形成动作数字研判、意图与具身后验、跨层因果对齐和代理权治理输出的四步闭环。

在本实施例的一种组合实施方式中，系统将输入污染评估与因果一致性计算组合执行。动作数字研判模块形成动作声明对象并提取第一特征序列，所述第一特征序列包括外部输入可信度、任务来源、调用来源和风险等级；意图与具身后验模块获取第二特征序列，所述第二特征序列包括授权链路完整性、外部输入来源、业务上下文和必要时的审批确认状态；因果验真模块在检测到提示注入、规则污染、任务链污染或外部依赖不可信时，将该输入源标记为不可信，并降低因果一致性；裁决与审计模块据此生成隔离会话、降权为人工审核、冻结任务链、撤销工具令牌或阻断执行的裁决令牌。

### 实施例七：沙箱预执行与动作后果校验

参见图6，在本实施例中，可采用同步具身验证路径，并结合沙箱预执行对具有不可逆或高影响后果的高风险动作进行动作后果校验。本发明可在动作最终影响真实系统之前，先在数字孪生沙箱、事务沙箱或权限沙箱中进行预执行。

系统获取预执行结果，包括资金变化、权限变化、数据外发范围、生产配置变化、物理设备状态变化和回滚可行性。随后，因果验真引擎将预执行结果与用户授权意图锚点和安全包络线进行比对。

若预执行显示动作后果超出授权范围，或系统无法保证回滚，则本发明输出阻断、人工审批或参数裁剪裁决。作为示例，系统可将“删除全部客户数据”的动作裁剪为“导出脱敏统计摘要”，或将“修改生产配置”的动作降级为“生成变更单并等待人工确认”。

### 实施例八：跨 Agent 协同治理

参见图5，在多 Agent 系统、工作流系统或 RPA 编排系统中，一个父执行主体可将任务拆分给多个子执行主体。传统审计通常只记录单个执行体的动作，难以识别意图在多跳传递中的漂移。

本发明在调用链上下文中记录父执行主体、子执行主体、任务拆分摘要、授权传递路径和每次裁决令牌。若某一子 Agent、工作流节点或自动化执行体被检测到提示注入、权限漂移或异常工具调用，系统可依据协作图谱对相关执行主体实施连锁降权，包括冻结同一任务链、撤销共享令牌、隔离关联会话或要求重新建立用户授权意图锚点。

### 实施例九：面向具身智能、物联网或车辆控制的增强验真

参见图3和图5，当 AI Agent、具身智能控制体、IoT 代理、车载代理或自动驾驶控制主体控制清洁机器人、门锁、车辆、无人机、机械臂或其它物理设备时，本发明可进一步引入设备传感器状态、环境状态、车辆状态、物理控制反馈和用户状态。

作为示例，家庭机器人 Agent 发出“打开房门”的动作请求时，系统可检查用户是否近期通过本地设备确认、门外声源或视觉状态是否异常、动作发生时间是否合理、是否存在远程提示注入来源。若出现用户意图锚点与环境状态冲突，如用户处于睡眠状态但 Agent 请求深夜开门，则判定为因果断裂并阻断动作。作为另一示例，自动驾驶控制主体准备执行远程接管、变道或靠边停车等动作时，系统可结合驾驶员或乘员状态、车辆环境、授权边界和安全策略输出裁决令牌。

### 实施例十：异步授权锚定路径

参见图7，在本实施例中，采用异步授权锚定路径，人类授权发生在动作执行之前，而自动化执行主体在之后的时间窗口内自动执行动作。作为示例，用户预先授权 Agent 自动续费指定范围内的订阅服务，企业审批人在工作流中批准夜间批量同步任务，或者设备持有人预先允许机器人在固定时段完成清洁任务。

当自动化执行主体在未来时间点发起动作时，系统不单独依据用户当前处于静默、离线、睡眠或不可交互状态而判定异常。系统首先提取第一特征序列，包括动作类型、作用对象、金额或资源范围、执行时段、任务来源、后果边界和可撤销状态。系统随后获取第二特征序列中的预授权锚点、授权时段、授权范围、授权风险等级、撤销记录、授权链路完整性和动作后果边界。

若当前动作的对象、范围、时段、风险等级和后果边界均落入预授权锚点，且授权链路未被撤销、篡改或污染，则可提高因果一致性并输出放行、限权或审计摘要对象。若当前动作超出授权范围、授权已撤销、动作后果不可回滚或任务链来源异常，则降低因果一致性，并输出重新验真、人工复核、限权、挂起或阻断的裁决令牌。

### 实施例十一：多端协同验证路径

参见图7，在本实施例中，采用多端协同验证路径，人类授权主体与自动化执行主体或真实执行端不在同一设备、同一网络或同一物理环境中。作为示例，用户在手机上确认云端财务 Agent 的付款请求，医生在远程操作台控制手术机器人，企业审批人在办公终端批准云端工作流执行，车辆控制责任主体在远端控制台对车载代理进行授权。

系统可在用户交互端采集人类授权意图、具身交互状态、设备在场状态、多端确认状态或意图脉冲响应；在执行端或执行环境采集动作声明对象、执行上下文、设备状态、环境状态、动作后果和结果回传；在网关端或平台端采集授权链路完整性、执行主体标识、策略状态和审计摘要。

因果验真模块将上述来自多端的特征进行时间对齐、动作链路阶段对齐、上下文对齐和授权链路对齐。若用户在 A 设备上的确认与 B 端执行动作的对象、参数、时段、环境状态和后果边界一致，则提高因果一致性；若执行端动作由被污染输入、异常链路或代理越权触发，而非由 A 端授权锚点驱动，或者 A 端确认内容与 B 端实际执行内容不一致，则判定跨端因果不一致，并输出限权、重新验真、冻结任务链、阻断或审计增强。

## 工业适用性

本发明可应用于企业 AI Agent 平台、AI 办公助手、财务 Agent、云运维 Agent、RPA 平台、企业工作流平台、金融交易代理、具身机器人、物联网控制平台、车辆控制系统、车载代理、自动驾驶控制系统、远程医疗设备控制系统、手术机器人控制系统和边缘控制节点等场景。其可通过 SDK、工具网关、插件运行时、企业代理网关、系统服务、安全沙箱、可信执行环境、边缘节点、机器人主权开关、车载控制节点或终端安全服务部署，用于在自动化执行主体发起高风险动作请求时进行人类真实意图验真、代理权边界控制和审计留痕，具备工业适用性。
