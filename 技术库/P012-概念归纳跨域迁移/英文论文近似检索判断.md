# P012英文论文近似检索判断

## 一、检索结论

围绕本案最小骨架进行了多轮英文论文检索，检索表达包括：

1. context action effect concept transfer；
2. situation action consequence concept learning；
3. interaction experience concept induction action effect；
4. sensorimotor experience concept action effects transfer；
5. object-action-effect affordance concept transfer；
6. action-effect-regulated symbols robot planning；
7. concept induction from interaction logs agent workflow；
8. sensorimotor contingencies concept transfer robot。

检索结果显示，英文论文中已经存在较成熟的三条相邻路线：

1. 机器人可供性路线：object-action-effect、affordance relation、cross-embodied affordance transfer；
2. 符号生成和动作效果规划路线：DeepSym、action-effect-regulated symbols、action effects for planning；
3. 概念迁移和数字智能体经验复用路线：concepts as cognitive programs、sensorimotor contingencies、workflow-induced structured experience、agent skill induction。

目前未发现单篇英文论文完整公开本案的全部组合特征，尤其未发现同时覆盖以下组合的文献：

1. 以交互经验记录作为输入，且该记录包括情境描述信息、动作信息和后果信息；
2. 基于多条记录中三类信息之间的对应关系形成概念标签；
3. 概念标签对应典型经验模式，且典型经验模式至少包括典型后果模式；
4. 基于概念标签及典型经验模式，在当前空间、当前执行主体或当前任务中确定可迁移经验；
5. 同时覆盖真实物理环境和真实数字环境，且不限定为机器人可供性、视觉概念、规划符号或工作流技能。

但是，英文论文层面的创造性压力明显高于中文专利检索结果，尤其是 DeepSym、Object-Action Complexes、Affordance Transfer 三条线，可能被作为组合评价基础。

## 二、高危英文论文

### 1. DeepSym: Deep Symbol Generation and Rule Learning for Planning from Unsupervised Robot Interaction

近似程度：高。

该论文公开机器人从交互经验中学习符号和规则。其交互经验包括 object/state、action、effect。论文明确描述机器人执行动作并记录结果效果，再基于该交互经验进行符号形成；其符号为 action-effect-regulated symbols，并用于后续规划。

与本案的接近点：

1. 均使用交互经验；
2. 均涉及动作和动作后效果；
3. 均从交互经验中形成高层符号或类别；
4. 均可用于后续规划或迁移。

可区分点：

1. DeepSym 的核心是从对象图像、动作和效果中发现 object/effect categories，并将规则转换为 PPDDL 规划规则；
2. DeepSym 未将概念标签定义为跨空间、跨主体或跨任务迁移的载体；
3. DeepSym 主要面向机器人操作和简化非机器人环境，并未统一覆盖真实物理环境和真实数字环境；
4. DeepSym 的迁移重点是符号规划规则和效果预测，不是概念标签对应的典型经验模式在不同空间、不同主体或不同任务中的迁移；
5. DeepSym 未强调典型经验模式至少包括典型后果模式、典型情境参数或典型动作参数，并作为独立可迁移经验被调用。

撰写影响：

DeepSym 是目前英文论文中对独权1压力最大的文献。如果独权1只写“从情境/动作/后果中形成概念并用于当前任务”，容易被其拉近。建议在独权或至少说明书核心段中强化“跨域差异下的迁移”，避免本案被理解为普通 action-effect symbol learning。

### 2. Object-Action Complexes: Grounded Abstractions of Sensorimotor Experience

近似程度：高。

OAC 将对象、动作、状态、结果、预期效果和层级行为组织在统一框架下，用于机器人基于真实交互经验形成符号化行为表示。

与本案的接近点：

1. 均以 sensorimotor experience 或真实交互为基础；
2. 均包括动作发生情境、动作和动作效果；
3. 均允许层级组织；
4. 均可用于预测、推理和规划。

可区分点：

1. OAC 是动作表征和认知单元框架，不是以概念标签及典型经验模式作为跨域迁移载体的方法；
2. OAC 并未明确形成“概念标签—典型经验模式—可迁移经验”的结构；
3. OAC 主要面向具身机器人，不覆盖真实数字环境中的 GUI、API、业务系统或虚拟世界运行实例；
4. OAC 没有将当前空间、当前执行主体或当前任务与历史记录之间的差异匹配作为核心迁移条件。

撰写影响：

说明书中应避免把本案写成抽象的 sensorimotor schema 或 action representation。应持续强调本案是从多条交互经验记录归纳概念标签，并将典型经验模式迁移到存在差异的当前域。

### 3. Cross-Embodied Affordance Transfer through Learning Affordance Equivalences

近似程度：高，主要影响跨主体从权。

该论文公开 robot-action-effect-object interaction data，并形成 latent affordance representation，用于不同机器人之间的 affordance transfer。

与本案的接近点：

1. 涉及 object/action/effect/agent；
2. 涉及跨机器人、跨本体迁移；
3. 关注 affordance equivalence 和能力差异下的迁移。

可区分点：

1. 该论文仍属于机器人可供性迁移，未覆盖真实数字环境；
2. 其迁移载体是 affordance equivalence 或 latent affordance representation，不是概念标签及典型经验模式；
3. 其重点是不同机器人之间的 affordance transfer，不是跨空间、跨主体和跨任务统一的概念层迁移；
4. 未形成面向任务处理模板、安全预判、隐私共享或概念生命周期管理的开放结构。

撰写影响：

从权6可以保留，但说明书中应把跨主体迁移写成“概念标签匹配+能力约束转换+典型经验模式迁移”，不要写成单纯 affordance equivalence。

### 4. Building Affordance Relations for Robotic Agents - A Review

近似程度：中高，作为基础综述威胁。

该综述明确指出机器人可供性常围绕 target object、actions、effects 三元素构建，并用于泛化。该文献不是单一破坏性文献，但能为审查员提供公知常识背景。

可区分点：

1. 可供性关系通常以物理对象和机器人动作作为中心；
2. 数据集和方法多集中于对象、动作和效果，且往往假设单一可供性或具体任务；
3. 本案不以对象可供性为终点，而以概念标签和典型经验模式作为跨空间、跨主体、跨任务迁移的中间层。

撰写影响：

背景技术中应保留“可供性学习或对象-动作-效果关系”的现有路线，并明确其通常不覆盖真实数字环境，也不必然形成面向跨域迁移的概念标签及典型经验模式。

### 5. Learning Labeled Robot Affordance Models Using Simulations and Crowdsourcing

近似程度：中。

该论文将 affordance 表示为 objects、actions、effects，并增加 human-understandable labels。其标签与 object-action-effect 相关。

与本案的接近点：

1. 有 O/A/E 三元关系；
2. 有 label；
3. 可用于机器人 affordance 理解。

可区分点：

1. 标签主要用于人类理解或自然语言 grounding；
2. 标签不是从多条交互经验记录中归纳出的跨域概念标签；
3. 未要求典型经验模式至少包括典型后果模式；
4. 未将标签作为当前空间、当前主体或当前任务中的可迁移经验确定依据。

### 6. Multi-Object Graph Affordance Network

近似程度：中。

该论文通过机器人交互观察效果来学习复合对象可供性，并用于目标导向规划。

可区分点：

1. 其领域集中于多对象物理操作；
2. 其核心是图神经网络表示复合对象 affordance；
3. 未覆盖数字执行体和真实数字环境；
4. 未形成通用的概念标签及典型经验模式迁移机制。

## 三、相关但不直接破坏的新兴路线

### 1. Beyond Imitation: Zero-shot task transfer on robots by learning concepts as cognitive programs

该论文强调从示意图输入输出中学习 concepts as cognitive programs，并迁移到真实机器人任务。其接近“概念迁移”方向。

区别在于：其概念是程序，输入为图像对或示意任务，不是多条交互经验记录中的情境、动作和后果对应关系；其迁移核心是 cognitive program 的执行，不是概念标签及典型经验模式。

### 2. Behavior Is Everything: Towards Representing Concepts with Sensorimotor Contingencies

该方向强调概念应由 sensorimotor contingencies 表示，属于本案哲学和认知科学层面的邻近背景。

区别在于：其主要是概念表征理论或实验系统，不等同于本案的交互经验记录、概念标签、典型经验模式和跨域迁移方法。

### 3. WISE-Flow: Workflow-Induced Structured Experience for Self-Evolving Conversational Service Agents

该论文从交互日志中归纳结构化 workflow 和 prerequisite-augmented action blocks，用于服务型智能体。

区别在于：其迁移对象是工作流和动作块，重点是对话服务流程复用，不是从情境、动作和后果共同约束形成概念标签，也未形成至少包括典型后果模式的典型经验模式。

### 4. Agent Skill Induction

该论文从 web navigation experiences 中归纳程序化技能，并可在网站之间复用。

区别在于：其核心是 program-based skills，不是概念标签及典型经验模式；其适用领域主要是 Web 任务，不覆盖物理执行主体与数字执行主体的统一概念迁移。

## 四、对权利要求的影响

英文论文检索后，不建议把本案退化成“概念归纳”或“动作效果学习”本身。该方向已有较多论文积累。

与此前中文近似判断相比，英文论文结果提示：独权1可以保持宽，但最好在独权1中更明确体现“跨域迁移”这一差异条件。否则，DeepSym 和 OAC 可能被用于主张：从交互经验的动作效果中形成符号或概念并用于规划，已经是现有研究方向。

建议考虑对权利要求1的 S4 做小幅收紧，使“迁移”不只是名称，而成为步骤中的明确技术条件。候选写法如下：

S4，基于匹配到的概念标签及其对应的典型经验模式，在所述当前情境信息、候选动作信息或待执行任务信息与至少部分交互经验记录所对应的历史情境、历史执行主体或历史任务存在差异的情况下，确定用于当前空间、当前执行主体或当前任务的可迁移经验。

上述修改的效果：

1. 不绑定具体算法、模型、机器人类型或数据库类型；
2. 仍保留物理世界、数字世界和未来执行主体形态；
3. 把本案从 DeepSym 的“同域符号学习+规划”中拉开；
4. 把发明点压回“概念层跨域迁移”，与发明名称一致；
5. 仍允许从权5、从权6继续展开具体跨空间和跨主体情形。

如果希望独权更宽，也可以不修改权利要求1，而是在说明书“与现有技术的进一步区别”中明确：本发明的重点不在于从单一环境中学习动作效果符号，而在于将多条交互经验记录归纳为概念标签及典型经验模式，并在当前域与历史域存在差异时确定可迁移经验。

## 五、背景技术更新建议

现有背景技术已经覆盖语义地图、知识图谱、行为概念、强化学习、端到端模型和可供性学习。结合英文论文检索，建议在后续正式说明书中进一步保持以下表达：

1. 对可供性学习的描述应包括 object-action-effect 或对象-动作-效果关系；
2. 对符号规划学习的描述应包括从动作效果中学习符号或规则；
3. 对概念迁移的描述应包括 cognitive programs 或 sensorimotor contingencies 这类高层概念表征；
4. 对数字执行体经验复用的描述应包括从交互日志中归纳 workflow、skill 或 action block；
5. 上述路线的共同不足应落在：未形成由情境、动作和后果共同约束的概念标签及典型经验模式，并在当前域与历史域存在差异时用于确定可迁移经验。

## 六、最终判断

英文论文层面没有发现能够直接覆盖本案全部核心组合的单篇论文。

但是，DeepSym 和 OAC 已经非常接近“从交互经验中的动作效果形成符号/概念并用于规划”的基础思想。因此，本案的真正稳定边界不应是“动作效果概念归纳”本身，而应是：

从多条交互经验记录中归纳概念标签及典型经验模式，并以该概念层作为跨空间、跨主体或跨任务迁移的载体。

如果团队希望母案独权更稳，建议小幅收 S4；如果团队希望母案独权最大化覆盖，则保留现有独权，但说明书必须把跨域差异、典型经验模式和数字环境覆盖写厚。
