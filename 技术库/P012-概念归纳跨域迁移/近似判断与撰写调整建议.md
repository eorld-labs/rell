# P012近似判断与撰写调整建议

## 一、总体结论

根据已检索并拉取的中文、外文近似全文，目前未发现单篇现有技术完整公开本案独立权利要求的全部核心特征。

本案的新颖性防线主要在于：交互经验记录包括情境描述信息、动作信息和后果信息；概念标签由上述三类信息之间的对应关系共同约束形成；典型经验模式至少包括典型后果模式；并基于概念标签及其典型经验模式确定当前空间、当前执行主体或当前任务中的可迁移经验。

主要风险不在单篇文献直接破坏新颖性，而在创造性组合评价。审查员可能将概念形成、抽象迁移、行为概念知识图谱、跨本体强化学习或语义地图类方案组合，用于主张本案属于常规技术拼接。因此，说明书应将本案与这些技术路线的差异写入背景技术、发明内容和区别说明中。

## 二、高危但可区分的近似技术

1. CN121481411A，基于机器学习的电子产品库存调配系统

该文献涉及渐进抽象、神经符号概念形成、抽象层次双向转换和基于抽象的迁移强化学习，是中文全文中最接近“概念形成加迁移”的方案。

可区分点在于：该文献的输入是库存及环境数据，输出是库存调配方案，其概念形成围绕库存数据和调配策略展开；未公开由执行主体的情境描述信息、动作信息和动作后可观测后果信息三者共同约束形成概念标签，也未要求典型经验模式至少包括典型后果模式。

2. CN118297149A，基于知识图谱和行为树的行为规划方法

该文献中行为概念的知识记录包括前置条件和执行结果，且用于扩展行为树。其接近“行为概念加执行结果”的方向。

可区分点在于：该文献使用预先构建的知识图谱进行行为树扩展，重点是行为规划；本案从多个交互经验记录中归纳概念标签及典型经验模式，并基于概念层进行跨空间、跨主体或跨任务迁移。该文献未公开从多条交互记录的情境、动作和后果共同特征中形成概念标签。

3. US9129158B1，Method and system for embedding visual intelligence

该文献公开从视频微动作中无监督学习时空动作概念、概念层级和因果关系，是外文中最接近“动作概念层级”的方案。

可区分点在于：该文献面向视频视觉理解和动作概念识别，核心是从输入视频像素、微动作和时空概念中进行视觉推理；本案面向执行主体的交互经验记录，后果信息是动作执行后的可观测结果，并作为概念形成和经验迁移的必要约束。

4. WO2025227355A1，Method and apparatus for reinforcement learning

该文献涉及跨本体无监督强化学习，通过多个本体的 state-action pairs、embodiment context 和 reward 调整统一策略。

可区分点在于：该文献的迁移载体是强化学习策略模型及其本体上下文；本案的迁移载体是概念标签及其典型经验模式。本案不以训练统一策略网络为必要步骤。

## 三、中等风险近似技术

1. KR20130010182A，Method for learning task skill and robot using thereof

该文献公开机器人从原始信息中提取动作模式，并根据动作模式之间的关联关系形成任务技能组。其接近技能学习和技能复用。

区别在于：该文献未将后果信息作为概念归纳的必要约束，也未形成至少包含典型后果模式的典型经验模式。

2. CN121835880A，跨空间、跨任务及跨本体学习的大模型与推理方法

该文献属于多模态大语言模型加动作策略头的 VLA/MLLM 路线，强调跨空间、跨任务和跨本体的统一模型控制。

区别在于：该文献通过模型结构、意图桥接和动作策略头实现泛化；本案通过结构化交互经验记录形成概念标签和典型经验模式进行迁移。

3. WO2023192497A1，Natural language control of a robot

该文献通过语言模型输出、技能描述、task-grounding measure 和 world-grounding measure 选择机器人技能。

区别在于：该文献关注自然语言指令到可执行技能的选择，未从多个交互经验记录中归纳概念标签，也未形成典型后果模式作为迁移依据。

## 四、低风险邻近技术

1. CN121453057A，语义地图创建及使用方法

该文献公开多模态数据、无监督聚类、大模型打标和语义地图构建。其接近“未知语义自动发现”。

区别在于：其输入主要是感知数据和空间位置信息，输出是自然语言语义标签和语义地图；未涉及动作信息及动作后的可观测后果信息，也未公开以典型后果模式为核心的经验迁移。

2. CN116881475A，概念知识图谱构建方法

该文献公开概念基础参数、特定参数及概念关系类型，属于概念图谱构建邻域。

区别在于：该文献的概念主要来自预定场景抽取及参数定义，并非由情境、动作和后果的交互经验对应关系形成。

3. US11625424B2 和 US20060020465A1

上述文献属于本体匹配、语义标准化和知识表示底层技术。其可作为组合背景，但单独杀伤力较低。

## 五、权利要求调整建议

1. 独立权利要求不建议收窄。

现有独权已经抓住最小必要特征：多个交互经验记录；情境描述信息、动作信息和后果信息；基于三者对应关系形成概念标签和典型经验模式；典型经验模式至少包括典型后果模式；基于概念标签及典型经验模式确定可迁移经验。该组合是区分近似案的主防线，不宜再加入具体算法、模型、聚类方式、知识图谱形式、语言模型或机器人形态。

2. 从权3是重要防御项，建议保留。

从权3明确概念标签不同于对象名称、对象类别或视觉外观标签，并强调由情境、动作和后果共同约束形成。该项直接区分语义地图、视觉分类和普通概念标签。

3. 从权5和从权6可以保持大胆。

从权5覆盖跨空间迁移，从权6覆盖跨主体迁移，是本案未来价值的主要延伸。虽然跨本体强化学习文献较近，但本案通过概念标签和典型经验模式迁移，而非策略参数迁移，因此不建议删除。

4. 从权7建议保留为防御性限定。

真实物理环境和真实数字环境的组合可以把本案从传统机器人可供性、视觉概念、物理对象操作中拉出来，并覆盖数字执行体。

5. 从权8建议保留。

概念更新、置信度调整、拆分、合并和待验证标记，是区别静态知识图谱和普通本体匹配的重要补强。

## 六、权利要求收放判断

### 6.1 不宜收窄的部分

权利要求1不宜加入以下限定：具体聚类算法、图谱结构、神经网络结构、大语言模型、强化学习模型、机器人类型、传感器类型、数据库类型、真实物理环境、真实数字环境、跨主体迁移或生命周期管理。上述内容均适合作为从属权利要求或说明书实施方式，不宜进入母案独权。

独权中的“概念标签”和“典型经验模式”也不宜限定为自然语言标签、知识图谱节点、向量簇、规则模板或任务模板。近似文献已经覆盖多种具体表达形态，独权应保持为功能性结构：由交互经验记录中的情境描述信息、动作信息和后果信息之间的对应关系形成，并至少关联典型后果模式。

### 6.2 应当保持的核心限定

独权中应当保留“情境描述信息、动作信息和后果信息”三元结构。该结构是区分语义地图、视觉标签、知识图谱、本体匹配、普通案例推理和强化学习样本的核心基础。

独权中应当保留“典型经验模式至少包括典型后果模式”。该限定把本案从普通概念归纳或标签聚类中拉出来，使概念层不是静态语义概念，而是含有动作后果经验的可迁移概念。

独权中应当保留“当前情境信息、候选动作信息或待执行任务信息”三类匹配入口。该写法兼容动作选择、任务规划、风险预判和数字执行体操作，适合母案级布局。

独权中应当保留“当前空间、当前执行主体或当前任务”的迁移出口。该写法覆盖跨空间、跨主体和跨任务三条产业化路径，且不绑定具体机器人或软件智能体形态。

### 6.3 可以大胆保留的部分

权利要求5的跨空间迁移可以保留。虽然近似文献涉及语义地图或跨环境学习，但其迁移对象通常是地图标签、策略参数或任务行为树。本案以概念标签及典型经验模式作为迁移载体，仍具有区别。

权利要求6的跨主体迁移可以保留。跨本体强化学习文献虽接近“不同执行主体之间迁移”，但其核心是策略模型或本体上下文。本案通过概念标签匹配和能力约束转换典型动作参数，不宜主动删除。

权利要求7的真实物理环境和真实数字环境可以保留为从属限定。该项有助于覆盖数字执行体、GUI智能体、API智能体和虚拟世界运行实例，同时不影响独权的宽度。

权利要求8的更新机制可以保留。该项对抗静态知识图谱、预设本体和普通标签体系具有价值，并为后续概念生命周期管理、冲突仲裁和重新验证预留支撑。

### 6.4 可以放入说明书而不进入权利要求的内容

概念标签的具体形成算法、标签命名方式、人工确认方式、语言模型辅助标注方式、图数据库实现方式、隐私脱敏方式、任务处理模板、概念生命周期管理、跨主体冲突仲裁、概念驱动安全预判等内容，建议主要放入说明书实施方式中。上述内容能够增加公开厚度和后续延展空间，但如果全部进入本案权利要求，可能削弱母案独权的覆盖范围。

## 七、背景技术更新建议

背景技术应当吸收近似案暴露出的四类现有路线：

1. 语义地图与多模态聚类打标路线；
2. 知识图谱、概念图谱和本体匹配路线；
3. 行为概念知识图谱与行为树规划路线；
4. 强化学习、跨本体策略迁移和大模型控制路线。

但不建议在说明书中点名具体专利号。宜采用技术路线描述方式，指出其共同不足：未从情境、动作和后果共同约束的交互经验记录中形成概念标签，也未将典型后果模式作为可迁移经验的核心组成部分。

## 八、后续撰写重点

说明书中应持续避免把本案写成以下类型：

1. 普通概念图谱构建；
2. 多模态聚类后大模型打标；
3. 知识图谱行为规划；
4. 强化学习策略迁移；
5. 大语言模型任务规划。

应当持续强调：

1. 概念标签由交互经验记录形成；
2. 交互经验记录至少包括情境、动作和后果；
3. 后果信息是动作后可观测结果；
4. 典型经验模式至少包括典型后果模式；
5. 可迁移经验由概念标签及典型经验模式确定。
