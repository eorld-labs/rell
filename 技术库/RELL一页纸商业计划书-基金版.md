# RELL 真实世界经验引擎一页纸商业计划书

版本：基金沟通外部版  
状态：核心专利方案已提交，开源 demo 与 Schema 已形成第一阶段产品雏形

## 一句话定位

RELL 面向机器人、Agent 和物理 AI，提供“真实世界经验引擎”：让自动化执行体在真实场景中能够复用经验、确认因果结果、识别不会做的任务，并在可控边界内形成可沉淀、可迁移、可审计的技能能力。

## 市场痛点

机器人和 Agent 正在从“生成答案”走向“执行动作”，但真实场景落地仍面临四个核心问题：

1. 场景适配成本高：同一技能换空间、对象、传感器或机器人本体后需要重新调试。
2. 执行过程不可控：许多系统只输出动作序列，缺少阶段推进、结果验真和失败恢复机制。
3. 经验难以复用：机器人执行后的成功经验、失败经验、补救经验和人类偏好没有形成可交易、可迁移的资产。
4. 安全边界不清：没有经验、缺少绑定或无法验证结果时，系统仍可能靠模型猜测执行，带来真实世界风险。

## 解决方案

RELL 不制造机器人本体，也不训练通用世界大模型，而是提供机器人落地所需的经验与执行中间层。

核心机制包括：

1. 过程模板：把“技能”从原子动作升级为阶段化物理过程，定义阶段序列、连续状态变量、阶段跃迁条件和参数槽。
2. 偶然绑定：把同一过程模板适配到当前对象、传感器、阈值、空间和用户偏好，降低跨场景部署成本。
3. 因果事实验真：不只判断“动作是否执行”，而是验证“目标事实是否成立”，例如杯中是否确实有水。
4. 最终成立状态回流：根据事实成立、冲突或失败结果，触发阶段推进、恢复执行、后续动作或经验回写。
5. 能力准入机制：没有经验、模板、绑定或验真条件时，系统声明不会执行，并生成待训练或待共创任务，避免盲目动作。

## 产品形态

1. 开源社区版：开放基础 Schema、最小 demo 和通用样例，建立事实标准和开发者生态。
2. 专业 SDK/API：面向机器人公司、Agent 平台和系统集成商，提供经验检索、过程模板调用、绑定生成、结果验真和审计接口。
3. 行业经验库与技能包：沉淀工厂、仓储、养老、家庭、园区等场景的可复用经验和经过验证的技能包。
4. 企业/行业版：提供私有化部署、行业规则、技能包管理、执行审计和治理接口。
5. 专利许可与联合方案：面向机器人公司、平台公司和行业客户提供技术授权、产品线授权或联合交付分成。

## 商业模式

1. SDK/API 订阅：按开发者、设备、执行体、调用量或部署实例收费。
2. 技能包与经验库授权：按行业、场景、技能包或机器人型号授权。
3. 私有化部署：收取部署费、年度维护费、模块授权费和扩容费用。
4. 行业共创服务：与机器人公司或场景方共同训练真实场景技能，形成样板项目和后续授权收入。
5. 专利许可：对关键过程模板、经验复用、因果验真与执行回流机制进行授权。

## 目标客户

1. 机器人公司：降低场景落地和技能适配成本。
2. 具身智能/VLA/世界模型团队：为模型输出提供真实执行前后的经验验证和回写层。
3. 系统集成商：提供行业技能包、部署工具和审计能力。
4. 企业自动化客户：用于高风险 Agent、RPA、IoT 或机器人执行链路的准入与治理。
5. 行业场景方：养老、仓储、工厂、园区、家庭服务等需要真实场景落地的客户。
