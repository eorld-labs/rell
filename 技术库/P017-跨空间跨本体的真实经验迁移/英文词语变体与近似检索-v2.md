# P017 英文词语变体与近似检索 v2

## 一、术语调整结论

P017 不宜使用“本体准入”“跨本体准入”作为主术语。该表述容易与 P008 的“主体能否进入空间、空间行动边界、空间准入”叙事混淆。

P017 应统一使用以下术语：

```text
本体能力画像
跨本体适配判断
执行可行性结果
基于本体能力画像的经验步骤执行可行性判断
```

对应英文建议：

```text
executor capability profile
embodiment capability profile
robot capability profile
embodiment-aware adaptation
cross-embodiment adaptation assessment
execution feasibility result
skill executability assessment
experience-step executability assessment
```

不建议作为 P017 主术语：

```text
embodiment admission
body admission
spatial admission
access admission
entry admissibility
```

原因：英文语境中 admission/admissibility 更容易落入 access control、spatial access、planning admissibility heuristic 或 safety admission，不如 executability / feasibility / applicability / adaptation assessment 精确。

## 二、英文词语变体

### 2.1 经验不变量契约

可用于英文检索的变体：

```text
"experience invariant contract"
"transferable experience invariant"
"experience invariant representation"
"invariant experience descriptor"
"embodiment-independent experience descriptor"
"reusable robot experience invariant"
"experience contract for robot skill transfer"
"topological invariant robot skill transfer"
"fact-based termination robot skill"
"robot skill termination condition invariant"
```

判断：

- “invariant feature space / invariant representation”是现有技术常用词，主要用于学习表征。
- “invariant contract”在形式化方法中存在，但未见其用于机器人经验迁移资产结构。
- P017 应把“contract”解释为经验记录中的数据约束结构，而不是形式化验证合同或软件契约。

### 2.2 本体能力画像

可用于英文检索的变体：

```text
"executor capability profile"
"robot capability profile"
"embodiment capability profile"
"robot embodiment profile"
"embodiment descriptor"
"morphology capability profile"
"robot morphology descriptor"
"end-effector capability profile"
"robot capability matching"
"skill capability matching"
```

判断：

- "capability profile" 和 "capability matching" 在工业机器人技能、生产系统和数字孪生中有公开。
- "embodiment profile" 在部分机器人技能 grounding 语境中出现，但更多用于描述机器人形态或生成代码适配。
- P017 的区别是本体能力画像不单独用于设备描述，而是与经验不变量契约和 RuntimeWorldState 共同生成执行可行性结果。

### 2.3 RuntimeWorldState

可用于英文检索的变体：

```text
"runtime world state"
"task-scoped world state"
"execution-time world state snapshot"
"ephemeral world state"
"task working memory robot world state"
"runtime state snapshot robot skill"
"temporary world state robot execution"
"world state snapshot skill execution"
```

判断：

- "world state" 和 "internal knowledge base" 是 TAMP 和技能规划常用词。
- P017 必须强调 runtime / task-scoped / ephemeral / snapshot，区别于长期世界模型、知识库或规划数据库。

### 2.4 执行可行性与适配判断

可用于英文检索的变体：

```text
"skill executability assessment"
"execution feasibility result"
"skill feasibility assessment"
"skill applicability assessment"
"robot skill applicability"
"experience step executability"
"embodiment-aware skill adaptation"
"capability-based skill matching"
"robot capability matching skill"
"skill-to-robot capability matching"
```

判断：

- skill applicability / skill precondition 是现有技术高频词。
- executability 在任务规划、机器人技能和世界模型中出现，但通常不与经验不变量契约、运行时快照和本体能力画像组合。

## 三、英文多轮检索记录

### 3.1 跨本体与机器人迁移

检索式：

```text
"cross embodiment" robot skill transfer morphology profile
"robot" "embodiment profile" "skill transfer"
"robot" "capability profile" "skill transfer" "world state"
"robot" "experience" "invariant" "skill transfer"
```

代表结果：

1. Transfer learning in robotics 综述提出机器人、环境和任务是机器人迁移学习的三个基本维度，并指出识别任务、环境和机器人之间的相同与差异仍是关键问题。
2. Cross-Embodiment Robot Manipulation Skill Transfer using Latent Space Alignment 关注不同机器人本体之间的技能迁移，但主线是潜在空间对齐。
3. Learn from Robot: Transferring Skills for Diverse Manipulation Via Cycle Generative Networks 关注跨机器人策略迁移，使用 latent action space 和 cycle generative network。
4. Learning Invariant Feature Spaces to Transfer Skills with Reinforcement Learning 使用 invariant feature spaces 在形态不同的 agent/robot 之间迁移技能。

压力判断：

这些文献对“跨本体迁移”名称有压力，但主线多是策略、轨迹、潜在空间或表征学习。P017 不主张学习策略，也不主张潜在空间对齐，而主张经验资产结构、能力画像和任务期世界状态共同完成执行可行性判断。

### 3.2 技能前置条件、效果模型和世界状态

检索式：

```text
"robot skill" "preconditions" "effects" "world state"
"robot skill" "applicability" "world state"
"robot skill" "executability" "capability"
"task and motion planning" robot current world state precondition effects skills
```

代表结果：

1. US12208522B2 公开 skill model、pre-condition model、effect model 和 final condition model，并使用这些模型构造任务 composition model，跟踪 system state evolution。
2. US11590651B2 公开 manipulation skill training，基于 precondition model 判断当前场景与示范场景的相似性，并可预测 skill effect。
3. Learning Model Preconditions for Planning with Multiple Models 将 skill precondition 描述为 skill applicability，并区分 precondition、model deviation estimator 和 effect model。
4. Concept for the automated adaption of abstract planning domains for specific application cases in skills-based industrial robotics 公开 abstract planning domain、specific planning domain、skills、preconditions、effects 和 internal knowledge base。

压力判断：

这些结果对 P017 的“目标因果事实反推前提链”“候选过程链生成”“世界状态更新”构成中高压力。因此上述内容不宜作为 P017 独权唯一核心，应作为从权或说明书种子。P017 独权应坚持“经验不变量契约 + 本体能力画像 + 任务期 RuntimeWorldState + 执行可行性结果”的组合。

### 3.3 Capability-based skill frameworks

检索式：

```text
"robot skill" "capability matching"
"capability-based frameworks" "robot skills"
"robot capability profile" "skill"
"skill-to-robot capability matching"
```

代表结果：

1. Capability-based Frameworks for Industrial Robot Skills: a Survey 讨论 hardware/vendor independence、skill applicability 和 available hardware functionalities mapping。
2. A capability and skill model for heterogeneous autonomous robots 讨论异构机器人 capability/skill model。
3. Concept for automated adaption of abstract planning domains 讨论 abstract skills、specific skills、specific planning domain 和机器人实际应用场景中的技能适配。

压力判断：

该方向对“本体能力画像/能力匹配”有压力。但这些方案通常围绕工业技能框架、技能参数化、能力匹配或规划域适配，不强调经验记录本身不保存绝对坐标、关节角和固定时长，也不强调 RuntimeWorldState 的任务期生命周期。

### 3.4 Patent search

检索式：

```text
site:patents.google.com "robot" "capability profile" "skill"
site:patents.google.com "robot" "embodiment profile"
site:patents.google.com "robot" "skill transfer" "world state" "precondition"
site:patents.google.com "robot" "invariant" "skill transfer"
site:patents.google.com "robot skill" "precondition model" "effect model"
```

代表结果：

1. US12208522B2：pre-condition model、effect model、final condition model、composition model。
2. US11590651B2：precondition model、effect model、object pose manifold、kinesthetic demonstration。
3. WO2025155646A1：physical skills transfer to robots，preconditions may include arm position、gripper state、scene properties。
4. US11498212B2：robot manipulation planning、symbolic abstraction and skill sequence。

压力判断：

专利侧主要压力集中在“技能模型 + 前置条件 + 效果模型 + 世界状态演化”。暂未发现将以下组合放在一起的单篇公开：

```text
experience invariant contract
+ current spatial semantic data
+ executor / embodiment capability profile
+ task-scoped runtime world state
+ embodiment-aware adaptation assessment / execution feasibility result
+ release of runtime world state after task completion
```

## 四、对权利要求 v2 的影响

### 4.1 术语影响

建议 v2 统一采用：

```text
执行可行性结果
适配可行性判断
跨本体适配判断
本体能力画像
```

避免：

```text
本体准入
跨本体准入
迁移准入
执行准入
```

### 4.2 字段影响

P017 的本体能力画像不应写入 P008 色彩过强的字段，例如：

```text
主体包络
移动半径
转向半径
可跨越高度
可进入最小宽度
可停靠面积
空间进入约束
```

这些更适合由 P008 或空间治理层承担。

P017 中建议保留：

```text
执行器可达范围
末端执行器类型
工具接口
抓取方式
执行器自由度
末端姿态范围
传感器视场
传感器坐标系
负载能力
速度范围
力控范围
控制接口
可用适配器能力
```

### 4.3 权利要求影响

P017 v2 独权应继续守最小必要主链：

```text
经验不变量契约
→ 当前空间语义数据
→ 当前执行体本体能力画像
→ RuntimeWorldState
→ 跨空间跨本体适配
→ 执行可行性结果
→ 执行闭环
→ RuntimeWorldState 更新与释放
```

目标因果事实反推前提链和显式空间路线保真仍建议放从权或说明书，不进入独权必要特征。

## 五、英文检索结论

英文语境下，P017 的主要近似压力不是“本体准入”，而是：

1. skill precondition / effect model / world state；
2. capability-based skill matching；
3. cross-embodiment policy or trajectory transfer；
4. invariant feature spaces for skill transfer。

P017 的区别应表述为：

```text
It does not learn or transfer a control policy, trajectory, latent action distribution, or generic skill precondition model.
It migrates an experience record by matching an experience invariant contract with current spatial semantics, an executor capability profile, and a task-scoped runtime world-state snapshot to produce an execution feasibility result.
```

中文撰写对应为：

P017 不学习或迁移控制策略、轨迹、潜在动作分布或普通技能前置条件模型，而是基于经验不变量契约、当前空间语义、本体能力画像和任务期运行时世界状态快照，对经验步骤生成执行可行性结果或绑定候选。
