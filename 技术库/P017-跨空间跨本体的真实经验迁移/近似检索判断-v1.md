# P017 近似检索判断 v1

## 一、检索目的

本轮检索用于在 P017 权利要求书 v1 成稿后，初步判断“跨空间跨本体的真实经验迁移”是否存在高相似公开方案，并据此提出权利要求收口建议。

检索结论仅为立项阶段初步判断，不替代正式专利数据库检索。后续正式提交前仍建议在 CNIPA、智慧芽、Incopat、Google Patents、WIPO 和英文论文数据库中继续扩大检索。

## 二、P017 核心技术特征提炼

P017 当前权利要求 v1 的核心组合为：

```text
待迁移经验记录
→ 经验不变量契约
→ 当前空间语义数据
→ 当前执行体本体能力画像
→ RuntimeWorldState 运行时世界状态快照
→ 基于目标因果事实生成或校验候选过程链
→ 基于不变量契约与本体能力画像生成绑定候选或执行准入结果
→ 交由执行闭环执行
→ 任务结束后释放 RuntimeWorldState，仅保留审计或经验摘要
```

区别性抓手：

1. 经验库保存可迁移不变量，而不是绝对坐标、关节角、固定时长或单一本体轨迹。
2. 经验迁移时同时读取当前空间语义和当前执行体本体能力画像。
3. RuntimeWorldState 是当前任务期间的运行时快照，不是长期世界数据库。
4. 目标因果事实可反推前提过程链。
5. 用户显式教入的空间路线可被保留，并接受因果校验。

## 四、近似文献与压力判断

### 4.1 机器人技能前后条件与世界状态模型

代表文献：

- “Does your Robot have Skills?” 提到技能的前置条件和后置条件可作为世界状态查询，用于判断是否可执行和是否成功。[来源](https://vbn.aau.dk/ws/files/69945674/ISR_2012_Paper_ID_1158_FINAL.pdf)
- “Integrating Mission, Logistics, and Task Planning for Skills-Based Robot Control” 公开技能输入参数和世界状态，输出状态变化，并由世界模型作为跨层工作记忆。[来源](https://ceur-ws.org/Vol-1782/paper_1.pdf)
- “Learning Composable Models of Parameterized Skills” 公开参数化技能的前置条件、效果、目标驱动反向搜索和对象关系特征。[来源](https://lis.csail.mit.edu/pubs/lpk/ICRA17.pdf)

相似点：

- 都涉及技能、世界状态、前置条件、后置效果或目标驱动规划。
- “Learning Composable Models” 对目标驱动反向搜索和对象关系表达构成较强近似压力。

差异点：

- 这些方案主要服务于 TAMP 或技能规划，不以“经验不变量契约 + 本体能力画像 + 任务期运行时世界状态快照”作为经验迁移闭环。
- 其世界模型通常是系统内共享知识/规划表示，未强调任务结束后释放的端侧运行时快照。
- 其技能参数和效果模型仍大量围绕参数化技能、对象位姿、机器人配置或可行性约束展开，未明确把经验存储政策限定为“不存绝对坐标、固定关节角、固定时长”。

判断：

中高压力。P017 不能把创新点写成普通“技能前置条件和后置条件”或“从目标反推前提”。这些内容应放从权。独权必须坚持经验迁移三件套。

### 4.2 机器人技能模板、轨迹模型和技能组合专利

代表文献：

- US12208522B2 公开从示范训练技能的轨迹模型、前置条件模型和最终条件模型，并按任务描述组合技能执行。[来源](https://patents.google.com/patent/US12208522B2/en)
- US11498212B2 公开学习多个操作技能的符号抽象，并根据目标规格确定技能序列。[来源](https://patents.google.com/patent/US11498212B2/en)
- US11590651B2 公开训练机器人操作技能模型，确定任务参数是附着于对象的参数还是自由参数，并基于示范数据训练技能模型。[来源](https://patents.google.com/patent/US11590651B2/en)

相似点：

- 均涉及技能模型、任务参数、前置条件/最终条件、技能组合。
- 对“经验记录 + 技能序列 + 当前任务执行”的宽泛写法构成压力。

差异点：

- 这些专利主要围绕示范学习、轨迹模型、技能模型训练或符号抽象组合。
- P017 的重心不是训练轨迹模型，也不是从示范学习技能参数，而是把已有经验以不变量契约形式迁移到当前空间和当前本体。
- P017 明确引入本体能力画像，并将其用于跨本体准入和绑定候选生成。

判断：

中高压力。权利要求应避免“训练技能模型”“轨迹模型”“技能符号抽象”作为主线；应强调已沉淀经验的迁移适配，而不是技能训练。

### 4.3 跨本体机器人技能迁移论文

代表文献：

- “Cross-Embodiment Robot Manipulation Skill Transfer using Latent Space Alignment” 关注不同形态机器人之间的控制策略迁移，将源/目标机器人的状态和动作空间对齐到共同潜在空间。[来源](https://arxiv.org/abs/2406.01968)
- “Trajectory Conditioned Cross-embodiment Skill Transfer” 通过稀疏光流轨迹作为跨本体运动提示，实现从人类示范视频到机器人动作的迁移。[来源](https://arxiv.org/abs/2510.07773)
- “Human-Humanoid Robots Cross-Embodiment Behavior-Skill Transfer...” 使用统一数字人模型、人体-对象交互图、运动重定向和动态微调，实现不同人形机器人平台间迁移。[来源](https://arxiv.org/abs/2412.15166)

相似点：

- 都关注 cross-embodiment skill transfer。
- 都承认不同机器人形态、动作空间和运动学差异是迁移难点。

差异点：

- 这些论文多聚焦策略、轨迹、潜在空间、视觉/视频示范或运动重定向。
- P017 不主张学习跨本体控制策略，而是主张经验资产的数据结构和迁移准入机制。
- P017 的本体能力画像不是神经网络条件输入，而是用于匹配经验不变量、空间语义和执行能力的可审计结构。

判断：

中等压力。对“跨本体技能迁移”名称有压力，但对 P017 的具体工程闭环压力较低。权利要求中应避免落入“跨本体策略学习/潜在空间对齐/轨迹重定向”。

### 4.4 机器人示范数据采集与语义标签

代表文献：

- CN120395873A 公开人类示范引导的机器人工厂操作数据采集，涉及语义解析、控制指令、工作空间限制、关节保护、电流过载阈值和传感器反馈。[来源](https://patents.google.com/patent/CN120395873A/zh)

相似点：

- 涉及人类示范、语义解析、机器人传感器数据、边界约束和操作数据。

差异点：

- 该文献以数据采集和控制指令生成为主，示例中仍包括原子操作序列、关节位姿和轨迹。
- P017 不保护数据采集过程，而保护经验沉淀后的不变量契约、当前本体能力画像和运行时世界状态共同完成迁移准入。

判断：

中低压力。对“教学生成经验”有背景压力，但不覆盖 P017 三件套。

### 4.5 轴不变量/逆运动学类方案

代表文献：

- CN108942942B 公开基于轴不变量的多轴机器人逆运动学建模与求解方法。[来源](https://patents.google.com/patent/CN108942942B/en)

相似点：

- 使用“不变量”概念，并涉及不同机械臂的运动学求解。

差异点：

- 该文献的不变量是运动学建模中的轴不变量。
- P017 的不变量是经验迁移层的数据契约，包括拓扑关系、试探方向/物理约束和事实终止条件，不限定具体逆运动学求解算法。

判断：

低压力。主要提醒 P017 不要把“本体能力画像”写成具体 IK 方法。

## 五、检索后对权利要求 v1 的判断

### 5.1 v1 的优点

1. 独权没有落在普通技能库、示范学习或轨迹模型上。
2. 将经验不变量契约、本体能力画像和 RuntimeWorldState 串成完整迁移闭环。
3. 目标因果事实反推和显式空间路线保真放在从权，避免被 TAMP/技能前后条件文献直接攻击独权。
4. 明确 RuntimeWorldState 的生命周期，区别于长期世界模型或共享知识库。

### 5.2 v1 的风险

1. 独权步骤较多，后续可考虑压缩独权为 6 步左右。
2. “目标因果事实、过程链或过程模板引用”与 P016 有交叉，说明书中需要强调 P017 是迁移准入层，不是执行闭环本身。
3. “执行闭环”表述较宽，需要在说明书中写明其可以是 P016 Runtime，也可以是其他返回事实成立状态的执行模块，避免被认为不清楚。
4. “经验不变量契约”是自定义术语，需要在说明书术语部分定义清楚，避免被审查员认为抽象。

## 六、建议调整方向

### 6.1 独权建议保留的最小必要特征

独权应坚持以下组合：

```text
经验记录包括经验不变量契约
当前空间语义数据
当前执行体本体能力画像
运行时世界状态快照
基于不变量契约和本体能力画像进行迁移适配/准入
执行期间更新运行时世界状态，任务结束后释放
```

目标因果事实反推和显式路线保真可以继续放从权，不必进独权最小必要闭环。

### 6.2 说明书必须展开的三块

1. 经验不变量契约实施例：倒水、抓取、插入各举一例，说明哪些信息不存，哪些信息存。
2. Executor Profile 实施例：人形机器人、轮式机械臂、数字执行体各一例，说明同一经验如何得出不同准入结果。
3. RuntimeWorldState 实施例：接水任务中位置、持有物、对象位置、已成立事实如何随过程步骤更新，任务结束后如何释放。

### 6.3 需要避免的写法

1. 避免把 P017 写成“学习机器人技能模型”。
2. 避免把 P017 写成“普通任务规划/动作前后条件”。
3. 避免把 P017 写成“跨本体策略学习/潜在空间对齐/轨迹重定向”。
4. 避免把 RuntimeWorldState 写成长期世界模型或全局知识库。
5. 避免把经验不变量契约写成单纯参数模板。

## 七、初步结论

本轮检索未发现单篇公开同时覆盖以下完整组合：

```text
经验不变量契约
+ 当前空间语义数据
+ 当前执行体本体能力画像
+ 当前任务期 RuntimeWorldState
+ 基于不变量契约和本体能力画像生成跨空间跨本体准入/绑定候选
+ 执行期间更新 RuntimeWorldState，任务结束后释放
```

最接近的现有技术分布在三个方向：

1. TAMP/技能前后条件/世界状态；
2. 跨本体策略或轨迹迁移；
3. 示范学习、技能模板和轨迹模型。

因此，P017 具备继续推进的价值。后续撰写说明书时，应把创造性集中在“经验资产如何从具体轨迹参数变成可迁移不变量，并通过本体能力画像和任务期运行时世界状态完成迁移准入”这一点上。

## 八、权利要求 v2 收口建议

结合当前中国专利审查环境和非正常申请风险，v1 的 20 条权利要求数量偏多，且系统、电子设备、存储介质、空间语义来源枚举和执行体类型枚举等内容容易造成铺量观感。建议采用 P016 的母案打法：权利要求克制，说明书展开充分。

### 8.1 v2 独权最小必要闭环

v2 独权建议仅保留以下主链：

```text
经验记录包括经验不变量契约
→ 获取当前空间语义数据
→ 获取当前执行体本体能力画像
→ 生成任务期 RuntimeWorldState
→ 基于不变量契约、空间语义、本体画像和 RuntimeWorldState 进行迁移适配/执行准入
→ 执行后依据产出事实和销毁事实更新 RuntimeWorldState
→ 任务结束后释放 RuntimeWorldState 并写入审计/经验摘要
```

该主链避免把独权写成普通任务规划、普通技能库或普通跨本体策略迁移。

### 8.2 v2 从权保留内容

从权保留：

1. 经验不变量契约的三类不变量；
2. 三类不变量的具体示例；
3. 不保存绝对坐标、关节角、固定执行时长等排除式限定；
4. 基于不变量契约与本体能力画像的匹配过程；
5. 本体能力画像字段；
6. RuntimeWorldState 字段和非长期数据库属性；
7. 产出事实/销毁事实更新；
8. 目标因果事实反推前提链；
9. 显式空间路线保真与因果校验。

### 8.3 放入说明书的内容

以下内容建议不进入当前权利要求，而作为说明书实施方式和分案种子：

1. 系统、电子设备、计算机可读存储介质；
2. SDK/API、云端、边缘端、机器人端部署方式；
3. 空间语义来源枚举；
4. 执行体类型枚举；
5. 执行闭环具体形态；
6. 抓取、倒水、插入、搬运、导航、数字执行体实施例；
7. Executor Profile 的不同机器人形态实施例；
8. 长程任务滚动执行上下文实施例。

### 8.4 v2 判断

v2 控制在 10 条权利要求以内，更适合当前阶段作为母案版本。其保护重心集中、非正常风险相对较低，同时通过说明书种子保留后续分案空间。
