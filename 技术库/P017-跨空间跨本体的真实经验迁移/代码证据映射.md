## 一、证据目标

本文件用于证明 P017“跨空间跨本体的真实经验迁移”已经在 RELL 样品中形成工程实现。证据重点不是 P016 已提交的阶段跃迁和目标因果事实验真，而是经验进入执行闭环之前的迁移、准入、状态对齐和路线校验机制。

## 三、可验证样例

### 3.1 短目标反推前提链

输入：

```text
到水源处接一杯水
```

预期：

```text
目标事实：cup_contains_water
反推链：move_to_counter -> pick_up_cup -> move_to_water_source -> fill_cup_at_water_source
```

证据意义：系统不是要求经验库存在同名短句经验，而是从目标事实和当前世界状态反推前提过程。

### 3.2 显式空间路线保真

输入：

```text
走到门旁边，再走到服务位，再去操作台拿杯子，去水源处倒杯水
```

预期：

```text
显式路线：move_to_doorway -> move_to_service_position -> move_to_counter -> pick_up_cup -> move_to_water_source -> fill_cup_at_water_source
目标事实：cup_contains_water
```

证据意义：用户显式教入的空间路标被保留，并通过因果层校验是否足以达成目标事实。

### 3.3 运行时世界状态更新

执行过程中：

```text
初始位置：region_floor_walkway
move_to_counter 后：executor.location_ref = region_counter_operation
pick_up_cup 后：executor.holding 包含 object_cup_white_mug
move_to_water_source 后：executor.location_ref = region_water_source
fill_cup_at_water_source 后：established_facts 包含 cup_contains_water
```

证据意义：运行时世界状态用于任务期间的状态对齐，用完即释放，不作为长期世界数据库。

### 3.4 经验不变量契约

经验记录包含：

```json
{
  "storage_policy": "store_invariants_not_concrete_parameters",
  "invariant_dimensions": [
    "topology_relation",
    "exploratory_direction_and_physical_constraint",
    "fact_based_termination_condition"
  ],
  "forbidden_storage": [
    "absolute_coordinates",
    "robot_specific_joint_angles",
    "fixed_execution_duration",
    "single_body_trajectory_without_binding_slots"
  ]
}
```

证据意义：经验从具体轨迹参数上升为可跨空间、跨本体迁移的不变量。

## 五、证据结论

P017 已经具备代码证据链。其代码实现与 P016 已提交方案不同，重点位于执行闭环之前的经验迁移和适配判断层：

```text
经验记录
→ 不变量契约
→ 当前空间语义
→ RuntimeWorldState
→ Executor Profile
→ 因果链/显式路线校验
→ 当前执行环境绑定候选
→ P016 Runtime 或其他执行闭环
```

因此，P017 适合作为新案立项，而不是仅作为 P016 的分案备忘。
