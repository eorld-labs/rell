# P017 英文定向查漏检索 v1

检索日期：2026-07-10

## 一、检索目的

本轮检索用于复核 P017 权利要求 v3 的 6 段主链是否在英文专利或学术语境中被单篇公开整体覆盖。

P017 当前主链为：

1. 获取待迁移经验记录，所述经验记录包括经验不变量契约；
2. 获取当前迁移上下文，包括当前空间语义数据、本体能力画像和任务意图；
3. 生成任务期运行时世界状态快照；
4. 基于经验不变量契约、当前空间语义数据、本体能力画像和任务期运行时世界状态快照，对经验步骤进行跨空间跨本体适配，生成绑定候选或执行可行性结果；
5. 可执行时交由开放式执行闭环执行，并依据因果产出事实和因果销毁事实更新快照；
6. 任务结束后删除或释放任务期运行时世界状态快照，并写入审计记录或经验记录。

## 五、对 P017 撰写的反推建议

### 1. 独权维持当前 6 段主链

当前独权不宜再加入 PDDL、技能 precondition/effect 模型、LLM 任务分解、视觉 affordance 检测等内容，否则会主动靠近已有英文近邻。

### 2. 说明书中增加区分性表述

建议说明书明确：

本发明不以学习或迁移控制策略、轨迹、关节角、动作分布、潜在动作空间或普通技能前置条件模型为必要方式，而是基于经验记录中的经验不变量契约，与当前空间语义数据、本体能力画像和任务期运行时世界状态快照进行适配，生成当前执行环境下的绑定候选或执行可行性结果。

### 3. “本体能力画像”不要单独拔高成唯一创新点

capability profile / capability-based skill matching 已有较多研究。本体能力画像应与经验不变量契约、空间语义和任务期快照组合表述。

### 4. “任务期运行时世界状态快照”的生命周期要写实

任务期生成、任务中更新、任务后删除或释放，是 P017 与普通世界模型、长期知识库、任务规划状态表的重要区别之一。该点应在说明书中给足实施例。

### 5. 不可执行/部分不可执行分支放说明书较合适

英文近邻中已有 infeasible plan、fallback、human intervention 的方向。P017 可在说明书中写明不可执行、部分不可执行、请求人工教学、替代经验搜索等分支，但不建议把这些分支全部塞入独权。

## 六、结论

从英文专利和学术查漏看，P017 当前 v3 的独权方向可以继续推进。

最大风险不是“完整主链被公开”，而是审查中可能把不同英文近邻拼接为：

- 技能 precondition/effect/world state；
- capability matching；
- cross-embodiment transfer；
- LLM/VLM task grounding；
- failure recovery/human intervention。

因此 P017 的抗辩核心应锁定为：

P017 不是普通技能规划、不是轨迹或策略迁移、不是单纯 capability matching、不是 LLM 任务分解，而是以经验记录中的经验不变量契约为迁移对象，并在当前空间语义、本体能力画像和任务期运行时世界状态快照的共同约束下生成执行可行性结果，在执行闭环返回事实状态后更新并释放任务期快照。

### 7.3 新增重点近邻

#### 7.3.1 US20230114561A1 / WO2021236506A1：Robotic demonstration learning device / Distributed robotic demonstration learning

该族专利公开 skill template，并明确提出 skill template 可以是 hardware agnostic。其方案允许在某些子任务中通过本地示教数据来细化 base control policy，同一 skill template 可用于不同任务或不同机器人模型。

风险判断：

- 对“技能模板、硬件无关、局部示教、跨机器人适配”压力较大。
- 但其核心仍是围绕示教数据和 base control policy 的训练/细化，不是以经验不变量契约作为迁移对象，也不要求生成任务期运行时世界状态快照并在任务后释放。
- 对 P017 的启示是：说明书中应明确 P017 不以训练或定制 control policy 为必要方式。

#### 7.3.2 A Hardware-Agnostic OPC UA Skill Model for Robot Manipulators and Tools

该论文公开基于 OPC UA 的硬件无关技能模型，目标是通过标准化接口让不同制造商机器人或工具可被同一 skill interface 控制。

风险判断：

- 对“硬件无关接口、机器人能力封装、技能复用”压力较大。
- 但其解决的是设备/工具功能的统一接口和工业组件互换，不是跨空间跨本体迁移真实经验记录。
- P017 中“执行闭环开放接口”和“本体能力画像”不要被写成单纯接口适配或 OPC UA 类 skill interface。

#### 7.3.3 A Case-Based Framework for Task Demonstration Storage and Adaptation

该方向以 case-based reasoning 存储单次示教作为 source case，在新环境包含不同对象、特征或配置时检索、适配和执行该 source case。

风险判断：

- 这是“经验记录/示教案例迁移”方向中较接近的近邻。
- 其公开重点在 case-based imitation learning 和 demonstration storage/adaptation，没有看到“经验不变量契约、本体能力画像、任务期快照生命周期、执行可行性结果”的组合。
- P017 要避免把经验记录说成简单“示教案例相似检索”，应强调经验不变量契约限定可迁移内容。

#### 7.3.4 Intuitive Task-Level Programming by Demonstration Through Semantic Skill Recognition

该论文公开在示教交互过程中持续更新 world model，语义技能识别算法查询对象关系、相对/绝对位姿，并基于 PDDL 标注的 conditions and effects 识别和选择技能。

风险判断：

- 对“空间语义、world model、对象关系、conditions/effects、示教识别技能”压力较大。
- 但其 world model 是持续维护的图数据库抽象层，主线是从示教中识别技能并生成任务级程序；P017 的任务期运行时世界状态快照强调任务期生成、执行中更新、任务后删除或释放。
- P017 说明书中应把“任务期快照”与长期 world model / knowledge graph / Neo4j 图数据库类方案区分清楚。

#### 7.3.5 Relational Learning for Skill Preconditions

该论文明确：机器人需要学习技能前置条件以判断某技能能否在给定环境中执行，尤其关注对象关系和不同形状、尺寸对象下的泛化。

风险判断：

- 对“技能可执行性、对象关系、跨情境判断”压力较大。
- 但其是学习技能前置条件模型，不是迁移经验不变量契约，也不涉及本体能力画像与任务期快照释放。
- P017 中“执行可行性结果”不应被解释为单纯 learned precondition model。

#### 7.3.6 SkillMaN

该论文公开带有 experiential knowledge 的机器人操作规划与执行框架，包含 workflow、轨迹、几何工具、推理能力，并能在动作 precondition 不满足时解释实际感知状态并反馈更新状态给 planner。

风险判断：

- 对“经验知识、技能执行、反馈状态给规划器、异常适应”压力较大。
- 但其经验知识仍围绕如何执行技能的 workflow/trajectory，未显示 P017 的经验不变量契约与任务期快照生命周期。
- 这是说明书抗辩时需要重点区分的英文近邻之一。

#### 7.3.7 ConditionNET

该论文用视觉语言模型学习动作 preconditions 和 effects，并用于执行监控、异常检测和恢复。

风险判断：

- 对“动作前置条件/效果、执行监控、恢复、倒水场景”有局部压力。
- 但其方案是数据驱动学习 preconditions/effects，不是经验记录跨空间跨本体迁移。
- P017 可在说明书中说明：目标因果事实、因果产出/销毁事实或事实终止条件可以由学习模型辅助判断，但本发明不以学习上述条件模型为必要特征。

#### 7.3.8 ASPIRE

NVIDIA ASPIRE 方向公开从执行反馈中修复 code-as-policy，保存 reusable skills，用闭环执行引擎、技能库和演化搜索形成持续学习系统。

风险判断：

- 对“可复用技能库、闭环执行、执行反馈、持续改进”有方向压力。
- 但其主线是 LLM/code-as-policy 技能发现与修复，不是以经验不变量契约和任务期快照完成跨空间跨本体迁移。
- P017 应保持“可迁移经验资产结构”而非“代码策略生成/修复”的定位。

### 7.4 换词检索后的补充结论

英文换词后找到的真正近邻比第一轮更多，但仍然集中在以下旧路线：

1. 硬件无关 skill interface；
2. 示教学习 / skill template / base control policy；
3. case-based task demonstration adaptation；
4. semantic skill recognition + world model + PDDL conditions/effects；
5. learned skill preconditions/effects；
6. experiential knowledge + planner feedback；
7. LLM/code-as-policy reusable skill library。

这些路线可以分别压住 P017 的某个局部，但没有一条把下列要素合成同一条不可绕主链：

```text
经验不变量契约
+ 当前空间语义数据
+ 本体能力画像
+ 任务期运行时世界状态快照
+ 跨空间跨本体适配
+ 绑定候选或执行可行性结果
+ 执行闭环返回事实状态后更新快照
+ 任务结束后删除或释放快照
```

因此，当前独权不建议继续扩张，也不建议为了规避这些近邻而进一步拆断主链。主链中的每一段都有作用：

- 去掉经验不变量契约，会落入普通 skill/template/case reuse；
- 去掉空间语义，会落入普通跨机器人能力适配；
- 去掉本体能力画像，会落入普通场景规划或经验检索；
- 去掉任务期快照，会落入长期 world model / PDDL / planner state；
- 去掉执行可行性结果，会落入普通绑定或参数实例化；
- 去掉事实更新与释放，会削弱与普通技能库、任务规划器、知识图谱的区别。
