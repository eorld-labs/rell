# P017 业务逻辑剥离攻防分析 v1

日期：2026-07-10

> 对应版本：`权利要求书-v3-6段最小闭环母案版.md`、`说明书提纲-v1-含分案种子.md`  
> 目的：预判审查、无效或竞对规避中可能出现的“这只是业务逻辑、案例复用、规则配置、任务上下文或抽象软件流程”攻击，并反推说明书中的技术性表达。  
> 结论：P017 不应被写成“把历史案例搬到新场景”的业务复用方案，而应被写成“将真实经验中的可迁移不变量从坐标、轨迹、关节角和固定时长中剥离出来，并在当前空间语义、本体能力画像和任务期运行时世界状态快照共同约束下生成执行可行性结果”的技术方案。

## 一、为什么需要单独分析

P017 已经完成近似检索、竞对规避和无效宣告视角的补强，但还需要单独处理一个软件类、AI 类专利中常见的审查风险：对方可能不从机器人技能迁移文献正面攻击，而是把本案降格为“业务逻辑剥离”或“经验案例参数化复用”。

这种攻击的典型说法是：

1. 经验记录只是历史案例。
2. 经验不变量契约只是规则模板或业务约束。
3. 当前空间语义数据只是当前上下文。
4. 本体能力画像只是设备配置表。
5. 任务期运行时世界状态快照只是任务状态表。
6. 执行可行性结果只是业务规则判断。
7. 执行闭环返回和任务后释放只是普通日志回写和内存释放。

如果说明书没有把“真实经验迁移执行”的技术语境写牢，P017 容易被审查员或无效请求人读成通用工作流、案例推理、规则引擎或参数绑定系统。

## 二、本案不能被写成什么

### 2.1 不能写成普通案例复用

危险写法：

1. 存储历史任务案例。
2. 查找相似案例。
3. 将案例参数替换为当前参数。
4. 执行替换后的步骤。

风险：

这会落入案例推理、RAG、经验库检索或业务流程复用的常见表达，无法体现 P017 的技术核心。

### 2.2 不能写成普通规则模板

危险写法：

1. 定义可复用规则。
2. 当前上下文满足规则时执行。
3. 不满足规则时输出失败。

风险：

这种表述可套用于审批、客服、工单、交易风控和业务规则引擎。P017 的经验不变量契约必须被写成从真实执行经验中剥离出的跨空间、跨本体可迁移结构，而不是任意人为规则。

### 2.3 不能写成普通设备能力匹配

危险写法：

1. 读取机器人能力清单。
2. 判断机器人是否支持某动作。
3. 支持则调用，不支持则拒绝。

风险：

该写法会退化为 capability matching 或机器人平台接入协议。P017 的本体能力画像不是孤立能力清单，而是与经验不变量契约、当前空间语义和任务期快照共同参与适配可行性判断。

### 2.4 不能写成普通任务上下文

危险写法：

1. 创建任务上下文。
2. 任务执行时更新上下文。
3. 任务结束清理上下文。

风险：

这会被理解为常规软件状态管理。P017 的任务期运行时世界状态快照必须被写成用于经验迁移适配的事实对齐和工作记忆结构，其生命周期隔离用于避免临时事实污染长期世界模型或后续任务判断。

### 2.5 不能写成普通成功失败分支

危险写法：

1. 判断步骤可执行。
2. 可执行则执行。
3. 不可执行则提示失败。

风险：

这会被视为普通流程控制。P017 的执行可行性结果应写成由经验不变量契约、空间语义、本体能力画像和任务期快照共同形成的控制结果，可表现为绑定候选、阻断、降级、替代经验搜索、人工确认或教学入口。

## 三、本案的技术性支点

P017 的技术性不在“把业务逻辑从代码中抽出来”，而在以下六个彼此耦合的技术支点：

1. 处理对象是机器人、仿真执行体、数字执行体或软件执行体的真实经验迁移执行，而不是普通业务流程。
2. 经验不变量契约限定历史真实经验中可跨空间或跨本体复用的结构性内容，并排除绝对坐标、专用关节角、固定执行时长和单一本体轨迹。
3. 当前空间语义数据用于将经验不变量契约中的对象、区域、关系、可供性、导航关系或空间约束落到当前执行环境。
4. 本体能力画像用于判断当前执行体能否满足经验步骤所需的动作、可达、末端执行器、传感器、控制精度、力控或安全限制。
5. 任务期运行时世界状态快照用于当前任务中的事实对齐、前提成立判断、执行反馈更新和任务后释放，而不是长期世界模型或普通日志。
6. 执行可行性结果和执行闭环事实回传共同决定经验步骤是否能执行、如何执行、何时阻断以及执行后哪些事实发生变化。

只要这六点写实，P017 就不是业务规则或抽象流程，而是跨空间跨本体经验迁移执行中的技术方案。

## 四、可能攻击路径与防守

### 4.1 攻击 A：经验记录只是历史业务案例

对方可能说：

P017 只是保存历史经验，再在当前任务中复用，和案例推理、知识库检索或 RAG 没有本质区别。

防守要点：

1. 经验记录不是任意历史案例，而是包含目标因果事实、过程链或过程模板引用以及经验不变量契约的结构化记录。
2. 当前复用的不是历史案例中的文本、坐标或动作轨迹，而是被经验不变量契约限定的可迁移结构。
3. 经验步骤必须经过当前空间语义、本体能力画像和任务期快照共同适配，不能直接照搬。
4. 经验复用结果表现为绑定候选或执行可行性结果，不是简单返回相似案例。

说明书补强表述：

> 本发明中的经验记录不同于普通业务案例、日志摘要或相似案例文本。经验记录至少包括用于限定可跨空间或跨本体复用内容的经验不变量契约，系统在当前任务中并非直接复制历史坐标、动作轨迹或执行时长，而是基于所述经验不变量契约与当前空间语义数据、本体能力画像和任务期运行时世界状态快照进行适配判断。

### 4.2 攻击 B：经验不变量契约只是业务规则

对方可能说：

经验不变量契约不过是人为写下的规则条件，例如“杯子要有水”“机器人要拿到杯子”，属于抽象规则。

防守要点：

1. 经验不变量契约不是任意业务规则，而是从真实执行经验中剥离出的可迁移执行约束。
2. 其内容包括拓扑关系不变量、试探方向与物理约束不变量、事实终止条件不变量。
3. 其技术作用是排除不可迁移的低层参数，并保留能够在新空间和新本体中重新绑定的结构。
4. 契约的成立依赖当前空间对象、本体能力和任务期事实，不是纯文本规则判断。

说明书补强表述：

> 经验不变量契约用于描述真实经验中在空间布局变化或执行体变化后仍可迁移的执行约束，其可以包括拓扑关系、试探方向、物理边界、传感器可观测终止条件或事实成立条件。该契约不同于用于审批、交易、调度或其他非执行场景的业务规则，其技术作用在于将历史经验从绝对坐标、关节角、固定时长和单一本体轨迹中剥离，并使该经验能够在当前空间和当前本体条件下重新绑定和校验。

### 4.3 攻击 C：当前空间语义数据只是上下文参数

对方可能说：

当前空间语义数据只是给规则引擎提供的一组上下文变量，例如对象位置、区域名称、当前状态。

防守要点：

1. 当前空间语义数据不是任意上下文，而是用于将经验不变量契约绑定到当前执行环境的空间结构输入。
2. 其至少可承载语义区域、对象、对象关系、可供性、导航关系或空间约束。
3. 绑定结果影响经验步骤能否执行、执行对象如何选择、路径或作用部位如何确定。
4. 空间语义数据可由世界模型、语义地图、场景图、数字孪生、传感器或外部系统提供，但其技术作用固定在迁移适配链中。

说明书补强表述：

> 当前空间语义数据不是仅用于展示或业务判断的上下文字段，而是用于将经验不变量契约中的对象、区域、关系、可供性或空间约束映射到当前执行环境的数据。该映射结果直接参与绑定候选生成和执行可行性判断。

### 4.4 攻击 D：本体能力画像只是设备配置表

对方可能说：

本体能力画像就是机器人配置文件、设备档案或 SDK capabilities。

防守要点：

1. 本体能力画像可以来源于配置文件或 SDK，但其发明作用不是设备描述。
2. 它用于判断经验步骤在当前本体上是否满足动作、可达范围、末端能力、传感器能力、控制精度、力控和安全约束。
3. 同一经验不变量契约面对不同本体能力画像会产生不同绑定候选、不同执行方式或不可执行结果。
4. 能力画像必须与经验不变量契约、空间语义和任务期快照共同作用，不能孤立理解。

说明书补强表述：

> 本体能力画像不同于单纯设备档案或平台接入清单。即使其字段可由 SDK、URDF、控制器参数或传感器配置提供，在本发明中其技术作用是参与经验步骤的跨本体适配可行性判断，并与经验不变量契约和当前空间语义数据共同确定当前执行体能否执行相应经验步骤。

### 4.5 攻击 E：任务期运行时世界状态快照只是任务状态表

对方可能说：

任何任务执行系统都有当前状态表，执行时更新，结束时释放，这不是技术创新。

防守要点：

1. 任务期快照不是普通任务状态表，而是经验迁移适配所需的事实对齐和工作记忆。
2. 它在当前任务期间承载已成立事实、待验证事实、事实冲突状态、执行体持有物、对象位置和当前过程节点。
3. 它依据执行闭环回传的因果产出事实和因果销毁事实更新。
4. 任务后释放的作用是避免任务期事实继续作为后续任务适配依据，降低状态污染。

说明书补强表述：

> 任务期运行时世界状态快照用于当前任务执行期间的事实对齐、前提判断和执行反馈更新，其与长期世界模型、共享知识库或普通日志不同。任务结束后，所述快照可以被物理删除、逻辑失效、解除任务绑定或停止作为后续任务适配判断依据，从而避免当前任务中的临时事实污染后续经验迁移判断。

### 4.6 攻击 F：执行可行性结果只是业务判断

对方可能说：

“可执行、不可执行、需确认”只是业务状态或用户提示，不能构成技术特征。

防守要点：

1. 执行可行性结果不是人工业务裁决，而是基于四类技术输入共同产生。
2. 该结果直接控制是否向执行闭环下发经验步骤、是否阻断危险动作、是否触发替代经验搜索或人工教学。
3. 绑定候选为空、候选被过滤、部分步骤不可执行、降级执行和人工确认触发都可以是执行可行性结果的实现形态。
4. 其技术效果是避免能力不足、空间绑定失败或事实前提不足时猜测执行。

说明书补强表述：

> 执行可行性结果不是面向用户展示的普通成功或失败提示，而是基于经验不变量契约、当前空间语义数据、本体能力画像和任务期运行时世界状态快照形成的执行控制结果。该结果用于决定经验步骤是否交由执行闭环执行，或触发阻断、降级、替代经验搜索、人工确认或补充教学。

### 4.7 攻击 G：执行闭环返回只是日志回写

对方可能说：

执行闭环返回状态并更新快照，只是把执行日志写回系统。

防守要点：

1. 回写对象不是普通日志，而是因果产出事实和因果销毁事实。
2. 更新结果会改变任务期快照中的已成立事实、待验证事实、冲突事实或不成立事实。
3. 更新后的事实会参与后续经验步骤的适配、前提补齐和不可执行判断。
4. 审计记录只是任务结束后的保留形态，不替代任务中事实状态更新。

说明书补强表述：

> 执行闭环返回状态不是仅用于日志记录，而是用于提取或确认经验步骤执行后的因果产出事实和因果销毁事实，并据此更新任务期运行时世界状态快照。更新后的快照继续参与当前任务后续经验步骤的适配判断和事实前提判断。

## 五、审查意见答复预案

### 5.1 如果审查员认为本案属于业务规则或抽象流程

可答复：

本案解决的不是历史案例复用或业务流程编排问题，而是真实经验中的坐标、轨迹、关节角和固定时长难以跨空间、跨本体复用的问题。权利要求中的经验不变量契约、当前空间语义数据、本体能力画像、任务期运行时世界状态快照、执行可行性结果和执行闭环事实回传，均服务于当前执行环境下的经验迁移适配和执行控制。

### 5.2 如果审查员认为经验不变量契约是人为规则

可答复：

经验不变量契约并非人为业务规则，而是从历史真实经验中抽取或保存的可迁移执行约束。其技术作用是排除不可迁移的绝对坐标、机器人专用关节角、固定执行时长和单一本体轨迹，并保留能够在当前空间语义和当前本体能力条件下重新绑定和校验的结构性内容。

### 5.3 如果审查员认为本体能力画像是普通设备档案

可答复：

本体能力画像可由设备档案、SDK 能力查询或运动学模型提供，但在本案中并不以设备描述本身为目的。其被用于判断经验不变量契约中的动作、可达、末端执行器、传感器、控制精度、力控和安全约束是否能由当前执行体满足，并与空间语义和任务期快照共同生成执行可行性结果。

### 5.4 如果审查员认为任务期快照是普通状态表

可答复：

任务期运行时世界状态快照不是通用任务状态表，而是当前任务期间用于经验迁移适配的事实对齐和工作记忆。它依据执行闭环回传的因果产出事实和因果销毁事实滚动更新，并在任务结束后删除、释放或失效，避免临时事实继续影响后续任务的经验迁移判断。

## 六、无效宣告中的防守重点

无效请求人可能把本案拆成：

1. 历史案例检索。
2. 规则模板。
3. 机器人能力匹配。
4. 当前任务状态。
5. 执行成功失败回写。
6. 任务结束释放资源。

我方应强调：

1. 单独看每个模块可能有现有技术基础，但 P017 的关键在于数据流和控制流耦合。
2. 经验不变量契约先限定历史真实经验中的可迁移内容，避免复用不可迁移参数。
3. 当前空间语义和本体能力画像将不变量落到当前执行环境和当前执行体。
4. 任务期快照提供当前任务事实对齐，并承接执行反馈。
5. 执行可行性结果阻断不满足空间、本体或事实前提的经验步骤。
6. 任务后释放快照避免临时事实污染长期世界模型或后续任务。

抗辩时不要把“经验不变量契约”“本体能力画像”“任务期快照”等术语作为孤立创新点，而要把它们串成真实经验迁移执行闭环。

## 七、竞对规避中的防守重点

竞对可能通过以下方式弱化技术性：

1. 把经验不变量契约叫作 recipe、skill template、case pattern、constraint set。
2. 把当前空间语义叫作 context、scene graph、world state、digital twin。
3. 把本体能力画像叫作 SDK capabilities、robot profile、URDF、device config。
4. 把任务期快照叫作 planner state、blackboard、working memory、runtime context。
5. 把执行可行性结果叫作 grounding result、action binding、candidate filtering。
6. 把执行闭环事实回传叫作 execution log、telemetry、callback。

防守时不要执着术语，而要抓实质：

1. 是否存在从历史经验中限定可迁移结构性内容的数据。
2. 是否用当前空间结构和当前执行体能力对该内容进行绑定或过滤。
3. 是否维护当前任务期间的事实状态用于适配和执行反馈。
4. 是否基于上述内容生成步骤能否执行的控制结果。
5. 是否在执行后用事实变化更新任务期状态。
6. 是否在任务结束后释放或失效该任务期状态。

## 八、对权利要求的影响

当前 v3 独权六段主链已经具备抵御“业务逻辑剥离”的关键锚点：

1. “经验不变量契约”防止落入普通案例复用。
2. “当前空间语义数据”防止落入单纯本体能力匹配。
3. “本体能力画像”防止落入单纯空间参数绑定。
4. “任务期运行时世界状态快照”防止落入长期世界模型或普通案例库。
5. “执行可行性结果”防止落入无控制结果的参数替换。
6. “执行闭环事实回传更新”防止落入一次性计划执行。
7. “任务结束后删除或释放”防止落入普通长期状态维护。

因此不建议为了防“业务逻辑剥离”继续扩张独权。更优策略是说明书写实技术语境，尤其写清经验不变量契约的来源、当前环境绑定、本体能力校验、任务期快照的技术用途、执行可行性结果的控制作用和任务后释放的状态隔离作用。

## 九、说明书必须补强的位置

建议在正式说明书中补强以下位置：

1. 背景技术：明确普通案例推理、规则引擎、任务状态表和设备能力匹配无法解决真实经验跨空间跨本体可控迁移。
2. 技术问题：写成不可迁移参数、能力不足猜测执行、任务期事实污染和跨本体适配失败，而不是“提高经验复用效率”。
3. 术语定义：每个核心术语都给出“不等同于”口径。
4. 具体实施方式：倒水、抓取、插入、导航实施例都要写“错误存法/正确存法”。
5. 执行可行性结果：写清其可以显式或隐式表现为绑定候选过滤、阻断、降级、替代搜索或人工确认。
6. 任务期快照：写清其与长期世界模型共存时的逻辑释放和停止作为后续任务适配依据。
7. 有益效果：每个效果都绑定到具体技术手段，避免空泛效率表述。

## 十、建议写入说明书的总防线段落

可在发明内容或具体实施方式开头加入以下段落：

> 本发明并非针对非技术业务流程、审批流程、工单流转或通用案例复用的规则配置方法。本发明所述经验记录、经验不变量契约、当前空间语义数据、本体能力画像、任务期运行时世界状态快照和执行可行性结果，均用于真实经验在当前空间和当前执行体条件下的迁移适配执行。其中，经验不变量契约限定历史真实经验中可跨空间或跨本体复用的拓扑关系、试探方向、物理约束或事实终止条件；当前空间语义数据和本体能力画像分别提供当前执行环境和当前执行体能力约束；任务期运行时世界状态快照提供当前任务期间的事实对齐和执行反馈更新；执行可行性结果用于决定经验步骤是否可交由执行闭环执行、是否需要阻断、降级、替代经验搜索、人工确认或补充教学。因此，本发明解决的是历史真实经验在不同空间布局和不同本体能力条件下的可控迁移执行问题。

## 十一、最终判断

“业务逻辑剥离”是 P017 在说明书起草阶段必须补的一条防线，但它不要求改变 v3 的独权结构。当前最优策略是：

1. 权利要求保持六段最小闭环，不继续把业务无关解释塞入独权。
2. 说明书把技术语境写牢，避免被读成案例检索、规则模板或普通任务状态管理。
3. 实施例必须出现可迁移不变量、当前空间绑定、本体能力约束、任务期事实快照、执行可行性结果和事实回传更新。
4. 答复口径围绕“不可迁移低层参数如何被剥离、当前环境如何重新绑定、能力不足如何阻断、任务期事实如何隔离”展开。

这样处理后，P017 的定位会更清楚：不是历史案例业务复用专利，而是真实经验跨空间跨本体迁移执行专利。
