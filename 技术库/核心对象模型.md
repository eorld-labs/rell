# 核心对象模型

## 一、定位

本文用于定义“空间经验世界模型与运行时治理底座”的核心对象。项目早期不急于建设完整大系统，而应优先保证所有经验、计划、动作和治理能力都有统一的数据承载方式。

核心对象模型的作用是：

1. 让空间语义、经验记录、任务记录、偏好记录和补救记录能够被统一检索和调用。
2. 让来源链、代理权预算、计划预验真、裁决令牌和审计摘要能够进入真实执行链路。
3. 让外部 Agent、机器人、VLA、工具网关、IoT 设备或企业系统能够通过 SDK/API 接入底座。

## 二、对象总览

底座至少包括以下对象：

1. 空间对象 `SpatialObject`
2. 空间区域 `SpatialRegion`
3. 空间关系 `SpatialRelation`
4. 治理规则 `GovernanceRule`
5. 执行主体 `ExecutionActor`
6. 情境对象 `ContextState`
7. 动作声明 `ActionDeclaration`
8. 经验记录 `ExperienceRecord`
9. 任务经验记录 `TaskExperienceRecord`
10. 补救经验记录 `RecoveryExperienceRecord`
11. 偏好记录 `PreferenceRecord`
12. 执行计划 `ExecutionPlan`
13. 步骤摘要 `StepSummary`
14. 计划预验真结果 `PlanPrecheckResult`
15. 来源链对象 `ProvenanceChain`
16. 代理权预算对象 `AgencyBudget`
17. 意图验真对象 `IntentVerification`
18. 物理控制裁决对象 `PhysicalControlDecision`
19. 保护策略声明对象 `ProtectionPolicyDeclaration`
20. 裁决令牌 `DecisionToken`
21. 审计摘要 `AuditSummary`

## 三、空间与治理对象

### 3.1 SpatialObject

用于表示空间中的对象实体。

建议字段：

```json
{
  "object_id": "obj_001",
  "object_type": "cup",
  "semantic_labels": ["container", "fragile"],
  "geometry_ref": "mesh_or_bbox_ref",
  "pose": {"x": 1.2, "y": 0.4, "z": 0.8},
  "owner_ref": "user_001",
  "sensitivity_level": "medium",
  "interaction_affordances": ["grasp", "move", "inspect"],
  "governance_tags": ["private_object", "confirmation_required"],
  "state": "formal"
}
```

### 3.2 SpatialRegion

用于表示可通行区、风险区、交互区、禁入区、共享区等。

建议字段：

```json
{
  "region_id": "region_001",
  "region_type": "shared_living_area",
  "functional_attributes": ["walkable", "interactive", "shared"],
  "access_policy_ref": "policy_001",
  "risk_level": "low",
  "allowed_actor_types": ["human", "home_robot"],
  "restricted_actions": ["high_speed_motion"]
}
```

### 3.3 GovernanceRule

用于表达对象、区域、主体、动作之间的治理约束。

建议字段：

```json
{
  "rule_id": "rule_001",
  "scope": {"region_id": "region_001"},
  "actor_condition": {"actor_type": "home_robot"},
  "action_condition": {"action_type": "move_object"},
  "decision": "candidate_state_required",
  "confirmation_required": true,
  "audit_required": true,
  "priority": 80
}
```

## 四、主体与动作对象

### 4.1 ExecutionActor

用于表示 Agent、机器人、车载代理、IoT 代理、工作流主体等。

建议字段：

```json
{
  "actor_id": "actor_001",
  "actor_type": "home_robot",
  "capability_profile": ["navigate", "grasp", "speak"],
  "autonomy_level": "limited_autonomy",
  "owner_ref": "user_001",
  "current_budget_ref": "budget_001",
  "trust_level": "verified",
  "control_boundary_ref": "boundary_001"
}
```

### 4.2 ActionDeclaration

用于在动作进入真实执行端前形成统一动作声明。

建议字段：

```json
{
  "action_id": "act_001",
  "actor_id": "actor_001",
  "action_type": "move_object",
  "target_refs": ["obj_001"],
  "parameter_digest": "move_to=table_center",
  "expected_outcome": "object_relocated",
  "risk_level": "medium",
  "rollback_capability": "partial",
  "context_ref": "ctx_001",
  "plan_ref": "plan_001",
  "source_chain_ref": "src_001"
}
```

## 五、经验对象

### 5.1 ExperienceRecord

对应 `{情境, 动作, 后果}`。

```json
{
  "experience_id": "exp_001",
  "context_ref": "ctx_001",
  "action_ref": "act_001",
  "outcome": {
    "outcome_type": "success",
    "state_delta": "cup_moved_without_spill",
    "risk_observed": "none"
  },
  "confidence": 0.86,
  "source_actor_ref": "actor_001",
  "environment_type": "real_physical",
  "created_at": "2026-06-28T12:00:00Z"
}
```

### 5.2 TaskExperienceRecord

对应 `{意图, 情境, 任务计划, 执行结果, 人类反馈}`。

```json
{
  "task_exp_id": "task_exp_001",
  "human_intent": "tidy_living_room",
  "context_ref": "ctx_001",
  "plan_ref": "plan_001",
  "execution_result": "partial_success",
  "human_feedback": "do_not_move_remote_control",
  "updated_steps": ["step_003"]
}
```

### 5.3 RecoveryExperienceRecord

对应 `{偏离情境, 补救动作, 补救结果}`。

```json
{
  "recovery_exp_id": "rec_001",
  "deviation_context": "object_slipped_during_grasp",
  "recovery_action": "reduce_force_and_regrasp",
  "recovery_result": "success",
  "original_action_ref": "act_001",
  "actor_capability_version": "gripper_v2"
}
```

### 5.4 PreferenceRecord

对应 `{情境, 动作, 人类偏好指示}`。

```json
{
  "preference_id": "pref_001",
  "human_ref": "user_001",
  "context_pattern": "living_room_evening",
  "action_pattern": "place_cup",
  "preference_signal": "prefer_table_center",
  "confidence": 0.74,
  "validity": "decay_enabled",
  "isolation_group": "household_user_001"
}
```

## 七、最小实现原则

早期实现不需要一次性覆盖所有字段，但必须保证以下最小对象可用：

1. `SpatialObject`
2. `ContextState`
3. `ActionDeclaration`
4. `ExperienceRecord`
5. `ExecutionPlan`
6. `PlanPrecheckResult`
7. `ProvenanceChain`
8. `AgencyBudget`
9. `DecisionToken`
10. `AuditSummary`

只要这十类对象跑通，项目就可以形成第一条工程闭环。
